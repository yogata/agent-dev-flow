## 目的

完了報告、GitHub本文、リンク、自然言語成果物の品質を維持し、文字化け・Markdown崩れ・AI生成特有の曖昧な文章を抑止する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0107-001 | agentdev command の完了報告は統一フォーマットに従うこと（SHALL） |
| REQ-0107-002 | git操作を行わない command は完了報告で git 永続化なしを明示すること（SHALL） |
| REQ-0107-003 | Issue/PR本文への書き込み後は読み戻し検証を行うこと（SHALL） |
| REQ-0107-004 | 読み戻し検証はエンコーディング、Markdown構造、必須セクションを確認すること（SHALL） |
| REQ-0107-005 | GitHub本文書き込みには `--body-file` 等のファイル指定を使用し、直接本文指定を避けること（SHALL） |
| REQ-0107-006 | GitHub本文内のリポジトリ参照リンクはGitHub URL形式を使用すること（SHALL） |
| REQ-0107-007 | コードブロック内とテンプレート変数プレースホルダーはリンク正規化対象外とすること（SHALL） |
| REQ-0107-008 | 自然言語成果物は具体性・検証可能性・過剰な一般論の回避を満たすこと（SHALL） |
| REQ-0107-009 | AI生成特有の空疎な表現、過度な美辞麗句、根拠のない断定を避けること（SHALL） |
| REQ-0107-010 | ファイル保存形式は UTF-8（BOMなし）と LF を基本とすること（SHALL） |
| REQ-0107-011 | コミットメッセージはプロジェクトの規約に従い、変更意図を明確に表すこと（SHALL） |

## 適用範囲

- **対象**: 完了報告、Issue/PR本文品質、リンク正規化、AI-slop抑止、gh CLI本文書き込み
- **対象外**: 個別template本文、コード実装、テスト実装
