## 目的

完了報告、GitHub本文、リンク、自然言語成果物の品質を維持し、文字化け・Markdown崩れ・AI生成特有の曖昧な文章を抑止する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0107-001 | agentdev command の完了報告は統一フォーマットに従うこと |
| REQ-0107-002 | git操作を行わない command は完了報告で git 永続化なしを明示すること |
| REQ-0107-003 | Issue/PR本文への書き込み後は読み戻し検証を行うこと |
| REQ-0107-004 | 読み戻し検証はエンコーディング、Markdown構造、必須セクションを確認すること |
| REQ-0107-005 | GitHub本文書き込みには `--body-file` 等のファイル指定を使用し、直接本文指定は対象外とすること |
| REQ-0107-006 | GitHub本文内のリポジトリ参照リンクはGitHub URL形式を使用すること |
| REQ-0107-007 | コードブロック内とテンプレート変数プレースホルダーはリンク正規化の対象外であること |
| REQ-0107-008 | 自然言語成果物は具体性・検証可能性・過剰な一般論の回避を満たすこと |
| REQ-0107-009 | 自然言語成果物は具体性・検証可能性を満たすこと。AI生成特有の空疎な表現・過度な美辞麗句・根拠のない断定は対象外とし、具体的・検証可能な記述を使用すること |
| REQ-0107-010 | ファイル保存形式は UTF-8（BOMなし）と LF を基本とすること |
| REQ-0107-011 | コミットメッセージはプロジェクトの規約に従い、変更意図を明確に表すこと |
| REQ-0107-026 | 完了報告は共通必須項目を含むこと |
| REQ-0107-033 | `case-close` の失敗系完了報告では、GitHub側完了状態、`.agentdev` 永続化状態、ブランチ・worktree削除状態を分離して報告すること |

### 完了報告 共通必須項目

REQ-0107-026 が求める共通必須項目は以下の項目とする:

- 完了コマンド
- 対象
- 結果
- 検証結果
- git 永続化
- 次のコマンド

## 適用範囲

- **対象**: 完了報告、Issue/PR本文品質、リンク正規化、AI-slop抑止、gh CLI本文書き込み、完了報告フォーマットの共通必須項目、完了報告における git 永続化状態の分離報告
- **対象外**: 個別template本文、種別 選択ロジック、種別 path、種別 schema、コード実装、テスト実装、command / skill 作成一般の設計原則化、learning item の作成、実出力 golden test / eval の導入、AgentDevFlow外の汎用テンプレート設計規約、Issue本文テンプレート・Issueコメントテンプレート・その他 skill/reference 内テンプレート、`case-update` の明示引数方式への変更、`agentdev-workflow-reporting`（廃止済み）の復元、実行時コマンドから SPEC への直接実行時依存
