## 目的

完了報告、GitHub本文、リンク、自然言語成果物の品質を維持し、文字化け・Markdown崩れ・AI生成特有の曖昧な文章を抑止する。

完了報告テンプレートは variant 単位で一意に参照できる構造であり、REQ-0107 の既存方針（完了報告SSoT・command側本文保持禁止・variant定義）に適合する。

完了報告variantは最終出力全文を一意に選択する単位である。共通契約は `docs/specs/artifact-contracts.md` が定義し、各 variant ファイルは command-local 構成である。

`req-save` に加え、`req-define`（Pattern別分岐）・`case-close`（失敗系分岐）・`case-update`（更新種別推論分岐）も variant 単位で分割される。

command 側は完了報告本文を持たず、使用する variant のファイルパス・選択条件・変数だけを保持する。

### 共通必須フィールド定義

REQ-0107-026 が検査する共通必須フィールドは以下の6項目とする:

- 完了コマンド
- 対象
- 結果
- 検証結果
- git 永続化
- 次のコマンド

## 適用範囲

- **対象**: 完了報告、Issue/PR本文品質、リンク正規化、AI-slop抑止、gh CLI本文書き込み、完了報告フォーマットの共通契約（`docs/specs/artifact-contracts.md` と command-local variant ファイル）、完了報告variantの定義単位・参照構造・command側の参照方式・docs-checkによる完了報告variant検査・共通必須フィールドの定義・`req-define`のPattern別variant分岐・`case-close`の失敗系variant分岐・`case-update`の更新種別推論とvariant分岐、command 定義における完了報告参照方式（variant path / 選択条件 / 置換変数）、command-local template による完了報告管理、`docs/specs/artifact-contracts.md` による完了報告共通契約定義
- **対象外**: 個別template本文、コード実装、テスト実装、command / skill 作成一般の設計原則化、learning item の作成、実出力 golden test / eval の導入、AgentDevFlow外の汎用テンプレート設計規約、Issue本文テンプレート・Issueコメントテンプレート・その他 skill/reference 内テンプレート、`case-update` の明示引数方式への変更、`agentdev-workflow-reporting`（廃止済み）の復元、runtime command から SPEC への直接実行時依存
