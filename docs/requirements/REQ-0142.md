## 目的

SPEC 再構築および配布物からの内部管理 ID 除去は、配布物の変更範囲と内部 ID の grep 結果のみを完了条件としていた。
これにより、内部 ID が除去された後も Markdown 構文破損、主要構造の重複、壊れた参照残骸、command / skill / SPEC 間の責務説明の矛盾が残る状態で「整合済み」と扱われるリスクがあった。
本 REQ は配布物 ID 除去後の完了条件として文意保持、構文健全性、責務整合を満たすべき状態として定義し、横断検査の観点を拡充することで、SPEC 再構築の完了判定の曖昧さを解消する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0142-001 | 配布物（src/opencode/commands/agentdev/、src/opencode/skills/agentdev-*/）から内部管理 ID（REQ-ID、ADR-ID、内部パス参照）を除去した後、当該配布物が Markdown 文書として構文破損を残さないこと |
| REQ-0142-002 | 配布物の frontmatter、タイトル、入力、出力、手順などの主要構造が、同一文書内で意図せず重複しないこと |
| REQ-0142-003 | 内部 ID 除去後に、壊れた括弧、壊れた参照表現、主語や目的語を失った文が配布物に残らないこと |
| REQ-0142-004 | command / skill / SPEC 間で、同一 command の責務説明が矛盾しないこと |
| REQ-0142-005 | case-open / case-run / case-close / case-auto の責務境界が、command 本体、command SPEC（docs/specs/commands/）、関連 skill の間で一致すること |
| REQ-0142-006 | 横断検査は、内部 ID の有無だけでなく、文意保持、構文健全性、責務整合の観点を含むこと |
| REQ-0142-007 | docs-check 等で既知 NG や false positive が残る場合、完了判定と混同せず、分類（false positive、pre-existing、今回修正対象）、理由、後続対象を明確にすること |

## 適用範囲

- **対象**: 配布物（src/opencode/commands/agentdev/、src/opencode/skills/agentdev-*/）から内部管理 ID を除去した後の文意保持、構文健全性、command / skill / SPEC 間の責務説明の整合性（特に case-open / case-run / case-close / case-auto）、横断検査（docs-check / inspect-docs / inspect-skills 等）の観点拡充
- **対象外**: case-* 系 command の新機能追加、実装計画の詳細化、Issue / PR 作成手順の新設、case-auto の実装方式そのものの再設計、case-run が担当する実装作業の具体的な実装内容、SPEC 再構築全体の再方針化、未確認の pre-existing NG の一括修正、case-update の責務境界の新規定義（関連 SPEC の整合確認対象として限定）
