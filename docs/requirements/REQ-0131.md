## 目的

case-close の完了ゲート、PR マージ、Issue クローズ、ドメイン状態永続化、実行後の intake / learning 候補回収、branch / worktree クリーンアップを定義する。
PR 本文からの intake / learning 分離回収と、ローカル変更検出時の安全な停止を含む。

| REQ-0131-005 | `case-close` は PR本文の capture セクションから intake / learning を分離回収し、intake 候補を intake ドメイン状態に、learning 候補を learning ドメイン状態に保存すること |

| REQ-0131-022 | case-close は最終 Wave 処理時（全子Issue クローズ完了）、Epic Issue 自体をクローズし Epic レベルの最終処理（intake/learning capture 含む）を実行すること |
| REQ-0131-023 | case-close は最終 Wave 以外の処理時、残 Wave 状況（完了 Wave、残 Wave、次実行可能 Issue）を通知すること |

| REQ-0131-026 | case-close は全 test strategy 項目が合格済みまたは Findings 記録済みであることを完了条件として確認すること。未処理の test strategy 項目が残る場合、完了扱いとしないこと |
| REQ-0131-027 | case-close は Epic Wave クローズ処理を準並列化すること。並列実行可能な処理と直列集約が必要な処理を分離すること。rebase による機械的コンフリクト解消は停止条件外とすること（REQ-0151-006 Level1）。rebase で解消不能なコンフリクトは実装変更を行わず case-auto へエスカレーションすること（REQ-0131-025、REQ-0151-002 Level2/3） |
| REQ-0131-028 | case-close は squash merge 実行前に PR の mergeable 状態を確認し、UNKNOWN の場合は mergeable になるまでポーリング待機すること（最大60秒、10秒間隔）。ポーリングには gh pr view --json mergeable,mergeStateStatus を使用し、最大待機時間超過時はマージを中止し構造化エラーとして報告すること |
| REQ-0131-029 | case-close は git main 同期時に worktree 状態（dirty tree）、並列実行による ref lock 競合、非 main ブランチ占有の3リスクを事前検出し、安全な代替同期手順（直列化待機、git fetch origin main:main による非チェックアウト同期）を選択すること |
| REQ-0131-030 | `case-close` は targeted docs guard（check_changed_docs.ts）の実行結果で files_checked が空の場合、検査見逃しのリスクとして警告を確認し、`--files` での再実行または対象ファイルの確認を行うこと |

> **case-close 責務境界再定義ノート（REQ-0151-007、ADR-0132 Level 1）**
>
> case-close の責務境界を「完了処理 + マージ時コンフリクトの機械的解消（rebase のみ、解消不能時は即エスカレーション、実装変更は行わない）」に再定義する。
> コンフリクト解消ロジックとクローズ処理の混在は構造的に起きない（マージ失敗時に case-close は Step 5-12 を実行しない）。
> case-merge の新規コマンド分離は行わない。

## 適用範囲

- **対象**: case-close、完了ゲート（要件、成果物、SPEC、DOC-MAP整合確認、ADR確認）、PR マージ（squash、リトライ、フォールバック、squash merge 失敗時 rebase による機械的コンフリクト解消、解消不能時 case-auto エスカレーション）、PR マージ前重複ファイル検出（ローカル未コミット変更 × PR 変更ファイル）、Issue クローズ、intake / learning 候補回収（PR本文→intake/learning ドメイン状態）、branch / worktree クリーンアップ、ローカル変更検出時の安全停止、乖離検出（docs全体grep）、Epic G04自動クローズ判定、完了条件チェックボックス評価、更新、force-with-lease制約、Epic Wave クローズ（現在 Wave の PR マージ、子Issue クローズ、最終 Wave で Epic クローズ、残 Wave 通知）
- **対象外**: case-run（REQ-0130）、case-open（REQ-0132）、Epic/Wave 並列実行オーケストレーション（REQ-0106）、workflow基本契約（REQ-0104）、intake / learning pipeline の詳細処理（REQ-0127 / 0128）
