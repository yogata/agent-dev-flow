## 目的

case-close Step 7 において、サブエージェントが生成した未コミット変更（一時ファイル・ログ・キャッシュ等）やプロセスロックが原因で `git worktree remove` が失敗する問題（Issue #344, #381, #389 で計3回発生）を予防する。また case-run において、PR作成後に worktree 内に残存する未コミット変更を検出し、対処する仕組みを追加する。

また、squash merge 後のローカルブランチ削除において、`git branch -d` が `not fully merged` エラーで失敗するケースに対する条件付き `-D` 許可の要件を明文化する。当該実装は既に case-close Step 7 に存在するが、正式なREQ要件としてのトレーサビリティが欠落しており、誤ったREQ参照（REQ-0038-002）が付与されていた。

## 要件

| ID | 要件 | RFC 2119 |
|----|------|----------|
| REQ-0037-001 | `case-close` Step 7 の .sisyphus/ クリーンアップ**前**に `git status --short` を実行し、未コミット変更の有無を検出すること（SHALL） | SHALL |
| REQ-0037-002 | 未コミット変更を検出した場合、当該変更が squash merge 済みか確認すること（SHALL）。squash merge 済みと確認できた場合は `git checkout .` で変更を破棄して後続の worktree remove に進むこと（SHALL） | SHALL |
| REQ-0037-003 | 未コミット変更が squash merge 済みと確認できない場合、警告を表示して停止すること（SHALL）。自動的な変更破棄は禁止する（REQ-0001-007） | SHALL |
| REQ-0037-004 | `git worktree remove` が Permission denied エラー（プロセスロック）で失敗した場合、ロックしているプロセスの特定方法（IDE・ターミナル等）を案内し、ユーザーの手動解消を待機すること（SHALL）。自動的なプロセスキル・強制削除は禁止する | SHALL |
| REQ-0037-005 | `case-run` Step 11 完了後・Step 12 開始前に、worktree 内で `git status --short` を実行し、未コミット変更を検出すること（SHALL） | SHALL |
| REQ-0037-006 | `case-run` で未コミット変更を検出した場合、変更内容を報告し、ユーザーの指示に従ってコミットまたは破棄すること（SHALL） | SHALL |
| REQ-0037-007 | `case-run` のクリーンアップ Step では .sisyphus/ ディレクトリの削除を行わないこと（SHALL） | SHALL |
| REQ-0037-008 | `case-close` Step 7 は、ローカルブランチ削除で `git branch -d` が `not fully merged` により失敗した場合、PR が squash merge 済みであることを GitHub 上の PR 状態（`gh pr view` の `state: MERGED`）で確認すること（SHALL） | SHALL |
| REQ-0037-009 | `case-close` Step 7 は、PR が merge 済みであることを確認できた場合に限り、`git branch -D` によるローカルブランチ強制削除を許可すること（SHALL） | SHALL |
| REQ-0037-010 | `case-close` Step 7 は、PR が merge 済みであることを確認できない場合、ローカルブランチを強制削除せず、警告を表示して停止すること（SHALL） | SHALL |
| REQ-0037-011 | `case-close.md` 内のブランチ削除処理（`git branch -d` 失敗時セクション）に付与された誤った REQ-0038 参照（`REQ-0038-002`）を、REQ-0037 の該当要件ID（REQ-0037-008〜010）に修正すること（SHALL） | SHALL |

## 適用範囲

### 対象

- `case-close.md` Step 7（ブランチ・worktree削除）
- `case-run.md` Step 11 と Step 12 の間に挿入する新規 Step

### 対象外

- `agentdev-git-worktree/SKILL.md`（別要件 REQ-0033-024, REQ-0033-025 で対応）
- `case-close.md` の他 Step
- `case-run.md` の他 Step
- .sisyphus/ ディレクトリのクリーンアップ（case-run 側）
