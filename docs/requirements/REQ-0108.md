## 目的

REQ/ADR/SPEC/DOC-MAP、command、skill、template、script の整合性を検査し、ガードレールとテストで品質を担保する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0108-001 | `integrity-check` は REQ/ADR/SPEC/DOC-MAP の横断的整合性を検査すること（SHALL） |
| REQ-0108-002 | `integrity-check` は active REQ と retired REQ を別集合として扱うこと（SHALL） |
| REQ-0108-003 | `integrity-check` は active REQ 一覧と実体ファイルの不一致を検出すること（SHALL） |
| REQ-0108-004 | `integrity-check` は retired REQ が現行要件判断に使われる導線を検出すること（SHOULD） |
| REQ-0108-005 | `integrity-check` は ADR↔REQ、Skill↔command、Template必須セクション、旧namespace残存を検査対象に含めること（SHALL） |
| REQ-0108-006 | 検査結果は finding ごとに分類し、推奨routeを提示すること（SHALL） |
| REQ-0108-007 | `integrity-check` は基準文書を直接自動更新しないこと（SHALL） |
| REQ-0108-008 | 再発原因・予防策として価値がある finding は learning 候補として扱うこと（SHOULD） |
| REQ-0108-009 | 要件・責務・ワークフロー変更が必要な finding は req-define 入力候補として扱うこと（SHALL） |
| REQ-0108-010 | `.agentdev` domain state 更新後は必要に応じて git 永続化すること（SHALL） |
| REQ-0108-011 | 読込・評価・ユーザー承認後に `git pull --ff-only` を行う設計では、読込時hashとpull後hashの一致を検証すること（SHALL） |
| REQ-0108-012 | agentdev command群は command / skill / template / epic の各層をカバーする体系的テストで品質を担保すること（SHALL） |

## 適用範囲

- **対象**: integrity-check、validation、guardrail scripts、体系的テスト、git永続化
- **対象外**: `.sisyphus/`、個別commandの詳細手順
