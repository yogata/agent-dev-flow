## 目的

`/repo/docs-check` は AgentDevFlow 本体リポジトリの文書 artifact に対する、検査対象を直接修正しない整合性検査として、存在・参照・索引・責務境界・旧構造残存を機械的に検査し、検出事項を report と intake 接続により後続処理へ渡す。現行REQは docs-check の恒久的な責務境界のみを保持し、checker 個別ルール・検出シグナル定義・誤検出抑制方式・作業履歴は integrity rule catalog、SPEC、skill reference、script、tests に配置する。

## 適用範囲

- **対象**: `/repo/docs-check` の検査対象を直接修正しない検査責務、artifact collection registry への委譲、report 出力、検出事項分類、基準 / 3層 gate / meta-integrity の責務境界、report から通常 intake item への接続、docs-check と inspect-docs / inspect-skills / inspect-promote の役割分担、体系的テスト、本体リポジトリ専用 command としての配布境界
- **対象外**: checker 個別ルール、検出シグナル定義、enum 値一覧、ルート表、誤検出抑制方式、パス解決アルゴリズム、検査データ具体値、script 内部実装、個別検出事項の修正、docs-check からの learning item / RU / REQ 直接生成、意味レビュー、作業履歴
