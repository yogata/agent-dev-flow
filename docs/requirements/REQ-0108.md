## 目的

`/repo/docs-check` は AgentDevFlow self-hosting repo の文書 artifact に対する、検査対象を直接修正しない整合性検査として、存在・参照・索引・責務境界・旧構造残存を機械的に検査し、finding を report と intake 接続により後続処理へ渡す。active REQ は docs-check の恒久的な責務境界のみを保持し、checker 個別ルール・検出シグナル定義・false positive 抑制方式・作業履歴は integrity rule catalog、SPEC、skill reference、script、tests に配置する。

## 適用範囲

- **対象**: `/repo/docs-check` の検査対象を直接修正しない検査責務、artifact collection registry への委譲、report 出力、finding 分類、baseline / 3層gate / meta-integrity の責務境界、report から通常 intake item への接続、docs-check と inspect-docs / inspect-skills / inspect-promote の役割分担、体系的テスト、self-hosting repo 専用 command としての配布境界
- **対象外**: checker 個別ルール、検出シグナル定義、enum 値一覧、route table、false positive 抑制方式、path resolution algorithm、fixture 具体値、script 内部実装、個別 finding の修正、docs-check からの learning item / RU / REQ 直接生成、semantic review、作業履歴
