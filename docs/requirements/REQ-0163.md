## 目的

agentdev コマンド群がサブエージェントへ委譲する際の category 選定と MUST NOT DO セクション記載の要件を定め、subagent が本来責務を超えてスコープ逸脱する事象を構造的に防ぐ。

Issue #1538 で `case-auto` から `case-open` を `category=writing` で委譲した際、subagent が文書監査ファイル生成や draft 作成といった case-open 責務外の作業へ逸脱した事象を契機とする。原因は category 名が「書く作業」を連想させ subagent の振る舞いを誘導したこと、および委譲プロンプトで MUST NOT DO が明示されずスコープ境界が弱かったことにある。`category=unspecified-high` と MUST NOT DO 強化で当該事象は解消したが、選定基準と記載要件が文書化されず、事務的手続きの委譲で毎回直面する。

本 REQ は選定ガイドラインと MUST NOT DO 必須化を要件化し、case-auto 委譲に限定せず subagent 委譲する全場面に適用可能な形で整備する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0163-001 | agentdev コマンドが subagent へ委譲する際、委譲先 command の責務（事務的手続き、実装作業、執筆作業等）と category 名の意味的距離を評価し、subagent の振る舞いを誤誘導しない category を選定すること。事務的手続き（Issue 作成、VERIFY、ラベル設定、状態遷移等）の委譲には `unspecified-high` を推奨し、`writing` category は執筆作業（docs 記述、article 作成等）のみに限定すること |
| REQ-0163-002 | agentdev コマンドから subagent へ委譲する全プロンプトに MUST NOT DO セクションを必須とすること。スコープ外作業の明示的禁止（当該 command 責務外のファイル作成、REQ/SPEC/src 修正の禁止、文書監査の禁止等）を列挙し、subagent がスコープ境界を推定せずに従えるようにすること |
| REQ-0163-003 | subagent 委譲プロンプトの標準テンプレートは category 選定基準と MUST NOT DO 記載要件を統合した形式をとること。case-auto、case-open、case-run、case-update、case-close の全委譲場面で共通適用可能な形とし、特定 command 名と category 名の意味的距離が大きい場合の注意事項を含むこと |

## 適用範囲

- **対象**: subagent 委譲を実施する agentdev command（case-auto、case-open、case-run、case-update、case-close）の委譲プロンプト、`agentdev-case-run-execution-adapter` SKILL の委譲契約、`agentdev-workflow-orchestration` references の subagent プロトコル記載
- **対象外**: `writing` category の発火スキル（`japanese-tech-writing` 等）無効化機構（別 Issue で具体化）、外部エージェント統合契約の委譲 API 詳細（REQ-0139）、委譲実行ハーネスの timeout、retry、context 管理（harness 責務、REQ-0162）、RU lifecycle（REQ-0105）
