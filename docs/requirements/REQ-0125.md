## 目的

AgentDevFlow の command と skill は runtime で疎結合に協調するため、Command→Skill 参照、Skill frontmatter、Skill 本文構造、references 利用、template / script 参照に過不足や妥当性の逸脱が生じ得る。`/agentdev/inspect-skills` はこれらを検査対象を直接修正せずに診断し、finding として報告することで、command / skill 間の参照妥当性を維持する。診断は修正を行わず、推奨 route の提示と finding file の生成に限定する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0125-001 | `/agentdev/inspect-skills` は public agentdev command として存在し、検査対象（Command/Skill 定義ファイル）を直接修正しない診断 command であること |
| REQ-0125-002 | 診断対象は Command 定義ファイル群、Skill 定義ファイル群、および必要に応じて関連する template / reference / script ファイル群であること |
| REQ-0125-003 | 診断観点は Command→Skill 参照妥当性、Skill frontmatter 整合、本文構造、references 利用、template / script 参照、粒度、段階的開示、責務境界、canonical name、内部構造依存を含むこと |
| REQ-0125-004 | 診断の詳細観点と判定基準は `agentdev-inspect-skills` skill に集約し、command 定義は薄い入口として skill への参照を持つこと |
| REQ-0125-005 | 出力は診断レポート（セッション内テキスト）、finding リスト（対象、観点、分類、根拠、推奨 route）、`.agentdev/inspect/inbox/inspect-skills-finding-{topic}.md`（finding file）に限定すること |
| REQ-0125-006 | finding file の出力先は `.agentdev/inspect/inbox/` 配下に限定すること |
| REQ-0125-007 | 許可される side effect は `.agentdev/inspect/inbox/inspect-skills-finding-*.md` の生成および `.agentdev/inspect/` 配下の git 永続化（commit / push）のみに限定すること（canonical docs 変更・REQ/ADR/SPEC 変更・Command/Skill/Template/Script 変更・Issue 作成・PR 作成・RU 保存・branch / worktree 操作は対象外） |
| REQ-0125-008 | finding に対する推奨 route の提示のみを行うこと（修正の実行は対象外） |
| REQ-0125-009 | `inspect-skills` は `.agentdev/inspect/` 配下の変更を git commit / push すること。変更なし時は commit/push せず完了報告で「変更なし」と報告すること |

## 適用範囲

- **対象**: `/agentdev/inspect-skills` コマンドの検出（Command→Skill 参照妥当性、Skill 構造、検査対象を直接修正しない診断、finding 出力、推奨 route 提示）
- **対象外**: inspect-docs（REQ-0109）、inspect-promote（REQ-0126）、docs-check（REQ-0108）、個別 finding の修正実行、機械的整合性検査（docs-check の担当）
