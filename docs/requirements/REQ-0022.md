## 目的

`.agentdev/` は git 管理される AgentDevFlow domain state である（REQ-0017-007/008）。しかし、`.agentdev/learning/**` または `.agentdev/intake/**` を更新するコマンド（learning-refine, learning-promote, case-close）は、更新後に当該変更を commit / push しないため、canonical state がローカルにのみ存在する状態になる。

本要件は、対象コマンドが `.agentdev/` 更新前にリモートと同期し、更新後に当該変更のみを scoped commit して push することで、canonical state を常にリモートと一致させる仕組みを提供する。

## 要件

### 実行前同期

| ID | 要件 |
|---|---|
| REQ-0022-001 | 対象コマンド（learning-refine, learning-promote, case-close）は、`.agentdev/` 更新前に `git pull --ff-only` を実行し、リモートの最新状態を取得しなければならない（SHALL） |
| REQ-0022-002 | pull が non-fast-forward、conflict、未コミット変更（dirty worktree）のいずれかで失敗した場合、コマンドは自動解消せず停止しなければならない（SHALL）。自動的なマージ、stash、reset は禁止する |
| REQ-0022-003 | 停止時はエラー種別（dirty_worktree / non_fast_forward / conflict / unknown_git_error）、停止理由、対象ブランチ、変更が検出されたファイル一覧、ユーザーが手動解消後に再実行すべきこと、必要に応じて git の元エラー出力を含む構造化エラーメッセージを表示しなければならない（SHALL） |

### 更新後の scoped commit

| ID | 要件 |
|---|---|
| REQ-0022-004 | 対象コマンドは `.agentdev/` 更新後、当該コマンドが変更したファイルだけを `git add` しなければならない（SHALL） |
| REQ-0022-005 | `.agentdev/` 以外の変更を commit に巻き込んではならない（MUST NOT） |
| REQ-0022-006 | commit メッセージは Conventional Commits 形式に従わなければならない（SHALL）。形式は `chore(agentdev): <summary>` とし、コマンド別の標準メッセージは以下の通りとする: learning-refine → `chore(agentdev): refine learning entries`、learning-promote → `chore(agentdev): promote learning findings`、case-close → `chore(agentdev): close case #N and capture domain state`（`#N` は case/issue 番号に置換） |
| REQ-0022-007 | case-close は learning capture と intake capture による `.agentdev/` 更新を同一 commit にまとめなければならない（SHALL）。1回の case-close 実行に由来する post-run domain state 更新として扱う |

### push

| ID | 要件 |
|---|---|
| REQ-0022-008 | commit 後に `git push` を実行しなければならない（SHALL） |
| REQ-0022-009 | push に失敗した場合、コマンドを完了扱いにしてはならない（MUST NOT）。エラー内容を報告し、ユーザーに手動対応を促す |

### 変更なしの場合

| ID | 要件 |
|---|---|
| REQ-0022-010 | `.agentdev/` に変更がない場合、commit / push を行ってはならない（MUST NOT） |

### 完了報告

| ID | 要件 |
|---|---|
| REQ-0022-011 | 完了報告に以下を含めなければならない（SHALL）: `.agentdev` 変更有無、変更ファイル一覧、commit hash（変更ありの場合）、push 成否、push しなかった理由（変更なし・push 失敗等） |

### 対象コマンドの変更ポイント

| ID | 要件 |
|---|---|
| REQ-0022-012 | learning-refine の Step 9（アーカイブ移動）の前に実行前同期（REQ-0022-001〜003）を追加し、Step 10（完了報告）の前に scoped commit + push（REQ-0022-004〜009）を追加し、完了報告に git 永続化の結果（REQ-0022-011）を含めるように変更しなければならない（SHALL） |
| REQ-0022-013 | learning-promote の Step 7（stub 生成）の前に実行前同期（REQ-0022-001〜003）を追加し、Step 9（完了報告）の前に scoped commit + push（REQ-0022-004〜009）を追加し、完了報告に git 永続化の結果（REQ-0022-011）を含めるように変更しなければならない（SHALL） |
| REQ-0022-014 | case-close の Step 9（learning capture）の前に実行前同期（REQ-0022-001〜003）を追加し、Step 9b（intake capture）の後に scoped commit + push（REQ-0022-004〜009）を追加し、完了報告（Step 11）に git 永続化の結果（REQ-0022-011）を含めるように変更しなければならない（SHALL）。learning capture と intake capture の変更は同一 commit にまとめる（REQ-0022-007） |

## 適用範囲

- **対象**:
  - `learning-refine` コマンド（`.opencode/commands/agentdev/learning-refine.md`）
  - `learning-promote` コマンド（`.opencode/commands/agentdev/learning-promote.md`）
  - `case-close` コマンド（`.opencode/commands/agentdev/case-close.md`）
  - `agentdev-workflow-reporting` スキルの完了報告フォーマット（git 永続化結果の表示項目追加）

- **対象外**:
  - 以外の agentdev コマンド（intake-capture, intake-from-github, intake-review, intake-promote, intake-open, integrity-check 等）への git 永続化の追加
  - git 永続化手順の共通 skill 化（3コマンドで重複後に別要件で検討）
  - `.agentdev/` 以外のパスの commit / push
  - 自動 conflict 解消や stash による回避
  - CI/CD パイプラインの変更
