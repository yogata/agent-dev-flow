## 目的

AgentDevFlow の ADR ライフサイクル、文書種別責務境界、runtime独立性、workflow 分類、状態モデルを正規化し、docs 永続文書と runtime 配布物の責務混線を解消する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0112-001 | ADR status 形式を正規化すること: `status: superseded` + frontmatter に `superseded_by: ADR-XXXX`（SHALL）。旧形式 `superseded-by:[ADR-XXXX]` は使用しない |
| REQ-0112-002 | ADR テンプレートから `## 実装後の記録` セクションを削除すること（SHALL） |
| REQ-0112-003 | `req-save` は合意済み ADR を `accepted` として作成すること（SHALL）。新規 ADR の初期 status は `proposed` とし、壁打ち合意後に `accepted` に遷移させる |
| REQ-0112-004 | `case-run` は同一ブランチ・同一 PR 内で ADR 変更を含めること（SHALL）。ADR 変更のみの独立 PR は原則として作成しない |
| REQ-0112-005 | `case-close` は ADR 差分の最終確認ゲートとして機能し、新規判断の追加は行わないこと（SHALL） |
| REQ-0112-006 | `accepted` status の ADR のみを現行根拠として使用すること（SHALL）。`proposed` / `superseded` / `deprecated` の ADR を現行要件判断の根拠として引用しない |
| REQ-0112-007 | docs 永続文書（REQ/ADR/SPEC/guides）から特定 RU-ID を根拠参照として引用しないこと（SHALL）。RU は `case-open` で削除される一時成果物であり、docs の永続根拠に不適である |
| REQ-0112-008 | REQ ファイルの Requirement Source セクションに RU パスを記録しないこと（SHALL）。req-save は RU パスを docs 永続文書に保存しない |
| REQ-0112-009 | `case-open` は RU 削除対象を docs 永続文書の Requirement Source ではなく、一時メタ情報から取得すること（SHALL） |
| REQ-0112-010 | `ru-sources` frontmatter フィールドを使用しないこと（SHALL） |
| REQ-0112-011 | work_type を `bugfix` / `feature` / `maintenance` / `docs_chore` の 4 値に固定すること（SHALL） |
| REQ-0112-012 | scale は `feature` のみ `standard` / `large` を持つこと（SHALL）。他の work_type は scale を持たない |
| REQ-0112-013 | workflow_route を保存せず、work_type + scale から都度導出すること（SHALL）。派生ルール: feature+standard → req_backed_case, feature+large → epic_case, その他 → direct_case |
| REQ-0112-014 | draft-meta に以下を保存しないこと（SHALL）: pattern, workflow_route, branch_type, labels, implementation_pattern, secondary_pattern, load_skills |
| REQ-0112-015 | Pattern A/B/C/D の分類を現行分類から退場させること（SHALL）。work_type + scale + workflow_route の3層分類に置き換える |
| REQ-0112-016 | runtime command が `docs/specs/system.md` / `docs/specs/patterns.md` に固定依存している構造を削除すること（SHALL）。runtime command は自己完結し、repo 内部 SPEC に依存しない |
| REQ-0112-017 | `case-run` は 150 行を目標とし、200 行を超える状態を維持しないこと（SHALL）。詳細判断は skill/reference に切り出す |
| REQ-0112-018 | `learning-*` / `intake-*` command を入口 command に縮小すること（SHALL）。詳細処理は skill/script に委譲する |
| REQ-0112-019 | `integrity-check` の frontmatter 検査において、implementation_pattern / secondary_pattern / load_skills の存在を禁止すること（SHALL） |
| REQ-0112-020 | command authoring standards も implementation_pattern / secondary_pattern / load_skills の frontmatter 記述を禁止すること（SHALL） |
| REQ-0112-021 | `docs/specs/` を repo-internal 設計文書として明記すること（SHALL）。runtime 配布物の依存先ではない |
| REQ-0112-022 | `docs/guides/` を人間向け navigation 層に位置づけること（SHALL）。規範的権限を持たない |
| REQ-0112-023 | 6 マイクロフェーズ（requirement/analyzed/created/in_progress/review/done）は状態管理モデルではなく説明用ラベルとすること（SHALL）。AgentDevFlow は全体横断の状態遷移モデルを持たない |
| REQ-0112-024 | `.agentdev/` を domain state として明確化し、配布物ではないことを明記すること（SHALL） |
| REQ-0112-025 | `.sisyphus/` を gitignore 対象とすること（SHALL） |
| REQ-0112-026 | `.sisyphus/drafts/` を一時領域として明確化すること（SHALL） |
| REQ-0112-027 | REQ / SPEC に workflow status を追加しないこと（SHALL） |
| REQ-0112-028 | intake promoted に route / status を追加しないこと（SHALL） |
| REQ-0112-029 | Issue / PR の状態を docs に複製しないこと（SHALL） |
| REQ-0112-030 | command-map を状態遷移エンジン化しないこと（SHALL） |

## 適用範囲

- **対象**: ADR ライフサイクル標準化、RU 参照排除、work_type 移行、command 薄型化、integrity 反転、docs 整合、状態モデル明確化。主対象 REQ: REQ-0101/0103/0104/0105/0108/0109。変更対象: ADR ファイル 6 個、SPEC 5 個、Commands 7 個、Skills 7 個
- **対象外**: 個別 command の詳細実装手順、テストコードの具体的記述、各 SPEC ファイルの本文内容
