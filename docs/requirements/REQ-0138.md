## 目的

`req-define`のdraftを、後続コマンド（`req-save`, `spec-save`, `case-open`, `case-auto`, `case-run`, `case-close`）が処理するための構造化合意結果ファイルへ変更する。
draftは緩い契約（soft contract: 生成元側標準、LLM推論前提、厳格なスキーマなし）として扱い、`artifact_actions`の有無で後続工程の実行可否を判定する。
これにより`work_type`固定分岐を廃止する。

## 要件

| REQ-0138-012 | 後続コマンドの入力処理は厳格なパーサー、スキーマバリデーターではなく、LLM推論による構造化フィールド解釈を前提とすること |

| REQ-0138-018 | req-define が生成し req-save が保存する最終 REQ ファイルは、REQ 必須セクション（目的/要件/適用範囲）を維持すること。補助セクション（SPEC候補等）の有無にかかわらず必須セクションは保存時にも維持されること（REQ-0136-009 から統合） |

| REQ-0138-024 | req-define は要件doc生成ステップを REQ 文書風 Markdown 生成から req_draft 構造化データ生成へ変更すること（REQ-0102-063 より分割） |
| REQ-0138-025 | req-define は draft-meta.spec-candidates と SPEC候補セクションを正とする旧出力をやめ、draft-data の artifact_actions のうち artifact: spec を正とすること（REQ-0102-064 より分割） |

## 適用範囲

- **対象**: req-define, req-save, spec-save, case-open, case-auto, case-run, case-close, draft出力契約, SSoT遷移
- **対象外**: 実装コードの変更、GitHub Issue/PR階層の決定（case-open責務）、外部エージェントの実装、設定
