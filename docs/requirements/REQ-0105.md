## 目的

作業候補、学び、要件化前の構造化入力を `.agentdev/` domain state として管理し、短期Issue化候補と要件化候補を混同しない。

## Design Notes

### backlog-review draft の status フィールドについて（INC-0019 対応注記）

REQ-0105-041 ~ 044 が参照する `status: reviewed` / `status: saved` は、backlog-review draft（`.sisyphus/drafts/`）の軽量ドキュメントライフサイクル注記である。これらは REQ-0108-123 が禁止するワークフロー状態管理モデル（6 マイクロフェーズによる進行管理）ではなく、backlog-review → backlog-save 間の受け渡し成立条件を示す 2 値フラグである。実際の draft 生成・状態遷移の実装は backlog-review / backlog-save コマンドが担う。
