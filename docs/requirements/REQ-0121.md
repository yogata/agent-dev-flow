## 目的

Runtime command 本体の規範語を原則使用しない方針を適用し、自然文での条件記述に置換する。併せて integrity 検査を、command 本体の語彙検査から責務境界違反または残存移行漏れ検査へ再定義する。REQ / SPEC / command / integrity 間で語彙ポリシーの矛盾を解消する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0121-001 | runtime command 本体では規範語を原則使用しない |
| REQ-0121-002 | 真の強制条件は、自然文での条件記述に置換する。「〜すること」「〜してはならない」「禁止」「停止する」「完了扱いにしない」などの表現を用いる |
| REQ-0121-003 | 強調・再宣言（commit message 形式の再宣言等）は削除する |
| REQ-0121-004 | Step / Guardrail と重複する再宣言は削除する |
| REQ-0121-005 | 禁止事項は「〜してはならない」「禁止」として自然文に置換する |
| REQ-0121-006 | 推奨・任意相当の内容は command 本体から原則排除し、必要なら注記・guide・SPEC・REQ に責務分離する |
| REQ-0121-007 | 高頻度 command（`case-run.md`, `case-close.md`, `req-save.md`）を優先適用対象とする |
| REQ-0121-008 | 規範語削減後も、必要な停止条件・禁止事項・検証条件が欠落していないことを確認する手順を含む |
| REQ-0121-009 | integrity 検査を command 本体の語彙検査から責務境界違反または残存移行漏れ検査へ再定義する |
| REQ-0121-010 | integrity 検査は command 本体の規範語残存を前提としない検査定義へ更新する |
| REQ-0121-011 | REQ / SPEC / command / integrity 間で語彙ポリシーの矛盾が残らないこと |

## 適用範囲

- **対象**: `src/opencode/commands/agentdev/*.md` の規範語、高頻度 command 優先、自然文置換基準、integrity 検査定義の再定義、語彙ポリシー整合性確認
- **対象外**: REQ 本文そのものからの必達要件全面禁止、SPEC の「現在有効な契約記述」からの必達要件一律排除、実装コードの振る舞い変更、完了報告 template 本文の全面書き換え、retired 文書の一括修正
