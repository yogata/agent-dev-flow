## 目的

AgentDevFlowの文書体系（REQ/ADR/SPEC/Guide/Report/DOC-MAP/Retired）における分類ルールを明確にし、文書の追加・更新時の同期不足、世代交代の不明確さ、分類の曖昧さに起因する不整合を防ぐ。分類ポリシーはSPEC（document-model.md）に集約し、各REQ/ADRは当該SPECを参照する構成とする。

## 要件

| ID | 要件 |
|---|---|
| REQ-0116-001 | document-model.md SPEC は、REQ/ADR/SPEC/Guide/Report/DOC-MAP/Retired の各分類の定義・責務・格納場所・権限モデルを含む Document Classification Policy セクションを定義すること（SHALL） |
| REQ-0116-002 | Document Classification Policy は、Document Authority Model（各分類の編集権限・承認フロー）を含むこと（SHALL） |
| REQ-0116-003 | Document Classification Policy は、Classification Decision Tree（文書追加時の分類判定フロー）を含むこと（SHALL） |
| REQ-0116-004 | Document Classification Policy は、Cross-document Projection Rules（原本から配置先への投影ルール）を含み、原本（`src/opencode/`）と配置先（`.opencode/`）の用語を定義すること（SHALL） |
| REQ-0116-005 | Document Classification Policy は、Reference Rules（文書間参照の書式とリンク規則）を定義すること（SHALL） |
| REQ-0116-006 | Document Classification Policy は、Lifecycle Rules（作成→アクティブ→改訂→廃止の状態遷移）を定義すること（SHALL） |
| REQ-0116-007 | Document Classification Policy は、Re-baseline Rules（全面改訂時の世代交代ルール、上書きではないことを明記）を定義すること（SHALL） |
| REQ-0116-008 | Document Classification Policy は、Synchronization Rules（APPEND/UPDATE操作時の同期要件）を定義し、APPEND/UPDATEが正しい操作であることと同期不足が問題であることを明記すること（SHALL） |
| REQ-0116-009 | Document Classification Policy は、Report を正式な文書分類として定義し、SPECから分離すること（SHALL） |
| REQ-0116-010 | Document Classification Policy は、ADRの定義を「将来の設計・運用・文書システムを制約する決定」に拡張すること（SHALL） |
| REQ-0116-011 | Document Classification Policy は、SPECにおける SHALL/MUST の使用を「現在有効な契約記述」に限定すること（SHALL） |
| REQ-0116-012 | Document Classification Policy は、Anti-patterns（分類ミスの典型的パターンとその検出方法）を列挙すること（SHALL） |
| REQ-0116-013 | Document Classification Policy は、Review/Check Mapping（分類ポリシーと docs-check/docs-review の対応関係）を含むこと（SHALL） |
| REQ-0116-014 | 廃止ADR（retired ADR）を現在の一次参照源として参照する場合、当該参照をアクティブな後継ADR/REQに更新すること。歴史的参照としては元のADR番号を保持可能（SHALL） |
| REQ-0116-015 | 原本（`src/opencode/`）と配置先（`.opencode/`）の用語を文書体系 SPEC に導入し、cross-document projection における方向性を明確にすること（SHALL） |

## 適用範囲

- **対象外**: 既存文書の実際の修正（Case 2 で対応）、command/skill/docs-check の変更（Case 3 で対応）、template の変更
