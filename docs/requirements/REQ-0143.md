## 目的

AgentDevFlow が管理する command 定義ファイルの Markdown 構成を標準化し、command 定義の読み取り、レビュー、整合性検査の基準を明確にする。

## 要件

| ID | 要件 |
|---|---|
| REQ-0143-001 | AgentDevFlow が管理する command 定義ファイルは、基盤 SPEC（`docs/specs/command-file-format.md`）で定義された command file format に従うこと |
| REQ-0143-002 | command file format の適用対象は `src/opencode/commands/agentdev/*.md`（AgentDevFlow 配布 command 原本）および `.opencode/commands/repo/*.md`（repo-local command）に限定すること |
| REQ-0143-003 | AgentDevFlow 適用プロジェクト（consumer project）の独自 command は command file format の強制対象外とすること |
| REQ-0143-004 | command SPEC（docs/specs/commands/*.md）と command 定義ファイル（src/opencode/commands/agentdev/*.md）の Step 番号構成は一致すること。先頭 Step 扱い、採番開始位置の詳細は SPEC（docs/specs/command-file-format.md）に配置すること |

## 適用範囲

- **対象**: AgentDevFlow が管理する command 定義ファイル（`src/opencode/commands/agentdev/*.md`、`.opencode/commands/repo/*.md`）の Markdown フォーマット標準化
- **対象外**: 各 command の処理内容変更、実装計画、責務再設計、個別 command SPEC の内容再設計、`docs/specs/commands/` のディレクトリ構成変更、`docs/specs/patterns.md` への統合、consumer project の独自 command への強制適用、`/repo/docs-check` の実装詳細
