## 目的

コマンド・スキル拡張時の後方互換性維持パターンを明文化し、既存フローを破壊する拡張を防止する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0111-001 | 既存コマンドの拡張時、既存Stepの内容を変更せず、分岐点のみで新規パスを注入するパターンを設計原則として定義すること（SHALL） |
| REQ-0111-002 | ディレクトリのリネームには `git mv` を使用し、Write/Deleteによる実装を禁止すること（SHALL） |
| REQ-0111-003 | 新規パスと既存パスを明示的に分離して記述すること（SHALL） |
| REQ-0111-004 | 既存 command を薄型化する場合、削除した詳細の移行先を明示し、移行先で同等の振る舞いを維持すること（SHALL） |
| REQ-0111-005 | 薄型化に伴い load_skills を削除する場合、削除対象 skill の参照方法を manual reference として残すこと（SHOULD） |

## 適用範囲

- **対象**: `agentdev-command-authoring` スキル references セクション、command 薄型化時の移行先明示と振る舞い維持、load_skills 削除時の manual reference 残し
- **対象外**: コマンド定義のSteps数・行数目安、load_skills整合性
