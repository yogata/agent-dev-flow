## 目的

Windows + junction 環境で `git worktree remove` が "Not a directory" エラーで失敗する問題に対し、agentdev-git-worktree skill の削除手順に junction 特有のエラーパターンとフォールバック手順を追記し、クロスプラットフォームでの削除信頼性を向上させる。

## 要件

| ID | 要件 |
|---|---|
| REQ-0117-001 | `src/opencode/skills/agentdev-git-worktree/references/worktree-operations.md` の削除手順セクションに、Windows + junction 環境での `git worktree remove` 失敗パターン（"Not a directory" エラー）の発生条件とフォールバック手順を追記すること。フォールバック手順は以下を含むこと: (1) `git worktree prune` の実行、(2) junction ディレクトリの手動削除、(3) ブランチ削除。Windows 固有の事象であることを明記すること |

## 適用範囲

- **対象**: `src/opencode/skills/agentdev-git-worktree/references/worktree-operations.md` の削除手順セクション
- **対象外**: worktree 作成手順、install-consumer-opencode.ps1 の junction 設定ロジック、REQ-0110（削除リトライ）の変更、他 skill/reference の変更
