## 目的

docs-check 機械検出により継続報告されている既知 NG を横断是正し、
check_integrity.test.ts fixture 経年劣化を解消し、正規化 PR のスコープ外残滓をクリーンアップし、
.agentdev/integrity/reports/ の git 管理状態を README 設計判断と一致させることで、
docs-check ノイズを一括低減し検証基盤の信頼性を回復する。
