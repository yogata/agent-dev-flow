## 目的

AgentDevFlow の標準ワークフロー（req-define → req-save → case-open → case-run → case-close）を維持しつつ、ユーザーが明示した場合に req-define 完了後の後続工程を最大限自走させる入口を提供する。

repo に閉じる変更に限り、エージェントが自律的に実装・検証・PR作成・merge・close まで進める。DB migration 実行、deploy、cloud apply 等、repo 外の実体を変更する操作は自走対象外とする。

## 適用範囲

- **対象**:
  - `/agentdev/case-auto` コマンド定義の新規追加
  - 入力解決（3段階優先順位）
  - 実行モデル（authoritative source reference による orchestration）
  - work_type に基づく工程分岐（feature / bugfix / maintenance / docs_chore）
  - 自走対象・自走対象外の判定ルール
  - 停止条件と停止時報告
  - ドキュメント整備（README / command reference / guides）
- **対象外**:
  - 既存の req-save / case-open / case-run / case-close の責務変更
  - req-save への --auto 等パラメータ追加
  - case-run への --auto-close 等パラメータ追加
  - 標準ワークフローの置換
  - req-define / 要件深掘りの自走化
  - repo 外の実体変更の自走化
  - DB migration 実行・deploy・cloud apply の自走化
  - 進行状態の永続化（停止時は再開可能コマンドを報告し、ユーザーが個別コマンドで再開）
