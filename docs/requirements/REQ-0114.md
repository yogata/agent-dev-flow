## 目的

AgentDevFlow の標準ワークフロー（req-define → req-save → case-open → case-run → case-close）を維持しつつ、ユーザーが明示した場合に req-define 完了後の後続工程を最大限自走させる入口を提供する。

repo に閉じる変更に限り、エージェントが自律的に実装・検証・PR作成・merge・close まで進める。DB マイグレーション実行、デプロイ、cloud apply 等、repo 外の実体を変更する操作は自走対象外とする。

## 適用範囲

- **対象**:
  - `/agentdev/case-auto` コマンド定義の構成
  - 入力解決（3段階優先順位）
  - 実行モデル（原本参照によるオーケストレーション）
  - work_type に基づく工程分岐（feature / bugfix / maintenance / docs_chore）
  - 自走対象・自走対象外の判定ルール
  - 停止条件と停止時報告
  - Epic Issue を制御単位として保持・読み取る振る舞い
  - case-auto Epic フロー クリーンアップ検証ゲート
  - case-auto 標準フロー 複数draft一括検出・逐次OU処理
  - case-open 相当処理の Epic Issue に残す実行制御情報
  - ドキュメント構成（README / command reference / guides）
  - case-auto の入力タイプ: 数値引数（Issue番号）・Issue URL
  - Issue番号/URL 入力時の工程分岐（case-run 相当 → case-close 相当）
  - 入力解決の優先順位拡張（Issue番号/URL が最優先）
  - 薄いオーケストレーター化（子Issue単位選択・委譲・永続状態ベース進行・再開）
  - Wave table「実行方法」列に基づく子Issue 並列選択・委譲
  - 並列 case-run 完了待機・部分失敗時の他Issue継続
  - Wave 間直列進行・前提失敗時の依存Issue pending維持
  - Epic Issue 本文の単一書き手制約
  - 完了報告のタイミング情報（開始時刻・終了時刻・所要時間）
- **対象外**:
  - 既存の req-save / case-open / case-run / case-close の責務への影響
  - req-save の --auto 等パラメータ
  - case-run の --auto-close 等パラメータ
  - 標準ワークフロー（独立維持、case-auto は並存）
  - req-define / 要件深掘りの自走化
  - repo 外の実体変更の自走化
  - DB マイグレーション実行・デプロイ・cloud apply の自走化
  - 進行状態の永続化（停止時は再開可能コマンドを報告し、ユーザーが個別コマンドで再開）
  - Issue 番号/URL の妥当性検証ロジック（case-run の既存ガードレール G05/G06 に委譲）
  - 複数 Issue 番号の同時指定
