## 目的

ローカル版で `agentdev-gh-cli` を上書きし、標準版と同一の手続き名で Issue/PR 相当の読み書きを Case ファイル（`.agentdev/cases/case-{NNNN}.md`）へ読み替える。
これにより、command/skill の分岐なしにローカル運用を可能にする差し替え基盤を提供する。
GitHub 非依存の抽象 backend 新設ではなく、GitHub 前提の gh-cli 手続き化と上書き可能化である。

## 要件

| ID | 要件 |
|---|---|
| REQ-0150-001 | ローカル版の `agentdev-gh-cli` は、標準版と同一の手続き名を提供し、Issue/PR 相当の読み書きを `.agentdev/cases/case-{NNNN}.md` に読み替えること |
| REQ-0150-002 | ローカル版では、PR 関連手続きを単純にスキップせず、PR 本文、PR 作成済み状態、merge 結果に相当する情報を Case ファイル上の対応セクションから読み書きできること |
| REQ-0150-003 | local 版 `agentdev-gh-cli` は別名スキルとせず、上位 command/skill は常に `agentdev-gh-cli` のみを参照すること |
| REQ-0150-004 | `src/opencode-local/agentdev-gh-cli/` を local 版 `agentdev-gh-cli` の原本とすること |
| REQ-0150-005 | `src/opencode-local/skills/` を作成しないこと |
| REQ-0150-006 | `case-schema/` を local 版 `agentdev-gh-cli` 配下の操作用定義に吸収すること |
| REQ-0150-007 | Case ファイル仕様の正本を docs/specs に置くこと（local 版 `agentdev-gh-cli` 配下の case schema は正本ではなく操作用定義） |
| REQ-0150-008 | GitHub 非依存の抽象 backend 新設を行わないこと（GitHub 前提の gh-cli 手続き化と上書き可能化であること） |

## 適用範囲

- **対象**: ローカル版 `agentdev-gh-cli` の位置づけ、手続き契約、Case ファイル対応づけ、case-schema 吸収、原本配置
- **対象外**: Case ファイルの詳細スキーマ（SPEC local-case-file.md）。手続き名と Case ファイルセクションの対応表（SPEC `agentdev-gh-cli`.md）。link mode の接続方式（REQ-0134）。標準版 `agentdev-gh-cli` の責務境界（REQ-0149）
