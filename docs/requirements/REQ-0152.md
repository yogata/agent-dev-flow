## 目的

gh CLI 直接呼出しの機械検出ルール（IR-053）を新設し、command/skill 定義への委譲漏れ込みを
docs-check の自動・機械的検出で担保する。inspect-skills エージェント実行型検出
（gh-direct-invocation-leak）と協調し、新規 command/skill 追加時の false negative リスクを低減する。

## 要件

| ID | 要件 |
|----|------|
| REQ-0152-001 | docs-check（check_integrity.ts）は command/skill 定義中の gh 直接記述（gh CLI 直接呼出し）を機械検出すること（IR-053 新規登録） |
| REQ-0152-002 | IR-053 の検出スコープ・除外対象は inspect-skills 診断観点（gh-direct-invocation-leak）および agentdev-gh-cli 手続き委譲基盤（REQ-0149-003 許容ファイル）と整合すること |

## 適用範囲

- **対象**: docs/specs/integrity-rule-catalog.md（IR-053 エントリ追加）、check_integrity.ts（IR-053 検出関数実装）、src/opencode/commands/agentdev/*.md・src/opencode/skills/agentdev-*/**/*.md（スキャン対象）
- **対象外**: inspect-skills 診断観点の変更（PR #1107 で正典化済み）、agentdev-gh-cli 許容ファイルの追加・削除、意味判断を要する委譲適否判定（inspect-skills / inspect-docs が担う）
