## 目的

要件（REQ）ファイルとアーキテクチャ決定記録（ADR）の管理規則を定義する。REQは領域別の総体として管理し、ADRは取り返しのつかない技術判断を記録する。両者の整合性を維持するための規約である。将来の標準構造として area-based current document 構造を定義し、段階的な移行方針を示す。

## 要件

| ID | 要件 |
|---|---|
| REQ-0004-001 | docs/requirements/ は現在有効な要件の Single Source of Truth (SSoT) として機能するものとする。将来の標準構造は README.md（運用説明）、INDEX.md（generated artifact）、core.md（初期・暫定area）、{area}.md（安定area）とする（SHALL） |
| REQ-0004-002 | INDEX.md は generated artifact であり、手動での編集を行わないものとする。INDEX.md の生成は req-file-manager スキルが管轄する（SHALL） |
| REQ-0004-003 | core.md は初期・暫定area（area_status: provisional）として定義する。unclassified.md は作成しないものとする（SHALL） |
| REQ-0004-004 | docs/requirements/sources/ および docs/requirements/areas/ ディレクトリは作成しないものとする（SHALL） |
| REQ-0004-005 | REQ-ID は REQ-{NNNN}（4桁ゼロ埋め）の安定IDであり、area情報をREQ-IDに含めないものとする。areaはファイル配置と frontmatter / generated INDEX で管理する（SHALL） |
| REQ-0004-006 | docs/requirements/{area}.md の frontmatter は最小構成（area_id / area_status / status / last_reviewed / review_policy）とし、owner_skill を含めないものとする（SHALL） |
| REQ-0004-007 | REQ単位の関連情報（REQ-ID / title / Status / Related artifacts / Related ADRs / Supersedes / Superseded by）は本文内に記述し、frontmatterフィールドとして管理しないものとする（SHALL） |
| REQ-0004-008 | REQの要件はRFC 2119言語（SHALL/SHOULD/MAY）で記述し、FR/NFRの区別を持たず単一テーブルに統合するものとする（SHALL） |
| REQ-0004-009 | REQの操作モデルは CREATE（新規作成）・APPEND（セクション追記）・UPDATE（内容変更）の3種類とし、判定基準に基づいて自動選択するものとする（SHALL） |
| REQ-0004-010 | req-file-manager スキルは requirements 構造・採番・area file 構造・メタデータ規約・frontmatter 規約・INDEX 生成方針・validation 方針・area review 方針を所有するものとする（SHALL） |
| REQ-0004-011 | 既存 REQ-{NNNN}.md 形式のファイルの一括移行はこの要件の対象外とする。移行は別 Issue で段階的に実施するものとする（SHALL） |
| REQ-0004-012 | ADR は取り返しのつかない技術判断のみを記録対象とし、ADR作成の要否は判定基準に基づいて評価するものとする（SHALL） |
| REQ-0004-013 | design-principles.md における丸数字（①②③）および「バイブス壁打ち」表記は、フェーズ名称（壁打ちフェーズ・構造的実行フェーズ・レビュー完了フェーズ）に補正するものとする（SHALL） |
| REQ-0004-014 | 要件管理構造の area-based 移行方針は ADR として記録するものとする（SHALL） |
| REQ-0004-015 | issue-save-req は現時点では REQ-{NNNN}.md へ保存し、{area}.md へ直接保存しないものとする（SHALL） |
| REQ-0004-016 | issue-save-req は現時点では INDEX.md を生成・更新しないものとする（SHALL） |
| REQ-0004-017 | docs/requirements/README.md は手書きINDEXとして当面更新対象に残るものとする（SHALL） |
| REQ-0004-018 | req-file-manager は current/future behavior を分離して記述するものとする（SHALL） |
| REQ-0004-019 | SPLIT は通常保存操作ではなく review/follow-up 候補とする。操作モデルには含めないものとする（SHALL） |
| REQ-0004-020 | SPLIT が必要でも保存可能範囲を実行し、保存全体を中止しないものとする（SHALL） |
| REQ-0004-021 | SPLIT 対象は完了報告で follow-up として明示するものとする（SHALL） |
| REQ-0004-022 | REQ-0004 操作モデルと req-file-manager 操作モードを整合させるものとする（SHALL） |

## 適用範囲

- **対象**: REQファイルの構造・操作モデル・採番規則、area-based current document構造、area file frontmatter規約、INDEX.md生成方針、ADRの作成基準、REQ/ADRの整合性管理、README インデックス管理
- **対象外**: 要件分析手法（req-analysisスキルの管轄）、仕様適合性検出（spec-complianceスキルの管轄）、docs/specs/ の内容管理、既存REQ-{NNNN}.mdファイルの一括移行
