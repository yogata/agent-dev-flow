## 目的

`.agentdev/intake/` は git 管理される AgentDevFlow domain state の一部である（REQ-0017-007/008, REQ-0022）。しかし、intake 系コマンド（intake-capture, intake-from-github, intake-review, intake-promote, intake-open）は、`.agentdev/intake/` を作成・移動・更新した後に当該変更を commit / push しないため、canonical state がローカルにのみ存在する状態になる。

本要件は、対象 intake 系コマンドが `.agentdev/intake/` 更新前にリモートと同期し、更新後に当該変更のみを scoped commit して push することで、canonical state を常にリモートと一致させる。REQ-0022 が learning-refine / learning-promote / case-close に対して定義した git 永続化パターンを、intake 系コマンドへ横展開するものである。

## 要件

### 実行前同期

| ID | 要件 |
|---|---|
| REQ-0025-001 | 対象 intake 系コマンドは、`.agentdev/intake/` を作成・移動・更新する前に `git pull --ff-only` を実行し、リモートの最新状態を取得しなければならない（SHALL） |
| REQ-0025-002 | pull が non-fast-forward、conflict、未コミット変更（dirty worktree）、その他の git error のいずれかで失敗した場合、コマンドは自動解消せず停止しなければならない（SHALL）。自動的なマージ、stash、reset は禁止する |
| REQ-0025-003 | 停止時はエラー種別（dirty_worktree / non_fast_forward / conflict / unknown_git_error）、停止理由、対象ブランチ、変更が検出されたファイル一覧、ユーザーが手動解消後に再実行すべきこと、必要に応じて git の元エラー出力を含む構造化エラーメッセージを表示しなければならない（SHALL） |

### 更新後の scoped commit

| ID | 要件 |
|---|---|
| REQ-0025-004 | 対象 intake 系コマンドは `.agentdev/intake/` 更新後、当該コマンドが変更したファイルだけを `git add` しなければならない（SHALL）。`git add` 対象は `.agentdev/intake/` 配下に限定する |
| REQ-0025-005 | `.agentdev/intake/` 以外の変更を commit に巻き込んではならない（MUST NOT） |
| REQ-0025-006 | commit メッセージは Conventional Commits 形式に従わなければならない（SHALL）。形式は `chore(agentdev): <summary>` とし、コマンド別の標準メッセージは以下の通りとする: intake-capture → `chore(agentdev): capture intake item`、intake-from-github → `chore(agentdev): capture intake items from github`、intake-review → `chore(agentdev): review and classify intake items`、intake-promote → `chore(agentdev): promote intake items`、intake-open → `chore(agentdev): issue intake items` |

### push

| ID | 要件 |
|---|---|
| REQ-0025-007 | commit 後に `git push` を実行しなければならない（SHALL） |
| REQ-0025-008 | push に失敗した場合、永続化未完了として扱い、コマンドを完了扱いにしてはならない（MUST NOT）。エラー内容を報告し、ユーザーに手動対応を促す |

### 変更なしの場合

| ID | 要件 |
|---|---|
| REQ-0025-009 | `.agentdev/intake/` に変更がない場合、commit / push を行ってはならない（MUST NOT） |

### 完了報告

| ID | 要件 |
|---|---|
| REQ-0025-010 | 対象 intake 系コマンドの完了報告に以下を含めなければならない（SHALL）: `.agentdev/intake` 変更有無、変更ファイル一覧、commit hash（変更ありの場合）、push 成否、push しなかった理由（変更なし・push 失敗等） |

### 対象コマンドの変更ポイント

| ID | 要件 |
|---|---|
| REQ-0025-011 | intake-capture の Step 4（保存）の前に実行前同期（REQ-0025-001〜003）を追加し、Step 4 と Step 5（完了報告）の間に scoped commit + push（REQ-0025-004〜008）を追加し、完了報告に git 永続化の結果（REQ-0025-010）を含めるように変更しなければならない（SHALL） |
| REQ-0025-012 | intake-from-github の Step 6（保存）の前に実行前同期（REQ-0025-001〜003）を追加し、Step 6 と Step 7（サマリーレポート提示）の間に scoped commit + push（REQ-0025-004〜008）を追加し、完了報告（Step 8）に git 永続化の結果（REQ-0025-010）を含めるように変更しなければならない（SHALL） |
| REQ-0025-013 | intake-review の Step 6（item の振り分け）の前に実行前同期（REQ-0025-001〜003）を追加し、Step 6 と Step 7（完了報告）の間に scoped commit + push（REQ-0025-004〜008）を追加し、完了報告に git 永続化の結果（REQ-0025-010）を含めるように変更しなければならない（SHALL） |
| REQ-0025-014 | intake-promote の Step 6（保存）の前に実行前同期（REQ-0025-001〜003）を追加し、Step 6 と Step 7（完了報告）の間に scoped commit + push（REQ-0025-004〜008）を追加し、完了報告に git 永続化の結果（REQ-0025-010）を含めるように変更しなければならない（SHALL） |
| REQ-0025-015 | intake-open の Step 13（artifact 削除）の前に実行前同期（REQ-0025-001〜003）を追加し、Step 13 と Step 14（完了報告）の間に scoped commit + push（REQ-0025-004〜008）を追加し、完了報告に git 永続化の結果（REQ-0025-010）を含めるように変更しなければならない（SHALL） |

## 適用範囲

- **対象**:
  - `intake-capture` コマンド（`.opencode/commands/agentdev/intake-capture.md`）
  - `intake-from-github` コマンド（`.opencode/commands/agentdev/intake-from-github.md`）
  - `intake-review` コマンド（`.opencode/commands/agentdev/intake-review.md`）
  - `intake-promote` コマンド（`.opencode/commands/agentdev/intake-promote.md`）
  - `intake-open` コマンド（`.opencode/commands/agentdev/intake-open.md`）
  - `agentdev-workflow-reporting` スキルの完了報告フォーマット（git 永続化結果の表示項目追加）

- **関連対象**（再定義しない）:
  - `/agentdev/case-close`
    - git 永続化は REQ-0022-014 により既に定義済み
    - 本要件では再定義しない
    - ただし intake item のファイル名規則・推奨セクション・frontmatter 禁止等の artifact contract は case-close 由来の intake item にも適用する

- **対象外**:
  - `case-close` コマンドの git 永続化（REQ-0022 で既に定義済み）
  - `learning-refine`、`learning-promote` コマンドの git 永続化（REQ-0022 で既に定義済み）
  - その他の agentdev コマンド（req-define, req-save, case-open, case-run, case-update, integrity-check 等）
  - git 永続化手順の共通 skill 化（REQ-0022 と本要件で重複後に別要件で検討）
  - `.agentdev/intake/` 以外のパスの commit / push
  - 自動 conflict 解消や stash による回避
  - CI/CD パイプラインの変更
