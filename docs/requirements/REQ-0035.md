## 目的

現行の `docs/requirements/views/*.md`（観点別体系化ビュー）と `docs/specs/*.md` の責務が重複しやすく、views が現在仕様の読み物になった場合は二重管理になる問題を解消する。views を廃止し、文書探索の入口として `docs/DOC-MAP.md` を導入する。また現在仕様の基準を `docs/specs/*.md` に明確化し、REQ / ADR / SPEC の基準境界を `agentdev-doc-map` スキルで定義する。

## 適用範囲

- **対象**:
  - `docs/DOC-MAP.md` の追加（文書探索・参照経路の入口）
  - `docs/requirements/views/*.md` の廃止（内容のDOC-MAP/specsへの移管後）
  - `docs/specs/*.md` の現在仕様基準化
  - `docs/specs/README.md` の追加または更新
  - `docs/requirements/README.md` の更新（views記述削除）
  - `docs/adr/README.md` の方針維持確認
  - `agentdev-doc-map` スキルの新設
  - `req-define` コマンドの探索順序変更（DOC-MAP追加・views除外）
  - `req-define` Step 4b の探索対象変更（views削除・DOC-MAP追加）
  - `case-run` コマンドへの DOC-MAP参照追加
  - `case-close` コマンドへの DOC-MAP参照追加
  - `integrity-check` コマンドの大幅強化（DOC-MAP検査、views残存検出、README↔実体不整合検出、finding分類・route提示）
  - ADR-0008 の新規作成（ADR-0007 の supersede）
  - REQ-0004 の部分UPDATE（views関連要件の supersede 明記）
  - REQ-0034 のUPDATE（Step 4b 探索対象変更）
  - DOC-MAP影響確認ルール（REQ/ADR/SPEC追加時）
- **対象外**:
  - `integrity-check` による自動修正（仕様判断を伴う可能性があるため）
  - views の継続利用（SPEC / DOC-MAP と責務重複するため廃止）
  - DOC-MAP の基準化（REQ/ADR/SPEC と競合するため）
  - req-define の docs 広範探索解禁（G05 の思想に反するため）
  - SPECへの要件基準移管（REQ基準境界を壊すため）
  - ADR README の廃止（ADR分類ビューとして既に機能しているため）

## 基準境界

| 種別 | 基準 | 責務 |
|---|---|---|
| REQ | `docs/requirements/REQ-{NNNN}.md` | 満たすべき要件 |
| ADR | `docs/adr/ADR-{NNNN}.md` | 採用した判断と理由 |
| SPEC | `docs/specs/*.md` | 現在仕様 |
| DOC-MAP | `docs/DOC-MAP.md` | 文書探索・参照経路の地図（参照用） |
| README | 各ディレクトリの入口 | 一覧・説明 |
| views | 廃止 | 使用しない |
