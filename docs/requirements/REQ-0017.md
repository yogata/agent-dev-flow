## 目的

本リポジトリを AgentDevFlow plugin として再定義し、AI agent-assisted development workflow の要件化・実行・レビュー・学習・未回収課題回収・整合性検査を扱う統一的な namespace に移行する。

現行の `/issue/issue-*` コマンド群は名前上 GitHub Issue 操作に見えるが、実態は開発ライフサイクル全体を扱っている。`/tips/*` は実態として learning pipeline であり、旧 backlog 系は正式化前の変更候補回収である。これらを AgentDevFlow plugin 内のサブドメイン（req / case / learning / intake / integrity）として統一する。

本要件は以下を達成する:

1. Plugin / product 表示名として `AgentDevFlow` を採用する
2. 公開コマンドの canonical namespace を `/agentdev/*` に統一する
3. Domain state directory を `.agentdev/` に分離する
4. 配布 skill の canonical prefix を `agentdev-*` に統一する
5. 旧 issue 系を req-* / case-* サブドメインに再編する
6. 旧 tips 系を learning-* サブドメインとして正式化する
7. 旧 backlog 系を intake-* サブドメインとして再定義する
8. artifact integrity check を integrity-check として導入する
9. `.sisyphus/` を Sisyphus runtime workspace、`.agentdev/` を AgentDevFlow domain state として分離する

## 適用範囲

### 対象

- `AgentDevFlow` の plugin / product 表示名と実装 namespace の定義
- `/agentdev/*` public command namespace の統一
- `.agentdev/` domain state directory と `.sisyphus/` runtime の責任分離
- `agentdev-*` skill prefix の原則定義
- req / case / learning / intake / integrity 各 domain への旧 command 移行
- intake item の一時保存 artifact としての定義
- artifact integrity check の read-only 棚卸し command としての新設
- 旧 command の完全削除（参照更新後）
- 参照ドキュメント（command-map / phases / ssot-transitions 等）の更新
- docs 用語の AgentDevFlow 用語への統一
- 子issue分割方針の定義

### 対象外

- AgentDevFlow の実行ロジックそのものの再設計
- Pattern A/B/C/D の判定基準変更
- REQ / ADR 管理方式の全面変更
- requirements の area-based current document 化
- GitHub Issue 以外の tracker 対応
- gh CLI 安全手順の変更
- issue description template のファイル名変更
- Sisyphus の plan / execution / evidence / archive 管理方式変更
- `archive-completed-plan` の削除
- リポジトリ名そのものの変更
- intake item の詳細フォーマット・状態管理・review 結果形式の固定
- `.agentdev/` の詳細サブディレクトリ構成の固定
- 個別 skill rename の新旧対応一覧の固定
- integrity-check の全検出対象の固定
- command-map による厳密な状態遷移管理
