## 目的

AgentDevFlow の検出コマンド群は `inspect` 命名で一貫しており、コマンド、ディレクトリ、skill、source_type、検出事項用語、ファイル名接頭辞（prefix）が全域で統一される。
旧 review 系コマンド、旧 diagnostics 系命名はコードベース、ドメイン状態、検出事項から完全に除去済みである。


`inspect` は「検出」に焦点を絞った軽量で一貫した命名である。
他の候補（diagnostics 系）は語として重く（診断、分析、原因特定、改善提案まで含むように見える）、単複揺れが発生するため採用しない。
命名上の中間状態は存在せず、不整合を残さない。

## 適用範囲

- **対象**:
 - `inspect-{docs,skills,promote}.md` コマンドファイル（`src/opencode/commands/agentdev/`）
 - `agentdev-inspect-skills/` スキル（`src/opencode/skills/`）
 - `inspect-docs/` 完了テンプレート（`src/opencode/commands/agentdev/templates/`）
 - `.agentdev/inspect/` ドメインディレクトリ（`inbox/`, `promoted/`, `archive/rejected/`）
 - `.agentdev/drafts/` の inspect 系検出事項ファイル（`inspect-docs-finding-*`, `inspect-skills-finding-*`）
 - draft type registry（`docs/specs/artifact-contracts.md`）
 - inspect 命名を参照する SPEC（`docs/specs/`: `workflow-contracts.md`, `system.md`, `integrity-contracts.md`, `document-model.md`）
 - inspect 命名を参照する REQ（`docs/requirements/`: `REQ-0103`, `REQ-0108`, `REQ-0109`, `REQ-0115`, `REQ-0116`, `REQ-0123`, `README.md`）
 - inspect 命名を参照する guides（`docs/guides/`: `command-selection.md`, `diagnostics-and-maintenance.md`, `troubleshooting.md`, `glossary.md`, `project-docs-and-specs.md`, `README.md`）
 - `docs/DOC-MAP.md`, `docs/README.md`
 - inspect 系命名を参照する他スキル（`src/opencode/skills/`: `agentdev-{req-file-manager,req-analysis,workflow-routing,workflow-lifecycle,spec-compliance,req-structure-diagnostics}/`）
 - 参照整合を保持する完了テンプレート（`integrity-check/standard.md`, `case-run.md`, `case-close.md`, `req-define.md`, `backlog-review.md`, `README.md`）
 - `.agentdev/README.md`（ドメイン状態説明の整合）

- **対象外**:
 - `docs/adr/ADR-0113.md`（旧命名移行決定の履歴記録、保持）
 - REQ-0103/0109/0115/0123 の changelog 行（履歴保持）
 - REQ-0103-145 自体（削除指示要件、保持）
 - `.agentdev/intake/archive/`（アーカイブ済み）
 - `docs/requirements/retired/`, `docs/requirements/mapping-table.md`（廃止履歴マッピング）
 - `.agentdev/backlog/req-units/RU-20260615-01.md`（RU 本体、要件化入力として保持）
 - `.opencode/`（gitignore、コンシューマー側投射先）
 - `req-restructure-review`（旧 review 系の更に旧名、別物）
 - `backlog-review`（別コマンド）
 - `req-structure-diagnostics`（`diagnostics` を含むが別スキル、inspect 対象外）
