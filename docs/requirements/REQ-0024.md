## 目的

agentdev コマンドの完了報告において、`completion-reports.md` の共通フォーマットが実行時に厳密適用されず、エージェントが汎用的な締め文にフォールバックする問題を解消する。

次工程提示の責務は各コマンド個別ではなく `agentdev-workflow-reporting` の共通完了報告フォーマットに集約されているため、`completion-reports.md` を中心とした横断修正として扱う。

## 要件

| ID | 要件 |
|---|---|
| REQ-0024-001 | 全 agentdev コマンドの完了報告では、`agentdev-workflow-reporting/reference/completion-reports.md` の該当セクションを必ず使用すること（SHALL）。独自フォーマット、要約のみの報告、汎用締め文への置換は禁止する（MUST NOT）。 |
| REQ-0024-002 | `completion-reports.md` に次工程が定義されているコマンドでは、完了報告に必ず `次のステップ: ...` を含めること（SHALL）。次工程の定義: `req-define` → `/agentdev/req-save` または `/agentdev/case-open`、`req-save` → `/agentdev/case-open`、`case-open` → `/agentdev/case-run {N}` または `/agentdev/case-run {epic_N}`、`case-run` → レビュー通過後 `/agentdev/case-close`、`case-update` → 更新種別に応じて `/agentdev/case-run {N}` 等、`intake-from-github` → `/agentdev/intake-open`、`intake-open` → `/agentdev/case-run {child...}` |
| REQ-0024-003 | 完了報告では以下の汎用締め文を禁止する（MUST NOT）。「次にやるべきことがあれば指示してください」「他に必要なことがあれば教えてください」「必要に応じて次の作業を指示してください」「次の指示をお待ちしています」。これらは次工程が定義済みの場面でユーザーに判断を戻し、ワークフロー遷移を曖昧にするため。 |
| REQ-0024-004 | `req-save` 完了報告では必ず `次のステップ: /agentdev/case-open` を出力すること（SHALL）。`req-save.md` 側にもこの制約を明示し、共通フォーマットの適用漏れを防ぐ。 |
| REQ-0024-005 | 各コマンドの完了報告ステップでは以下を明示すること（SHALL）: (1) `completion-reports.md` の該当セクションを使用する、(2) 変数のみ置換する、(3) 最終出力を独自に再構成しない、(4) 次工程行を削除しない。各コマンド定義のインライン完了報告コードブロックは削除し、`completion-reports.md` 参照に一本化する（SHALL）。対象: `req-define`、`req-save`、`case-open`、`case-run`、`case-update`、`case-close`、`intake-from-github`、`intake-open` |
| REQ-0024-006 | `case-close` のような終端コマンドでは次工程を出さない（SHALL）。ただし汎用締め文も使わない（MUST NOT）。完了状態、変更有無、git 永続化結果、capture 結果を出して終了する。 |
| REQ-0024-007 | コマンド定義・スキル定義のレビューでは、完了報告が次を満たすことを確認する観点を追加すること（SHALL）: `completion-reports.md` の該当フォーマットと一致する、次工程が定義済みなら `次のステップ` がある、禁止された汎用締め文がない、終端コマンドでは次工程を捏造していない |
| REQ-0024-008 | `intake-open.md` の `load_skills` に `agentdev-workflow-reporting` を追加すること（SHALL）。現状欠落しており、エージェントが `completion-reports.md` を参照できないため。 |
