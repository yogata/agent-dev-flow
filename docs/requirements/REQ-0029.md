## 目的

intake-open コマンドは現在、引数なし実行時に promoted artifact が1件なら自動検出、2件以上なら候補一覧を表示して停止する仕様である。しかし、intake-promote が複数の artifact を同時に生成することがあり、その都度手動でファイル名を指定して再実行する必要がある。

本要件は、引数なし実行時に promoted artifact を全件自動処理する一括処理モードを導入し、オペレーション効率を向上させる。同時に、artifact 単位のエラーで処理全体が停止しないよう継続処理戦略を採用し、完了報告で処理結果を一覧可能にする。

## 要件

### 一括検出・処理

| ID | 要件 |
|---|---|
| REQ-0029-001 | intake-open の Step 1（Artifact 検出）において、引数なし + 対象 artifact が 0件の場合はエラー停止すること（SHALL） |
| REQ-0029-002 | 引数なし + 対象 artifact が 1件以上の場合、`.agentdev/intake/promoted/intake-open/*.md` をファイル名昇順で全件取得し、順次処理すること（SHALL） |
| REQ-0029-003 | ファイル名引数が指定された場合は、指定された artifact のみを処理すること（SHALL） |
| REQ-0029-013 | 現行 Step 1 の「引数なし + promoted artifact が2件以上 → 候補一覧表示して停止」の動作を廃止すること（MUST NOT） |

### artifact 単位の処理

| ID | 要件 |
|---|---|
| REQ-0029-004 | 各 artifact ごとに、単一 Issue または Epic + 子 Issue の構成判定を行うこと（SHALL） |
| REQ-0029-005 | 各 artifact は、Issue 作成および VERIFY が成功した場合のみ削除すること（SHALL） |
| REQ-0029-006 | artifact 単位の処理失敗時は、当該 artifact を残したうえで後続 artifact の処理を継続すること（SHALL） |

### git 操作

| ID | 要件 |
|---|---|
| REQ-0029-007 | git pull --ff-only を全 artifact 処理の前に1回だけ実行すること（SHALL） |
| REQ-0029-008 | git pull --ff-only が失敗した場合は、artifact 処理を開始せず全体停止すること（SHALL） |
| REQ-0029-009 | 全 artifact 処理後に、`.agentdev/intake/` 配下の変更を1 commit / 1 push すること（SHALL） |
| REQ-0029-010 | git push が失敗した場合は完了扱いにせず、構造化エラーとして報告すること（SHALL） |

### 完了報告

| ID | 要件 |
|---|---|
| REQ-0029-011 | 完了報告に以下を含めること: 成功件数、失敗件数、削除済み artifact 一覧、残存 artifact 一覧、作成 Issue 番号一覧、git 永続化結果（SHALL） |
| REQ-0029-012 | 完了報告フォーマットは `agentdev-workflow-reporting` の完了報告フォーマット（completion-reports.md）に従うこと（SHALL） |

## 適用範囲

### 対象

- `.opencode/commands/agentdev/intake-open.md` の Step 構造変更（Step 1 の検出ロジック変更、ループ構造の導入、git 操作の再配置、完了報告の更新）
- `.opencode/skills/agentdev-workflow-reporting/reference/completion-reports.md` の intake-open 完了時フォーマット更新（一括処理結果の報告フォーマット追加）

### 対象外

- intake-promote、intake-review、intake-capture 等の他 intake コマンド
- case-open コマンド
- promoted artifact のフォーマット変更
- Issue テンプレート（issue_desc_*.md）の変更
- agentdev-gh-cli スキル
- agentdev-epic-tracker スキル
