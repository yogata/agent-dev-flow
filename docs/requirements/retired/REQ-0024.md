## 目的

全 agentdev コマンドの最終出力を、ユーザーが即座に判断できる短い完了報告に統一する。
完了報告は `agentdev-workflow-reporting/reference/completion-reports.md` を SSoT とし、共通必須フィールド（完了コマンド、対象、結果、検証結果、git 永続化、次のコマンド）を全コマンドで統一する。

現状の問題として、Step 別実行結果が長く出る、Todo 一覧が最後に出る、サブエージェント名、モデル名、実行時間が混入する、完了報告が最後に表示されない、終端コマンドで次のコマンドが明示されない、コマンドごとに完了報告の粒度が揺れる、といった課題がある。
ユーザーにとって重要なのは詳細ログではなく、最後に表示される短い完了報告である。

## 共通完了報告フォーマット

### 基本形（単一コマンド）

```markdown
✅ {command} 完了

完了コマンド: /agentdev/{command}
対象: {対象リソース}
結果:
  - {主要結果1}
  - {主要結果2}
  - {主要結果3}
検証結果: {✅ OK / ⚠️ 注意 / ❌ NG}
git 永続化: {変更なし（commit/push スキップ） / commit: {hash}, push: {成功/失敗} / 該当なし}
次のコマンド: {/agentdev/... / なし}
```

### 条件分岐がある場合

```markdown
✅ {command} 完了

完了コマンド: /agentdev/{command}
対象: {対象リソース}
結果:
  - {主要結果1}
  - {主要結果2}
検証結果: ✅ OK
git 永続化: {結果}
次のコマンド:
  - {条件1}: /agentdev/{command}
  - {条件2}: /agentdev/{command}
```

### case-close 正規例

```markdown
✅ case-close 完了

完了コマンド: /agentdev/case-close
対象: Issue #800 / PR #801
結果:
  GitHub 完了:
    - PR #801 を squash merge
    - Issue #800 をクローズ済み
  domain state:
    - ブランチ・worktree を削除済み
    - .agentdev 変更なし
検証結果: ✅ OK（1件の書き込み操作を検証済み）
git 永続化: 変更なし（commit/push スキップ）
次のコマンド: なし
```

### Epic Orchestrator 集約完了報告

```markdown
✅ case-run 完了（Epic Orchestrator）

完了コマンド: /agentdev/case-run
対象: Epic #{epic_N}
結果:
  | Wave | Issue | 状態 | PR | 備考 |
  |------|-------|------|----|----|
  | 1 | #{child1_N} | ✅ 完了 | #{pr1_N} | — |
  | 2 | #{child2_N} | ✅ 完了 | #{pr2_N} | — |

  成功: {success_count}件 / 失敗: {fail_count}件 / スキップ: {skip_count}件
検証結果: {✅ OK / ⚠️ 注意 / ❌ NG}
git 永続化: {変更なし（commit/push スキップ） / commit: {hash}, push: {成功/失敗} / 該当なし}
次のコマンド: {レビュー通過後: /agentdev/case-close / なし}
```
