## 目的

Runtime command 本体は規範語を原則使用せず、自然文での条件記述で構成される。
integrity 検査は command 本体の語彙検査ではなく、責務境界違反または残存移行漏れの検査として定義される。
REQ / SPEC / command / integrity 間で語彙ポリシーが矛盾しない。

## 要件

| ID | 要件 |
|---|---|
| REQ-0121-001 | runtime command 本体では規範語を原則使用しない |
| REQ-0121-002 | 真の強制条件は自然文での条件記述である。「〜すること」「〜してはならない」「禁止」「停止する」「完了扱いにしない」などの表現を用いる |
| REQ-0121-003 | 強調、再宣言（commit message 形式の再宣言等）は runtime command 本体に含まれない |
| REQ-0121-004 | Step / Guardrail と重複する再宣言は runtime command 本体に含まれない |
| REQ-0121-005 | 禁止事項は「〜してはならない」「禁止」の自然文で記述される |
| REQ-0121-006 | 推奨、任意相当の内容は command 本体に原則含まれず、必要な場合は注記、guide、SPEC、REQ に責務分離される |
| REQ-0121-007 | 高頻度 command（`case-run.md`, `case-close.md`, `req-save.md`）が本要件の優先対象である |
| REQ-0121-008 | runtime command は、必要な停止条件、禁止事項、検証条件の欠落がない状態を維持すること |
| REQ-0121-009 | integrity 検査は command 本体の語彙検査ではなく、責務境界違反または残存移行漏れの検査として定義される |
| REQ-0121-010 | integrity 検査は command 本体の規範語残存を前提としない検査として定義される |
| REQ-0121-011 | REQ / SPEC / command / integrity 間で語彙ポリシーの矛盾が残らないこと |

## 適用範囲

- **対象**: runtime command 本体（`src/opencode/commands/agentdev/*.md`）の規範語構成、高頻度 command 優先、自然文記述基準、integrity 検査の定義、語彙ポリシー整合性
- **対象外**: REQ 本文、SPEC、ADR の規範語（※旧 REQ-0122 対象範囲、REQ-0122 は完了につき retire 済み）、REQ 本文そのものの必達要件、SPEC の「現在有効な契約記述」の必達要件、実装コードの振る舞い、完了報告 template 本文、retired 文書
