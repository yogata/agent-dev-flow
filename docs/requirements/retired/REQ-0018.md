## 目的

`agentdev-req-analysis` スキルに、要件確定前の未決分岐を証拠ベースで解消する壁打ちメソドロジーを追加する。
現行の壁打ちメソドロジーは深掘り、合意形成、ステークホルダー視点を提供するが、壁打ち中に発生する未決分岐を依存順に解消する方法、1問1論点の制約、証拠で解決できる事項とユーザー判断事項の分離が不足している。
未決分岐解消を親概念として既存メソドロジーを再構造化し、壁打ち品質を担保する。

## 要件

### 未決分岐の抽出と整理

| ID | 要件 |
|---|---|
| REQ-0018-001 | `agentdev-req-analysis` は、壁打ち中の論点を未決分岐として扱い、目的、対象、対象外、粒度、永続化、責務境界、判定基準の観点で整理すること（SHALL） |
| REQ-0018-002 | `agentdev-req-analysis` は、未決分岐間の依存関係を判定し、依存元の分岐から順に解消する順序ルールを定義すること。優先順は 目的 → 対象 → 対象外 → 責務境界 → 粒度 → 永続化 → 判定基準 とし、既存資料により上位分岐が確定済みの場合は未確定の最上位分岐から開始すること（SHALL） |

### 質問運用ルール

| ID | 要件 |
|---|---|
| REQ-0018-003 | `agentdev-req-analysis` は、1回の質問で1つの未決分岐のみを扱う「1問1論点」ルールを定義すること（SHALL） |
| REQ-0018-004 | `agentdev-req-analysis` は、ユーザーへ質問する前に、明示入力ファイル、関連REQ、関連ADR、specs、command/skill定義で解決可能かを確認する evidence-first ルールを定義すること（SHALL） |

### 回答分類と反映

| ID | 要件 |
|---|---|
| REQ-0018-005 | `agentdev-req-analysis` は、壁打ち結果を Confirmed / Inferred / Unknown / User Decision / Out of Scope に分類すること（SHALL） |
| REQ-0018-006 | `agentdev-req-analysis` は、Confirmed のみを要件docへ反映し、Inferred / Unknown / User Decision は要件doc外の壁打ち出力で区別して提示すること（SHALL） |
| REQ-0018-007 | `agentdev-req-analysis` は、Inferred を要件doc本文、要件テーブル、適用範囲に混入させず、推定事項として根拠付きで提示すること（SHALL） |
| REQ-0018-008 | `agentdev-req-analysis` は、ユーザーが Inferred を明示的に承認または修正した場合のみ、Confirmed に昇格して要件docへ反映すること（SHALL） |
| REQ-0018-009 | `agentdev-req-analysis` は、壁打ち結果を `目的` / `要件` / `適用範囲` に射影するルールを定義すること（SHALL） |

### Unknown の取扱い

| ID | 要件 |
|---|---|
| REQ-0018-010 | `agentdev-req-analysis` は、要件成立に必要な Unknown が残存している場合、壁打ちを終了せず、追加調査、ユーザー判断、対象外化のいずれかで処分すること（SHALL） |
| REQ-0018-011 | `agentdev-req-analysis` は、Unknown を要件doc本文、要件テーブル、適用範囲の対象に混入させないこと（SHALL） |
| REQ-0018-012 | `agentdev-req-analysis` は、Unknown が今回の要件化対象外と判断された場合、Unknown のまま残さず Out of Scope に再分類すること（SHALL） |

### 既存メソドロジーの再構造化

| ID | 要件 |
|---|---|
| REQ-0018-013 | `agentdev-req-analysis` は、既存の壁打ちメソドロジーを未決分岐解消メソドロジーの配下に再構造化し、深掘り、合意形成、ステークホルダー視点をそれぞれ未決分岐の抽出、解消、確定判定に対応付けること（SHALL） |
| REQ-0018-014 | `agentdev-req-analysis` は、既存の深掘り、合意形成、ステークホルダー視点の内容を削除せず、役割を明確化したうえで再配置すること（SHALL） |

### 責務境界

| ID | 要件 |
|---|---|
| REQ-0018-015 | `agentdev-req-analysis` は、REQファイルの採番、CREATE/APPEND/UPDATE、frontmatter更新、README更新を扱わないこと（SHALL） |
| REQ-0018-016 | `agentdev-req-analysis` は、実装計画、タスク分割、コード変更方針の確定を扱わないこと（SHALL） |
| REQ-0018-017 | `agentdev-req-analysis` は、ADR閾値に達する可能性がある判断候補を抽出するが、ADRファイルの作成、更新は扱わないこと（SHALL） |

## 適用範囲

- **対象**:
 - `.opencode/skills/agentdev-req-analysis/SKILL.md` の壁打ちメソドロジーセクションの再構造化
 - 未決分岐抽出、依存順解消、1問1論点、evidence-first、回答分類、射影ルール、Unknown取扱いの追加
 - 既存要素（深掘り、合意形成、ステークホルダー視点）の再配置
 - `/agentdev/req-define` から参照される壁打ち品質基準の拡充

- **対象外**:
 - REQファイル作成、更新処理（`agentdev-req-file-manager` の管轄）
 - ADRファイル作成、更新処理（`agentdev-adr-file-manager` の管轄）
 - 実装計画、タスク分割、コード変更方針
 - コード実装
 - 新規コマンド追加
 - 外部 skill の vendoring
 - `CONTEXT.md` 系ファイルの導入
 - 要件分析、チェックボックス品質基準、ADR閾値判定ブリッジ、テンプレート参照セクションの変更
