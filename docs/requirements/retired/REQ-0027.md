## 目的

learning pipeline の各 artifact（inbox.md / archive.md / evaluation-report.md / promoted/）の lifecycle 責務を明確化し、command 間の責任境界を利用者が誤解しないようにする。

現行の command 定義（learning-refine / learning-promote）と skill 定義（agentdev-learning-pipeline）には、各 artifact の役割について部分的な記述が散在している。しかし、体系的な lifecycle 責務分類が存在せず、特に以下の誤解が生じやすい:

- `archive.md` が「終端履歴」ではなく「living pool」である点
- `evaluation-report.md` が「長期履歴」ではなく「refine/promote 間の境界 artifact」である点
- `promoted/` が「直接実装対象」ではなく「req-backlog 経由で RU 化する staging」である点
- `inbox.md` が「永続ストレージ」ではなく「未整理 entry の active queue」である点

本要件は、実装ロジックを変更せず、既存 lifecycle の意味を明文化する。

## 要件

### Artifact 責務分類

| ID | 要件 |
|---|---|
| REQ-0027-001 | `.agentdev/learning/inbox.md` を、未整理 learning entry の active queue として定義すること（SHALL） |
| REQ-0027-002 | `learning-refine` 成功後、処理対象 entry を `.agentdev/learning/archive.md` に移動し、`inbox.md` をヘッダーのみにクリアすること（SHALL） |
| REQ-0027-003 | `.agentdev/learning/archive.md` を、終端履歴ではなく、評価済み learning entry の living pool として定義すること（SHALL） |
| REQ-0027-004 | `archive.md` は `learning-promote` の入力として参照されることを明記すること（SHALL） |
| REQ-0027-005 | `archive.md` は `learning-promote` の prune により更新され得ることを明記すること（SHALL） |
| REQ-0027-006 | `learning-promote` は、昇華判定により staged / rejected / duplicate に分類された entry を archive.md から prune 対象にできること（SHALL） |
| REQ-0027-007 | `learning-promote` は、deferred および未処理 entry を prune 対象外とし、archive.md に残すこと（SHALL） |
| REQ-0027-008 | `.agentdev/learning/evaluation-report.md` を、`learning-refine` から `learning-promote` へ判断材料を渡すための境界 artifact として定義すること（SHALL） |
| REQ-0027-009 | `evaluation-report.md` は毎回上書きし、長期履歴として扱わないことを明記すること（SHALL） |
| REQ-0027-010 | `.agentdev/learning/promoted/` を、`learning-promote` が生成する Requirement Source stub の staging 領域として定義すること（SHALL）。`req-backlog` が promoted artifact を RU 化し、RU を req-define に渡す |
| REQ-0027-011 | `promoted/` の stub を `req-backlog` → RU → `req-define` の流れで扱うことを明記すること（SHALL） |
| REQ-0027-012 | `promoted/` の stub を `.opencode/` や実装コードへ直接反映してはならないことを明記すること（MUST NOT） |
| REQ-0027-013 | raw learning item を runtime command / skill の直接参照対象にしないこと。学びは `learning-promote` により Requirement Source stub 化し、`promoted/` → `req-backlog` → RU → `req-define` のルートを通じて反映すること（SHALL） |

### Command 責任境界

| ID | 要件 |
|---|---|
| REQ-0027-014 | `learning-refine` の責務を以下として明記すること（SHALL）: (1) inbox.md の entry を分類、評価する (2) evaluation-report.md を生成する (3) inbox.md の entry を archive.md へ移動する |
| REQ-0027-015 | `learning-promote` の責務を以下として明記すること（SHALL）: (1) evaluation-report.md を主入力として昇華判定する (2) archive.md を参照する (3) Requirement Source stub を promoted/ に生成する (4) staged / rejected / duplicate を prune する |
