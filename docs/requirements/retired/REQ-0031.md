## 目的

GitHub Issue / PR / comment に出力される Markdown 本文内のリポジトリ内ファイル参照を、常に正しい GitHub URL として出力する。
#379 のような個別 Issue のリンク修正ではなく、今後生成されるすべての GitHub 本文で同種の誤リンクを発生させないことを目的とする。

現行の各コマンド（case-open / case-run / case-close / case-update / intake-open）はテンプレートを読み込んで GitHub 本文を生成するが、本文内のリポジトリ内ファイル参照を GitHub URL に正規化する規則を持たない。
agentdev-workflow-templates はテンプレート構造と選定ルールを持つが、テンプレート変数に埋め込むファイル参照の URL 形式を定義していない。
agentdev-gh-cli の VERIFY はエンコーディング、Markdown 構造、必須セクションを検証するが、リンク先 URL の妥当性は検証していない。

本要件は出力コンテキスト規則の追加であり、既存 Issue の過去本文の自動修正は行わない。

## 要件

| ID | 要件 |
|---|---|
| REQ-0031-001 | GitHub Issue 本文、Issue コメント、PR 本文、Epic Issue 本文、Parent Issue 本文など、GitHub 上に書き込む Markdown 本文では、リポジトリ内ファイル、ディレクトリ参照を正規 GitHub URL として出力すること（SHALL）。 |
| REQ-0031-002 | GitHub 上に書き込む Markdown 本文では、リポジトリ内ファイル参照のリンク先として `../docs/...`、`docs/...`、`.opencode/...` などの相対パスまたは裸パスを Markdown link URL に使用してはならない（MUST NOT）。 |
| REQ-0031-003 | リポジトリ内ファイルを参照する場合、URL は `https://github.com/{owner}/{repo}/blob/{branch}/{path}` 形式に正規化すること（SHALL）。 |
| REQ-0031-004 | リポジトリ内ディレクトリを参照する場合、URL は `https://github.com/{owner}/{repo}/tree/{branch}/{path}` 形式に正規化すること（SHALL）。 |
| REQ-0031-005 | `.opencode` 配下の command / skill / template / reference を参照する場合、URL path に `.opencode/` を含めること（SHALL）。 |
| REQ-0031-006 | REQ を参照する場合、現行構成では `docs/requirements/REQ-{NNNN}.md` を参照先とし、`.opencode` を付与してはならない（MUST NOT）。 |
| REQ-0031-007 | ADR を参照する場合、現行構成では `docs/adr/ADR-{NNNN}.md` を参照先とし、`.opencode` を付与してはならない（MUST NOT）。 |
| REQ-0031-008 | branch は既定で `main` を使用すること（SHALL）。将来 default branch 取得が導入された場合でも、生成本文では実在する branch 名を使用すること。 |
| REQ-0031-009 | `case-open` は、Issue 本文および Issue コメント生成時に、リポジトリ内ファイル参照を正規 GitHub URL に変換すること（SHALL）。 |
| REQ-0031-010 | `case-run` は、PR 本文生成時に、リポジトリ内ファイル参照を正規 GitHub URL に変換すること（SHALL）。 |
| REQ-0031-011 | `case-close` は、Issue コメント、Issue 本文、Parent Issue 本文を更新する際に、リポジトリ内ファイル参照を正規 GitHub URL に変換すること（SHALL）。 |
| REQ-0031-012 | `intake-open` は、Issue / Epic / 子Issue / Epic 更新本文を生成する際に、リポジトリ内ファイル参照を正規 GitHub URL に変換すること（SHALL）。 |
| REQ-0031-013 | `case-update` は、Issue 本文、Issue コメント、レビューNGコメント、および REQ 更新に伴う Issue 本文同期を行う際に、リポジトリ内ファイル参照を正規 GitHub URL に変換すること（SHALL）。 |
| REQ-0031-014 | `agentdev-workflow-templates` は、GitHub 本文向けのリポジトリ内ファイル参照 URL 規則を SSoT として定義すること（SHALL）。 |
| REQ-0031-015 | `agentdev-gh-cli` の VERIFY 操作は、GitHub 書き込み本文に含まれるリポジトリ内ファイル参照リンクについて、相対リンク、裸パス、誤った `.opencode` 付与、`blob`/`tree` 種別誤りを検出すること（SHALL）。 |
| REQ-0031-016 | `integrity-check` は、command / skill / template 内に GitHub 本文へ相対リンクを出力させる指示が残っていないかを静的検査できること（SHOULD）。 |
| REQ-0031-017 | リポジトリ内 Markdown ファイル同士のリンクは、本要件の禁止対象外とする。README や docs 内の相対リンクは維持してよい（SHALL）。 |
| REQ-0031-018 | 既存 Issue の過去本文を自動修正することは本要件の対象外とする（MUST NOT）。必要な場合は別要件で GitHub body audit / migration として扱う。 |
| REQ-0031-019 | `agentdev-gh-cli` VERIFY 操作における裸パス検出は、Markdown link URL と backtick 内パスのうち `docs/`、`.opencode/`、`../docs/`、`../.opencode/` で始まるものを対象とすること（SHALL）。検出対象は `https://github.com/` 等の正規 URL 形式に変換すべきパスである |
| REQ-0031-020 | 裸パス検出の除外条件は以下の通りとすること（SHALL）: (a) `http://` / `https://` で始まる URL、(b) `#` アンカーのみのリンク、(c) `{owner}` / `{repo}` / `{branch}` / `{path}` 等のテンプレート変数プレースホルダーを含む URL、(d) コードブロック（` ``` ` で囲まれた領域）内のパス |

## 適用範囲

- **対象**:
 - case-open: Issue 本文、Issue コメントを生成する
 - case-run: PR 本文を生成する
 - case-close: Issue コメント、Issue 本文、Parent Issue 本文を更新する
 - case-update: Issue 本文更新、Issue コメント追加、レビューNGコメント追加、REQ 更新に伴う Issue 本文同期を行う
 - intake-open: intake artifact から Issue / Epic / 子Issue を生成、更新する
 - 対象テンプレート:
 - Issue 本文テンプレート: `issue_desc_feature.md`, `issue_desc_bug.md`, `issue_desc_epic.md`, `issue_desc_child.md`
 - Issue コメントテンプレート: `issue_comment_*`
 - PR 本文テンプレート: `pr_desc.md`
 - 対象スキル: agentdev-workflow-templates（SSoT）、agentdev-gh-cli（VERIFY）
- **対象外**:
 - 既存 GitHub Issue / PR / comment の一括修正
 - README / docs 内 Markdown の相対リンク禁止
 - GitHub URL の到達性をネットワークアクセスで毎回検査すること
 - REQ/ADR の配置変更
 - `.opencode` 配下への REQ/ADR 移動
 - Issue/PR/comment テンプレート構造の再設計

## 正規化例

| 参照対象 | 正しいリンク |
|---|---|
| REQ | `https://github.com/yogata/agent-dev-flow/blob/main/docs/requirements/REQ-0024.md` |
| ADR | `https://github.com/yogata/agent-dev-flow/blob/main/docs/adr/ADR-0001.md` |
| `.opencode` 配下ファイル | `https://github.com/yogata/agent-dev-flow/blob/main/.opencode/skills/agentdev-workflow-reporting/reference/completion-reports.md` |
| `.opencode` 配下ディレクトリ | `https://github.com/yogata/agent-dev-flow/tree/main/.opencode/commands/agentdev/` |

## 受け入れ条件

- [ ] #379 と同種の Issue 本文生成時に、REQ/ADR リンクが `../docs/...` ではなく `https://github.com/yogata/agent-dev-flow/blob/main/docs/...` 形式で出力される
- [ ] `.opencode` 配下のファイル参照は `blob/main/.opencode/...` 形式で出力される
- [ ] `.opencode` 配下のディレクトリ参照は `tree/main/.opencode/...` 形式で出力される
- [ ] REQ/ADR 参照に `.opencode` が誤って付与されない
- [ ] GitHub 本文に出力する Markdown link URL として `../docs/...` が残らない
- [ ] GitHub 本文に出力する Markdown link URL として `.opencode/...` の裸パスが残らない
- [ ] `agentdev-gh-cli` VERIFY で、相対リンク、裸パス、`blob`/`tree` 種別誤りを検出できる
- [ ] リポジトリ内 Markdown ファイル同士の相対リンクは検出対象外として維持される
- [ ] `agentdev-gh-cli` VERIFY が `docs/`、`.opencode/`、`../docs/`、`../.opencode/` で始まる裸パスを検出できる
- [ ] テンプレート変数プレースホルダー（`{xxx}`）とコードブロック内パスが検出対象外として除外される
