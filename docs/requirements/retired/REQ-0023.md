## 目的

learning-promote が生成する staging stub（`.agentdev/learning/promoted/*.md`）は、req-backlog → req-define → case-open → case-close のライフサイクルを通じて取り込みされる。
現在、staging stub がどの Issue で取り込みされたかの追跡機能と、取り込み後のアーカイブ機能が存在しない。

本要件は、staging stub の取り込み追跡（req-backlog / req-define / case-open / case-close 間でのパス伝播）と、取り込み後のアーカイブ（case-close での移動、取り込み記録追記）を実現する。

## 要件

### req-define: Requirement Source セクション

| ID | 要件 |
|---|---|
| REQ-0023-001 | req-define は、入力に `.agentdev/learning/promoted/*.md`（直接または RU 経由）が含まれる場合、生成する REQ 文書の `## 適用範囲` セクションの後に `## Requirement Source` セクションを追加しなければならない（SHALL） |
| REQ-0023-002 | `## Requirement Source` セクションには、入力となった staging stub のファイルパスをリスト形式で記録しなければならない（SHALL） |
| REQ-0023-003 | `## Requirement Source` セクションの機構は、learning staging stub 以外の入力ソースにも適用可能な汎用形式とするべきである（SHOULD） |

### case-open: Issue 本文転記

| ID | 要件 |
|---|---|
| REQ-0023-004 | case-open は、REQ 文書に `## Requirement Source` セクションが存在する場合、その内容を Issue 本文の対応セクションに転記しなければならない（SHALL） |

### case-close: imported 判定

| ID | 要件 |
|---|---|
| REQ-0023-005 | case-close は、Issue 本文に staging stub パスが記録されている場合、`agentdev-learning-pipeline` skill を load しなければならない（SHALL） |
| REQ-0023-006 | case-close は、Issue 本文から staging stub のファイルパスを抽出し、当該ファイルが `.agentdev/learning/promoted/` に存在することを確認しなければならない（SHALL） |
| REQ-0023-007 | staging stub の imported 判定は、以下の 4 条件を全て満たす場合にのみ imported としなければならない（SHALL）: (1) Issue に stub パスが記録されている、(2) 対応 PR がマージ済み、(3) Issue が completed としてクローズされている、(4) case-close が正常完了している |
| REQ-0023-008 | imported 判定で、staging stub の内容と Issue/PR の内容の意味的一致検証を実行してはならない（MUST NOT） |
| REQ-0023-009 | staging stub パスが Issue 本文に記録されていない場合、archive 処理を実行してはならない（MUST NOT） |
| REQ-0023-010 | case-close は、imported 判定の結果を完了報告に含めなければならない（SHALL） |

### staging stub archive

| ID | 要件 |
|---|---|
| REQ-0023-011 | imported と判定された staging stub を `.agentdev/learning/promoted/archive/` に移動しなければならない（SHALL） |
| REQ-0023-012 | 移動前に、stub 本文の末尾に `## 取り込み記録` セクションを追記しなければならない（SHALL） |
| REQ-0023-013 | `## 取り込み記録` セクションには以下を記録しなければならない（SHALL）: 取り込み元 Issue 番号、マージ済み PR 番号、取り込み日（YYYY-MM-DD）、処理（case-close） |
| REQ-0023-014 | archive 先に同名ファイルが既存の場合、上書きしてはならない（MUST NOT） |
| REQ-0023-015 | imported と判定された stub のみを archive 対象としなければならない（SHALL） |
| REQ-0023-016 | 不採用の stub は archive 対象外とし、自動処理してはならない（MUST NOT） |
| REQ-0023-017 | archive 処理は case-close の Step 9c の前に実行しなければならない（SHALL） |
| REQ-0023-018 | archive 処理の結果は Step 9c の `.agentdev` commit に含めなければならない（SHALL） |

### 取り込み記録形式

`## 取り込み記録` セクションの記述形式:

```
## 取り込み記録

| 項目 | 値 |
|------|-----|
| Issue | #N |
| PR | #N |
| 取り込み日 | YYYY-MM-DD |
| 処理 | case-close |
```

 - 不採用 stub の自動処理（ユーザー手動対応）
