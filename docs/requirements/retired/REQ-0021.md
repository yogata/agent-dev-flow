## 目的
<!-- 【必須】 -->

AgentDevFlow のガードレールのうち、機械的に判定できる検査を自然言語記述からスクリプトに移行し、検査の再現性と信頼性を向上させる。あわせて、スキル同梱スクリプトの配置方針、実装言語、共通 CLI 契約を定義する。

現在、integrity-check、gh-cli 書き込み検証、template 検証、skill 構造検証などのガードレールはスキルやコマンド内の自然言語で記述されており、エージェントが読み落とすリスクがある。本要件はこれらをスクリプト化し、決定的な検査を可能にする。

本要件は REQ-0017（AgentDevFlow plugin namespace 統一と learning / intake / integrity の正式化）の子要件であり、REQ-0017-031「詳細検査対象は子issueで整理する」を具体化するものである。

## 要件
<!-- 【必須】 -->

### スクリプト化の境界

| ID | 要件 |
|---|---|
| REQ-0021-001 | スキルとスクリプトの責務を以下に分離するものとする（SHALL）: スキル＝判断基準、例外判断、責務境界、ユーザー確認要否、意味的妥当性判断。スクリプト＝形式検査、存在確認、参照整合性、Markdown 構造検査、差分抽出、証跡収集、機械可読レポート生成 |
| REQ-0021-002 | 自然言語で記述されたガードレールのうち、機械的に判定可能な検査はスクリプトとして実装するものとする（SHALL） |
| REQ-0021-003 | 意味的な妥当性判定、ユーザー承認要否判断、最終分類判断はスクリプト化せずスキルの責務として扱うものとする（SHALL） |

### 実装言語

| ID | 要件 |
|---|---|
| REQ-0021-004 | スクリプトの標準実装言語は TypeScript (bun runtime) とする（SHALL） |
| REQ-0021-005 | Bun runtime により TypeScript ファイルを直接実行するものとする（SHALL） |
| REQ-0021-006 | 既存 Python スクリプトは、安定稼働しており TypeScript 化の効果が薄い場合、直ちに移植しない（MAY） |
| REQ-0021-007 | 新規スクリプトを Shell / PowerShell で実装しないものとする（SHALL） |
| REQ-0021-008 | 新規 Python スクリプトを標準ルートに配置しないものとする（SHALL） |

### 配置方針

| ID | 要件 |
|---|---|
| REQ-0021-009 | TypeScript スクリプトは原則として該当 skill 配下の `scripts/` ディレクトリに配置するものとする（SHALL） |
| REQ-0021-010 | `.opencode/` 直下に `src/`、`tests/`、`tsconfig.json` を新規配置しないものとする（SHALL） |
| REQ-0021-011 | 既存 `.opencode/package.json` を依存管理に利用するものとする（SHALL） |
| REQ-0021-012 | `tsconfig.json` は初期配置せず Bun runtime による直接実行とするものとする（SHOULD）。IDE 型チェック支援が必要になった段階で最小限の `tsconfig.json` を検討する |
| REQ-0021-013 | テストは skill-local に併置し `bun test` で実行するものとする（SHOULD） |
| REQ-0021-014 | 共通ライブラリが必要になった場合のみ、別途 shared scripts 方針を定義する（MAY）。初期実装では skill-local を優先する |

### agentdev-integrity skill

| ID | 要件 |
|---|---|
| REQ-0021-015 | `agentdev-integrity` skill を新設するものとする（SHALL） |
| REQ-0021-016 | `agentdev-integrity` skill は以下を持つものとする（SHALL）: 検査対象定義、検査カテゴリ定義、NG/warning/info 分類基準、レポート schema、`.agentdev/integrity/reports/` への出力規約、intake item 化候補ルール、validator 呼び出し規約、false positive を許容し false negative を減らす検査方針 |
| REQ-0021-017 | `/agentdev/integrity-check` command は薄い入口として維持し、個別検査ロジックは `agentdev-integrity` skill に委譲するものとする（SHALL）。command にはユーザー向け入口、Input/Output、高レベル Steps、read-only guardrail、skill 利用宣言、レポート保存先、完了報告のみを置く |
| REQ-0021-018 | 横断的な整合性検査の orchestration と report schema は `agentdev-integrity` に集約するものとする（SHALL） |
| REQ-0021-019 | 各 domain skill は自分の artifact 仕様を保持してよいが、integrity-check 用 validator scripts は `agentdev-integrity/scripts/` から実行するものとする（SHALL） |
| REQ-0021-020 | 各 skill 配下に integrity scripts を分散配置し `integrity-check` command が順次呼び出す構成は採用しない（SHALL） |

### 共通 CLI 契約

| ID | 要件 |
|---|---|
| REQ-0021-021 | スキル同梱スクリプトは非対話実行とするものとする（SHALL） |
| REQ-0021-022 | スキル同梱スクリプトは `--help`、`--json`、`--dry-run` オプションを持つものとする（SHALL） |
| REQ-0021-023 | スキル同梱スクリプトは入力パスを引数で受け取るものとする（SHALL） |
| REQ-0021-024 | スキル同梱スクリプトは JSON 出力と Markdown report 出力を持つものとする（SHALL） |
| REQ-0021-025 | stdout を機械可読出力に、stderr を診断メッセージに使用するものとする（SHALL） |
| REQ-0021-026 | exit code を 0（問題なし）、1（検査NG）、2（入力不正・実行エラー）に分けるものとする（SHALL） |
| REQ-0021-027 | スキル同梱スクリプトは破壊的変更を行わないものとする（SHALL） |
| REQ-0021-028 | 自動修正機能を追加する場合は検査モードとは別モードに分離するものとする（SHOULD） |

## 適用範囲
<!-- 【必須】 -->

- **対象**:
  - AgentDevFlow ガードレールのうち機械的検査可能な対象のスクリプト化（6 種）
  - スキル同梱スクリプトの配置方針（skill-local scripts/）
  - 実装言語の標準化（TypeScript / bun runtime）
  - 共通 CLI 契約の定義
  - `agentdev-integrity` skill の新設と検査体系の定義
  - 初期実装対象（Phase 1: 4 スクリプト）の要件定義
  - 次段階対象（Phase 2: 2 スクリプト）の概要定義
  - `.agentdev/integrity/reports/` へのレポート出力
  - `.opencode/package.json` を用いた依存管理

- **対象外**:
  - learning / intake artifact の storage 再設計
  - req-define の壁打ち判断そのもの
  - ADR 要否判断
  - 要件の粒度調整
  - スコープ変更要否判断
  - 本筋外発見を intake / learning のどちらに送るかの最終判断
  - `spec-bug` / `impl-bug` / `scope-creep` の最終判断
  - ドキュメント内容の意味的な妥当性判定
  - ユーザー承認が必要かどうかの判断
  - Epic 実行順序 SSoT を別ファイル化すること
  - `.opencode/` 直下を汎用 TypeScript project root にすること
  - 既存 Python スクリプトの TypeScript 移植
  - 自動修正機能の実装
