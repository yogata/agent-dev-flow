## 目的

REQファイルの運用に伴い、興味対象の変化、分散、肥大化、ドリフトが中長期的に発生する。
これらREQ体系上の歪みを、通常の短期intakeとは異なる回収サイクルで管理するため、REQ再構成候補をraw intake itemとして独立した保存先に蓄積できるようにする。
保存先の分離により、短期処理対象と長期蓄積対象の混在を防ぎ、レビュー、棚卸しの見通しを担保する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0050-001 | 通常の raw intake item は `.agentdev/intake/inbox/` 直下に保存すること（SHALL） |
| REQ-0050-002 | REQ再構成候補の raw intake item は `.agentdev/intake/inbox/req-restructure/` に保存すること（SHALL） |
| REQ-0050-003 | REQ再構成intakeは、通常intakeと同じ本文フォーマットを維持し、frontmatter、状態値、重複排除キーを追加してはならないこと（SHALL） |
| REQ-0050-004 | req-save は、REQ保存処理中に同じ興味対象のREQ分散、特定REQの責務肥大、分割候補、統合候補、移動候補、重複、不要化、ドリフトを検知した場合、通常のREQ保存処理を妨げず REQ再構成intake を `.agentdev/intake/inbox/req-restructure/` に保存できること（SHOULD） |
| REQ-0050-005 | case-close は、完了処理中にREQ再構成候補を検知した場合、通常の post-run intake capture と同様に REQ再構成intake を `.agentdev/intake/inbox/req-restructure/` に保存できること（SHOULD） |
| REQ-0050-006 | REQ再構成intakeは、保存時点でREQの分割、統合、移動、廃止を要求するものではなく、将来のREQ再構成レビューの入力材料として扱うこと（SHALL） |
| REQ-0050-007 | REQ再構成intakeには、対象REQ、観測した構造上の問題、根拠となる要件行、セクション、編集文脈、想定される再構成方向を記録すること（SHOULD） |
| REQ-0050-008 | intake-review は `.agentdev/intake/inbox/` 直下の通常intakeと `.agentdev/intake/inbox/req-restructure/` 配下のREQ再構成intakeを区別して一覧、レビューできること（SHOULD） |
| REQ-0050-009 | REQ再構成intakeは、通常の短期Issue化候補ではなく、REQ再構成レビューの入力候補として扱うこと（SHALL） |
| REQ-0050-010 | intake-promote または後続導線は、REQ再構成intakeを通常の `req-backlog` 向け短期作業候補と混同して処理してはならないこと（SHALL） |
| REQ-0050-011 | REQ再構成intakeのレビュー結果は、採用時は `.agentdev/intake/accepted/req-restructure/` に、却下時は `.agentdev/intake/archive/rejected/req-restructure/` に配置すること（SHALL） |
| REQ-0050-012 | intake-review はREQ再構成intakeの採用、却下判定で、通常intakeのIssue化可否とは独立した判定基準（REQ再構成レビューへの入力適性）を適用すること（SHALL） |

## 適用範囲

- **対象**:
 - `.agentdev/intake/inbox/req-restructure/` ディレクトリの追加
 - `.agentdev/intake/accepted/req-restructure/` ディレクトリの追加
 - `.agentdev/intake/archive/rejected/req-restructure/` ディレクトリの追加
 - req-save における REQ再構成候補検知、保存導線
 - case-close の post-run intake capture における REQ再構成候補の保存先分岐
 - intake-review における通常intake / REQ再構成intakeの区別、独立判定
 - intake-promote における REQ再構成intakeの req-backlog 除外
 - capture-boundaries における intake / learning / REQ再構成intake の境界補足
 - agentdev-req-file-manager における REQ再構成候補の検知観点追加（SPLIT / MERGE / MOVE / DUPLICATE / RETIRE / DRIFT）
- **対象外**:
 - raw intake item 本文フォーマットの全面変更
 - frontmatter、status、重複排除キーの導入
 - REQ再構成レビュー自体の詳細手順
 - REQファイルの実際の分割、統合、移動、廃止
 - 通常intakeの既存ライフサイクル全面再設計
 - learning pipeline との統合
 - REQ再構成レビュー専用コマンドの作成
