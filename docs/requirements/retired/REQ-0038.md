## 目的

case-run/case-close の実行プロセスで、外部副作用を伴う操作（GitHub API 呼び出し、git pull）の信頼性不足と、実装とドキュメントの整合性確認の自動化が不十分なため、3件の品質問題が発生した。
本要件は、チェックボックス更新時の API 呼び出し結果確認の必須化、git pull 前のローカル変更事前チェック、および実装完了後の docs/specs/ キーワード grep による矛盾自動確認を追加し、case 実行プロセス全体の信頼性を向上する。

### 背景

以下の3つの品質問題が case-run/case-close の実行信頼性に関連して発生した:

1. **チェックボックス更新未実行**（Issue #787）: case-run Step 7 で `gh issue edit` が実際には実行されず、todo のみ completed にマークされた
2. **git pull 失敗、ローカル変更残存**（PR #788）: case-close Step 8b で `git pull --ff-only` がローカル変更と競合して失敗。事前チェックなし
3. **docs/specs/system.md 旧仕様の矛盾**（Issue #815, REQ-0030）: case-close で検出されたが、case-run 段階での予防仕組みがない

## 要件

### case-run Step 7: チェックボックス更新の信頼性強化

| ID | 要件 |
|---|---|
| REQ-0038-001 | `case-run` Step 7 は、チェックボックス更新時（`[ ]` → `[x]`）に `gh issue edit --body-file` の実行結果を確認してから todo を completed にマークすること（SHALL） |
| REQ-0038-002 | `case-run` Step 7 は、todo の completed マークと `gh issue edit` の API 呼び出しを同一ツール呼び出しブロック内で実行すること（SHALL） |
| REQ-0038-003 | `case-run` Step 7 は、`gh issue edit` の実行が失敗した場合、該当チェックボックスを `[x]` に更新せず、エラーを表示して停止すること（SHALL） |

### case-close Step 8b: git pull 前のローカル変更事前チェック

| ID | 要件 |
|---|---|
| REQ-0038-004 | `case-close` Step 8b は、`git pull --ff-only` 実行前に `git status --porcelain` でローカル変更の有無を確認すること（SHALL） |
| REQ-0038-005 | `case-close` Step 8b は、ローカル変更が検出された場合、当該ファイルを `git checkout --` でリセットしてから pull を実行すること（SHALL） |
| REQ-0038-006 | `case-close` Step 8b のローカル変更リセットは、PR で削除されたファイルに限定して実行すること（SHALL） |

### case-run Step 6: docs/specs/ キーワード grep 自動確認

| ID | 要件 |
|---|---|
| REQ-0038-007 | `case-run` Step 6 は、実装完了後、ローカル実装 diff からキーワードを自動抽出し、`docs/specs/` で grep して矛盾する記述がないか自動確認すること（SHALL） |
| REQ-0038-008 | `case-run` Step 6 の実装キーワード自動抽出は、PR diff ではなく、worktree 内のローカル実装 diff（対象ブランチと `origin/main` の差分、または未コミット差分を含む作業差分）から行うこと（SHALL） |
| REQ-0038-009 | `case-run` Step 6 で docs/specs/ に矛盾する記述が検出された場合、実装計画の関連ドキュメント更新確認の対象に含めること（SHALL） |
| REQ-0038-010 | `case-run` は、Step 11b 以降で PR diff が取得可能な場合、ローカル diff 由来の抽出結果と矛盾しないかを確認してよい（MAY） |
| REQ-0038-011 | `case-run` は、PR diff で新規キーワードまたは未計画のスキーマ変更が見つかった場合、提出処理を進めず work plan / 関連ドキュメント確認へ戻すこと（SHALL） |

### 既存フォーマット維持

| ID | 要件 |
|---|---|
| REQ-0038-012 | 本要件の変更は、既存の完了報告フォーマット（`agentdev-workflow-reporting`）およびエラー処理フォーマットを維持すること（SHALL） |
