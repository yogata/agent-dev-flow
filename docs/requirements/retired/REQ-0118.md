## 目的

AgentDevFlow command/skill では、source path と runtime path の混同が繰り返し発生する問題に対し、subagent が worktree 内でファイルを編集する際の安全制約が command-authoring / skill-authoring に含まれ、パス・参照誤用を予防する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0118-001 | `src/opencode/skills/agentdev-command-authoring/SKILL.md` は subagent edit safety 制約を含むこと。制約内容は以下を含むこと: (1) worktree 内制約: subagent は worktree root 外のファイルを編集しない、(2) パスプレフィクス確認: 編集前に worktree root からの相対パスであることを検証する、(3) ファイル存在確認: 編集対象ファイルの存在を事前に確認する。source path（`src/opencode/...`）と runtime path（`.opencode/...`）の混同防止が明示されていること |
| REQ-0118-002 | `src/opencode/skills/agentdev-skill-authoring/SKILL.md` は subagent edit safety 制約を含むこと。REQ-0118-001 と同一の3制約（worktree 内制約・パスプレフィクス確認・ファイル存在確認）を含むこと |

## 適用範囲

- **対象**: `agentdev-command-authoring` SKILL.md、`agentdev-skill-authoring` SKILL.md
- **対象外**: workflow-orchestration/references/subagent-protocol.md への影響（整合性確認は推奨）、他 skill への影響、agentdev-req-analysis への影響
