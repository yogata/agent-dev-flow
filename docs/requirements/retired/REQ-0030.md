## 目的

AgentDevFlow のコマンド・スキル・テンプレートに対する体系的なテストスイートを実装し、system.md のテスト要件（SHALL）との乖離を解消する。

現在、スクリプト単体テストは5ファイル全てに存在するが、コマンド .md、スキル SKILL.md、テンプレート .md の構造検証テストと、コマンド実行フローのE2Eテストが存在しない。
これにより、リファクタリング時のデグレード検出が困難であり、回帰テストによる品質担保が機能していない。

本要件は、Phase B（構造検証テスト）と Phase C（E2Eテスト）の2段階でテストスイートを構築する。
Phase A（スクリプト単体テスト）は既存5スクリプト全てにテストが存在するため、本要件の実装対象外とする（Epic 本文に前提として記載）。

## 要件

### Phase B: 構造検証テスト

| ID | 要件 |
|---|---|
| REQ-0030-001 | 全コマンド定義ファイル（.opencode/commands/agentdev/*.md、README.md を除く）に対して、frontmatter 必須フィールドの存在確認テストを実装する（SHALL） |
| REQ-0030-002 | 全コマンド定義ファイルに対して、Steps セクション構造の確認と 確認テストを実装する（SHALL） |
| REQ-0030-003 | 全スキル定義ファイル（.opencode/skills/*/SKILL.md）に対して、frontmatter 必須フィールド（name, description）の存在確認テストを実装する（SHALL） |
| REQ-0030-004 | 全スキル定義ファイルに対して、USE FOR / DO NOT USE FOR セクションの存在確認と See Also 参照先の存在確認テストを実装する（SHALL） |
| REQ-0030-005 | 全テンプレートファイル（.opencode/skills/*/templates/*.md）に対して、frontmatter 必須項目と `<!-- 【必須】 -->` 付きセクションの存在確認テストを実装する（SHALL） |
| REQ-0030-006 | 既存の integrity validator（check_integrity / lint_skills / check_templates）で既にカバーされている検査項目と重複しないよう、ギャップ分析に基づいて新規テストを追加する（SHALL） |
| REQ-0030-007 | 構造検証テストは `bun test` で実行し、全て PASS すること（SHALL） |
| REQ-0030-008 | 構造検証テストのスクリプトは対応する scripts/ ディレクトリに `*.test.ts` として配置すること（SHALL） |

### Phase C: E2Eテスト

| ID | 要件 |
|---|---|
| REQ-0030-009 | 全14コマンドに対して、コマンド実行の正常系E2Eテストを実装する（SHALL） |
| REQ-0030-010 | E2Eテストはテストフィクスチャ（入力ファイル、ディレクトリ状態）を用意し、コマンド実行後の成果物（生成ファイル、更新ファイル等）の存在と内容の妥当性を検証すること（SHALL） |
| REQ-0030-011 | E2Eテストはコマンド実行時の異常系（必須入力不足、存在しないファイル参照、前提条件違反等）についても検証すること（SHALL） |
| REQ-0030-012 | E2Eテストは `bun test` で実行し、全て PASS すること（SHALL） |
| REQ-0030-013 | E2Eテストスクリプトは system.md のテスト配置規約（同ディレクトリ `*.test.ts`）に従うこと（SHALL） |

### 共通要件

| ID | 要件 |
|---|---|
| REQ-0030-014 | テストファイルは配布対象外とする（SHALL） |
| REQ-0030-015 | `.opencode/tests/` ディレクトリを新設せず、既存の配置規約に従うこと（SHALL） |

## 適用範囲

- **対象**:
 - 14コマンド定義ファイル（.opencode/commands/agentdev/*.md）の構造検証
 - 20スキル定義ファイル（.opencode/skills/*/SKILL.md）の構造検証
 - 16テンプレートファイル（.opencode/skills/*/templates/*.md）の構造検証
 - 14コマンドのE2Eテスト（正常系、異常系）
 - テストフィクスチャの作成
 - 既存 integrity validator と重複しないギャップ分析

- **対象外**:
 - 既存5スクリプトテストの品質レビュー、拡充（Phase A: 完了済み前提）
 - CI（GitHub Actions 等）へのテスト組み込み
 - テストカバレッジの測定、報告
 - 配布時のテスト除外処理（system.md で既に規定済み）
 - AI エージェントの動作検証（コマンド定義に従って AI が正しく動作することの検証）
 - スキルの意味的妥当性検証（構造的完全性の検証に留める）
