## 目的

AgentDevFlow の整合性維持には、用語・表記・構造・参照レベルの機械的チェックと、docs 全体の意味ベースレビューの両方が必要である。`/repo/docs-check` は repo-local な機械検査を担い、`/agentdev/inspect-docs` は SPEC 本文を含む `docs/` 全体の意味整合レビューを担う。docs-* command suite は、局所予防・機械チェック・意味レビュー・是正ルーティングを一貫した workflow として構成され、命名・責務境界が一貫している。

## 適用範囲

- **対象**: `/repo/docs-check` コマンド（`/repo/integrity-check` の後継）の要件、`/agentdev/inspect-docs` コマンドの要件、`/agentdev/req-restructure-review` の `/agentdev/inspect-docs` への統合、req-* / case-* workflow（req-define, req-save, case-run, case-update, case-close）の補強要件、是正ルーティング（docs-check → inspect-docs → req/case/RU → docs-check rule 追加 feedback loop）、gate-levels.md の finding route における `/agentdev/inspect-docs` の包含、vocabulary-registry.md と検出ルールの同期、現行不整合パターンの fixture / check rule seed 扱い
- **対象外**: 現行不整合の個別修正そのもの、`.sisyphus/` の全体監査対象化、semantic review による自動修正・自動 RU 作成・自動 Issue 作成、`/repo/semantic-integrity-review` の新設、`/agentdev/req-restructure-review` の独立 command としての存続、互換 shim を前提とした段階的導入
