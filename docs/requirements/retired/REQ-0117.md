## 目的

Windows + junction 環境で `git worktree remove` が "Not a directory" エラーで失敗する場合、agentdev-git-worktree skill は junction 特有のエラーパターンとフォールバックを含み、クロスプラットフォームでの削除信頼性を担保する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0117-001 | `src/opencode/skills/agentdev-git-worktree/references/worktree-operations.md` の削除セクションは、Windows + junction 環境での `git worktree remove` 失敗パターン（"Not a directory" エラー）の発生条件とフォールバックを含むこと。フォールバックは以下を含むこと: (1) `git worktree prune` の実行、(2) junction ディレクトリの手動削除、(3) ブランチ削除。Windows 固有の事象であること |

## 適用範囲

- **対象**: `src/opencode/skills/agentdev-git-worktree/references/worktree-operations.md` の削除セクション
- **対象外**: worktree 作成、install-consumer-opencode.ps1 の junction 設定ロジック、REQ-0110（削除リトライ）、他 skill/reference への影響
