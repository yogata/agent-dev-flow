## 目的

AgentDevFlowの文書体系（REQ/ADR/SPEC/Guide/Report/DOC-MAP/Retired）の分類ルールは明確であり、文書の追加、更新時の同期不足、世代交代の不明確さ、分類の曖昧さに起因する不整合を防ぐ。
分類ポリシーはSPEC（document-model.md）に集約され、各REQ/ADRは当該SPECを参照する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0116-001 | document-model.md SPEC は、REQ/ADR/SPEC/Guide/Report/DOC-MAP/Retired の各分類の定義、責務、格納場所、権限モデルを含む Document Classification Policy セクションを持つこと |
| REQ-0116-002 | Document Classification Policy は、Document Authority Model（各分類の編集権限、承認フロー）を含むこと |
| REQ-0116-003 | Document Classification Policy は、Classification Decision Tree（文書追加時の分類判定フロー）を含むこと |
| REQ-0116-004 | Document Classification Policy は、Cross-document Projection Rules（原本から配置先への投影ルール）を含み、原本（`src/opencode/`）と配置先（`.opencode/`）の用語が含まれること |
| REQ-0116-005 | Document Classification Policy は、Reference Rules（文書間参照の書式とリンク規則）を含むこと |
| REQ-0116-006 | Document Classification Policy は、Lifecycle Rules（作成→アクティブ→改訂→廃止の状態遷移）を含むこと |
| REQ-0116-007 | Document Classification Policy は、Re-baseline Rules（全面改訂時の世代交代ルール、上書きではないこと）を含むこと |
| REQ-0116-008 | Document Classification Policy は、Synchronization Rules（APPEND/UPDATE操作時の同期要件）を含み、APPEND/UPDATEが正しい操作であり、同期不足が問題であること |
| REQ-0116-009 | Report は正式な文書分類であり、SPECから分離されていること |
| REQ-0116-010 | ADRは「将来の設計、運用、文書システムを制約する決定」を対象とすること |
| REQ-0116-011 | SPECにおける 必達要件 の使用は「現在有効な契約記述」に限定されること |
| REQ-0116-012 | Document Classification Policy は、Anti-patterns（分類ミスの典型的パターンとその検出方法）を含むこと |
| REQ-0116-013 | Document Classification Policy は、Review/Check Mapping（分類ポリシーと docs-check/inspect-docs の対応関係）を含むこと |
| REQ-0116-014 | 廃止ADR（retired ADR）を現在の一次参照源として参照する場合、当該参照はアクティブな後継ADR/REQを指すこと。歴史的参照としては元のADR番号を保持できること |
| REQ-0116-015 | 原本（`src/opencode/`）と配置先（`.opencode/`）の用語が文書体系 SPEC に含まれ、cross-document projection における方向性が明確であること |
| REQ-0116-016 | SPEC における 必達要件 limitation は、規範語制限ではなく「SPEC に新規要件を置かない」「現在仕様、契約記述に限定する」責務境界であること |
| REQ-0116-017 | guide の規範表現検出は、語彙ベースではなく「guide が要件本文または契約本文を保持していないか」の検査であること |

## 適用範囲

- **対象**: document-model.md SPEC の分類ポリシーセクション、REQ-0116（本要件）、REQ-0101/0112 の分類関連事項、ADR-0103 spec の分類拡張
- **対象外**: 既存文書の実際の修正（Case 2 で対応）、command/skill/docs-check の変更（Case 3 で対応）、template の変更
