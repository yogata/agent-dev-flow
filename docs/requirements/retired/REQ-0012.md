## 目的

AgentDevFlow の自然言語成果物における文章品質の最低基準を定義する。
AI-slop（一見整っているが実務上の判断材料にならない文章）を検出、排除し、読者が判断、実行、検証できる文章を保証するための要件である。
本要件は `agentdev-no-ai-slop-writing` Skill が実装する品質基準の規範（normative source）である。

## 要件

| ID | 要件 |
|---|---|
| REQ-0012-001 | README、docs/guides/specs、REQ/ADR本文、command/skill descriptions、Issue/PR本文、完了報告、設計説明、intake/learning中間成果物の自然言語成果物は、AI-slopを含まないこと（SHALL） |
| REQ-0012-002 | AI-slopを以下の10基準のいずれかに該当する文章と定義すること（SHALL）: (1) 主語がない — 誰が何をするか不明、(2) 結論がない — 読み手が次に何をすべきか分からない、(3) 根拠と判断が分離されていない — なぜその判断に至ったか追跡不能、(4) 抽象的で具体的な操作に落とせない — 実行可能な手順に変換できない、(5) 既存の実装、ドキュメント、要件との関連がない — 文脈から浮いている、(6) 読者の次動作がない — 読後に何も起きない、(7) 根拠のない推測がある — 事実と推論が混在、(8) 比喩や雰囲気の水増しがある — 実質なしで文字数を稼ぐ、(9) ユーザーの同意を前提としていない確認がある — 形式的な確認で実質的な合意を回避、(10) もっともらしい一般論で具体的事実を置き換えている — 固有の文脈を一般化で回避 |
| REQ-0012-003 | 自然言語出力を以下の順序で構成すること（SHALL）: (1) 結論、(2) 判断理由、(3) 根拠、(4) 具体的な対応、(5) 不明点または残リスク。この順序を崩すのは読者にとってその方が明確になる場合のみとする（MAY） |
| REQ-0012-004 | 自然言語出力の前に以下の11ルールを適用してレビューすること（SHALL）: (1) 結論を先頭に書く、(2) 主語を明示する、(3) 判断と根拠を分離する、(4) 抽象語を条件、操作、判定基準、成果物、責務、失敗条件、参照先に置換する、(5) 根拠なき推測で不明点を埋めない、(6) 推論は「推論」と明記する、(7) 不明点は「不明」と明記する、(8) 同じ内容の言い換えを削除する、(9) 水増し、無条件の同調、不要な結びを削除する、(10) 外部事実、仕様、数値、引用、最新情報には根拠を付ける、(11) 読者が承認、却下、修正指示、実装、レビュー、Issue化、後続調査のいずれかを実行できる状態にする |
| REQ-0012-005 | 検出、置換ルールセットに以下の9カテゴリを含むこと（SHALL）: (1) 結論を遅らせる表現、(2) 抽象的な評価語、(3) 曖昧な行動表現、(4) 主語がない表現、(5) 根拠なし推測、(6) 無条件の同調、(7) 比喩、雰囲気語、(8) 水増しの締め文、(9) 「できるようにする」の濫用 |
| REQ-0012-006 | 各検出カテゴリが Severity（High/Medium/Low）、Detection pattern、Why it is AI-slop、Rewrite rule、Before、After、Allowed exceptions の構造を持つこと（SHALL） |
| REQ-0012-007 | 検出、置換ルールの対象言語を以下の優先度とすること（SHALL）: 主対象（SHALL）は日本語のAI-slop表現、補助対象（SHOULD）は英語AI-slop表現。英語対象は frontmatter description、Conventional Commits、固有名詞、技術仕様名、引用、外部テンプレートに限定すること |
| REQ-0012-008 | 以下の場合を検出対象外とすること（SHALL）: (1) ユーザー発言の引用、(2) 既存文書の引用、(3) 禁止表現そのものの批評、説明、(4) 不確実性が実際に存在し根拠、条件、残リスクを明示している場合、(5) 法務、契約、監査上の断定回避が必要な場合 |
| REQ-0012-009 | 本要件の実装を `agentdev-no-ai-slop-writing` Skill が行うこと（SHALL）。Skill は本要件を規範的ソースとして実装し、整合性を保つこと（SHALL） |

## 適用範囲

- **対象**: AI-slopの定義、検出基準（10基準）、出力構造原則（5順序）、Pre-output reviewルール（11ルール）、検出カテゴリ（9カテゴリ）とその構造定義、対象言語の優先度、適用除外条件
- **対象外**: REQ/ADR番号付与、APPEND/UPDATE/CREATE判定、ADR必要性判定、command手順設計、Issue/PR CRUD、コード実装、テスト実行、カジュアルな文章、広告、詩
