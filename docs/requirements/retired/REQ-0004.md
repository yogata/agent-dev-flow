> **履歴として残す理由**: area-based移行方針撤回の判断経緯、views 廃止から DOC-MAP への移行履歴

## 目的

要件（REQ）ファイルとアーキテクチャ決定記録（ADR）の管理規則を定義する。REQは `REQ-{NNNN}.md` を永続基準とする。ADRは `ADR-{NNNN}.md` を判断履歴の基準とし、本文を area / topic / view 別ファイルへ再配置しない。両者の整合性を維持するための規約である。area-based current document 構造への移行方針は撤回し、per-file 構造を永続標準とする。

> **注**: `views/*.md`（観点別体系化ビュー）は REQ-0035 / ADR-0008 により廃止された。文書探索の入口は `docs/DOC-MAP.md` に、現在仕様の基準は `docs/specs/*.md` に移行した。
