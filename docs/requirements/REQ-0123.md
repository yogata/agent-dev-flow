## 目的

`agentdev-workflow-lifecycle` は宣言的定義のみを提供するスキルであり（REQ-0103-095）、コマンド固有手続きは目的別スキル（`agentdev-issue-management`, `agentdev-intake-pipeline`, `agentdev-backlog-integration`, `agentdev-req-structure-diagnostics`）が分担して保持する。
SKILL.md の「宣言的定義のみ。
手順、手続きは含まない」宣言と実体は一致し、5コマンド（case-open, intake-from-github, intake-promote, backlog-review, inspect-docs）の DO NOT USE FOR 衝突参照は存在しない。

本来件は REQ-0103-095（workflow-lifecycle 責務限定）が定める状態を具体化する。
REQ-0103-095 は何を除外するか（WHAT）を定義済みであり、本要件はどのスキルが保持するか（WHERE）を定義する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0123-001 | `agentdev-issue-management`（操作用 Skill）は GitHub Issue 作成、更新、リンク、確認の安全手順、検証手順に限定すること。case-open 固有の Epic / 子Issue フロー、RU削除タイミング、Issue本文構造は case-open command 本文の責務とする（REQ-0103-105 準拠） |
| REQ-0123-002 | `agentdev-intake-pipeline`（知識ベース Skill）は GitHub intake 抽出ロジック（期間解釈、データ取得、構造検出、LLM解析、item生成）と promote 時のレビュー、分類、整形、振り分け基準を提供すること |
| REQ-0123-003 | `agentdev-backlog-integration`（知識ベース Skill）は採用済み成果物の統合、分割判定、矛盾検出、RU生成ルール、depends_on 依存解決基準を提供すること |
| REQ-0123-004 | `agentdev-req-structure-diagnostics`（知識ベース Skill）は REQ参照ID整合性確認、第一参照導線確認、現行、廃止境界確認、6観点診断（SPLIT/MERGE/MOVE/DUPLICATE/RETIRE/DRIFT）、未処理artifact確認、問題候補出力スキーマを提供すること |
| REQ-0123-012 | `workflow-lifecycle/references/upstream-handoff.md` は `agentdev-workflow-lifecycle/references/` に配置されること。上位への引き継ぎはライフサイクル判定の一部として workflow-lifecycle スキルに属すること |
| REQ-0123-014 | 5コマンド（case-open, intake-from-github, intake-promote, backlog-review, inspect-docs）の本文内の `agentdev-workflow-lifecycle` への参照は、対応する新スキル名であること。参照は skill 名レベルにとどめること（REQ-0103-090, REQ-0103-093 準拠） |
| REQ-0123-016 | `agentdev-workflow-lifecycle` の DO NOT USE FOR は REQ-0103-095 の除外リストであること。実体と宣言は一致すること |
| REQ-0123-017 | reference 内容に REQ-0113（Skill References SPEC分離基準）に該当する SPEC 相当記述が含まれる場合、当該 SPEC 相当記述は `docs/specs/` に配置されること。実行時コマンド/skill は自己完結すること（REQ-0113-002 準拠） |
| REQ-0123-019 | `agentdev-workflow-lifecycle` の See Also は新スキルとの関連を記述し、補助導線に限定すること（REQ-0103-096 準拠） |
