## 目的

Command / Skill / Template / Script の責務境界を明確にし、重複記述・責務混線・参照不整合を防止する。

## 適用範囲

- **対象**: command / skill / template / script の責務、配置、命名、namespace、implementation pattern 分類、pattern 別責務境界と skill 要件の参照整合性、Command 薄型化基準（行数、Steps 数、deprecated shim）、load_skills prompt budget（即時必要制限、pattern 別目安上限、常時 load 禁止）、Skill 入口カード化（SKILL.md 構成、progressive disclosure、references/ canonical）、共通処理の中央集約（Git 同期等の共通化、集約先優先順位）、command-local template の配置・責務（`.opencode/commands/agentdev/templates/{command}/{variant}.md`）、完了報告 template の SPEC 管轄
- **対象外**: 個別commandの詳細手順、テストケース本文、`.sisyphus/` 運用、個別 implementation pattern の詳細定義（lifecycle reference 側に集約）、個別 command の具体的な縮小目標行数、個別 SKILL.md の具体的な分割方法、共通処理の具体的な配置先ファイルパス、completion report 以外の command-local template 種別
