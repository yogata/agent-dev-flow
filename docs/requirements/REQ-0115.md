## 目的

AgentDevFlow の整合性維持には、用語・表記・構造・参照レベルの機械的チェックと、docs 全体の意味ベースレビューの両方が必要である。現行体系では `/repo/integrity-check` が repo-local な機械検査を担い、`/agentdev/req-restructure-review` が REQ 体系再構成を担っているが、SPEC 本文を含む `docs/` 全体の意味整合レビューが独立して存在しない。また docs 整合性に関する command suite としての命名・責務境界も一貫していない。

本要件は、同種の不整合を今後発生させないため、`docs-*` command suite として局所予防・機械チェック・意味レビュー・是正ルーティングを一貫した workflow として定義する。

## 適用範囲

- **対象**: `/repo/docs-check` コマンド（`/repo/integrity-check` から改名）の要件定義、`/agentdev/docs-review` コマンド（新設）の要件定義、`/agentdev/req-restructure-review` から `/agentdev/docs-review` への統合方針、req-* / case-* workflow（req-define, req-save, case-run, case-update, case-close）の補強要件、是正ルーティング（docs-check → docs-review → req/case/RU → docs-check rule 追加 feedback loop）、gate-levels.md への finding route 追加、vocabulary-registry.md との検出ルール同期、現行不整合パターンの fixture / check rule seed 化
- **対象外**: 現行不整合の個別修正そのもの、`.sisyphus/` の全体監査対象化、semantic review による自動修正・自動 RU 作成・自動 Issue 作成、`/repo/semantic-integrity-review` の新設、`/agentdev/req-restructure-review` の独立 command 維持、互換 shim を前提とした段階移行
