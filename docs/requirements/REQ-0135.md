## 目的

agent-dev-flow 管理ドラフトの配置先・draft type registry・コマンド間受け渡しに関する要件

## 要件

| ID | 要件 |
|---|---|
| REQ-0103-126 | agent-dev-flow が生成・保存・再読込・コマンド間受け渡しするドラフトファイルは `.agentdev/drafts/` 配下に配置すること |
| REQ-0103-127 | ドラフト配置対象には、req-define から req-save へ渡す要件ドラフト、req-save 前の保存対象ドラフト、agent-dev-flow コマンドが後続コマンドへ渡す中間ドラフトを含めること |
| REQ-0103-128 | agent-dev-flow は `.sisyphus/` 配下をドラフトの生成先・保存先・入力元・コマンド間受け渡し先・ライフサイクル管理対象・削除対象・canonical・draft source として扱わないこと |
| REQ-0103-129 | `.agentdev/drafts/` は、draft type registry に登録された draft type の中間成果物のみを配置する場所とすること |
| REQ-0103-131 | 各 command は、入力 draft の draft_type が自 command の allowed input に該当するかを確認すること。producer/consumer/next を個別 file frontmatter から読んで整合性判定する必要はないこと |
| REQ-0103-132 | 標準 draft type は `req_draft` の1種のみとすること。`requirements-review-finding` および旧 `skill_review_finding` に相当する Skill/Command 参照妥当性の検出結果は inspect lifecycle（`.agentdev/inspect/inbox/`）へ出力すること |
| REQ-0103-133 | req_draft は req-define が生成し、req-save と case-open が消費する、保存前の要件ドラフトとする（file pattern: `.agentdev/drafts/req-draft-{topic}.md`） |
| REQ-0103-134 | inspect-skills の検出結果は inspect lifecycle（`.agentdev/inspect/inbox/inspect-skills-finding-{topic}.md`）へ出力する中間成果物とする。旧 `skill_review_finding` draft type は存在せず、inspect lifecycle に統合されている |
| REQ-0103-138 | req-define は `.agentdev/inspect/inbox/inspect-skills-finding-*.md` を明示入力として受け付け、未確認事項や採否未確定事項は要件本文から除外すること |
| REQ-0103-139 | inspect-skills の許可される side effect は `.agentdev/inspect/inbox/inspect-skills-finding-*.md` の生成および `.agentdev/inspect/` 配下の git 永続化（commit / push）のみとすること（canonical docs 変更・REQ/ADR/SPEC 変更・Command/Skill/Template/Script 変更・RU 保存・Issue 作成・PR 作成・branch / worktree 操作は対象外） |

## 適用範囲

- **対象**: .agentdev/drafts/ 配置ルール・draft type registry・.sisyphus/ 除外
- **対象外**: Command/Skill responsibility boundary、source/projection・sync・repo type・consumer install、inspect domain state と inspect command lifecycle
