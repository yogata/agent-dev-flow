## 目的

git worktree 削除は、ファイルハンドル解放遅延に起因する Permission denied エラーに対し、リトライ機構、事前 cleanup、prune フォールバック、tracked files 復元による信頼性を担保する。Windows + ジャンクション環境で発生する "Not a directory" エラーに対するジャンクション特有のフォールバック手順を含み、クロスプラットフォームでの削除信頼性を担保する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0110-002 | リトライ後も失敗する場合、警告を表示して停止すること |
| REQ-0110-003 | リトライの記述はOS非依存の汎用記述であること |
| REQ-0110-004 | worktree 削除リトライ前に、untracked files のみを対象とする cleanup を実行すること |
| REQ-0110-007 | untracked-only cleanup は tracked file を保護し、cleanup 対象を untracked-only に限定すること |
| REQ-0110-008 | Windows+junction 環境の worktree で `.opencode/` が空の場合（junction 未伝播 + `.gitignore` で `.opencode/agentdev-*` が追跡対象外）、source/projection は手動両辺編集を行い同期スクリプトに依存しないこと。driver subagent 引き継ぎプロンプトにこの制約を必須項目として明記すること。詳細手順は SKILL（agentdev-workflow-orchestration）に配置すること |

## 適用範囲

- **対象**: `agentdev-git-worktree` スキルにおける worktree 削除の信頼性（リトライ、cleanup、prune フォールバック、tracked files 復元、Windows + ジャンクション環境の "Not a directory" エラーフォールバック）
- **対象外**: worktree 作成、ブランチ操作、install-consumer-opencode.ps1 のジャンクション設定ロジック、他 skill/reference への影響
