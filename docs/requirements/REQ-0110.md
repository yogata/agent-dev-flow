## 目的

git worktree 削除時のファイルハンドル解放遅延に起因する Permission denied エラーへの対策として、汎用的なリトライ機構を定義する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0110-001 | `git worktree remove` が Permission denied で失敗した場合、短い待機を挟んで最大2回リトライすること（SHALL） |
| REQ-0110-002 | リトライ後も失敗する場合、警告を表示して停止すること（SHALL） |
| REQ-0110-003 | リトライ手順はOS非依存の汎用記述とすること（SHALL） |

## 適用範囲

- **対象**: `agentdev-git-worktree` スキルの worktree 削除手順
- **対象外**: worktree 作成手順、ブランチ操作、OS固有の診断機能
