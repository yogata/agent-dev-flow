## 目的

git worktree 削除は、ファイルハンドル解放遅延に起因する Permission denied エラーに対し、リトライ機構、事前 cleanup、prune フォールバック、tracked files 復元による信頼性を担保する。Windows + junction 環境で発生する "Not a directory" エラーに対する junction 特有のフォールバック手順を含み、クロスプラットフォームでの削除信頼性を担保する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0110-001 | `git worktree remove` が Permission denied で失敗した場合、短い待機を挟んで最大3回リトライすること |
| REQ-0110-002 | リトライ後も失敗する場合、警告を表示して停止すること |
| REQ-0110-003 | リトライの記述はOS非依存の汎用記述であること |
| REQ-0110-004 | worktree 削除リトライ前に、untracked files のみを対象とする cleanup を実行すること |
| REQ-0110-005 | リトライ全失敗後、`git worktree prune` によるフォールバッククリーンアップを試行すること |
| REQ-0110-006 | worktree 削除失敗からの復旧時、当該 worktree 内で `git checkout .` により tracked files の復元を試行すること |
| REQ-0110-007 | untracked-only cleanup の実行にあたり、tracked file を削除してはならないこと |
| REQ-0110-008 | worktree 削除が Windows + junction 環境で "Not a directory" エラーを返す場合、(1) `git worktree prune` の実行、(2) junction ディレクトリの手動削除、(3) ブランチ削除 のフォールバック手順を含むこと。当該事象は Windows 固有であること |

## 適用範囲

- **対象**: `agentdev-git-worktree` スキルにおける worktree 削除の信頼性（リトライ、cleanup、prune フォールバック、tracked files 復元、Windows + junction 環境の "Not a directory" エラーフォールバック）
- **対象外**: worktree 作成、ブランチ操作、install-consumer-opencode.ps1 の junction 設定ロジック、他 skill/reference への影響
