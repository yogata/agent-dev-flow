## 目的

git worktree 削除時のファイルハンドル解放遅延に起因する Permission denied エラーへの対策として、リトライ機構、事前 cleanup、prune フォールバック、tracked files 復元を含む包括的な削除信頼性向上を定義する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0110-001 | `git worktree remove` が Permission denied で失敗した場合、短い待機を挟んで最大3回リトライすること |
| REQ-0110-002 | リトライ後も失敗する場合、警告を表示して停止すること |
| REQ-0110-003 | リトライ手順はOS非依存の汎用記述とすること |
| REQ-0110-004 | worktree 削除リトライ前に、untracked files のみを対象とする cleanup を実行すること |
| REQ-0110-005 | リトライ全失敗後、`git worktree prune` によるフォールバッククリーンアップを試行すること |
| REQ-0110-006 | worktree 削除失敗からの復旧時、当該 worktree 内で `git checkout .` により tracked files の復元を試行すること |
| REQ-0110-007 | untracked-only cleanup の実行にあたり、tracked file を削除してはならないこと |

## 適用範囲

- **対象**: `agentdev-git-worktree` スキルの worktree 削除手順（リトライ、cleanup、prune フォールバック、tracked files 復元）
- **対象外**: worktree 作成手順、ブランチ操作、OS固有の診断機能
