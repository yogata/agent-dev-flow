## 目的

AgentDevFlow の command/skill が GitHub CLI（gh）コマンドを直接記述している問題を解消し、agentdev-gh-cli skill の手続き委譲へ集約する責務境界を確立する。
GitHub I/O 操作を一箇所に集約し、command は workflow に、domain skill は意味処理に集中できる構成にする。
併せて、ローカル版が agentdev-gh-cli を差し替え可能にするための抽象化基盤を提供する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0149-001 | GitHub Issue/PR 操作を行う command/skill は、gh コマンドを直接記述せず、agentdev-gh-cli の手続きへ委譲すること |
| REQ-0149-002 | agentdev-gh-cli は、Issue 作成、Issue 本文読込、Issue 本文更新、Issue コメント追加、PR 本文読込、PR merge、Issue close、VERIFY に相当する手続きを提供すること（詳細な手続き名、引数、戻り値は SPEC agentdev-gh-cli.md） |
| REQ-0149-003 | 標準版の agentdev-gh-cli は、各手続きの既定実装として gh コマンドを実行すること |
| REQ-0149-004 | agentdev-gh-cli は I/O 手続きと VERIFY を担当し、本文生成、完了判定、Epic 依存判定、capture 分類を担当しないこと |
| REQ-0149-005 | agentdev-gh-cli は薄いルーティング入口とし、操作契約と標準版手順を references に分離すること（詳細は SPEC agentdev-gh-cli.md） |
| REQ-0149-006 | command は workflow 順序、停止条件、完了判定を担当し、GitHub CLI の具体的フラグ、一時ファイル、エンコーディング対策、再読込 VERIFY を直接記述しないこと |
| REQ-0149-007 | domain skill は本文生成、完了条件評価、Epic 表解析、capture 分類等の意味処理を担当し、GitHub CLI の具体操作を直接記述しないこと |
| REQ-0149-008 | GitHub 版の既存 workflow は、agentdev-gh-cli 委譲後も従来と同等の Issue/PR 操作結果を維持すること |

## 適用範囲

- **対象**: agentdev-gh-cli の責務境界、手続き委譲規律、command/skill の I/O 責務分離、差し替え可能性の基盤
- **対象外**: agentdev-gh-cli の具体的な手続き名一覧、引数、戻り値、VERIFY 観点の詳細（SPEC agentdev-gh-cli.md）。ローカル版 agentdev-gh-cli の実装（REQ-0150）。link mode の導入方式（REQ-0134）。REQ-0103-078 が定義する load 関係の宣言（REQ-0103 に維持）
