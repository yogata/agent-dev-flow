## 目的

AgentDevFlow の要件・判断・仕様・参照入口を、読み手が迷わない基準構造として維持する。active REQ は現在満たすべき要件を表し、retired REQ は履歴参照に限定する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0101-001 | active REQ は `docs/requirements/REQ-{NNNN}.md` 直下に配置すること（SHALL） |
| REQ-0101-002 | retired REQ は `docs/requirements/retired/REQ-{NNNN}.md` に配置し、現行要件判断に使用しないこと（SHALL） |
| REQ-0101-003 | REQ ID は4桁ゼロ埋めの安定IDとし、active / retired を問わず再利用しないこと（SHALL） |
| REQ-0101-004 | REQ の要件行IDは `REQ-{NNNN}-{MMM}` 形式で一意に識別すること（SHALL） |
| REQ-0101-005 | REQ frontmatter は `id`、`title`、`created`、`updated`、`tags` を含めること（SHALL） |
| REQ-0101-006 | active REQ は関心対象の総体を表し、追加要件は原則として既存 active REQ への APPEND / UPDATE で扱うこと（SHALL） |
| REQ-0101-007 | 新規 active REQ を作成する場合、既存 active REQ に吸収できない独立した関心対象である理由を記録すること（SHALL） |
| REQ-0101-008 | ADR は取り返しのつかない技術判断の記録として `docs/adr/ADR-{NNNN}.md` に配置すること（SHALL） |
| REQ-0101-009 | SPEC は現在仕様の基準として `docs/specs/*.md` に配置し、REQ/ADR の判断内容を代替しないこと（SHALL） |
| REQ-0101-010 | `docs/DOC-MAP.md` は文書探索入口であり、REQ/ADR/SPEC の内容を代替しないこと（SHALL） |
| REQ-0101-011 | 文書間に矛盾がある場合、active REQ を優先すること（SHALL） |
| REQ-0101-012 | `docs/requirements/README.md` は active REQ の第一参照一覧、retired REQ の履歴入口、移行表への導線を提供すること（SHALL） |
| REQ-0101-013 | 旧REQから新REQへの移行有無は `docs/requirements/mapping-table.md` に記録すること（SHALL） |
| REQ-0101-014 | guides は利用者向け読み物であり、REQ/ADR/SPEC/DOC-MAP の内容を代替しないこと（SHALL） |
| REQ-0101-015 | `.sisyphus/` 配下の成果物は現行REQ体系の管理対象に含めないこと（SHALL） |

## 適用範囲

- **対象**: REQ/retired REQ/ADR/SPEC/DOC-MAP/guides の基準境界、REQ移行表、ID規約
- **対象外**: 各コマンドの実行手順、コード実装、`.sisyphus/` 運用
