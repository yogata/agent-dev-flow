## 目的

AgentDevFlow の要件・判断・仕様・参照入口を、読み手が迷わない基準構造として維持する。active REQ は現在満たすべき要件を表し、retired REQ は履歴参照に限定する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0101-001 | active REQ は `docs/requirements/REQ-{NNNN}.md` 直下に配置すること（SHALL） |
| REQ-0101-002 | retired REQ は `docs/requirements/retired/REQ-{NNNN}.md` に配置し、現行要件判断に使用しないこと（SHALL） |
| REQ-0101-003 | REQ ID は4桁ゼロ埋めの安定IDとし、active / retired を問わず再利用しないこと（SHALL） |
| REQ-0101-004 | REQ の要件行IDは `REQ-{NNNN}-{MMM}` 形式で一意に識別すること（SHALL） |
| REQ-0101-005 | REQ frontmatter は `id`、`title`、`created`、`updated`、`tags` を含めること（SHALL） |
| REQ-0101-006 | active REQ は関心対象の総体を表し、追加要件は原則として既存 active REQ への APPEND / UPDATE で扱うこと（SHALL） |
| REQ-0101-007 | 新規 active REQ を作成する場合、既存 active REQ に吸収できない独立した関心対象である理由を記録すること（SHALL） |
| REQ-0101-008 | ADR は将来の設計・運用・文書システムを制約する決定の記録として `docs/adr/ADR-{NNNN}.md` に配置すること（SHALL） |
| REQ-0101-009 | SPEC は現在仕様の基準として `docs/specs/*.md` に配置し、REQ/ADR の判断内容を代替しないこと（SHALL） |
| REQ-0101-010 | `docs/DOC-MAP.md` は文書探索入口であり、REQ/ADR/SPEC の内容を代替しないこと（SHALL） |
| REQ-0101-011 | 文書間に矛盾がある場合、active REQ を優先すること（SHALL） |
| REQ-0101-012 | `docs/requirements/README.md` は active REQ の第一参照一覧、retired REQ の履歴入口、移行表への導線を提供すること（SHALL） |
| REQ-0101-013 | 旧REQから新REQへの移行有無は `docs/requirements/mapping-table.md` に記録すること（SHALL） |
| REQ-0101-014 | guides は人間向けナビゲーション層であり、REQ/ADR/SPEC/DOC-MAP の内容を代替しないこと（SHALL）。docs/guides/ は README.md（入口）と 10 ガイド（quickstart, command-selection, req-case-flow, intake-learning-backlog-flow, diagnostics-and-maintenance, artifacts-and-state, project-docs-and-specs, consumer-project-setup, troubleshooting, glossary）で構成する |
| REQ-0101-015 | `.sisyphus/` 配下の成果物は現行REQ体系の管理対象に含めないこと（SHALL） |
| REQ-0101-016 | retired REQ の参照は current primary reference（現行要件判断に使われる参照）と historical reference（移行表・履歴説明・置換関係の参照）を区別して扱うこと（SHALL）。`mapping-table.md` 内の migration 用 retired REQ 参照は historical reference とし、現行要件判断に使用されている導線の検査対象外とすること |
| REQ-0101-017 | 利用者が現在の入力状態から次に実行すべきコマンドを選べる入口表を README またはコマンド選択ガイドに設けること（SHALL） |
| REQ-0101-018 | コマンド系統の説明において「本流」「支流」「合流」のような比喩的上下関係を構造名として使用しないこと（SHALL） |
| REQ-0101-019 | ~~workflow-overview は概念説明に先立ち、前工程・生成物・次工程の関係を生成物中心で示すこと（SHALL）~~ （historical: workflow-overview は REQ-0101-025 により削除されたため、本要件は実質的に artifacts-and-state ガイド等に再配分済み） |
| REQ-0101-020 | root README は identity、入口表、参照先リンク、最小限のクイックスタートに限定し、guide や command README と重複する概念説明やフロー説明を含めないこと（SHALL） |
| REQ-0101-021 | 対象文書から、現行要件のように読める旧 REQ 参照、非推奨 command を現行 producer とする説明、canonical ではない path 参照を除去または現行化すること（SHALL） |
| REQ-0101-022 | `.agentdev/README.md` は domain state の早見表とし、各 path について状態、producer、consumer / next command、retention / removal condition を記載すること（SHALL） |
| REQ-0101-023 | `.agentdev/README.md` は `.agentdev/` を永続 domain state、`.sisyphus/` を runtime workspace と明記し、working draft が canonical domain state ではないことを示すこと（SHALL）。ADR status 正規化形式（`status` + `superseded_by`）の記載基準を含める |
| REQ-0101-024 | 旧 `artifact-model.md` の内容は新ガイド（artifacts-and-state 等）に再配分し、docs/guides/ は README.md（ガイド入口）と 10 ガイドファイルで構成する人間向けナビゲーション層とすること（SHALL） |
| REQ-0101-025 | 旧 `domain-state-lifecycle.md` の内容は新ガイド（artifacts-and-state, diagnostics-and-maintenance 等）に再配分し、旧 3 ガイド（workflow-overview, artifact-model, domain-state-lifecycle）は削除すること（SHALL） |
| REQ-0101-026 | `.opencode/commands/agentdev/README.md` は概念説明の重複を避け、各コマンドの command、primary input、primary output、next command / next action を一覧化するリファレンスとして構成すること（SHALL） |
| REQ-0101-027 | ガイドは規範文書ではなく人間向けナビゲーション層であり、ガイド内に MUST/SHALL 規範表現、load_skills / implementation_pattern / Pattern 詳細、REQ番号注記を含めないこと（SHALL） |
| REQ-0101-028 | ADR status 形式は `status: superseded` + frontmatter に `superseded_by: ADR-XXXX` とし、旧形式 `superseded-by:[ADR-XXXX]` を使用しないこと（SHALL） |
| REQ-0101-029 | docs 永続文書の Requirement Source セクションに RU-ID（RU-XXXX 等）を記載しないこと（SHALL）。RU は一時成果物であり docs の永続根拠に不適である |
| REQ-0101-030 | `.sisyphus/` を `.gitignore` に追加し、runtime workspace として追跡対象外とすること（SHALL） |

| REQ-0101-032 | 参照パス不存在（10件）を修正すること（SHALL）。各 reference の行方を確認し、新規作成または参照元を修正する。対象: case-close.md の references/git-common-procedures.md, references/capture-boundaries.md、case-run.md の capture-boundaries.md、docs-check.md の git-common-procedures.md、req-define.md の templates/doc_requirement.md (FPR-007) |

| REQ-0101-037 | `docs/specs/workflow-contracts.md` および `docs/specs/document-model.md` に workflow status 禁止ルールを追記すること（SHALL）。`docs/guides/artifacts-and-state.md` の既存記載と整合させる (FPR-034) |

## 適用範囲

- **対象**: REQ/retired REQ/ADR/SPEC/DOC-MAP/guides の基準境界、REQ移行表、ID規約、README / command-selection / command README / artifact-model / domain-state-lifecycle / DOC-MAP の責務境界と内容基準、`.agentdev/README.md` の早見表形式、旧 REQ 参照・obsolete path 参照の除去
- **対象外**: 各コマンドの実行手順、コード実装、`.sisyphus/` 運用、各ドキュメントの具体的な本文内容、新規ドキュメントの追加
