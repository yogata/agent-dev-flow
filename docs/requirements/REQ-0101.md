## 目的

AgentDevFlow の要件、判断、仕様、参照入口を、読み手が迷わない基準構造として維持する。
現行REQは現在満たすべき要件を表し、廃止REQは履歴参照に限定する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0101-001 | 現行REQは `docs/requirements/REQ-{NNNN}.md` 直下に配置すること |
| REQ-0101-002 | 廃止REQは `docs/requirements/retired/REQ-{NNNN}.md` に配置し、履歴参照用途のみに使用すること |
| REQ-0101-003 | REQ ID は4桁ゼロ埋めの安定IDとし、現行、廃止を問わず一意性を維持すること |
| REQ-0101-004 | REQ の要件行IDは `REQ-{NNNN}-{MMM}` 形式で一意に識別すること |
| REQ-0101-005 | REQ frontmatter は `id`、`title`、`created`、`updated` を含めること |
| REQ-0101-006 | 現行REQは関心対象の総体を表し、追加要件は原則として既存REQへの APPEND / UPDATE で扱うこと |
| REQ-0101-007 | 新規REQを作成する場合、既存REQに吸収できない独立した関心対象である理由を記録すること |
| REQ-0101-008 | ADR は将来の設計、運用、文書システムを制約する決定の記録として `docs/adr/ADR-{NNNN}.md` に配置すること |

| REQ-0101-010 | `docs/DOC-MAP.md` は文書探索入口であり、REQ/ADR/SPEC への導線を提供すること |
| REQ-0101-011 | 文書間に矛盾がある場合、現行REQを優先すること |

| REQ-0101-017 | 利用者が現在の入力状態から次に実行すべきコマンドを選べる入口表を README またはコマンド選択ガイドに設けること |

| REQ-0101-020 | root README は identity、入口表、参照先リンク、最小限のクイックスタートに限定し、概念説明やフロー説明は guide や command README へ委譲すること |

| REQ-0101-022 | `.agentdev/README.md` はドメイン状態の早見表とし、各パスについて状態、生成元、利用元 / 次に実行するコマンド、保持、削除条件を記載すること |

| REQ-0101-028 | ADR status 形式は `status: superseded` + frontmatter に `superseded_by: ADR-XXXX` とし、旧形式 `superseded-by:[ADR-XXXX]` を使用しないこと |

## 適用範囲

- **対象**: REQ/廃止REQ/ADR/SPEC/DOC-MAP/guides の基準境界、REQ移行表、ID規約、README / command-selection / command README / artifact-model / domain-state-lifecycle / DOC-MAP の責務境界と内容基準、`.agentdev/README.md` の早見表形式、現行文書における旧REQ参照、旧パス参照の扱い、ADR記述対象境界（意思決定のみ、作業手段除外）、REQ/ADR/SPEC の主たる文意の肯定文記述原則、境界条件としての否定文の許容範囲
- **対象外**: 各コマンドの実行手順、コード実装、`.sisyphus/` 運用、各ドキュメントの具体的な本文内容、新規ドキュメントの追加
