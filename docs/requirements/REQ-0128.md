## 目的

再発防止、抽象化、反映価値のある知見を `.agentdev/learning/` に蓄積し、8軸（8-axis）評価と HITL（人の判断を挟む）確定を経て採用済み成果物として要件化パイプラインに供給する。
learning パイプラインのドメイン状態管理と promote 判定を定義する。

## 要件

| ID | 要件 |
|---|---|
| REQ-0128-001 | learning パイプラインは専用のドメイン状態を持つこと。ドメイン状態は `inbox.md`（未整理エントリ）, `deferred.md`（評価済み・再評価候補の living pool）, `evaluation-report.md`（8軸評価レポート）, `promoted/`（採用済み成果物）で構成されること |
| REQ-0128-002 | learning は再発防止、抽象化、反映価値のある知見を蓄積すること |
| REQ-0128-004 | `learning-promote` の内部フェーズ（normalize → classify → 8-axis eval → evaluation-report）は独立したコマンドではなく、`learning-promote` 内の処理ステップであること |
| REQ-0128-005 | evaluation-report は `learning-promote` 内部の評価根拠として生成される中間成果物であること |
| REQ-0128-006 | `learning-promote` は HITL 確定ステップを持ち、promote / prune の最終確定は HITL 確定ステップを経て行うこと（AI 単独での確定は対象外） |
| REQ-0128-007 | `learning-promote` の完了報告は 8軸評価サマリ、判定結果（promote/defer/reject/duplicate）、後続経路を提示すること |
| REQ-0128-008 | `learning-promote` において staged / rejected / duplicate のいずれかの判定が確定した `deferred.md` entry は、判定確定時に `deferred.md` から prune すること。詳細な HITL 境界、自動実行条件は REQ-0147-006/007 に従うこと |
| REQ-0128-009 | `deferred.md` は deferred / 未処理 / 再評価対象の entry を保持する living pool として維持すること。staged / rejected / duplicate の prune 後も deferred / 未処理 / 再評価対象の entry を残置すること |

## 適用範囲

- **対象**: learning-promote、`.agentdev/learning/` ドメイン状態、learning inbox.md、evaluation-report.md、8軸評価、HITL 確定、promote/defer/reject/duplicate 判定
- **対象外**: intake パイプライン（REQ-0127）、backlog-review の RU 生成（REQ-0129）、RU ライフサイクル（REQ-0105）、learning の 8軸評価ディメンションの変更、evaluation-report スキーマの変更（配置先変更のみ）、個別 learning 項目の内容、case-run / case-close からの learning capture 回収（REQ-0130 / 0131）
