## 目的

配布物（src/opencode/commands/, src/opencode/skills/）が .opencode/skills/ 配下のスキルを参照する場合、新規 clone 環境でスキル不在に陥る。.opencode/skills/ 配下は .gitignore で除外されており、配布物の自己完結性（self-contained）が崩れる。配布物が依存するスキルを src/opencode/skills/ に昇格（配布物化）する方針と、repo-local 専用スキルとの境界を確立し、docs-check で未トラックスキル参照を機械検出して新規 clone 環境でのスキル不在を恒久的に防ぐ。

## 要件

| ID | 要件 |
|---|---|
| REQ-0159-001 | 配布物（src/opencode/commands/, src/opencode/skills/）が参照する .opencode/skills/ 配下のスキルは、src/opencode/skills/ に昇格（配布物化）すること。昇格の基準（配布物が依存するか否か）と昇格手順は文書化すること |
| REQ-0159-002 | repo-local 専用スキル（配布物が依存しないもの）は .opencode/skills/ のまま管理し、.gitignore による除外を継続すること。配布物と repo-local の境界は明確に文書化すること |
| REQ-0159-003 | docs-check（check_integrity.ts）は、配布物が参照する未トラックスキル（.opencode/skills/ 配下で git 管理対象外）を検出すること。検出された場合は src 昇格を促すメッセージを出力すること |
| REQ-0159-004 | 配布物化されたスキル（src/opencode/skills/ 配下）の SKILL.md frontmatter description は USE FOR / DO NOT USE FOR trigger 記述を必ず持つこと。lint_skills.ts が trigger 記述欠落を WARNING として検出する対象を、配布スキル全般に適用する。repo-local スキル（.opencode/skills/ 配下）は本要件の対象外とする |

## 適用範囲

- **対象**: 配布物（src/opencode/commands/, src/opencode/skills/）が参照するスキルの配置方針、配布物と repo-local スキルの境界、docs-check による未トラックスキル参照の機械検出
- **対象外**: 個別スキルの昇格作業そのもの（実装修復作業として実施）、.gitignore ホワイトリスト方式（src 昇格による一元管理を採用し不採用）
