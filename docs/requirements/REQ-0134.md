## 目的

AgentDevFlow の配布基盤（source/projection 分離・同期スクリプト・repo type 定義・consumer install モデル）に関する要件

## 要件

| ID | 要件 |
|---|---|
| REQ-0103-048 | AgentDevFlow本体のcommand/skill/template sourceのcanonical locationは`src/opencode/`であること |
| REQ-0103-049 | `.opencode/`はruntime projectionであること。本体repoではself-hosting projectionとして機能する |
| REQ-0103-050 | source→projection同期スクリプト（sync script）が存在すること |
| REQ-0103-051 | sync scriptはdry-run / check / applyの3モードを持つこと。dry-runは変更を予測のみ。checkはdivergenceを検出。applyは同期を実行 |
| REQ-0103-052 | sync scriptはsourceとprojectionのdivergenceを検出・報告できること |
| REQ-0103-053 | 初回導入用 migration script が one-time 実行可能であること |
| REQ-0103-054 | migration script は導入完了後に削除可能であること |
| REQ-0103-055 | stale symlink / broken symlink / projection target不整合を検出できること |
| REQ-0103-061 | 4種のrepo type（self-hosting / consumer-with-agentdev / consumer-local / plugin-future）が定義されていること |
| REQ-0103-062 | `.opencode/`の意味がrepo typeにより異なること。self-hostingではprojection、consumerではproject-local customization入口 |
| REQ-0103-063 | AgentDevFlow予約名との衝突を防ぐproject-local naming rulesが定義されていること |
| REQ-0103-064 | symlink-based installationを推奨すること。plugin/npm/package化は将来optionとして扱い、copy-based installationを含めて現在の要件の対象外とすること |
| REQ-0103-065 | sync scriptの適用範囲がrepo typeごとに定義されていること |
| REQ-0103-072 | Consumer repository では agent-dev-flow 本体を `.agentdev-plugin/` に clone することを推奨する。`.agentdev/` に clone してはならない |
| REQ-0103-073 | agent-dev-flow repo 自身は `.agentdev/` を domain state として使用すること（`.agentdev-plugin/` は consumer repository 専用である） |
| REQ-0103-074 | sync script（REQ-0103-050）を self-hosting projection 用（`sync-self-opencode.ps1`）と consumer install 用（`install-consumer-opencode.ps1`, `check-consumer-opencode.ps1`）に分離し、`scripts/` 配下に配置すること |
| REQ-0103-075 | sync script は `scripts/` 配下に配置すること。root 直下への配置は対象外とし、`sync-opencode.ps1` も `scripts/` 配下に配置される |
| REQ-0103-076 | consumer install の対象は AgentDevFlow の public runtime artifacts（`.opencode/commands/agentdev/`、`.opencode/skills/agentdev-*/`）に限定すること |
| REQ-0103-077 | consumer repository の推奨 `.gitignore` において `.agentdev-plugin/` と `.sisyphus/` と `.opencode/commands/agentdev/` と `.opencode/skills/agentdev-*/` を含めること。`.agentdev/` を git 管理対象に含めること |

## 適用範囲

- **対象**: source/projection layout・sync/migration script・repo type・consumer install
- **対象外**: Command/Skill responsibility boundary、drafts配置・draft type registry、inspect domain state と inspect command lifecycle
