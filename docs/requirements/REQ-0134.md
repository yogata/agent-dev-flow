## 目的

AgentDevFlow の配布基盤（原本/配置先分離、同期スクリプト、リポジトリ種別定義、導入先インストールモデル）に関する要件

## 要件

| ID | 要件 |
|---|---|
| REQ-0103-048 | AgentDevFlow本体の command/skill/template の原本の正規の配置場所は `src/opencode/` であること |
| REQ-0103-049 | `.opencode/` は実行時の配置先であること。本体リポジトリでは配置先として機能する |
| REQ-0103-050 | 原本→配置先の同期スクリプト（sync script）が存在すること |
| REQ-0103-051 | sync script は `dry-run` / `check` / `apply` の3モードを持つこと。`dry-run` は変更を予測のみ、`check` は乖離を検出、`apply` は同期を実行する |
| REQ-0103-052 | sync script は原本と配置先の乖離を検出、報告できること |
| REQ-0103-053 | 初回導入用の移行 script が一回限り実行可能であること |
| REQ-0103-054 | 移行 script は導入完了後に削除可能であること |
| REQ-0103-055 | 失効シンボリックリンク、リンク切れシンボリックリンク、配置先リンク先不整合を検出できること |
| REQ-0103-061 | 5種のリポジトリ種別（`self-hosting` / `consumer-with-agentdev` / `consumer-local` / `consumer-generated` / `plugin-future`）が定義されていること。`consumer-generated` はローカル版 OpenCode を導入する利用側リポジトリ（詳細は REQ-0141 参照） |
| REQ-0103-062 | `.opencode/` の意味がリポジトリ種別により異なること。`self-hosting`（本体リポジトリ）では配置先、導入先リポジトリではプロジェクト固有カスタマイズの入口 |
| REQ-0103-063 | AgentDevFlow予約名との衝突を防ぐプロジェクト固有の命名規則が定義されていること |
| REQ-0103-064 | シンボリックリンクベースのインストールを推奨すること。plugin/npm/package 化は将来の選択肢として扱い、コピーベースのインストールを含めて現在の要件の対象外とすること |
| REQ-0103-065 | sync script の適用範囲がリポジトリ種別ごとに定義されていること |
| REQ-0103-072 | 導入先リポジトリでは agent-dev-flow 本体を `.agentdev-plugin/` に clone することを推奨する。`.agentdev/` に clone してはならない |
| REQ-0103-073 | agent-dev-flow リポジトリ自身は `.agentdev/` をドメイン状態として使用すること（`.agentdev-plugin/` は導入先リポジトリ専用である） |
| REQ-0103-074 | sync script（REQ-0103-050）を本体リポジトリの配置先用（`sync-self-opencode.ps1`）と導入先インストール用（`install-consumer-opencode.ps1`, `check-consumer-opencode.ps1`）に分離し、`scripts/` 配下に配置すること |
| REQ-0103-075 | sync script は `scripts/` 配下に配置すること。root 直下への配置は対象外とし、`sync-opencode.ps1` も `scripts/` 配下に配置される |
| REQ-0103-076 | 導入先インストールの対象は AgentDevFlow の公開実行時成果物（`.opencode/commands/agentdev/`、`.opencode/skills/agentdev-*/`）に限定すること |
| REQ-0103-077 | 導入先リポジトリの推奨 `.gitignore` で `.agentdev-plugin/` と `.opencode/commands/agentdev/` と `.opencode/skills/agentdev-*/` を含めること。`.agentdev/` を git 管理対象に含めること |
| REQ-0103-155 | `src/opencode-local/` は local 版 `agentdev-gh-cli` の原本領域であり、`src/opencode/` とは独立に管理されること（ADR-0131, REQ-0141, REQ-0150 参照）。同期スクリプト（REQ-0103-050）の管理対象外とする |
| REQ-0103-156 | 通常版とローカル版は、`.opencode/` 配下を仕様管理リポジトリへ接続する link mode を利用すること（ADR-0131 参照） |
| REQ-0103-157 | 通常版では、`.opencode/commands/agentdev/` および `.opencode/skills/agentdev-*/` を `src/opencode/` 配下へ接続すること |
| REQ-0103-158 | ローカル版では、`agentdev-gh-cli` 以外の command/skill を `src/opencode/` 配下へ接続し、`.opencode/skills/agentdev-gh-cli/` だけを `src/opencode-local/agentdev-gh-cli/` へ接続すること（REQ-0150 参照） |
| REQ-0103-159 | `runtime-overrides/` を設けないこと |
| REQ-0103-160 | `.opencode/` 実パス確認は、link mode を一律停止するのではなく、意図した link target かどうかを確認する方式とすること（ADR-0131 参照） |

## 適用範囲

- **対象**: 原本/配置先レイアウト、同期/移行 script、リポジトリ種別、導入先インストール、link mode による接続方式
- **対象外**: Command/Skill 責務境界、drafts配置、draft type registry、inspect ドメイン状態と inspect コマンドライフサイクル。link mode の接続手順の技術詳細（SPEC runtime-package-boundary.md）
