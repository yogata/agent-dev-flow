## 目的

`/agentdev/inspect-docs` および `/agentdev/inspect-skills` が検出した inspect finding は、採用・保留・却下の分類を経て初めて後続工程に渡される。`/agentdev/inspect-promote` は inspect finding を promote / defer / reject に分類し、採用した finding を promoted artifact として昇格させることで、検出と要件化の間に人間の判断を挟む。分類と昇格はユーザーの明示的な承認に基づき、自動的な promoted artifact 生成を行わない。

## 要件

| ID | 要件 |
|---|---|
| REQ-0126-001 | `/agentdev/inspect-promote` は public agentdev command として存在し、`.agentdev/inspect/inbox/*.md` の inspect finding を入力とすること |
| REQ-0126-002 | 各 finding を以下の分類基準で評価し、promote / defer / reject を判定すること: 明確な不整合 → promote（RU 化対象）、不整合の有無・採否・範囲・優先度・正とする情報源が未確定 → defer（intake 送付候補）、誤検知・対応不要 → reject、具体的修正対象を持たない再発防止知見 → defer（learning 送付候補） |
| REQ-0126-003 | promoted artifact はユーザーの明示的な承認を得た後に生成すること。分類結果を提示し、承認を得てから処理すること |
| REQ-0126-004 | promote された finding は `.agentdev/inspect/promoted/` へ保存し、元の inbox file を削除すること |
| REQ-0126-005 | reject された finding は `.agentdev/inspect/archive/rejected/` へ移動すること |
| REQ-0126-006 | defer された finding は `.agentdev/inspect/inbox/` に残置し、intake / learning 送付の推奨を報告すること |
| REQ-0126-007 | docs-check rule / fixture 追加候補は promoted artifact の要件化方向または受け入れ条件に含めること |
| REQ-0126-008 | inbox が空の場合は「対象なし」と報告して終了すること |
| REQ-0126-009 | `inspect-promote` は `.agentdev/inspect/` 配下の変更（promoted / archive/rejected への移動・inbox 削除を含む）を git commit / push すること。変更なし時は commit/push せず完了報告で「変更なし」と報告すること |

## 適用範囲

- **対象**: `/agentdev/inspect-promote` コマンドの分類・昇格（promote / defer / reject 分類、HITL 承認、promoted artifact 生成、reject / defer 処理）
- **対象外**: inspect-docs（REQ-0109）、inspect-skills（REQ-0125）、docs-check（REQ-0108）、promoted artifact の RU 化（backlog-review の担当）、個別 finding の検出実行
