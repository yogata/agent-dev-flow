## 目的

`/agentdev/inspect-docs` および `/agentdev/inspect-skills` が検出した inspect 検出事項は、採用、保留、却下の分類を経て初めて後続工程に渡される。
`/agentdev/inspect-promote` は inspect 検出事項を promote / defer / reject に分類し、採用した検出事項を採用済み成果物として昇格させることで、検出と要件化の間に人間の判断を挟む。
分類と昇格はユーザーの明示的な承認に基づき、自動的な採用済み成果物生成を行わない。

## 要件

| ID | 要件 |
|---|---|
| REQ-0126-001 | `/agentdev/inspect-promote` は public agentdev command として存在し、`.agentdev/inspect/inbox/*.md` の inspect 検出事項を入力とすること |
| REQ-0126-002 | 各検出事項を以下の分類基準で評価し、promote / defer / reject を判定すること: 明確な不整合 → promote（RU 化対象）、不整合の有無、採否、範囲、優先度、正とする情報源が未確定 → defer（intake 送付候補）、誤検知、対応不要 → reject、具体的修正対象を持たない再発防止知見 → defer（learning 送付候補） |
| REQ-0126-003 | 採用済み成果物はユーザーの明示的な承認を得た後に生成すること。分類結果を提示し、承認を得てから処理すること |
| REQ-0126-004 | promote された検出事項は `.agentdev/inspect/promoted/` へ保存し、元の inbox ファイルを削除すること |
| REQ-0126-005 | reject された検出事項は削除すること |
| REQ-0126-006 | defer された検出事項は `.agentdev/inspect/inbox/` に残置し、intake / learning 送付の推奨を報告すること |
| REQ-0126-007 | docs-check rule / 検査データ追加候補は採用済み成果物の要件化方向または受け入れ条件に含めること |
| REQ-0126-008 | inspect-promote コマンドは inbox が空の場合「対象なし」と報告して終了すること |
| REQ-0126-009 | `inspect-promote` は `.agentdev/inspect/` 配下の変更（promoted/ への保存、inbox 削除を含む）を git commit / push すること。変更なし時は commit/push せず完了報告で「変更なし」と報告すること |
| REQ-0126-010 | `inspect-promote` は明示的なオプトイン（`--auto`）により、自動 promote 対象カテゴリ（機械的に特定可能で移行先 SPEC が一意に定まる高確信度検出事項。詳細は `docs/specs/foundations/workflow-contracts.md` 参照）に合致する検出事項を HITL（人の判断を挟む）承認を経ずに `.agentdev/intake/promoted/` へ投入し、intake/backlog パイプラインへ流入させること。`--auto` 未指定時は自動 promote を行わないこと。自動 promote 実行ログを記録し、誤検知取り消し手順を整備すること（REQ-0136-016） |

## 適用範囲

- **対象**: `/agentdev/inspect-promote` コマンドの分類、昇格（promote / defer / reject 分類、HITL 承認、採用済み成果物生成、reject / defer 処理、`--auto` オプトインによる高確信度検出事項の自動 promote、intake/promoted/ 投入、実行ログ、誤検知取り消し）
- **対象外**: inspect-docs（REQ-0109）、inspect-skills（REQ-0125）、docs-check（REQ-0108）、採用済み成果物の RU 化（backlog-review の担当）、個別検出事項の検出実行
