## 目的

case-run の実行信頼性、作業用 worktree での実装、PR 作成、実装充足・乖離ゲート（QG-3）を定義する。case-run の capture 責務境界と、case-close への capture 情報引き継ぎ手段を含む。

## 要件

| ID | 要件 |
|---|---|
| REQ-0130-001 | `case-run` は準備・実装・提出の再開可能なフェーズを提供すること |
| REQ-0130-002 | `case-run` は作業用 worktree で実装し、PRを作成すること |
| REQ-0130-003 | `case-run` はチェックボックス更新など外部書き込みの成功を確認してから完了扱いにすること |
| REQ-0130-004 | `case-run` は変更影響候補を実装計画に組み込むこと |
| REQ-0130-006 | `case-run` は本筋外の検出事項をPR本文の `## Findings / Capture候補` セクションに `### intake` と `### learning` 小見出しで分類して記録すること |
| REQ-0130-007 | case-run の QG-3 が PR 作成直前の実装充足・乖離ゲートに限定されていること。品質メトリクス収集・docs全体grep・Document Classification Policy全体確認・case-update連携が QG-3 に含まれないこと |
| REQ-0130-008 | `case-run` における `.agentdev/intake/inbox/` および `.agentdev/learning/inbox.md` の扱いは読み取り専用とすること |
| REQ-0130-009 | `case-run` から `case-close` への capture 情報の引き継ぎは PR 本文のみを経由すること（一時会話コンテキスト・ローカル変数・中間ファイル経由の引き継ぎは対象外） |
| REQ-0130-010 | `case-run` は単一 Issue または単一 Wave（Epic Issue 指定時: 現在 ready な Wave の子Issue を並列実行）を処理し、他Issue の実装履歴や Epic 全体の実装過程を前提としないこと。Epic 全体（複数 Wave）の処理は行わない。Wave 境界（PR マージ必要）は case-close の責務であり、case-run は1 Wave の実行（PR作成まで）で return すること |
| REQ-0130-011 | `case-run` は1 Issue 単位で実装作業をサブエージェントへ委譲できること |
| REQ-0130-012 | 子Issue の完了・失敗・停止結果は親コンテキストではなく永続状態（Issue / PR / `.agentdev/`）に記録されること |
| REQ-0130-013 | case-run は実装時に判明した SPEC レベル詳細を PR 本文の SPEC確定候補セクションに記録すること |
| REQ-0130-015 | 実行担当サブエージェントは .agentdev/intake/・.agentdev/learning/ を直接変更せず PR 本文に記録すること |
| REQ-0130-016 | case-run は OpenCode プラグインとして導入された oh-my-openagent のエージェント型（Sisyphus-Junior）を task() 経由で利用すること。CLI subprocess（bunx oh-my-openagent run）は使用しない |
| REQ-0130-017 | case-run は Issue 実装を task(subagent_type="Sisyphus-Junior", load_skills=["ulw-loop"]) で委譲すること。委譲プロンプトは ulw-loop スキルを使用すること（"/ulw-loop Implement Issue #N: <Issue本文>"）。CLI subprocess（bunx oh-my-openagent run）は廃止する |
| REQ-0130-018 | Sisyphus-Junior(ulw-loop) に渡す「要件」は Issue 本文であり、実行計画方向性の参考情報が含まれ得ること |
| REQ-0130-019 | Sisyphus-Junior(ulw-loop) は ulw-loop による目標分解・evidence-backed 実行・品質ゲートを含む全工程を担当すること |
| REQ-0130-020 | 【廃止】デフォルトタイムアウト1時間規定は廃止。CLI subprocess 廃止に伴い、Sisyphus-Junior の各ツール呼び出しごとの120秒 timeout が適用される |
| REQ-0130-021 | 【廃止】PR URL 不在時の gh pr list --head <branch> フォールバック規定は廃止。Sisyphus-Junior が直接 PR 作成を行うため、PR URL は task() result として返却される |
| REQ-0130-022 | 【廃止】ハーネス不在時の case-run エラー停止規定は廃止（CLI subprocess 廃止に伴う）。oh-my-openagent は OpenCode プラグインとして導入済みであり、エージェント型（Sisyphus-Junior）が利用不可の場合は task() 起動失敗として検知される |
| REQ-0130-023 | case-run は Wave 実行時、各子Issue の worktree 作成前に `git fetch origin` を実行し origin/main の鮮度を確認すること。詳細手順は SKILL（agentdev-git-worktree）に配置すること |
| REQ-0130-024 | 【廃止】bunx モジュール解決エラー時の npx フォールバック規定は廃止。CLI subprocess 廃止に伴い bunx/npx フォールバックは不要 |
| REQ-0130-025 | case-run は Sisyphus-Junior の task() 起動失敗・異常終了時は即 failed とせず「実装完了・検証未完了」として扱い、worktree の `git status` で未コミット変更を確認し、残留箇所の grep と手動修正を行うこと。詳細手順は SKILL（agentdev-case-run-execution-adapter）に配置すること |

## 適用範囲

- **対象**: case-run、作業用 worktree 実装、PR 作成、QG-3（PR作成直前ゲート）、本筋外検出事項の PR 本文記録、capture 責務境界（inbox 非変更・PR本文経由引き継ぎ）、チェックボックス外部書き込み確認、変更影響候補の実装計画組み込み、実装差分（diff）の SPEC 矛盾確認、単一 Issue/Wave スコープ実行・サブエージェント委譲・永続状態ベース結果記録、OpenCode プラグイン型エージェント（Sisyphus-Junior）の task() 経由利用・ulw-loop 委譲プロンプト・Epic Wave 並列実行（最大5件）
- **対象外**: case-close（REQ-0131）、case-open（REQ-0132）、Epic/Wave 並列実行オーケストレーション（REQ-0106）、ワークフロー基本契約・work_type分類（REQ-0104）、intake / learning パイプライン（REQ-0127 / 0128）、REQ再構成の詳細手順
