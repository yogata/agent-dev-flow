## 目的

複数の文書に同じ説明が異なる粒度で存在する状態を解消し、各文書の責務と適切な粒度を定義する。

現在、root README / system.md / command-map.md / artifact-boundaries.md / commands README の各ファイルに、他のファイルと重複する説明が異なる粒度で存在する。また、一部のファイルには本来その文書の責務を超える詳細な手順や仕様が含まれている。

本要件は、11件の確認された記述問題を修正し、8文書の責務判断基準を確立する。ファイル配置の変更は行わない。

### 文書責務の判断基準

各文書が持つべき粒度を以下のとおり定義する。本要件の修正はすべてこの判断基準に従うものとする。

| 文書 | 責務 | 粒度 |
|------|------|------|
| root README | 入口。最小限のフロー、リンク、最小限のディレクトリ構造 | 最小 |
| docs README | ハブ。インデックスリンクのみ。REQ/ADR の全量記載なし | リンクのみ |
| domain-state-lifecycle.md | .agentdev/ 状態遷移の SSoT | SSoT |
| command-map.md | コマンド接続、I/O SSoT、次ステップ。手順は含めない | SSoT |
| system.md | 現在仕様の要約 + 詳細 reference へのリンク。手順・エラーメッセージは含めない | 要約 |
| 各 command | Input / Output / 高レベル Steps / command 固有 Guardrails | 定義 |
| 各 skill | 再利用可能な判断基準、schema、protocol、lifecycle | 定義 |
| capture-boundaries.md | intake / learning の責任境界のみ | SSoT |

### Post-Run Capture 帰属マッピング

system.md から削除する Post-Run Capture 内容の正しい帰属先を以下のとおり定義する。

| 内容 | 帰属先 |
|------|--------|
| Post-Run Capture 全体の分岐基準 | capture-boundaries.md |
| case-run 中の本筋外発見の扱い | case-run.md は概要、詳細は capture-boundaries.md 参照 |
| case-close での learning item 保存 | case-close.md + agentdev-learning-capture |
| case-close での intake item 保存 | case-close.md |
| intake item 形式・保存先 | intake 系 command / domain-state-lifecycle.md / capture-boundaries.md |
| system.md | Post-Run Capture の概要と参照リンクのみ |
