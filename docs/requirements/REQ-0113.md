## 目的

skill / skill references 内の SPEC 相当記述を SPEC（`docs/specs/`）に分離し、runtime 配布物と repo-internal な現在仕様の責務境界を明確にする（REQ-0103-035, 036, 045, 047）。runtime command / skill が `docs/specs/` を直接の実行時依存として参照しない境界を維持する。skill sourceは`src/opencode/skills/`に配置し、`.opencode/skills/`はruntime projectionである（REQ-0103-008）。

## 要件

| ID | 要件 |
|---|---|
| REQ-0113-001 | skill references に含まれる SPEC 相当記述は `docs/specs/` へ移管すること。SPEC 相当とは、現在仕様・状態モデル・artifact lifecycle・workflow contract・command topology・schema・frontmatter contract・finding category / route / report schema・文書種別間の責務境界・repo-internal な設計説明を含む |
| REQ-0113-002 | SPEC へ移管後、runtime command / skill は移管内容を直接の実行時依存として参照してはならない。runtime 実行時に必要な最小情報は command または skill references に自己完結させること |
| REQ-0113-003 | skill に残置してよい記述は以下に限定すること: runtime 実行時に参照する短い判断表、USE FOR / DO NOT USE FOR、command 実行時のチェックリスト、操作時の注意、失敗回避のための実務知識 |
| REQ-0113-004 | 分離対象は全 skill / skill references を確認対象とすること |
| REQ-0113-005 | SPEC 相当記述の移管先は以下の既存 SPEC を優先し、新規 SPEC の作成は既存 SPEC で責務を表現できない場合に限定すること: `artifact-contracts.md`（artifact 入出力契約・配置・完了報告共通契約）、`integrity-contracts.md`（検査契約・finding 分類・report schema）、`workflow-contracts.md`（workflow lifecycle・transition・route・artifact lifecycle） |
| REQ-0113-006 | SPEC 移管に伴い skill references が不要になった場合、当該 reference ファイルを削除すること |
| REQ-0113-007 | SPEC 移管に伴い skill references の内容が縮小された場合、移管済み内容への参照を残さず自己完結させること |
| REQ-0113-008 | SPEC 相当記述の移管は Epic 単位で一括管理し、子 Issue に分割して実施してよい |
| REQ-0113-009 | runtime command は他 skill の references 内部構造に直接依存しない（REQ-0103-090 と整合する command からの参照境界制約である） |

## 適用範囲

- **対象**: skill / skill references 内の SPEC 相当記述の分離基準、移管対象の定義、移管後の runtime 配布物の自己完束制約、移管先 SPEC の選択基準
- **対象外**: 個別 skill の具体的な移管内容の詳細（子 Issue で管理）、新規 SPEC の設計（既存 SPEC で対応）、SPEC 移管の実行スケジュール・優先順位
