## 目的

skill / skill references と SPEC（`docs/specs/`）は責務境界で分離され、SPEC 相当記述は SPEC に配置される（REQ-0103-035, 036, 045, 047）。runtime 配布物と repo-internal な現在仕様の責務境界は明確である。runtime command / skill は `docs/specs/` を直接の実行時依存として参照しない。skill source は `src/opencode/skills/` に配置され、`.opencode/skills/` は runtime projection である（REQ-0103-008）。

## 要件

| ID | 要件 |
|---|---|
| REQ-0113-001 | skill references に含まれる SPEC 相当記述は `docs/specs/` に配置されること。SPEC 相当とは、現在仕様・状態モデル・artifact lifecycle・workflow contract・command topology・schema・frontmatter contract・finding category / route / report schema・文書種別間の責務境界・repo-internal な設計説明を含む |
| REQ-0113-002 | runtime command / skill は SPEC を直接の実行時依存から除外し、runtime 実行時に必要な最小情報を command または skill references に自己完結させること |
| REQ-0113-003 | skill に保持してよい記述は以下の範囲である: runtime 実行時に参照する短い判断表、USE FOR / DO NOT USE FOR、command 実行時のチェックリスト、操作時の注意、失敗回避のための実務知識 |
| REQ-0113-004 | 本要件の対象は全 skill / skill references であること |
| REQ-0113-005 | SPEC 相当記述は以下の既存 SPEC への配置を優先し、新規 SPEC は既存 SPEC で責務を表現できない場合のみである: `artifact-contracts.md`（artifact 入出力契約・配置・完了報告共通契約）、`integrity-contracts.md`（検査契約・finding 分類・report schema）、`workflow-contracts.md`（workflow lifecycle・transition・route・artifact lifecycle） |
| REQ-0113-006 | skill references の内容が SPEC への配置により不要になった場合、当該 reference ファイルは削除すること |
| REQ-0113-007 | skill references の内容が SPEC への配置により縮小される場合、自己完結していること |
| REQ-0113-008 | SPEC 相当記述の配置は Epic 単位で追跡対象であり、子 Issue 単位での実施を許容する |
| REQ-0113-009 | runtime command は他 skill への参照を skill 名レベルにとどめること（REQ-0103-090 と整合する command からの参照境界制約である） |

## 適用範囲

- **対象**: skill / skill references 内の SPEC 相当記述の SPEC 配置、配置対象の定義、runtime 配布物の自己完束制約、配置先 SPEC の選択
- **対象外**: 個別 skill の具体的な配置内容の詳細（子 Issue で管理）、新規 SPEC の設計（既存 SPEC で対応）、SPEC 配置の実行スケジュール・優先順位
