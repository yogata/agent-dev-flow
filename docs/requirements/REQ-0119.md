## 目的

コマンド、スキル、サブエージェントの責務境界で、コマンドは薄く（公開API、Step見出し、委譲接続点のみ）、詳細手順はスキルに、探索、検査、分類はサブエージェントに、最終判断、保存、副作用は親エージェントが保持する構造を満たす。Step 表記は整数に統一され、委譲の最小契約と返却形式が定義されている。

## 要件

| ID | 要件 |
|---|---|
| REQ-0119-001 | コマンドは、公開 API、入力、出力、親エージェントの責務、Step 見出し、委譲接続点に限定すること |
| REQ-0119-002 | Step 専用の詳細手順、確認手順、禁止事項、返却形式、失敗時の扱いは、スキルの SKILL.md または references/ に配置されること |
| REQ-0119-003 | 長い判定基準、例、分類表、詳細な失敗時手順は、スキルの references/ に配置されること |
| REQ-0119-004 | 親エージェントは、次 Step へ進む判断、保存、Issue / PR 更新、commit、push、ユーザー確認を保持すること |
| REQ-0119-008 | 小さい確認、作成、更新直後の VERIFY、Issue / PR 更新に密接な確認は、親 Step 内に含めること |
| REQ-0119-009 | 広い探索、局所検査、候補抽出、分類、大きなファイル確認は、サブエージェント化候補とすること |
| REQ-0119-010 | サブエージェントは、`pass` / `warn` / `fail` / `partial` を基本とした判定結果を返すこと |
| REQ-0119-011 | サブエージェントは、判定結果、調査結果の入力のみを提供し、保存、Issue / PR 更新、commit、push、ユーザー確認は親エージェントの責務に属すること |
| REQ-0119-012 | サブエージェントの返却に、副作用がないことを示す情報を含めること |
| REQ-0119-013 | 成果物本文（Issue 本文、PR 本文、commit message、保存対象ファイル本文、テンプレート成果物）は原文のまま（verbatim）で返すこと。判定結果、調査過程、中間ログ、読解メモは要約、成果物パス、根拠、親判断事項、capture 候補へ圧縮して返すこと。サブエージェント出力に一律 verbatim 制約を課さないこと |
| REQ-0119-014 | intake / learning 候補は capture 候補として親エージェントへ返すこと |
| REQ-0119-015 | `lightweight-delegation` は主要パターンではなく、主要な実装分類に重ねる委譲の扱いとして位置づけること |
| REQ-0119-016 | `manager-orchestrator` は、複数子タスクを統括する特殊な委譲形態であり、軽量な単一タスク委譲とは区別されること |
| REQ-0119-017 | `delegation_type` / `on_result` は必須の受け渡し項目（envelope）に含まれないこと。必要な場合のみ、簡潔なラベルや親側の扱いとして記述する |
| REQ-0119-018 | ADR / SPEC 上の委譲時最小契約は、`inputs`、`side_effect_boundary`、`output_contract`、`capture_handoff` を中心に構成されること |
| REQ-0119-019 | command 本文では英語ラベルを増やさず、日本語の説明文で委譲接続点を記述すること |
| REQ-0119-020 | 現行 docs / 原本 の旧 Findings / Intake 系見出しは `Findings / Capture候補` であること |
| REQ-0119-021 | 配置先側または integrity rule の旧語検出文字列は、検出目的が分かる形で残存を許容する |
| REQ-0119-022 | ADR-0111 は `status: superseded` であり、ADR-0112 を後継として参照すること |
| REQ-0119-023 | ADR-0112 は `status: accepted` であり、実装分類と委譲境界を包括する ADR であること。`delegation_type` 分類は SPEC に属すること |
| REQ-0119-024 | 対象ドキュメント全体は REQ-0119-001 から REQ-0119-023 の基準に適合していること |
| REQ-0119-028 | REQ-0119 が責務分界REQとして維持され、履歴、廃止理由、ADR の status 作業結果が要件本文から外されていること |
| REQ-0119-029 | command / skill / サブエージェント責務境界が REQ-0119 または SPEC で一貫していること |
| REQ-0119-030 | agentdev-architecture-advisory は req-define のアーキテクチャ確認を担う公開スキルであり、case-run 実行、momus 確認、ファイル編集を対象外とすること |
| REQ-0119-031 | agentdev-case-run-execution-adapter は case-run の外部実行手段接続に責務を限定する公開スキルであり、req-define のアーキテクチャ確認、ADR 判断、ワークフロー状態管理、Issue 完了条件評価を対象外とすること。SKILL.md はハーネス非依存の抽象 IF とし、ハーネス固有の実装詳細は references/<harness>.md に配置すること。プロジェクトルートの AGENTS.md でハーネスを選定、指定すること。将来別ハーネスを選ぶプロジェクトは references 増設で対応すること |

## 適用範囲

- **対象**:
  - `src/opencode/commands/agentdev/*.md`（全13コマンド）
  - `src/opencode/skills/agentdev-*/SKILL.md` 及び `references/*.md`（詳細手順の配置先）
  - `docs/adr/ADR-0111.md`（`status: superseded`）
  - `docs/adr/ADR-0112.md`（`status: accepted`、実装分類と委譲境界を包括）
  - `docs/specs/workflow-contracts.md`（委譲契約）
  - `docs/specs/artifact-contracts.md`（`delegation_type` 分類の配置先）
  - `docs/specs/artifact-responsibilities.md`（親/サブエージェント責務）
  - `docs/specs/rule-ownership.md`（検査ルール）
  - `src/opencode/skills/agentdev-command-authoring/references/command-authoring-standards.md`（作成基準）
  - `docs/DOC-MAP.md`（導線）
  - `docs/requirements/retired/REQ-0111.md`（廃止済み）
  - 関連 integrity rule / template / guide

- **対象外**:
  - サブエージェント実行基盤そのものの実装
  - OpenCode / Codex / Claude Code ごとの差分吸収
  - 実際の並列実行制御の新規実装
  - Issue / PR 作成
  - 既存コマンドの外部的な目的変更
  - ユーザー未承認の保存、commit、push
  - inspect-docs 全体の再設計
  - `.agentdev/` への保存責務の変更
  - サブエージェントへの最終判断の委譲
  - capture 候補の保存処理をサブエージェントへ持たせること
  - 旧語検出用途で残す検査文字列の機械的削除
