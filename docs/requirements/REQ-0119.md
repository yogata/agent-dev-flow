## 目的

コマンド・スキル・サブエージェントの責務境界を再基準化し、コマンドは薄く（公開API・Step見出し・委譲接続点のみ）、詳細手順はスキルへ、探索・検査・分類はサブエージェントへ、最終判断・保存・副作用は親エージェントが保持する構造を確立する。併せて Step 表記を整数に統一し、委譲の最小契約と返却形式を定める。

## 要件

| ID | 要件 |
|---|---|
| REQ-0119-001 | コマンドは、公開 API、入力、出力、親エージェントの責務、Step 見出し、委譲接続点に限定し、詳細手順を保持しないこと |
| REQ-0119-002 | Step 専用の詳細手順、確認手順、禁止事項、返却形式、失敗時の扱いは、スキルの SKILL.md または references/ へ移管すること |
| REQ-0119-003 | 長い判定基準、例、分類表、詳細な失敗時手順は、スキルの references/ に置くこと |
| REQ-0119-004 | 親エージェントは、次 Step へ進む判断、保存、Issue / PR 更新、commit、push、ユーザー確認を保持すること |
| REQ-0119-005 | トップレベル Step は整数のみとし、小数 Step を廃止すること |
| REQ-0119-006 | サブステップは N-M 形式（例: 10-1, 10-2）を許容する。英字サブステップ（10a, 10b）は使用しないこと |
| REQ-0119-007 | case-run.md の旧小数 Step は独立 Step へ昇格し、後段の近接 Step を統合してトップレベル Step 数を 12 以下に収めること |
| REQ-0119-008 | 小さい確認、作成・更新直後の VERIFY、Issue / PR 更新に密接な確認は、サブエージェント化せず親 Step 内に含めること |
| REQ-0119-009 | 広い探索、局所検査、候補抽出、分類、大きなファイル確認は、サブエージェント化候補とすること |
| REQ-0119-010 | サブエージェントは、`pass` / `warn` / `fail` / `partial` を基本とした判定結果を返すこと |
| REQ-0119-011 | サブエージェントは、保存、Issue / PR 更新、commit、push、ユーザー確認を行わないこと |
| REQ-0119-012 | サブエージェントの返却に、副作用がないことを示す情報を含めること |
| REQ-0119-013 | 成果物本文（Issue 本文、PR 本文、commit message、保存対象ファイル本文、テンプレート成果物）は verbatim で返すこと。判定結果、調査過程、中間ログ、読解メモは要約・成果物パス・根拠・親判断事項・capture 候補へ圧縮して返すこと。サブエージェント出力の一律 verbatim 制約は廃止すること |
| REQ-0119-014 | intake / learning 候補は保存せず、capture 候補として親エージェントへ返すこと |
| REQ-0119-015 | `lightweight-delegation` は primary pattern としないこと。主要な実装分類に重ねる委譲の扱いとして整理すること |
| REQ-0119-016 | `manager-orchestrator` は、複数子タスクを統括する特殊な委譲形態として扱い、軽量な単一タスク委譲と混同しないこと |
| REQ-0119-017 | `delegation_type` / `on_result` は必須 envelope から外すこと。必要な場合のみ、簡潔なラベルや親側の扱いとして記述する |
| REQ-0119-018 | ADR / SPEC 上の委譲時最小契約は、`inputs`、`side_effect_boundary`、`output_contract`、`capture_handoff` を中心に整理すること |
| REQ-0119-019 | command 本文では英語ラベルを増やさず、日本語の説明文で委譲接続点を記述すること |
| REQ-0119-020 | current docs / source の旧 Findings / Intake 系見出しは `Findings / Capture候補` へ統一すること |
| REQ-0119-021 | projection 側または integrity rule の旧語検出文字列は、検出目的が分かる形で残存を許容する |
| REQ-0119-022 | ADR-0111 を `status: superseded` とし、ADR-0112 を後継として参照すること |
| REQ-0119-023 | ADR-0112 を `status: accepted` とし、実装分類と委譲境界を包括する ADR として再定義すること。`delegation_type` 分類は SPEC へ降格すること |
| REQ-0119-024 | REQ-0119-001 から REQ-0119-023 に基づく再基準化を、対象ドキュメント全体に同時に適用すること |
| REQ-0119-025 | REQ-0111（Command authoring 後方互換性維持原則）を retire すること。`docs/requirements/REQ-0111.md` を `docs/requirements/retired/` へ移動すること。REQ-0111 の条項は他REQへの吸収なしで廃止とすること。理由: REQ-0119 の破壊的再基準化が REQ-0111 の中核原則（既存Step変更禁止）と矛盾し、復元用タグの存在により後方互換の拘束力が既に緩んでいるため |
| REQ-0119-026 | REQ-0103-089 の `REQ-0111-004` への参照を更新すること。REQ-0111 retire に伴い、薄型化時の移行先明示・振舞維持の要件は REQ-0103 自身または REQ-0119 の該当条項（REQ-0119-002、REQ-0119-003）を参照する形に修正すること |

## 適用範囲

- **対象**:
  - `src/opencode/commands/agentdev/*.md`（全13コマンド）
  - `src/opencode/skills/agentdev-*/SKILL.md` 及び `references/*.md`（移管内容受け取り）
  - `docs/adr/ADR-0111.md`（superseded 化）
  - `docs/adr/ADR-0112.md`（accepted 化・再定義）
  - `docs/specs/workflow-contracts.md`（委譲契約更新）
  - `docs/specs/artifact-contracts.md`（delegation_type 降格先）
  - `docs/specs/artifact-responsibilities.md`（親/サブエージェント責務更新）
  - `docs/specs/rule-ownership.md`（検査ルール更新）
  - `src/opencode/skills/agentdev-command-authoring/references/command-authoring-standards.md`（作成基準更新）
  - `docs/DOC-MAP.md`（導線更新）
  - `docs/requirements/REQ-0111.md`（retire → `docs/requirements/retired/` へ移動）
  - 関連 integrity rule / template / guide

- **対象外**:
  - サブエージェント実行基盤そのものの実装
  - OpenCode / Codex / Claude Code ごとの差分吸収
  - 実際の並列実行制御の新規実装
  - Issue / PR 作成
  - 既存コマンドの外部的な目的変更
  - ユーザー未承認の保存、commit、push
  - docs-review 全体の再設計
  - `.agentdev/` への保存責務の変更
  - サブエージェントへの最終判断移譲
  - capture 候補の保存処理をサブエージェントへ持たせること
  - 旧語検出用途で残す検査文字列の機械的削除
