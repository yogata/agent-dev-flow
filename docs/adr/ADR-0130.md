# ADR-0130: agentdev-gh-cli を差し替え可能な I/O 境界として確立

## 背景

AgentDevFlow の既存 command/skill は GitHub CLI（gh）コマンドを直接記述しており、Windows/PowerShell/エンコーディング/--body-file/VERIFY の扱いが各ファイル（12ファイル）に分散している。
ローカル版で GitHub Issue/PR を使わない場合、この直接記述が差し替えの障害となる。
既存の agentdev-gh-cli は Windows PowerShell 環境でのエンコーディング安全化を目的とする狭いスキルであり、全 GitHub I/O を集約する I/O 境界としては機能していなかった。

## 決定

1. command/skill は gh コマンドを直接記述せず、agentdev-gh-cli の手続きへ委譲する。
2. agentdev-gh-cli は I/O 手続きと VERIFY を担当する中央集権的な I/O 境界である。
本文生成、完了判定、Epic 依存判定、capture 分類は担当しない。
3. agentdev-gh-cli は薄いルーティング入口とし、操作契約と標準版手順を references に分離する。
4. ローカル版は agentdev-gh-cli を `src/opencode-local/agentdev-gh-cli/` で差し替え可能とする。
上位 command/skill は常に agentdev-gh-cli のみを参照し、別名スキルを参照させない。
5. ローカル版は同一手続き名で Issue/PR 相当の読み書きを Case ファイルへ読み替える。
PR 関連操作をスキップせず、Case ファイルの対応セクションで代替する。
6. GitHub 非依存の抽象 backend は新設せず、GitHub 前提の gh-cli 手続き化と上書き可能化とする。

## 結果、影響

### 正の影響

- GitHub I/O が一箇所に集約され、command は workflow に、skill は意味処理に集中できる。
- Windows/PowerShell/文字化け/一時ファイル/--body-file/VERIFY の扱いが各 command から排除される。
- ローカル版は agentdev-gh-cli 差し替えだけで対応でき、command/skill 全体の分岐が不要になる。

### 負の影響

- 全 command/skill の gh 直接記述を委譲へ書き換える移行コストが発生する（12ファイル）。
- agentdev-gh-cli の責務が拡大し、SPEC の全面改訂が必要。

## 判断根拠

差し替え可能な I/O 境界の確立は、将来のコマンド追加時も縛る制約であり、「将来の設計、運用、文書システムを制約する決定」（`agentdev-adr-guidelines` False Negative 防止基準）に該当する。
ADR-0107/0112 の適用、具体化であるが、新規の責務境界（全 gh I/O の集約、担当/非担当宣言、差し替え可能性）を確立するため例外として ADR を認める。
詳細な手続き名、引数、戻り値は SPEC（agentdev-gh-cli.md）へ、責務境界の宣言は REQ-0149 へ配置する。
