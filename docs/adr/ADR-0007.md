# ADR-0007: REQ/ADR基準構造と分類ビュー運用の再定義

## 背景

ADR-0004 で area-based current document 構造への段階的移行方針を定めた。しかし、以下の課題が顕在化した:

1. area-based 構造への移行コストが高く、実装優先度が上がらない
2. `REQ-{NNNN}.md` の per-file 構造が実際の運用で十分機能している
3. ADR についても本文を area / topic / view 別に再配置する構造は不要との認識に至った
4. 探索性の改善は本文の移動ではなく、分類ビュー（README.md の拡充、views/*.md の導入）で対応可能

## 決定

以下の方針に変更する:

1. **REQ基準は `REQ-{NNNN}.md`**: per-file 構造を永続標準とし、area-based 移行方針を撤回する
2. **views/*.md を補助ビュー**: `docs/requirements/views/*.md` を観点別体系化ビューとして導入する。基準は `REQ-{NNNN}.md` であり、views は補助
3. **ADR基準は `ADR-{NNNN}.md`**: ADR本文を area / topic / view 別に移動しない
4. **README.md を分類ビューに拡充**: `docs/adr/README.md` に Status View、Topic View、Decision Map、Related REQ を追加し、探索性を改善する
5. **ADR→Issue 逆参照は原則禁止**: ADRを実装履歴に引きずらせない
6. **case-close の ADR更新責務は排除**: case-close はADR本文をAPPEND/UPDATEしない

## 代替案

1. **area-based 構造を継続**: 移行コストが高く、per-file で十分なため不採用
2. **views/*.md を generated artifact に**: 手動管理の柔軟性を優先し、generated は必須としない
3. **ADR→Issue 逆参照を許可**: ADRの独立性が損なわれるため不採用

## 結果・影響

- `REQ-{NNNN}.md` が永続基準として確定し、area-based 移行の計画が不要になる
- `views/*.md` により観点別の要件俯瞰が可能になる
- `docs/adr/README.md` の分類ビュー拡充によりADRの探索性が改善される
- `agentdev-req-file-manager` スキルから area-based 前提の削除が必要（REQ-0004-081/085, retired）
- `req-save.md` コマンド定義の area-based 注記削除・views 影響確認追加が必要（REQ-0004-083/084, retired）
- ADR-0004 は `superseded-by:[ADR-0007]` に遷移する
