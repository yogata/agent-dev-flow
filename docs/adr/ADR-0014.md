# ADR-0014: ADR / SPEC 再分類基準

## 背景

ADR と SPEC の責務境界があいまいだった。SPEC に「なぜそう決めたか」の記述が混入し、ADR に「現在の仕様」が記述されることがあった。また SPEC は agent-dev-flow レポジトリ専用の文書であることが明示されていなかった。

## 決定

以下の再分類基準を定める:

- **SPEC**: 「現在の構造・契約・ルール」を記述する。agent-dev-flow レポジトリ専用であり、runtime 配布物の依存先ではない
- **ADR**: 「なぜその決定をしたか」の記録。SPEC とは責務を分離する
- **Guides**: 人間向けナビゲーション層。規範的権限を持たない
- **DOC-MAP**: 案内のみ。規範的権限を持たない

各 SPEC の冒頭に「docs/specs は agent-dev-flow レポジトリ専用」宣言を含める。

## 代替案

- SPEC を runtime 配布物に含める: SPEC はレポジトリ内部の管理文書であり、配布物に含めると管理コストが増大する
- ADR と SPEC を統合する: 「なぜ」と「何」の関心が異なるため分離が適切

## 結果・影響

- 新規 SPEC（document-model.md, artifact-contracts.md, workflow-contracts.md, integrity-contracts.md）の作成が必要
- design-principles.md に runtime/authoring 分離原則を追記する必要がある
- 既存 ADR との整合性確認が必要

## 関連する決定

- ADR-0013: runtime/authoring 関心分離 — 本決定の基盤
- ADR-0015: docs/specs 非runtime依存宣言 — SPEC 配布範囲の補完
