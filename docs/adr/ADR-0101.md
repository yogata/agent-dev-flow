# ADR-0101: AgentDevFlow plugin namespace 統一

## 背景

本リポジトリの `/issue/issue-*` コマンド群は名前上 GitHub Issue 操作に見えるが、実態は開発ライフサイクル全体（要件定義・実行・レビュー・学習・未回収課題回収）を扱っている。`/tips/*` は実態として learning pipeline であり、旧 backlog 系は正式化前の変更候補回収である。

この namespace の不一致は以下の問題を引き起こす:
- 新規ユーザーが「Issue 操作ツール」と誤認する
- コマンド追加時に適切な namespace が不明確
- 実態と名称の乖離が学習コストを増加させる

## 決定

1. 配布 plugin / product の表示名を `AgentDevFlow` とする
2. 公開コマンドの canonical namespace を `/agentdev/*` に統一する
3. `/agentdev/*` 配下に req / case / learning / intake の domain を置く（integrity domain の検査コマンドは後に `/repo/docs-check` へ移行: ADR-0106, REQ-0115）
4. AgentDevFlow domain state の canonical directory を `.agentdev/` とする
5. AgentDevFlow 配下の skill は `agentdev-*` prefix を付ける
6. 旧 command は deprecated wrapper を残さず完全削除する

> ※Command Migration table（旧→新の対応一覧）はREQ-0103（Artifact責任分界）を参照。
> ※Domain State Directory（`.sisyphus/`と`.agentdev/`の責務詳細）はREQ-0103およびREQ-0105を参照。

## 代替案

1. **段階的移行（deprecated wrapper）**: 旧 command に deprecated ラベルを付けて新 command と併存 → 採用却下。2系統のメンテナンスコストとユーザー混乱を回避するため、完全削除を選択
2. **namespace に `devflow` を使用**: より短い namespace → 採用却下。`AgentDevFlow` の方が「AI agent-assisted development」の意図を明確に表現するため
3. **`.agentdev/` を `.sisyphus/` のサブディレクトリにする**: → 採用却下。Sisyphus runtime と AgentDevFlow domain state の関心が異なるため、トップレベル分離が適切

## 結果・影響

- 全コマンドの rename と参照更新が必要（9子issue に分割）
- 既存の `load_skills` 参照の全面更新が必要
- docs 全体の用語更新が必要
- 移行期間中の旧コマンド参照は不可（完全削除方式）
- Sisyphus 命名規則（`.sisyphus/`）は維持
