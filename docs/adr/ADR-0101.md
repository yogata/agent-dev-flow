# ADR-0101: AgentDevFlow プラグイン名前空間の統一

## 背景

本リポジトリの `/issue/issue-*` コマンド群は名前上 GitHub Issue 操作に見えるが、実態は開発ライフサイクル全体（要件定義、実行、レビュー、学習、未回収課題回収）を扱っている。
`/tips/*` は実態として Learning パイプラインであり、旧 Backlog 系は正式化前の変更候補回収である。

この名前空間の不一致は以下の問題を引き起こす:
- 新規ユーザーが「Issue 操作ツール」と誤認する
- コマンド追加時に適切な名前空間が不明確
- 実態と名称の乖離が学習コストを増加させる

## 決定

1. 配布プラグイン、プロダクトの表示名を `AgentDevFlow` とする
2. 公開コマンドの正規の名前空間を `/agentdev/*` に統一する
3. `/agentdev/*` 配下に req / case / learning / intake のドメインを置く（integrity ドメインの検査コマンドは後に `/repo/docs-check` へ移行: ADR-0106, REQ-0115）
4. AgentDevFlow ドメイン状態の正規ディレクトリを `.agentdev/` とする
5. AgentDevFlow 配下のスキルには `agentdev-*` プレフィックスを付ける
6. 旧コマンドは完全削除し、実行時コマンドは新名前空間のみで構成する（非推奨ラッパーの併存は結果、影響を参照）

> ※コマンド移行テーブル（旧→新の対応一覧）は REQ-0103（Artifact 責任分界）を参照。
> ※ドメイン状態ディレクトリ（`.agentdev/`の責務詳細）は REQ-0103 および REQ-0105 を参照。

## 代替案

1. **段階的移行（非推奨ラッパー）**: 旧コマンドに非推奨ラベルを付けて新コマンドと併存 → 採用却下。2系統のメンテナンスコストとユーザー混乱を回避するため、完全削除を選択
2. **名前空間に `devflow` を使用**: より短い名前空間 → 採用却下。`AgentDevFlow` の方が「AI エージェント支援開発」の意図を明確に表現するため
3. **`.agentdev/` を他ディレクトリのサブディレクトリにする**: → 採用却下。実行時一時領域と AgentDevFlow ドメイン状態の関心が異なるため、最上位分離が適切

## 結果、影響

- 全コマンドのリネームと参照更新が必要（9子 Issue に分割）
- docs 全体の用語更新が必要
- 移行期間中の旧コマンド参照は不可（完全削除方式）
