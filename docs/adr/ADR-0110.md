# ADR-0110: DOC-MAP 導入と requirements/views 廃止

## 背景

ADR-0007 で `views/*.md` を観点別体系化ビューとして導入した。しかし、以下の課題が顕在化した:

1. `views/*.md` と `docs/specs/*.md` の責務が重複しやすく、views が現在仕様の読み物になった場合は二重管理になる
2. views は分類索引テーブルに留まっており、REQ へのリンク集以上の価値を提供していない
3. 文書探索の入口として、views よりも全体像を示す DOC-MAP が適している
4. 現在仕様の説明は `docs/specs/*.md` に一本化する方が明確

## 決定

以下の方針に変更する:

1. **views/*.md を廃止**: `docs/requirements/views/*.md` を削除し、内容を DOC-MAP/specs に移管する
2. **DOC-MAP.md を導入**: `docs/DOC-MAP.md` を文書探索・参照経路の入口として追加する。DOC-MAP は参照用とする
3. **SPEC を現在仕様の基準化**: `docs/specs/*.md` を現在仕様の基準として明確化する
4. **agentdev-doc-map スキルを新設**: 基準境界と探索順序を定義する
5. **REQ/ADR 基準は維持**: `REQ-{NNNN}.md` / `ADR-{NNNN}.md` を引き続き基準とする

## 代替案

1. **views を継続**: specs と責務重複するため不採用
2. **DOC-MAP を基準化**: REQ/ADR/SPEC と競合するため不採用
3. **SPEC へ要件基準移管**: REQ基準境界を壊すため不採用

## 結果・影響

- `views/*.md` の内容は DOC-MAP（分類索引）に移管される
- `docs/specs/*.md` が現在仕様の明確な基準となる
- `docs-check` に DOC-MAP 検査・views 残存検出・finding 分類が追加される
- `req-define` / `case-run` / `case-close` が DOC-MAP を参照するようになる
- ADR-0007 の views 導入決定が supersede される
- REQ-0004 (retired) の views 関連要件が supersede される
