# ADR-0108: Orchestration skill 作成基準の導入

## 背景

従来、コマンドとオーケストレーションスキルの関係が暗黙的であり、1 コマンド = 1 オーケストレーションスキルの原則が適用されるケースとそうでないケースが混在していた。このため、小さいコマンドに不要なスキルが作成されたり、大きな状態機械を持つコマンドがスキル化されず肥大化したりする問題があった。

## 決定

オーケストレーションスキルは状態機械・プロトコル・再利用境界単位で作成する。以下の条件のいずれかを満たす場合にオーケストレーションスキル化を認める:

- 大きな状態機械を持つ場合
- 再開ポイント検出が必要な場合
- CI ループを持つ場合
- Wave スケジューリングを持つ場合
- サブエージェントプロトコルを持つ場合

条件非該当の小規模コマンドは本文完結を基本とする。

## 代替案

1. **1:1 原則の維持**: 全コマンドにオーケストレーションスキルを必須化 → 小さいコマンドに過剰
2. **原則なし**: 判断を現場任せ → 一貫性なし

## 結果・影響

- agentdev-workflow-orchestration と agentdev-learning-pipeline は新設（条件に合致）
- agentdev-learning-capture はキャプチャ専用として維持（オーケストレーションスキル化なし）
- agentdev-command-authoring は判断基準への入口を提供

- ADR-0002（廃止 — 本判断の前身）
- ADR-0107（コマンド・スキル・テンプレート・スクリプト責任分界の正式定義 — 補完関係）
