# ADR-0106: /repo/* Namespace for Repo-Local Tooling

## 背景

`agentdev-integrity` contains repo-specific integrity checks (vocabulary registry, source/projection validation) that are not applicable to consumer projects. No existing namespace category for repo-local-only tooling exists. `.agentdev/` has been used for static registries (vocabulary-registry.md) which contradicts its domain state purpose.

## 決定

1. Introduce `/repo/*` command namespace and `repo-*` skill prefix for self-hosting repo-only tooling.
2. Move `integrity-check` to `/repo/integrity-check` and `agentdev-integrity` to `repo-agentdev-integrity`.
3. Move vocabulary registry to repo-local skill references.

> ※移動対象の具体的パスリスト、vocabulary registry移動先の詳細はREQ-0103、REQ-0108を参照。

4. Exclude repo-local artifacts from sync-opencode.ps1 junction management.

## 結果・影響

- Consumer projects will no longer receive integrity-check or agentdev-integrity.
- Self-hosting repo maintains its own audit tooling independently.
- Future repo-local tools (e.g., repo-specific automation) have a clear namespace.
- `.agentdev/` returns to pure domain state role.
