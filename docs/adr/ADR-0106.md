# ADR-0106: リポジトリローカルツールのための /repo/* 名前空間

## 背景

`agentdev-integrity` はリポジトリ固有の整合性検査（語彙レジストリ、ソース、プロジェクション検証）を含んでおり、これらは利用プロジェクトには適用できない。
リポジトリローカル専用ツールのための名前空間カテゴリが存在しない。
`.agentdev/` は静的レジストリ（vocabulary-registry.md）の配置に使われており、これは同ディレクトリのドメイン状態という目的と矛盾する。

## 決定

1. 本体リポジトリ専用ツールのための `/repo/*` コマンド名前空間と `repo-*` スキルプレフィックスを導入する。
2. `integrity-check`（旧名）を `/repo/docs-check` に、`agentdev-integrity` を `repo-agentdev-integrity` に移動する。
3. 語彙レジストリをリポジトリローカルのスキル `references/` に移動する。

> ※移動対象の具体的パスリスト、語彙レジストリ移動先の詳細は REQ-0103、REQ-0108 を参照。

4. リポジトリローカルの成果物を sync-opencode.ps1 のジャンクション管理対象から除外する。

## 結果、影響

- 利用プロジェクトには docs-check や agentdev-integrity が配布されなくなる。
- 本体リポジトリは独自の監査ツールを独立に維持する。
- 将来のリポジトリローカルツール（例: リポジトリ固有の自動化）に明確な名前空間が与えられる。
- `.agentdev/` が純粋なドメイン状態の役割に戻る。

## 関連する決定

- ADR-0020（廃止（本判断の前身））
- ADR-0101（AgentDevFlow プラグイン名前空間の統一（リポジトリローカル名前空間と正規名前空間の境界定義））
- ADR-0105（OpenCode Source / Projection 分離（リポジトリローカル成果物をソース、プロジェクション同期対象外にする根拠））
