# ADR-0002: Orchestration skill作成基準の導入

## Context

従来、command と orchestration skill の関係が暗黙的であり、1 command = 1 orchestration skill の原則が適用されるケースとそうでないケースが混在していた。このため、小さい command に不要な skill が作成されたり、逆に大きな状態機械を持つ command が skill 化されず肥大化したりする問題があった。

## Decision

`1 command = 1 orchestration skill` を原則とはしない。以下の条件を満たす場合にのみ orchestration skill 化を認める:

- 大きな状態機械を持つ場合
- 再開ポイント検出が必要な場合
- CI loop を持つ場合
- Wave scheduling を持つ場合
- サブエージェント protocol を持つ場合

小さい command ごとの不要な 1:1 orchestration skill 追加は行わない。Orchestration skill は command 単位ではなく、状態機準・protocol・再利用境界単位で作成する。

## Alternatives

1. **1:1 原則の維持**: 全 command に orchestration skill を必須化 → 小さい command に過剰
2. **原則なし**: 判断を現場任せ → 一貫性なし

## Consequences

- agentdev-workflow-orchestration と agentdev-learning-pipeline は新設（条件に合致）
- agentdev-learning-capture は capture 専用として維持（orchestration skill 化なし）
- agentdev-command-authoring は判断基準への入口を提供
