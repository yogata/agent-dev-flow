# ADR-0001: Command/Skill/Template/Script責任分界の正式定義

## Context

現在、一部の command に詳細な状態機械、schema、分類表、テンプレート選定ルール、共通安全手順が入り込んでおり、Command / Skill / Template / Script の責任分界が曖昧になっている。
artifact-boundaries.md には Command / Skill / SPEC / Template の 4 種が定義されているが、Script の責務が未定義であり、Command が肥大化しやすい構造になっている。

## Decision

artifact-boundaries.md を拡張し、Command / Skill / Template / Script の 4 種アーティファクトとして再定義する。

- **Command**: 公開APIと高レベル実行骨格（原則 100行以内、Steps 5〜12個）
- **Skill**: 再利用可能な判断基準と大きな状態機械
- **Template**: 出力構造とプレースホルダー
- **Script**: 決定的でテスト可能な処理

design-principles.md にこの責任分界を要約として記載する。
また `agentdev-command-authoring` skill を新設し、command 作成時の基準を明確にする。

## Alternatives

1. **現状維持**: artifact-boundaries.md を更新せず個別対応 → 肥大化が継続
2. **1 command = 1 orchestration skill の原則**: すべての command に対応 skill を必須化 → 小さい command に不要なオーバーヘッド
3. **Script なしの 3 種**: 決定的処理を Skill に含める → Skill の責務が曖昧になる

## Consequences

- Command が薄くなり、責務が明確になる
- orchestration skill は状態機械、protocol、再利用境界単位で作成される
- 既存の良い分離（req-analysis / req-file-manager 等）は維持される
- 新規 skill: `agentdev-command-authoring`, `agentdev-workflow-orchestration`, `agentdev-learning-pipeline`
