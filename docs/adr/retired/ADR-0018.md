# ADR-0018: runtime 独立性

## 背景

runtime command（個別プロジェクトで実行される commands/skills）が `docs/specs/system.md` や `docs/specs/patterns.md` に固定依存していた。SPEC の変更が runtime の動作に影響する可能性があった。また skill references/ に runtime と authoring 資料が混在し、runtime 配布物に不要な authoring-only 情報が含まれていた。

ADR-0015（docs/specs 非runtime依存宣言）と ADR-0016（skill references runtime-only 制約）が関連するが、個別の宣言にとどまり、runtime 独立性の包括的な判断として統合する必要があった。

> ※ADR-0015/0016統合の詳細、具体的なSPEC参照禁止リストはREQ-0103、REQ-0108を参照。

## 決定

runtime 配布物は自己完結し、agent-dev-flow レポジトリの内部 docs（SPEC/guides）に依存しないことを宣言する:

1. runtime command は `docs/specs/` のファイルパスを直接参照しない
2. skill references/ は runtime 配布物のみを含める（authoring-only 資料は除外）
3. 必要な判断基準は SKILL.md 本体または references/ に自己完結させる

## 代替案

1. SPEC を runtime 配布物に含める → SPEC は repo-internal であり配布に不適
2. SPEC のうち runtime 関連部分だけを抽出 → 管理コストが増大し、切り出し漏れのリスクが高い
3. 現状維持（ADR-0015/0016 の個別宣言） → 包括的な判断として不十分

## 結果・影響

- ADR-0016（skill references runtime-only 制約）を本判断に統合し supersede する
- ADR-0015（docs/specs 非runtime依存宣言）の内容も本判断に統合する
- integrity-check に runtime の SPEC 依存検出を追加可能
- case-run の薄型化が促進される（SPEC 参照の削除による行数削減）

## 関連する決定

- ADR-0016: skill references runtime-only 制約（superseded by 本判断）
- ADR-0015: docs/specs 非runtime依存宣言（本判断に統合）
- ADR-0013: runtime/authoring 関心分離（本判断の基盤）
- ADR-0017: 文書種別責務境界（本判断と補完関係）
