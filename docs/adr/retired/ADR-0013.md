# ADR-0013: runtime / authoring 関心分離

## Context

AgentDevFlow の command frontmatter には 廃止フィールドが含まれていた。
これらは agent-dev-flow レポジトリの開発用メタデータであり、個別プロジェクトで command を実行する際には不要である。
しかし integrity-check で必須検査の対象となっており、runtime と authoring の関心が混在していた。

## Decision

command frontmatter を runtime 配布物として扱い、許可フィールドを description と agent のみに制限する。
廃止フィールドは frontmatter から削除し、dev メタデータは design-principles.md や skill references など agent-dev-flow レポジトリ内の管理物に限定する。

> ※禁止フィールドの具体的一覧、integrity検査の必須→存在禁止へのreversal詳細仕様はREQ-0103、REQ-0108を参照。

runtime 配布物は自己完結し、agent-dev-flow レポジトリの dev メタデータに依存しないことを保証する。

## Alternatives

- frontmatter に廃止フィールドを残しつつ optional 化する: 廃止予定のフィールドを残すと移行コストが継続する
- 別ファイルに dev メタデータを分離する: 管理対象が増え、整合性維持のコストが上がる

## Consequences

- 全 command frontmatter から廃止フィールドを削除する必要がある
- integrity-check の該当検査を「必須」から「存在禁止」に reversal する必要がある
- REQ-0103、REQ-0108 の該当 items を更新する必要がある
- design-principles.md に分類定義を移譲する必要がある
