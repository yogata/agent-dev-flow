# ADR-0113: diagnostics workflow 導入と review 系コマンド完全削除

> **status: deprecated** (2026-06-16, agentdev-system-reorganization OU-07)
>
> 本 ADR は `diagnostics-*` 命名を採用したが、その後 inspect-* への全面改名（REQ-0124）により現行判断の根拠として読めなくなった。`diagnostics` 命名自体がすでに `inspect` に置き換えられており、本 ADR の決定内容は現行基盤に反映されていない。現行の検出コマンド群の状態は REQ-0103 / REQ-0124 / SPEC 側に記録されている。本 ADR は履歴参照としてのみ扱い、現行根拠として引用しないこと。

## 背景

AgentDevFlow の診断機能は `docs-review` と `skill-review` の2コマンドで構成されていた。これらには以下の問題があった。

1. **語の衝突**: `review` は行為名として広く、`docs-review` / `skill-review` / `backlog-review` など既存コマンド名と衝突しやすい
2. **finding RU化経路の分断**: 診断 finding の行き先が intake / req-define 入力 draft / docs-check rule に分断され、経路が一貫していなかった
3. **promoted 概念の不統一**: intake / learning は promoted を持つが、diagnostics には同等の概念がなかった

## 決定

### 1. diagnostics を一級概念として導入

`diagnostics` を診断 workflow の一級概念として導入する。コマンド体系を `diagnostics-*` に統一する:

- `diagnostics-docs`: docs 系意味整合診断（旧 `docs-review`）
- `diagnostics-skills`: command/skill 参照妥当性・skill 構造診断（旧 `skill-review`）
- `diagnostics-promote`: diagnostic finding の分類・採用
- `diagnostics-commands`: 予約拡張点（本次件では対象外）

### 2. docs-review / skill-review の完全削除

deprecated shim なしで完全削除する（ドキュメント記述 + 実体ファイル）。REQ-0115-039「互換 shim を前提とした段階移行は行わず、即座にあるべき姿へ移行すること」に合致する。

### 3. diagnostics domain state の導入

`.agentdev/diagnostics/` を domain state として導入する:

- `inbox/`: 未分類 diagnostic finding
- `promoted/`: 採用済み・RU 化対象
- `archive/rejected/`: reject 済み

### 4. promoted 概念の統一

`promoted/` の意味を intake / learning / diagnostics で「採用済み・RU 化対象」に統一する。

### 5. finding routing の整理

- 明確な不整合 → diagnostics promoted → RU 化対象（intake に回さない）
- 未確定 finding → intake
- 修正対象を持たない知見 → learning
- docs-check rule / fixture 候補 → 独立 route とせず RU の要件化方向または受け入れ条件に含める

## 根拠

### 完全削除 vs deprecated shim

REQ-0115-039 が「互換 shim を前提とした段階移行は行わず、即座にあるべき姿へ移行すること」を定めている。deprecated shim を選択すると REQ-0115-039 との競合が発生し、shim の保守コストも発生する。完全削除は REQ-0115-039 に合致し、diagnostics workflow 導入の全体構造を単純化する。

### review → diagnostics への改名

`review` は `backlog-review` 等、既存コマンドと衝突する。`diagnostics` はドメイン固有であり、衝突を回避できる。
