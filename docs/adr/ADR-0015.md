# ADR-0015: docs/specs 非runtime依存宣言

## 背景

docs/specs/ は agent-dev-flow レポジトリの管理文書であるが、runtime 配布物（commands/skills）が SPEC の内容に依存しているかのような実装が存在していた。SPEC の変更が runtime の動作に影響する可能性があった。

## 決定

docs/specs/ は agent-dev-flow レポジトリ専用の管理文書であり、runtime 配布物（個別プロジェクトで実行される commands/skills）の依存先ではないことを宣言する。

runtime 配布物は自己完結し、agent-dev-flow レポジトリの SPEC を参照しなくても動作する。

## 代替案

- SPEC を runtime 依存として扱う: 配布物がレポジトリ内部文書に依存すると、バージョン管理と配布が複雑化する
- SPEC を廃止する: 現在仕様の基準として有用であり、廃止は非現実的

## 結果・影響

- SPEC を参照していた runtime 処理（integrity-check 以外）を自己完結化する必要がある
- 各 SPEC の冒頭に非runtime依存宣言を追加する
- integrity-check は SPEC を参照するが、これは agent-dev-flow レポジトリ内でのみ実行されるため問題ない
