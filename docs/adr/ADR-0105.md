# ADR-0105: OpenCode ソース・プロジェクション分離

## 背景

AgentDevFlow のコマンド/スキル/テンプレートのソースが `.opencode/` 直下に存在し、ソースコード（編集対象）と実行時プロジェクション（実行対象）が同一ディレクトリに混在している。
これにより以下の問題がある:
- ソース変更の影響範囲が不明確
- 利用プロジェクトへの配布方法が定義できない
- 整合性検査の対象がソースとプロジェクションで区別できない

## 決定

1. AgentDevFlow 本体のコマンド/スキル/テンプレートのソースを `src/opencode/` に配置する
2. 本体リポジトリの `.opencode/` は本体プロジェクションとする
3. `.opencode/` は実行時プロジェクションとして維持する（実行時入口を提供）
4. agentdev コマンド名前空間および `agentdev-*` スキルプレフィックスを本体用予約名とする
5. 初回移行は移行スクリプトで実行し、日常同期は同期スクリプトで実行する。
   両スクリプトの責務を分ける（移行は一回限り、同期は継続的）

## 結果・影響

### 正の影響

- ソース変更は `src/opencode/` で行い、`.opencode/` はプロジェクションとして自動同期される
- 利用プロジェクトでは `.opencode/` がプロジェクトローカルのカスタマイズ入口として機能する
- 整合性検査がソースとプロジェクションを区別できる

### 負の影響

- シンボリックリンクが利用できない環境ではコピーベースのプロジェクションへのフォールバックが必要
- 開発ワークフローに同期手順が追加される
- 既存のパス参照を含む docs の更新が必要

> ※シンボリックリンク・コピーフォールバックの詳細仕様、移行手順、Windows環境リスクの詳細は REQ-0103、REQ-0108 を参照。
