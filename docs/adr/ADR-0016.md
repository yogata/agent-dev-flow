# ADR-0016: skill references runtime-only 制約

## 背景

skill の references/ ディレクトリには runtime で必要な資料と authoring（開発用）資料が混在していた。runtime 配布物が authoring-only 資料に依存しているかのような状態になっていた。

## 決定

skill references/ は runtime 配布物のみを含める。authoring-only 資料（command-authoring, skill-authoring 等）は references/ に含めない。

runtime で必要な資料: 使用条件、判定基準、テンプレート、チェックリスト等
authoring-only 資料: 作成ガイド、設計原則の詳細説明等

## 代替案

- references/ に authoring-only 資料も残す: 配布サイズの増大と runtime/authoring の境界曖昧化
- 別ディレクトリに authoring 資料を配置: 既存 references/ との二重管理リスク

## 結果・影響

- 各 skill の references/ から authoring-only 資料を除去する必要がある
- integrity-check に references/ 存在確認（runtime のみ）を追加する
- authoring-only 資料は design-principles.md や別管理領域に移譲する
