# ADR-0003: issue-req入力の抽象化

## Context

issue-req の Input / Guardrails が自然文・Issue URL・エラーログに限定されており、learning-promote が生成する promoted artifact や要件化前の設計メモなど、要件ソースとして扱うべきファイルを入力として受け取れない状態だった。また、learning-promote の出力を issue-req に渡す前提がある一方で、promoted 専用ロジックを追加すると issue-req の汎用性が損なわれる懸念があった。

## Decision

issue-req の Input に「ユーザーが明示した入力ファイル」を追加する。source file の種類に依存しない汎用的な入力扱いとし、promoted 専用分岐や専用ロジックは追加しない。

対象ファイル:
- 要件化前の設計メモ
- 調査メモ
- 反映指示書
- learning-promote が生成した promoted artifact（Requirement Source 形式）
- その他変更内容・背景・制約・完了条件を含む source file

Guardrails は以下のように修正する:
- ユーザーが明示した入力ファイルは read-only で参照可能
- docs 配下の広範な探索は禁止（例外: 明示入力ファイルと docs/requirements/** の read-only 参照）
- inbox.md / archive/active.md を直接ロードしない

## Alternatives

1. **promoted 専用分岐の追加**: learning-promote 出力のみ特別扱い → 他の入力ソースに拡張性なし
2. **入力ファイル制限の維持**: 自然文・Issue URL・エラーログのみ → promoted artifact を渡せない

## Consequences

- issue-req が汎用的な要件ソース入力を受け取れるようになる
- learning-promote の promoted artifact が Requirement Source 形式で issue-req に渡される
- inbox.md / archive/active.md の直接ロードは引き続き禁止
- promoted artifact → req-backlog（RU 生成）→ issue-req → issue-save-req → issue-create → issue-work の流れが確立（historical: req-backlog は REQ-0105-038 で廃止、コマンド名は ADR-0005 で /agentdev/* に改名）

> **歴史注記**: 上記の `req-backlog`、`issue-req`、`issue-save-req`、`issue-create`、`issue-work` は旧コマンド名・旧フロー名である。現在のフローは promoted artifact → backlog-save（RU 生成）→ req-define → req-save → case-open → case-run → case-close に移行済み（ADR-0005, REQ-0105）。
