# ADR-0107: コマンド・スキル・テンプレート・スクリプト責任分界の正式定義

## 背景

現在、一部のコマンドに詳細な状態機械、スキーマ、分類表、テンプレート選定ルール、共通安全手順が入り込んでおり、コマンド・スキル・テンプレート・スクリプトの責任分界が曖昧になっている。
artifact-boundaries.md にはコマンド、スキル、SPEC、テンプレートの 4 種が定義されているが、スクリプトの責務が未定義であり、コマンドが肥大化しやすい構造になっている。

## 決定

artifact-boundaries.md を拡張し、コマンド・スキル・テンプレート・スクリプトの 4 種アーティファクトとして再定義する。

- **コマンド**: 公開 API と高レベル実行骨格（原則 100行以内、ステップ 5〜12個）
- **スキル**: 再利用可能な判断基準と大きな状態機械
- **テンプレート**: 出力構造とプレースホルダー
- **スクリプト**: 決定的でテスト可能な処理

design-principles.md にこの責任分界を要約として記載する。
また agentdev-command-authoring スキルを新設し、コマンド作成時の基準を明確にする。

## 代替案

1. **現状維持**: artifact-boundaries.md を更新せず個別対応 → 肥大化が継続
2. **1 コマンド = 1 オーケストレーションスキルの原則**: すべてのコマンドに対応スキルを必須化 → 小さいコマンドに不要なオーバーヘッド
3. **スクリプトなしの 3 種**: 決定的処理をスキルに含める → スキルの責務が曖昧になる

## 結果、影響

- コマンドが薄くなり、責務が明確になる
- オーケストレーションスキルは状態機械、プロトコル、再利用境界単位で作成される
- 既存の良い分離（req-analysis / req-file-manager 等）は維持される
- 新規スキル: agentdev-command-authoring, agentdev-workflow-orchestration, agentdev-learning-pipeline
