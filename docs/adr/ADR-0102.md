# ADR-0102: 実行時 / 編集時 関心分離

## 背景

AgentDevFlow のコマンドフロントマターには廃止フィールドが含まれていた。これらは agent-dev-flow リポジトリの開発用メタデータであり、個別プロジェクトでコマンドを実行する際には不要である。しかし docs-check で必須検査の対象となっており、実行時と編集時の関心が混在していた。

## 決定

コマンドフロントマターの許可フィールドは description と agent のみとする（実行時配布物として扱う）。開発用メタデータは design-principles.md やスキル `references/` など agent-dev-flow リポジトリ内の管理物に配置し、実行時配布物は自己完結する。

> ※禁止フィールドの具体的一覧、整合性検査の「必須」→「存在禁止」への反転の詳細仕様は REQ-0103、REQ-0108 を参照。

## 代替案

- フロントマターに廃止フィールドを残しつつ任意化する: 廃止予定のフィールドを残すと移行コストが継続する
- 別ファイルに開発用メタデータを分離する: 管理対象が増え、整合性維持のコストが上がる

## 結果・影響

- 全コマンドフロントマターから廃止フィールドを削除する必要がある
- docs-check の該当検査を「必須」から「存在禁止」に反転させる必要がある
- REQ-0103、REQ-0108 の該当項目を更新する必要がある
- design-principles.md に分類定義を移譲する必要がある
