# ADR-0134: 配布物依存スキルの src 昇格方針

## 背景

.opencode/skills/ 配下のスキルは .gitignore で除外されている。配布物（src/opencode/commands/, src/opencode/skills/）が .opencode/skills/ 配下のスキルを参照する場合、新規 clone 環境でスキルが不在になり配布物の自己完結性（self-contained）が崩れる。既存の配布物依存スキル（japanese-tech-writing 等）は src/opencode/skills/ へトラック化済みだが、同種の参照済み未トラックスキルが他にも残存する可能性がある。

## 決定

配布物が依存するスキルは src/opencode/skills/ に昇格（配布物化）する。repo-local 専用スキル（配布物が依存しないもの）は .opencode/skills/ のまま管理し、.gitignore による除外を継続する。.gitignore ホワイトリスト方式は導入せず、src 昇格による一元管理を採用する。

## 代替案

.gitignore ホワイトリスト方式（.opencode/skills/ 配下をホワイトリストで個別にトラック管理）。不採用。src 昇格により配布物依存スキルを src/opencode/skills/ へ一元化する方が、配布物と repo-local の境界が明確になり、.gitignore の運用負荷も低減する。

## 結果、影響

配布物の自己完結性（self-contained）の原則を担保し、新規 clone 環境でのスキル不在を防ぐ。配布物依存スキルの特定は git ls-files 突合により機械的に行え、docs-check（check_integrity.ts）が未トラックスキル参照を検出して src 昇格を促す（REQ-0159-003）。repo-local 専用スキルは .opencode/skills/ で継続管理し、配布対象外を維持する。
