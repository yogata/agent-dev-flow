# ADR-0104: 実行時独立性

## 背景

実行時コマンド（個別プロジェクトで実行されるコマンド、スキル）が `docs/specs/system.md` や `docs/specs/patterns.md` に固定依存していた。SPEC の変更が実行時の動作に影響する可能性があった。またスキルの `references/` に実行時資料と編集時資料が混在し、実行時配布物に不要な編集時専用情報が含まれていた。

ADR-0015（docs/specs の実行時非依存宣言）と ADR-0016（スキル `references/` の実行時専用制約）が関連するが、個別の宣言にとどまり、実行時独立性の判断として統合する必要があった。

> ※ADR-0015/0016 統合の詳細、具体的な SPEC 参照禁止リストは REQ-0103、REQ-0108 を参照。

## 決定

実行時配布物は自己完結を原則とする:

1. 実行時コマンドの判断基準は SKILL.md 本体または `references/` に自己完結させる
2. スキルの `references/` は実行時配布物のみを含める（編集時専用資料は agent-dev-flow リポジトリ内の別領域に配置）
3. 実行時配布物は agent-dev-flow リポジトリの内部文書（SPEC/guides）から独立して動作する（SPEC/guides は ADR-0103 に従いリポジトリ内部管理文書）

## 代替案

1. SPEC を実行時配布物に含める → SPEC はリポジトリ内部文書であり配布に不適
2. SPEC のうち実行時関連部分だけを抽出 → 管理コストが増大し、切り出し漏れのリスクが高い
3. 現状維持（ADR-0015/0016 の個別宣言） → 判断として不十分

## 結果、影響

- ADR-0016（スキル `references/` の実行時専用制約）を本判断に統合し置き換える
- ADR-0015（docs/specs の実行時非依存宣言）の内容も本判断に統合する
- docs-check に実行時の SPEC 依存検出を追加可能
- case-run の薄型化が促進される（SPEC 参照の削除による行数削減）

## 関連する決定

- ADR-0018（廃止（本判断の前身））
- ADR-0016（廃止（スキル `references/` の実行時専用制約、本判断に統合））
- ADR-0015（廃止（docs/specs の実行時非依存宣言、本判断に統合））
- ADR-0102（実行時・編集時の関心分離（本判断の基盤））
- ADR-0103（文書種別責務境界（本判断と補完関係））
