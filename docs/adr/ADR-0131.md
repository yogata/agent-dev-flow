# ADR-0131: ローカル版導入方式を link mode へ統一し生成方式を廃止

## 背景

ADR-0126 は `src/opencode-local/` を生成時ソース領域とし、AI エージェントが変換プロンプトに従ってローカル版 command/skill を `.opencode/` に直接生成する方式（generate-and-place）を採用した。`generated_by` 識別子による上書き保護とジャンクション環境での生成停止を安全制約として宣言した。

ADR-0130（agentdev-gh-cli を差し替え可能な I/O 境界として確立）により、ローカル版は command/skill 全体を生成せず、agentdev-gh-cli の link 先だけを差し替えれば済む。この場合、直接生成方式の変換プロンプト・全削除再生成・`generated_by` 上書き保護は不要になり、link mode（`.opencode/` 配下を src 配下へ接続）へ統一できる。

## 決定

1. 通常版とローカル版の両方を link mode（`.opencode/` 配下を src 配下へ接続）に統一する。
2. 通常版では `.opencode/commands/agentdev/` と `.opencode/skills/agentdev-*/` を `src/opencode/` 配下へ接続する。
3. ローカル版では agentdev-gh-cli 以外を `src/opencode/` 配下へ接続し、`.opencode/skills/agentdev-gh-cli/` だけを `src/opencode-local/agentdev-gh-cli/` へ接続する。
4. ローカル版全体生成（transform/generate.md による変換プロンプト実行）を廃止する。transform/ と generation-flow.md は廃止候補とする。
5. `generated_by` 識別子による上書き保護（ADR-0126 決定 #2）は廃止する。link による接続のため上書き問題が発生しない。
6. `.opencode/` 実パス確認は、ジャンクション環境での一律停止（ADR-0126 決定 #3）から、意図した link target かどうかを確認する方式へ見直す。
7. `src/opencode/` は GitHub 版専用原本（ADR-0126 決定 #4）という不変量は維持する。ローカル版は `src/opencode/` を変更せず、agentdev-gh-cli のみ `src/opencode-local/` から接続する。

## 結果・影響

### 正の影響

- 通常版とローカル版の導入方式が統一される。導入・更新方式が二系統化しない。
- 変換プロンプトの品質検証が不要になる。原本がそのまま接続されるため変換品質問題が消滅する。
- agentdev-gh-cli 差し替えだけでローカル版が構成できる。

### 負の影響

- 変換プロンプト資産（transform/generate.md, review.md）が不要になる。
- runtime-package-boundary.md の consumer-generated 定義（生成された実ディレクトリ・非ジャンクション）を更新する必要がある。
- ADR-0126 の決定 #1〜#4 を覆すため、履歴の理解に ADR-0131 の参照が必要になる。

## 関連する決定

- supersedes: ADR-0126（ローカル版 OpenCode 生成基盤: source model 拡張と生成安全性制約）
- relates-to: ADR-0105（OpenCode ソース・プロジェクション分離）
- relates-to: ADR-0130（agentdev-gh-cli を差し替え可能な I/O 境界として確立）

## 判断根拠

ADR-0126 の決定 #1〜#4 を根本的に覆す変更であり、UPDATE ではなく supersede が適切。決定の根幹（生成方式→link 方式）が覆るため、ADR-0126 の APPEND/UPDATE では対応できない。

ローカル版導入方式の根本転換は「将来の設計・運用・文書システムを制約する決定」（`agentdev-adr-guidelines` False Negative 防止基準）である。link mode 自体は可逆だが、現行契約（REQ-0141-007 直接生成、032 変換スクリプト不使用、033 全削除再生成）と変換プロンプト資産を破棄する点で、文書体系・運用モデルの根本変更であり、ADR で判断根拠を残さないと履歴が失われる。

link mode 採用の宣言は REQ-0134（REQ-0103-156〜160）へ、要件の詳細は REQ-0141 へ、導入方式の技術詳細は SPEC（runtime-package-boundary.md, local-generation.md）へ配置する。
