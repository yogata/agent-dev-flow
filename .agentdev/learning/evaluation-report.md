### 問題クラス1: Windows環境でのgh CLI日本語文字化け

- **根本原因**: Windows環境（PowerShell 7）の標準出力パイプライン・ファイルI/Oがgh CLIのUTF-8出力を正しくハンドリングせず、Shift-JISやBOM付きUTF-8への変換が発生する
- **再発条件**: Windows環境でgh CLIの`--body-file`、`--json`出力、`>`リダイレクトをPowerShell経由で扱い、日本語を含む場合
- **予防策**: (1) gh CLI読み取りはPython subprocess経由でUTF-8バイトとして直接取得、(2) 書き込みは`gh api --field body=@file`を使用、(3) PowerShellの`>`リダイレクトと`--body-file`の組み合わせを禁止するルールをagentdev-gh-cliスキルに明記
