# Node.jsテンプレートリテラルのクォート競合回避策追加

## 背景

Windows環境で`node -e`内にテンプレートリテラル（バッククォート）を含むJavaScriptコードを渡す際、PowerShell・Node.js・JQ式のクォート階層が競合しパースエラーが発生する。agentdev-gh-cliには既にクォート競合回避策（軽量回避策・退避策）が記載されているが、テンプレートリテラルの具体的な禁止・推奨事項がない。

## 問題

agentdev-gh-cliのWindows回避策セクションにテンプレートリテラルに関する具体的なガイダンスがないため、一時スクリプト内でテンプレートリテラルを使用した際にPowerShellのクォート解釈と競合する。

## 望ましい変更

agentdev-gh-cliのWindows回避策セクション（Section 3 項目7付近）に以下を追加:
- 一時スクリプト（`.js`ファイル）内ではテンプレートリテラルの使用を禁止せず、インライン`node -e`内でのテンプレートリテラル使用を禁止する
- 推奨替代: 配列の`join('\n')`を使用した文字列構築

## 制約

- `.js`スクリプトファイル内でのテンプレートリテラル使用は禁止しない（クォート競合が発生しないため）
- `node -e`インライン内でのテンプレートリテラル使用のみを禁止する
- 配列joinへの替代は可読性を損なわない範囲で推奨
