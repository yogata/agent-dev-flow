# 新旧REQ適用運用ルールの文書化

## 背景

REQ要件行の「振る舞い単位記述」と「Step番号参照の排除」のガイドをPR #975で追記したが、既存REQへの遡及適用は別Issueの範囲とした。「既存REQは現状維持・新規REQから適用」の運用ルールがあちこちで暗黙仮定されているが、この運用ルールが文書化されていない。整備時に暗黙に後方互換性を損なうリスクがある。

## 問題

「既存REQは現状維持・新規REQから適用」の運用ルールが AGENTS.md / REQ-0101 / ガイドに文書化されていない。要件行記述内容（REQ-0102-006/007/023）は AGENTS.md L41 に定義済みだが、新ガイドラインの適用タイミング（新規のみ vs 既存遡及）が不明示。

## 望ましい変更

AGENTS.md または REQ-0101 に「新ガイドライン（要件行記述基準・HOW除去等）は新規REQから適用し、既存REQは別途整備Issueがない限り現状維持とする」運用ルールを明文化する。

## 対象範囲

### 対象
- `AGENTS.md`（編集ガードレールセクション）
- または `docs/guides/` 配下の REQ 運用ガイド

### 対象外
- 既存REQ ファイルの遡及修正（別Issue範囲）
- REQ-0102-006/007/023 の要件行記述基準自体の変更
