# command-authoring テンプレート3層防御パターンの明文化

## 背景

「テンプレートに従って生成」という指示だけではエージェントがテンプレート構造を無視する問題が発生（評価スコア 31/40）。必須セクションの明示（SPECIFY）と検証ゲート（VERIFY）がないとテンプレート準拠が担保されない。

## 問題

コマンド定義は「テンプレートに従って生成」と指示するのみで、(1) テンプレートに必須/任意の区別がなく、(2) コマンドに必須セクション一覧がなく、(3) 検証ゲートがないため、エージェントがテンプレート構造を無視した出力を生成する。

## 望ましい変更

`agentdev-command-authoring` スキルに以下を反映:
- テンプレート使用箇所すべてに READ→SPECIFY→VERIFY の3層防御パターンを明文化
- コマンド設計原則として「テンプレート参照時は必須セクション一覧を明示し、検証ゲートを設ける」を追加
- Guardrails にテンプレート準拠チェック要件を追加

## 対象範囲

### 対象

- `.opencode/skills/agentdev-command-authoring/SKILL.md`

### 対象外

- 各コマンドの個別 Guardrails（本スタブは設計原則の明文化のみ）
- テンプレートファイル自体（<!-- 【必須】 --> マーカーは既に追加済み）
