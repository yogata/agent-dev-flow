# テスト戦略セクションのスコープ管理改善

## 背景

case-close の G08 完了ゲートが完了条件・テスト戦略両セクションの全 `[ ]` を検出して停止するが、case-open でテスト戦略に「実装 PR で達成不可能なテスト項目」が含まれることがあり、case-close が停止する問題が 2 回発生した。根本原因は case-open がテスト戦略を生成する際に「実装 PR で達成可能か」を判定していないことにある。

## 問題

1. case-open がテスト戦略セクションに実装 PR のスコープ外のテスト項目（E2E テスト等）を含める
2. case-run がスコープ外の `[ ]` チェックボックスをそのまま残し、case-close G08 でエラー停止する
3. case-open にテスト戦略項目の「実装 PR で達成可能性」判定基準がない

## 望ましい変更

- case-open のテスト戦略生成時に「実装 PR で達成可能か」を判定基準に含める
- 達成不可能なテスト項目はテスト戦略セクションに含めない、または「別途確認」注記付きのテキスト（チェックボックス以外）として出力する
- case-run の完了 Step でスコープ外の `[ ]` を検出した場合、チェックボックス形式を外すまたは削除する

## 対象範囲

### 対象

- `.opencode/commands/agentdev/case-open.md` Step 2（Issue 本文生成・テスト戦略セクション）
- `.opencode/commands/agentdev/case-run.md` Step 7（チェックボックス更新・完了確認）

### 対象外

- case-close Step 2（G08 自体の挙動変更は含まない）
- テンプレートファイル自体の変更

## 反映先候補

| 種別 | パス | 変更内容 |
|------|------|----------|
| command | `.opencode/commands/agentdev/case-open.md` | Step 2 にテスト戦略出力時の「実装 PR で達成可能か」判定基準を追加。達成不可能な項目はチェックボックス形式にしない |
| command | `.opencode/commands/agentdev/case-run.md` | Step 7 にスコープ外 `[ ]` の検出・処理（テキスト形式への変換または削除）を追加 |

## 既存対策確認

- **確認結果**: 既存対策あり（部分的）
- **該当ファイル**: `.opencode/commands/agentdev/case-close.md` Step 2（自律解決判定）
- **ギャップ分類**: fix gap
- **ギャップ詳細**: case-close に自律解決判定（検証適用不能・代替検証実施）の仕組みは存在するが、根本原因である case-open のテスト戦略出力品質にフィルタがない。case-run にもスコープ外項目のクリーンアップがない

## 制約

- G08 の完了ゲート自体（全 `[ ]` を検出）の挙動は変更しない。G08 は正しいセーフティネットとして機能している
- テスト戦略セクション自体を削除するのではなく、内容を適切にスコープ管理する
- E2E テスト等の重要な項目は「別途確認」として残す（情報の損失を防ぐ）

## 受け入れ条件

- [ ] case-open にテスト戦略出力時の「実装 PR で達成可能か」判定基準が記載されている
- [ ] 達成不可能なテスト項目はチェックボックス形式（`- [ ]`）以外の形式で出力される
- [ ] case-run にスコープ外 `[ ]` の検出・処理 Step が追加されている

## 元learning item / 根拠

- **要約**: case-open が達成不可能なテスト項目を含め、case-close G08 が停止する
- **根拠**: 2件の実発生（Issue #381/PR #382, Issue #393/PR #394）。G08 がテスト戦略のスコープ外 `[ ]` を検出してエラー停止。case-open が E2E テスト等をテスト戦略に含めた
- **再発条件**: case-open がテスト戦略にスコープ外項目を含め、case-close を実行する場合
- **横展開可能性**: AgentDevFlow パイプラインを使用する場合に発生可能
