# RU: Command frontmatter 禁止キー除去とフォーマット規約違反の網羅確認

## 背景

Command frontmatter に、現行フォーマット規約では使用しない `load_skills` や `implementation_pattern` が残存している。

確認済みの対象として、`case-open` / `case-run` / `case-update` / `case-close` に `load_skills` が残り、`intake-promote` / `learning-promote` に `implementation_pattern` が残っている。

ユーザーは、これらについて該当部分だけを除去し、あわせて全フォーマット規約を対象に、違反する全ファイルを網羅的に確認することを求めている。

## 問題

Command frontmatter に現行規約外のキーが残ると、以下が発生する。

- command 定義と integrity-check の判定基準が不一致になる
- 古い運用要素が command 定義上に残り続ける
- 形式違反の検出対象が一部の既知キーに偏り、他のフォーマット規約違反が見落とされる

今回の修正は、`load_skills` / `implementation_pattern` の除去だけで完了してはならない。全フォーマット規約を対象に、違反ファイルを網羅確認する必要がある。

## Source Summary

チャット内で合意済みの内容は以下。

- `load_skills` / `implementation_pattern` は command frontmatter から除去する
- 除去は該当キー部分のみに限定する
- 同一趣旨の禁止キーが他にも残っていれば確認対象に含める
- 全フォーマット規約を対象に、違反する全ファイルを網羅的に確認する
- 未承認の保存、commit、push は行わない

## 統合理由

`load_skills` / `implementation_pattern` の除去は局所修正だが、同時に「全フォーマット規約違反の網羅確認」が求められている。

このため、単なる command frontmatter cleanup ではなく、以下を一体の RU として扱う。

- 既知の command frontmatter 禁止キー除去
- 同種の禁止キー残存確認
- Command 以外を含むフォーマット規約違反の網羅確認
- 修正対象と報告対象の切り分け

## 要件化の方向

本RUは、既存の integrity / validation / command format 関連REQへの APPEND または UPDATE として扱う。

要件化では、以下を明文化する。

1. Command frontmatter では、現行規約外キーを使用しない
2. 既知の `load_skills` / `implementation_pattern` は該当キーのみ削除する
3. `secondary_pattern` など同一カテゴリの禁止キーが存在する場合も違反として検出する
4. 全フォーマット規約を対象に、違反ファイルを網羅確認する
5. 修正対象と、確認のみ・別RU候補の不整合を分けて報告する
6. ユーザー未承認の保存、commit、push は行わない

- Issue 作成

## 受け入れ条件

- `load_skills` が残存していた command frontmatter から、該当キーのみ削除されている
- `implementation_pattern` が残存していた command frontmatter から、該当キーのみ削除されている
- `secondary_pattern` など同一カテゴリの禁止キーが存在する場合、違反として検出・報告されている
- 既知の6 command ファイル以外にも、全 command frontmatter が確認されている
- Command 以外のフォーマット規約も対象に確認されている
- 違反ファイル一覧が、修正済み・未修正・別RU候補に分けて報告されている
- 修正差分は、ユーザー依頼に直結する該当キー除去に限定されている
- 隣接する無関係な記述、体裁、表現が変更されていない
- 保存、commit、push はユーザー承認後にのみ実施される
