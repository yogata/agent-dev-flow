# RU: runtime command から非必須参照記述を除去する

## 背景
AgentDevFlow の runtime command は、実行時にエージェントへ直接読まれる主要プロンプト資産である。現状、一部 command には Step や Guardrail に直接結び付かない補助的な参照記述が含まれており、実行骨格に寄与しないトークン消費が発生している。

本セッションでは、トークン最適化の観点から「真に必要不可欠な記述以外は削除する」方針が合意された。また、個別 command への対症対応ではなく、リポジトリ全体へ適用する前提で、別エージェントが安全に実行できる計画を策定する必要があることが確認された。

## 問題
runtime command に、以下のような実行骨格外の補助記述が混在しうる。

- 独立した補助参照一覧
- Step に統合されていない「念のため」の参照
- 実行タイミングや判断条件が本文内で閉じていない覚え書き

これらは command を薄い dispatcher として保つ方針と緊張し、runtime 実行時のトークン効率を低下させる。一方で、無差別削除を行うと Step や Guardrail の成立に必要な参照根拠まで失うリスクがある。

## Source Summary
本セッションで以下が合意された。

- `Manual References` のような独立補助節は中途半端である
- 真に必要な参照は Step または Guardrail に組み込むべきである
- 念のために残すだけの記述は不要である
- トークン最適化の観点から、真に必要不可欠な記述以外は削除する
- 適用対象は repo 全体だが、実行は別エージェントが行えるように計画化する
- 無差別削除ではなく、runtime command を一次対象として段階的に進める

## 統合理由
本件は単なる文言整理ではなく、runtime command の責務境界とトークン効率を揃えるための要件である。個別 command ごとの判断に委ねると、補助参照の残存基準が揺れ、command ごとの品質差と再発を招く。repo 全体で一貫した削除基準と実行手順を持たせることで、runtime prompt の最小化と安全な保守を両立できる。

## 要件化の方向
runtime command を対象に、実行骨格に直接寄与しない補助参照記述を除去する。必要な参照は Step または Guardrail に統合し、独立した補助参照節や覚え書きは残さない。実施は repo-wide cleanup として扱うが、影響範囲と安全性の観点から対象面を段階化し、まず `src/opencode/commands/agentdev/*.md` を一次対象とする。
