# RU: 既存配布物の導入先未解決参照を段階解消する

## 背景

AgentDevFlow 配布対象の command 定義および skill 定義は、導入先プロジェクトで解決可能な参照のみを含む必要がある。

チャット内では、既存の配布物領域に多数の導入先未解決参照が残っていることが共有された。提示値では、`src/opencode/commands/agentdev/**/*.md` に 138 件、`src/opencode/skills/agentdev-*/**/*.md` に 114 件、合計 252 件の違反が存在する。

件数はチャット内提示値であり、保存前の全量再検証値ではない。ただし、少なくとも配布対象 command / skill に REQ/ADR ID 固定参照が残っている問題は合意済みである。

## 問題

既存配布物内に、導入先プロジェクトで解決不能な参照が多数残っている。

これにより、配布物が本体リポジトリ内部文書、repo-local command / skill、原本側パス、内部 docs に依存し、導入先で自己完結しない。

一括置換で処理すると、文意喪失、根拠消失、実行時判断基準の欠落、command / skill / SPEC の責務混線が発生する可能性が高い。

## Source Summary

チャット内で以下に合意した。

- 既存違反解消は、検出ルール追加とは別 RU として扱う。
- 既存違反は cleanup ではなく、既存原則違反の段階解消として扱う。
- 一括修正ではなく、docs-check report / intake / backlog 経由で領域別に処理する。
- command / skill からは REQ/ADR ID などの固定参照を除去する。
- 実行時に必要な判断基準は、配布物内で自己完結させる。
- 根拠IDや設計根拠は `docs/specs/{commands,skills}/` 側へ寄せる。
- PR #1318 由来の追加分は既存252件全体とは分離し、増分違反の先行修正として扱う。

## 統合理由

既存違反は複数ファイル・複数種別に分散しているが、根本原因は「配布物が導入先で解決できない内部参照を保持している」ことにある。

そのため、REQ/ADR ID、`src/opencode/`、`/repo/*` などの個別パターンごとに即時修正 RU を分けるのではなく、段階解消の運用単位として統合する。

ただし実作業は一括ではなく、検出結果を元に領域別・ファイル群別に分割する。

## 要件化の方向

既存配布物内の導入先未解決参照を、docs-check report / intake / backlog を経由して段階的に解消する。

対象領域は以下とする。

- `src/opencode/commands/agentdev/**/*.md`
- `src/opencode/skills/agentdev-*/**/*.md`
- `src/opencode/skills/agentdev-*/references/**/*.md`
- `src/opencode/skills/agentdev-*/SKILL.md`

解消方針は以下とする。

- 配布 command / skill から REQ/ADR ID 固定参照を除去する。
- 配布 command / skill から `src/opencode/` 参照を除去し、導入先で解決可能な `.opencode/` 相対パスまたはシンボリックパスへ置換する。
- 配布 command / skill から `/repo/*` command 参照および `repo-*` skill 参照を除去する。
- 配布 command / skill から本体 docs / SPEC / guide への実行時依存参照を除去する。
- 実行時に必要な判断基準は SKILL.md 本体または references/ に自己完結させる。
- 根拠ID、設計根拠、履歴説明、SPEC 的詳細は `docs/specs/{commands,skills}/` または内部 docs 側に保持する。
- command / skill 本文では、ID 参照ではなく意味名・実行時ルール名で表現する。

- 検出ルール自体の新設

- PR #1318 由来の増分違反のみの先行修正
- 文意を確認しない機械的一括置換

- 実装手順の詳細化
- 会話URLの記録

## 受け入れ条件

- docs-check report または同等の検出結果から、既存配布物内の導入先未解決参照が一覧化される。
- 既存違反が領域別またはファイル群別に intake / backlog 化される。
- 一括置換ではなく、意味保持を確認できる粒度で段階解消される。
- command / skill から REQ/ADR ID 固定参照が除去される。
- command / skill から `src/opencode/` 参照が除去される。
- command / skill から `/repo/*` および `repo-*` 参照が除去される。
- command / skill から導入先で存在保証のない本体 docs / SPEC / guide 参照が除去される。
- 実行時に必要な判断基準は配布物内に自己完結している。
- 根拠ID、設計根拠、履歴情報は内部 docs / SPEC 側に残り、配布物側に固定依存として残らない。
- 修正後、導入先プロジェクトで解決不能な参照が新規に増えていないことを docs-check または delta guard で確認できる。
