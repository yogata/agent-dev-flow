# RU: req-define 実装フェーズ誤遷移防止

## 背景

今回のセッションログ分析では、`/agentdev/req-define` 実行中に、要件整理フェーズを超えて作業計画生成および実装修正へ横滑りしたことが確認された。

現行 `req-define` は、要件整理・要件doc生成を担う壁打ちフェーズであり、出力は `.sisyphus/drafts/req-draft-{topic-slug}.md` またはセッション内要件docに限定される。

`docs/**`、`src/opencode/**`、`.agentdev/**`、`.sisyphus/plans/**` の編集や、後続工程の代行は責務外である。

## 問題

`req-define` 実行中に、ユーザーがファイル内容・ログ・コマンド定義・現状分析を提示した場合、agent がそれを実装開始意図として誤認し、後続工程へ遷移する余地がある。

この結果、以下の問題が発生する。

- `req-define` が `.sisyphus/drafts/**` 以外を作成・編集する
- `req-define` が `.sisyphus/plans/**` に作業計画を生成する
- `req-define` が `docs/**` または `src/opencode/**` を編集する
- 明示的な後続コマンド実行なしに、保存・Issue作成・実装修正・PR作成へ進む
- 後続工程の責務境界を通らないまま変更が発生する

## Source Summary

チャット内で以下を合意した。

- 真の再発防止対象は `req-define` から実装フェーズへの誤遷移である
- `req-define` に実装遷移禁止ゲートを追加する必要がある
- `req-define` は `.sisyphus/drafts/**` 以外を編集してはならない
- `docs-check` または repo-local integrity に、`req-define` のフェーズ逸脱検査を追加する必要がある
- ユーザーがファイル内容・ログ・コマンド定義を貼っただけでは、実装開始意図として扱わない

## 統合理由

本RUは、command phase boundary の逸脱防止を扱う。

`req-define` は要件定義フェーズ、後続コマンドは保存・Issue作成・実行・完了処理を担う。今回の不具合は、この責務境界を越えて `req-define` が後続工程へ進んだことにある。

したがって、REQ-0102 を主対象として `req-define` の責務境界を補強し、REQ-0108 に検査を追加するのが妥当である。

## 要件化の方向

### req-define の実装遷移禁止

`req-define` は、ユーザー入力またはセッション文脈から実装開始意図を推定してはならない。

ユーザーがファイル内容、diff、ログ、コマンド定義、現状コード、分析メモを貼った場合でも、それだけを根拠に実装・保存・Issue作成・PR作成へ進んではならない。

後続工程の実行には、明示コマンドを必要とする。

### req-define の編集範囲制限

`req-define` は `.sisyphus/drafts/**` 以外のファイルを作成・編集・削除してはならない。

明示的に対象外とする。

- `.sisyphus/plans/**`
- `docs/**`
- `src/opencode/**`
- `.agentdev/**`

### plan 生成禁止

`req-define` は作業計画、実装タスク、subagent 実行計画、PR/Issue 作成計画を生成してはならない。

必要な後続作業は、要件docの後続候補または関連ドキュメント更新候補として保持し、実行は後続コマンドに委譲する。

### implementation intent 検出時の停止

`req-define` 実行中に agent が implementation intent を検出した場合、実装せず停止する。

停止時には以下を報告する。

- 現在の要件doc状態
- 実装に進むために必要な明示コマンド
- `req-define` 自身では後続工程を代行しないこと

### integrity 検査

`docs-check` または repo-local integrity は、`req-define` command 定義に以下が含まれていないことを検査する。

- `.sisyphus/plans/**` への出力許可
- `docs/**` への編集許可
- `src/opencode/**` への編集許可
- `.agentdev/**` への編集許可
- 後続工程の代行記述
- implementation intent を根拠に実装へ遷移する記述

## 主対象REQまたは変更対象候補

### 主対象REQ

- `REQ-0102`
  - `req-define` の責務・出力・フェーズ境界を定義する要件として APPEND する

### 変更対象候補

- `src/opencode/commands/agentdev/req-define.md`
  - Guardrails に実装遷移禁止を追加
  - `.sisyphus/drafts/**` 以外の編集禁止を強化
  - implementation intent 検出時の停止動作を追加
  - 後続工程は明示コマンドに委譲することを明記

- `REQ-0108`
  - `req-define` のフェーズ逸脱を検出する integrity / docs-check 要件を APPEND

- repo-local integrity / docs-check 関連定義
  - `req-define` に禁止された編集先・後続工程代行記述が混入していないことを検査する

## 対象外

- 後続コマンドの責務変更
- Issue 階層判定ロジックの変更
- RU 削除条件の変更
- 後続工程の `req-define` への取り込み
- `.sisyphus/plans/**` を `req-define` の正式出力にする変更
- 実装手順・コード差分案

## 受け入れ条件

- `REQ-0102` に、`req-define` が実装開始意図を推定して後続工程へ進んではならない要件が追加されている
- `REQ-0102` に、`req-define` の編集可能範囲が `.sisyphus/drafts/**` に限定される要件が追加されている
- `REQ-0102` に、`req-define` が `.sisyphus/plans/**`、`docs/**`、`src/opencode/**`、`.agentdev/**` を作成・編集・削除してはならない要件が追加されている
- `src/opencode/commands/agentdev/req-define.md` の Guardrails に、実装遷移禁止、後続工程代行禁止、明示コマンド必須が記載されている
- `src/opencode/commands/agentdev/req-define.md` に、implementation intent 検出時は実装せず停止し、次に実行すべき明示コマンドを報告する手順が記載されている
- `REQ-0108` または対応する integrity 定義に、`req-define` のフェーズ逸脱記述を検出する検査要件が追加されている
- `req-define` から後続工程を代行できる記述が残っていない
