## Sources

`gh-cli-windows-encoding.md`（learning）より:

- **観測**: case-auto 自走中（Issue #932 / PR #933）に Windows PowerShell 環境から gh CLI の `--body-file` / `--title` で日本語を含む Issue 本文を作成した際、GitHub 上の本文が文字化け。原因調査の結果、一時ファイル自体は UTF-8 (BOM なし) で正しく書かれていたが、PowerShell のコンソール出力エンコーディングが shift_jis（chcp 932）だったため、gh CLI がファイル読み込み・引数渡し時にエンコーディング変換を起こした。READ 操作（Node.js execSync）では再発せず、WRITE 操作（`--body-file` / `--title`）でのみ発生。
- **ギャップ**: `agentdev-gh-cli` SKILL Section 1（禁止事項）と Section 2（標準手順）には `[System.IO.File]::WriteAllText` で UTF-8 (BOMなし) を書き出す規定はあるが、コンソールエンコーディング初期化手順が不在。ファイルが UTF-8 でもコンソールが shift_jis の場合に文字化けする。
- **影響**: Windows 環境デフォルトコンソールエンコーディングが shift_jis のため、AgentDevFlow を Windows で利用する全ユーザーに影響。case-open / case-close 等、gh CLI を呼ぶ全コマンドで潜在的に発生。
