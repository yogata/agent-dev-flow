## Source Summary

ADR体系はREQ体系より整理されているが、以下の是正が必要。

1. ADR-0113 は `diagnostics-*` 導入と review 系コマンド削除を主題にしており、現行の `inspect-*` 方針および「命名移行はADR対象外」という判断と矛盾する。
2. ADR-0110 は DOC-MAP を探索入口にする意思決定としては妥当だが、表題・決定本文で `requirements/views 廃止` が主題化している。
3. ADR README の Current Baseline View が `accepted/proposed` と説明しながら `superseded` の ADR-0111 を含めている。
4. ADRの現行根拠は accepted ADR に限定し、deprecated / superseded ADR は履歴参照へ落とす必要がある。

## 統合理由

これは個別ADR本文の文言修正ではなく、ADRの現行根拠セットと文書種別責務境界を整える再編成である。REQ再編成とは独立に実行できるが、REQ-0101 / REQ-0112 / document-model のADR境界と整合させる必要がある。

4. ADR境界の検査・参照整合を確認する。
   - accepted ADR に、削除・廃止・移行・完全削除そのものを主題にしたものが残っていないことを確認する。
