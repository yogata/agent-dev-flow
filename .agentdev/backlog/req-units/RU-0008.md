# RU: Agent Skills 仕様に合わせた `references/` 統一

## 背景

AgentDevFlow の command / skill 定義において、skill 配下の参照資料ディレクトリ名が `reference/` と `references/` で混在している。

Agent Skills 公式仕様では、skill directory の任意ディレクトリとして `references/` が定義されている。Directory structure、Optional directories、File references の各説明で `references/` が使われている。

現行 AgentDevFlow では、以下の混在がある。

- `agentdev-skill-authoring` は `reference/` を構造例として記述している
- `agentdev-workflow-lifecycle` は `reference/ 構成一覧` を持つ
- `agentdev-workflow-reporting` は `reference/ 構成一覧` を持つ
- `agentdev-workflow-routing` は `reference/ 構成一覧` を持つ
- `agentdev-no-ai-slop-writing` は `reference/forbidden-phrases.md` を参照している
- `agentdev-command-authoring` は `references/command-authoring-standards.md` を参照している
- `agentdev-learning-capture` は `references/capture-entry-template.md` と `references/example.md` を参照している
- `integrity-check` / `agentdev-integrity` は `reference/*.md` と `references/*.md` の両方を正規スキャン対象として扱っている

## 問題

Agent Skills 仕様上の正規ディレクトリ名である `references/` と、AgentDevFlow 内部の `reference/` が不一致である。

このため、skill authoring 基準、既存 skill の実体配置、command からの参照、integrity-check の検査対象定義が一意になっていない。

また、`integrity-check` が `reference/` と `references/` の両方を正規構造として扱っているため、仕様不一致を検出できない。

一部 command は `capture-boundaries.md` などを裸ファイル名で参照しており、どの skill 配下の参照資料かが不明確である。

## Source Summary

- Agent Skills 公式仕様では、任意ディレクトリとして `references/` が定義されている
- AgentDevFlow 内部には `reference/` と `references/` の両方が存在する
- `reference/` は AgentDevFlow 内部の旧・誤規約として扱う
- `references/` を正規構造として統一する

## 統合理由

本件は単なる typo 修正ではなく、以下を一体で扱う必要がある。

- Agent Skills 仕様への準拠
- skill authoring 基準の修正
- 既存 skill 配下ディレクトリの移動
- command / skill 内の参照パス更新
- integrity-check の検査対象定義変更
- 旧構造 `reference/` 残存検出の追加

個別に分けると、移行途中に `reference/` と `references/` が再び併存し、integrity-check が正しく検出できないため、一体のRUとして扱う。

## 要件化の方向

AgentDevFlow の skill 配下参照資料ディレクトリ名を `references/` に統一する。

`reference/` は旧構造として扱い、正規の artifact category から外す。既存の `reference/` 配下ファイルは `references/` に移動する。

`integrity-check` は `references/` を正規構造としてスキャンし、`reference/` が残っている場合は obsolete-structure として検出する。

## 主対象REQまたは変更対象候補

主対象は REQ-0108 または REQ-0103 の APPEND / UPDATE 候補。

- REQ-0108: integrity-check の検査観点・対象構造に関する変更
- REQ-0103: command / skill 構造・AgentDevFlow command protocol に関する変更

既存REQ照合の結果、より適切な対象REQがある場合はそちらへ APPEND / UPDATE する。
