# RU: summary系 docs の鮮度維持と docs-check coverage 拡張

## 背景

`guides/`、`DOC-MAP`、`adr/README.md`、一部 SPEC には、利用者向けの索引や要約として canonical 内容の再掲が存在する。

これらは利便性が高い一方、active REQ 範囲、ADR 定義、status 語彙、タイトル要約が authoritative source とずれやすい。

## 問題

`project-docs-and-specs.md`、`glossary.md`、`patterns.md`、`system.md`、`adr/README.md` に stale な range・定義・status・タイトルが残っている。

`docs-check` の REQ range 鮮度検査は一部ファイルしか見ておらず、summary / index 文書の drift を十分に捕捉できていない。

## Source Summary

- guides と DOC-MAP は navigation layer であり、canonical 内容の代替ではない。
- それでも summary / index として最小限の再掲は行われており、ここに stale 情報が残ると誤誘導になる。
- docs-check は一部 summary doc の stale REQ range を見るが、coverage に穴がある。
- ADR README のタイトル・要約も本文との同期が崩れうる。

## 統合理由

本件は個別修正文書が複数にまたがるだけでなく、summary / index の drift を継続検知する docs-check coverage 拡張が必要である。

`REQ-0108` と `REQ-0115` へ APPEND し、summary doc の機械検査と docs-review route を整備するのが妥当である。

## 要件化の方向

以下を APPEND する。

- `docs-check` は summary / index 文書における active REQ range / count、旧 status 語彙、旧 command 名、旧 ADR 定義の残存を検査する。
- 検査対象には少なくとも `project-docs-and-specs.md`、`glossary.md`、`patterns.md`、`system.md` の summary 節、`docs/adr/README.md` を含める。
- `docs-check` で機械検
査しにくい summary 過多や説明過剰は docs-review の route 対象として提示する。
- summary / index 文書は canonical table の複製を増やしすぎない方針を明文化する。

## 対象外

以下は本RUの対象外とする。

- DOC-MAP 自体の役割再設計
- guides 全廃や summary 文書の全面削除
- README 群の情報設計を一からやり直すこと

## 受け入れ条件

- summary / index 文書の stale REQ range / count / 定義 / 旧語彙を docs-check が検出できる。
- 対象 summary / index 文書の stale 情報が authoritative source と同期されている。
- summary / index の説明過多や canonical 重複について、docs-review の route が明示されている。
