# inspect 系コマンドへの git 永続化フェーズ追加と git pull 方式の --ff-only 統一

## 背景

`.agentdev/` は README と AGENTS.md で git 管理対象と明記されている（domain state）。intake 系（`intake-capture`, `intake-promote`）と `learning-promote` は、成果物を `.agentdev/` 配下に出力した後に git pull → commit → push の「Git & Completion」フェーズを実装し、作業ツリーに未コミットファイルを残留させない運用となっている。

一方、inspect 系3コマンド（`inspect-docs`, `inspect-skills`, `inspect-promote`）は `.agentdev/inspect/` 配下に finding や promoted artifact を生成するが、git commit / push を一切行わない。特に `inspect-skills` は Guardrail G04 で「commit / push / branch / worktree 操作を行わない」と明示的に禁止しており、`agentdev-inspect-skills` SKILL.md の「Read-Only 制約」も同一の禁止を記述している。

チャット内で、inspect 系を実行した後に `.agentdev/inspect/` 配下がコミットされていなかった事象を確認。intake/learning とは実装に差があるか調査し、是正方針を合意した。

## 問題

- `inspect-docs`, `inspect-skills` を実行すると `.agentdev/inspect/inbox/` に finding が生成されるが、未コミットのまま作業ツリーに残留する。複数セッション・複数人で実行すると競合や不整合の温床になる。
- `inspect-promote` を実行すると inbox → promoted / archive/rejected へのファイル移動が発生するが、これも commit されない。
- `inspect-skills` G04 と `agentdev-inspect-skills` SKILL.md「Read-Only 制約」が「commit / push 禁止」を明示しており、read-only 診断の原則と `.agentdev/` の git 管理性が矛盾している。
- `learning-promote` の git pull が `--rebase`、`intake-*` が `--ff-only` で不揃い。共通 template `templates/common/git-error-messages.md` の「Git 同期エラー（pull --ff-only 失敗）」形式と learning だけ不整合。
- `templates/inspect-skills/standard.md`, `templates/inspect-promote/standard.md` が存在せず、`inspect-skills.md` Step 6 / `inspect-promote.md` Step 7 が「完了報告 template に従って出力」と参照しているのと不整合。`check_integrity.ts` の完了報告フォーマット検証でも検知対象。

## Source Summary

チャット内で以下を合意した。

- inspect 系3コマンド（inspect-docs / inspect-skills / inspect-promote）には git pull / commit / push のフェーズが存在せず、`.agentdev/inspect/` 配下が未コミットで残留する。
- intake-capture（Step 3-1, 4-1 直書き）、intake-promote（Phase 5 で `agentdev-intake-pipeline` スキルへ委譲）、learning-promote（Phase 6 Step 11, 15 直書き）は全て git pull → commit → push を持つ。
- 是正方針は **A案（全 inspect コマンドへ一律追加）**。read-only 原則を「副作用 = 成果物生成 + その git 永続化」へ再定義する。intake-capture が「保存専用」を謳いながら commit/push している前例と整合。
- git pull 方式は **`--ff-only` で統一**。`learning-promote` Step 11 の `--rebase` を `--ff-only` に変更する。

## 統合理由

1:1（単一論点）。inspect 系 git 永続化追加と git pull 方式の `--ff-only` 統一は、いずれも「`.agentdev/` 配下の git 永続化の一貫性回復」という単一目的。分離しても独立した要件として成立しないため、単一の Requirement Unit として扱う。

## 要件化の方向

req-define は以下を検討する:

1. **inspect-docs.md**: Steps に「実行前同期（`git pull --ff-only`）」と「`.agentdev/inspect/` 変更の commit と push」を追加。commit message は `chore(agentdev): capture inspect-docs finding`。commit 対象は `.agentdev/inspect/` のみ。共通 template `templates/common/git-error-messages.md` を参照。基本原則の副作用定義を「finding 生成 + その git 永続化」に拡張。
2. **inspect-skills.md**: 同構造の git セクション追加。commit message は `chore(agentdev): capture inspect-skills finding`。**G04 を「commit / push は `.agentdev/inspect/` 配下の永続化のみ許可。branch / worktree 操作は禁止」に修正**。基本原則の副作用定義を拡張。
3. **inspect-promote.md**: HITL 承認後のファイル操作（promoted / archive/rejected 移動、inbox 削除）後に git commit する構造を追加。commit message は `chore(agentdev): promote inspect findings`。`git add .agentdev/inspect/` で追加/変更/削除を一括 index 化（ファイル移動も表現）。
4. **agentdev-inspect-skills/SKILL.md**: 「Read-Only 制約」セクションの「commit, push を行わない」を「`.agentdev/inspect/` 配下の永続化のみ許可」に整合修正。branch/worktree/Issue/PR 作成の禁止は維持。
5. **templates/inspect-docs/standard.md**: 行11 `git 永続化: 該当なし` を `git 永続化: commit: {hash}, push: {成功/失敗}` に変更。
6. **templates/inspect-skills/standard.md 新設**: intake-capture / learning-promote と同形式（結果・検証結果・git 永続化・次のコマンド セクション）。
7. **templates/inspect-promote/standard.md 新設**: 同上。
8. **learning-promote.md**: Step 11 の `git pull --rebase` を `git pull --ff-only` に統一。
9. **制約**: intake-capture / intake-promote / learning-promote の git セクションの構造リファクタリングは行わない（学習 `--rebase → --ff-only` の変更のみ）。`agentdev-inspect-pipeline` 等 pipeline スキルの新設は別件。CI/CD（`.github/workflows/`）導入も別件。

work_type は enhancement（maintenance）を想定。自己適用リポジトリ内のコマンド・スキル・テンプレート整合性の是正。

## 受け入れ条件

- `inspect-docs` / `inspect-skills` / `inspect-promote` を実行後、`.agentdev/inspect/` 配下の変更が commit / push されること。
- commit message が `chore(agentdev): capture inspect-docs finding` / `capture inspect-skills finding` / `promote inspect findings` の形式であること。
- commit 対象が `.agentdev/inspect/` 配下のみであり、他のパスを巻き込まないこと。

- 各コマンドの基本原則の副作用定義が「成果物生成 + その git 永続化」に拡張されていること。
- `templates/inspect-docs/standard.md` の git 永続化欄が `commit: {hash}, push: {成功/失敗}` を含むこと。
- `templates/inspect-skills/standard.md` と `templates/inspect-promote/standard.md` が新設され、intake-capture / learning-promote と同形式（結果・検証結果・git 永続化・次のコマンド セクション）を含むこと。

- read-only 診断の本質（canonical docs / REQ / ADR / SPEC / Command / Skill / Template / Script の変更禁止、Issue / PR 作成禁止、worktree / branch 操作禁止、自動修正禁止、intake / learning / RU 処理禁止）は維持されていること。
