# RU: skill reference に残る禁止参照・旧用語・旧パスを除去する

## 背景

`src/opencode/skills/*/references/*.md` は、配布対象 skill の一部として runtime に読み込まれる詳細参照ファイルである。

現行方針では、配布対象 command / skill / reference に以下を残さない。

- 固定 `REQ-NNNN` / `ADR-NNNN` 参照
- 他 skill の内部 `references/*.md` path や Section 番号への直接依存
- command 固有 Step 番号への skill 側依存
- 廃止済み namespace / 旧 path / 旧運用例
- agent-dev-flow 管轄外とする `.sisyphus/` への実行導線

`src/opencode/skills/*/references/*.md` について、SKILL.md から宣言されている reference 13件を確認した結果、複数の残存が確認された。

## 問題

skill reference に禁止参照・旧用語・旧パスが残ると、以下が発生する。

- consumer project に存在しない REQ / ADR への解決不能参照が配布される
- skill 間の責務境界が崩れ、他 skill の内部 Section / reference path に依存する
- command 固有 Step 番号が skill reference に固定され、command 更新時に skill reference が drift する
- `.sisyphus/` に agent-dev-flow が関与する実行手順として読める
- 旧 namespace / 旧運用例が現行導線として誤読される

確認済みの主な残存は以下である。

- `agentdev-git-worktree/references/worktree-operations.md`
  - 固定 `REQ-0117-001`
  - Issue / PR 番号を含む履歴参照
  - `.sisyphus/` cleanup 手順
- `agentdev-learning-capture/references/example.md`
  - 固定 `REQ-0106`
  - `retired ADR-0003`
  - `docs/requirements/REQ-0106.md`
  - `git worktree remove -f` を自律対応例・予防策として記載
- `agentdev-backlog-integration/references/integration-judgment.md`
  - 他 skill 内部 reference path `agentdev-workflow-lifecycle/references/upstream-handoff.md`
- `agentdev-intake-pipeline/references/intake-promotion.md`
  - 他 skill 内部概念 `Split Rule` への直接依存
- `agentdev-issue-management/references/issue-operation-safety.md`

  - `.sisyphus/tmp/` 一時ファイル path
- `agentdev-workflow-orchestration/references/self-healing-and-errors.md`
  - command 固有 Step 番号
  - `Section 1`
  - 旧 namespace 例 `/issue/*`, `/tips/*`, `issue-*`, `tips-*`, `reference/`
  - `.sisyphus/` grep 除外指定
- `agentdev-git-worktree/references/git-common-procedures.md`

## 統合理由

検出内容はいずれも、配布対象 skill reference の参照境界・用語境界・runtime path 境界の同一問題である。

個別ファイルごとに分けるより、`src/opencode/skills/*/references/*.md` を対象とする1つの整理単位として扱う方が、後続の `req-define` で既存 REQ-0103 系の責務境界・配布境界要件と照合しやすい。

## 要件化の方向

`src/opencode/skills/*/references/*.md` から、配布対象 reference として不適切な固定参照・内部依存・旧語・旧 path を除去または現行表現へ置換する。

要件化では、以下を変更後仕様として扱う。

- skill reference は固定 `REQ-NNNN` / `ADR-NNNN` を含まない
- skill reference は他 skill の内部 `references/*.md` path に依存しない
- skill reference は他 skill の Section 番号に依存しない
- skill reference は command 固有 Step 番号を安定契約として保持しない
- skill reference は旧 namespace / 廃止 path を現行導線として記載しない
- skill reference は `.sisyphus/` を agent-dev-flow 管轄の作業領域として扱う実行導線を持たない
- 例示が必要な場合は、固定IDではなく `{REQ-ID}`、`{ADR-ID}`、`{issue_number}`、`{pr_number}` 等のプレースホルダーを使う
- 禁止語・旧語を説明する必要がある場合は、検出ルール・履歴説明・禁止条件として誤読されない形に限定する

## 主対象REQまたは変更対象候補

主対象候補:

- `REQ-0103`: command / skill / template / script の責務境界、配布対象 artifact の参照境界、skill 内部参照境界

副対象候補:

- `REQ-0101`: `.sisyphus/` 管轄外・文書体系境界
- `REQ-0104`: workflow / command protocol と `.sisyphus/` 非SSoT境界
- `REQ-0108`: integrity 検査観点としての残存参照検出

既存要件に同等の禁止が存在する場合、新規REQ作成ではなく既存REQへの UPDATE / APPEND または実体修正の対象として扱う。
