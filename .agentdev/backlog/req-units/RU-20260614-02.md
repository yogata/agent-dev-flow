# RU: AgentDevFlow 本体向け handoff marker の明確化

## 背景

consumer repository で AgentDevFlow 配布物由来の不具合・改善要求を発見することがある。

この場合、consumer repository 固有の要件として Issue 化・実装するのではなく、AgentDevFlow 本体へ取り込む対象として扱う必要がある。

現行では `apply_in_current_project: false` という metadata が使われているが、命名が否定形であり、AgentDevFlow 本体へ取り込むことが自明ではない。

## 問題

`apply_in_current_project: false` は、現在 repository で処理しないことは示すが、AgentDevFlow 本体へ渡すことを明示しない。

また、通常 RU にまで routing 用 metadata を記載すると、RU frontmatter が不要に肥大化する。

handoff 対象ではない通常 RU に `false` を明示する運用も不要であり、通常 RU と handoff RU の区別を曖昧にする。

## Source Summary

チャット上で以下を合意した。

- `apply_in_current_project` は命名不良であり、仕様バグとして扱う。
- `upstream_handoff: true` は汎用的すぎる。
- `agent-dev-flow: true` は、何が true なのか不明である。
- `agentdev_handoff: true` を採用する。
- `agentdev_handoff: true` は、consumer repo で AgentDevFlow 本体側の修正が必要な場合だけ RU frontmatter に記載する。
- 通常 RU には `agentdev_handoff` を記載しない。
- `agentdev_handoff: false` も記載しない。
- AgentDevFlow 本体内で見つけた AgentDevFlow 本体の仕様バグは handoff ではなく、自プロジェクト内の通常要件として扱う。

## 統合理由

この変更は、consumer repository と AgentDevFlow 本体 repository の責務境界を明確にする。

`agentdev_handoff: true` を routing marker として定義することで、consumer repository では停止・報告し、AgentDevFlow 本体では通常 workflow 入力として処理する分岐を明確にできる。

また、通常 RU には marker を記載しないことで、frontmatter を通常要件の情報に限定できる。

## 要件化の方向

`apply_in_current_project: false` を正式な routing marker として扱わない。

AgentDevFlow 本体向け handoff を示す metadata として、`agentdev_handoff: true` を定義する。

`agentdev_handoff: true` は条件付き marker であり、通常 RU / 要件doc の必須 frontmatter ではない。

consumer repository の workflow は、`agentdev_handoff: true` を検出した場合、現在 repository で Issue 化・実装せず、AgentDevFlow 本体への handoff 対象として報告する。

AgentDevFlow 本体 repository の workflow は、`agentdev_handoff: true` を停止条件として扱わない。取り込み元の履歴 metadata として扱い、通常の req/case workflow 入力として処理する。

## 主対象REQまたは変更対象候補

主対象候補:

- workflow contract 系 SPEC
- case-open の入力判定・停止条件
- upstream handoff 関連 reference

変更対象候補:

- `apply_in_current_project` を参照する command / skill / reference
- RU frontmatter または handoff metadata の説明箇所

## 対象外

以下は対象外とする。

- 複数 upstream repository を扱う汎用 routing 機構
- `upstream_repository` の RU frontmatter 追加
- `consumer_repository_action` の RU frontmatter 追加
- 通常 RU への `agentdev_handoff: false` 付与
- AgentDevFlow 本体内の通常仕様バグを handoff として扱うこと
- Issue 作成、PR 作成、実装手順の策定

## 受け入れ条件

- `agentdev_handoff: true` が AgentDevFlow 本体向け handoff marker として定義されている。
- `agentdev_handoff` は通常 RU / 要件doc の必須 frontmatter ではないと定義されている。
- `agentdev_handoff: false` は通常時に記載しないと定義されている。
- consumer repository では `agentdev_handoff: true` を検出した場合に Issue 化・実装せず停止し、AgentDevFlow 本体への handoff 対象として報告すると定義されている。
- AgentDevFlow 本体 repository では `agentdev_handoff: true` を停止条件として扱わず、通常の req/case workflow 入力として処理すると定義されている。
- `apply_in_current_project` を正式 marker として使う記述が残らない。
- `upstream_repository` / `consumer_repository_action` を RU frontmatter に追加する前提になっていない。
