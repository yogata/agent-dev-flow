## Source Summary

integrity-check の検査ロジックが、現行ドキュメント構造と例外対象を十分に反映していない。結果として、実不整合ではない項目が NG / Warning として大量に報告され、レポートの優先度判断を難しくしている。

## 統合理由

以下は同じ「validator の判定ルールと現行 artifact 形式のズレ」に属するため、1つの RU として扱う。

- REQ README index 抽出条件の不一致
- completion report registry 形式の解釈不一致
- migration table における retired REQ 参照の除外不足
- レポート上の大量過検出によるノイズ増大

これらを個別に修正すると、別カテゴリの過検出が残り続け、integrity-check の出力を意思決定に使いにくい状態が継続する。

## 要件化の方向

REQ-0108（Integrity / Validation / Tests）への APPEND / UPDATE 候補として扱う。

integrity-check は、現行 artifact の正規形式を検査ルールに反映し、過検出を抑制する必要がある。

### 要件化候補

- `docs/requirements/README.md` の active REQ index は、Markdown table 内のリンク形式 `[REQ-NNNN](REQ-NNNN.md)` を正規に抽出すること。
- completion report の存在確認は、個別 `## {command}` セクションではなく、現行の `Variant Registry` と `completion-reports/{command}/{variant}.md` 実体の整合性を確認すること。
- `docs/requirements/mapping-table.md` は retired REQ 参照検査の例外対象とし、旧REQ移行対応表としての参照を Warning 扱いしないこと。
- retired REQ 参照検査は、履歴・移行・retired index 目的の文書と、現行要件として参照している文書を区別すること。
- integrity report は、検査ロジック由来の疑義がある場合に `recommendation: integrity-check-gap` などの分類を明示すること。
