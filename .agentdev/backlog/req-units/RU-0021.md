# RU: Command / Skill 外部ファイル参照の存在検査と template パス正規化

## 背景

AgentDevFlow の public commands は `.opencode/commands/agentdev/` に配置され、skills は `.opencode/skills/` に配置される。

Command 固有の完了報告テンプレートは、command-local template として以下に配置する。

```markdown
.opencode/commands/agentdev/templates/{command}/{variant}.md
```

Issue / PR / コメント本文テンプレートは、引き続き以下に配置する。

```markdown
.opencode/skills/agentdev-workflow-templates/templates/
```

## 問題

複数の command 定義で、完了報告 template を以下のように相対参照している。

```markdown
templates/{command}/{variant}.md
```

この表記は、実行時に `.opencode/skills/agentdev-workflow-templates/templates/` 配下を探索する誤動作を誘発する。

実体ファイルは command-local template として存在しているため、欠落ファイルを skill-local templates 側に新規作成してはならない。

また、現行の `agentdev-integrity` は、Command / Skill 定義内の明示的な外部ファイル参照を全件抽出し、参照先実体と照合する検査を十分に持っていない。

## Source Summary

この RU は、以下のチャット合意に基づく。

- 完了報告 template の参照は command-local template の repo 相対パスへ統一する
- `templates/{command}/{variant}.md` の曖昧な相対参照は使用しない
- `.opencode/skills/agentdev-workflow-templates/templates/` に完了報告テンプレートを追加して解決しない
- `integrity-check` に Command / Skill 外部ファイル参照の存在検査を追加する
- 参照欠落時は、該当 file / line / evidence / expected を含む NG として報告する

## 統合理由

同じ根本原因を持つため、1 RU に統合する。

根本原因は、Command / Skill 定義に書かれた外部ファイル参照の解決規則が不明確であり、integrity-check がその不整合を機械的に検出できていないこと。

個別 command の template パス修正だけでは、同種の参照欠落が再発する。したがって、既存参照の正規化と integrity-check の検査拡張を同一 RU で扱う。

## 要件化の方向

REQ-0103（Artifact責任分界）または REQ-0108（Integrity / Validation / Tests）への UPDATE / APPEND 候補とする。

方向性は以下。

1. Command-local template 参照の canonical path を明確化する
2. Command / Skill 定義内の外部ファイル参照を integrity-check の検査対象に追加する
3. 存在しない参照、曖昧な template 参照、旧配置前提の参照を NG として検出する

## 主対象REQまたは変更対象候補

### 主対象REQ候補

- REQ-0103: Command / Skill / Template / Script の責務境界、template 配置契約
- REQ-0108: integrity-check の検査カテゴリ、参照存在検査、レポート schema

### 変更対象候補

- `.opencode/commands/agentdev/*.md`
- `.opencode/skills/agentdev-integrity/SKILL.md`

- 必要に応じて `docs/specs/artifact-contracts.md`

## 要件候補

### 1. Command-local template 参照の正規化

AgentDevFlow の command 定義は、完了報告 template を参照する場合、以下の repo 相対パスを使用すること。

```markdown
.opencode/commands/agentdev/templates/{command}/{variant}.md
```

以下の曖昧な相対参照は禁止する。

```markdown
templates/{command}/{variant}.md
```

### 2. 既存 command の完了報告 template 参照修正

以下の command 定義内の完了報告 template 参照を command-local full path に修正する。

- `req-define.md`
- `req-save.md`
- `case-open.md`
- `case-run.md`
- `case-update.md`
- `case-close.md`
- `intake-capture.md`
- `intake-from-github.md`
- `intake-review.md`
- `intake-promote.md`
- `learning-refine.md`
- `learning-promote.md`
- `backlog-review.md`
- `backlog-save.md`
- `req-restructure-review.md`
- `integrity-check.md`

### 3. integrity-check の参照存在検査追加

`agentdev-integrity/scripts/check_integrity.ts` は、以下を対象に明示的な外部ファイル参照を抽出し、参照先の存在を検査すること。

- `.opencode/commands/agentdev/*.md`
- `.opencode/skills/*/SKILL.md`
- `.opencode/skills/*/references/*.md`
- `.opencode/skills/*/scripts/*`
- `.opencode/skills/*/templates/*.md`
- `.opencode/commands/agentdev/templates/**/*.md`

### 4. 抽出対象

以下を外部ファイル参照として抽出する。

- Markdown link: `[text](path)`
- backtick path: `` `path/to/file.md` ``
- `template: ...`
- `templates/...`
- `references/...`
- `scripts/...`
- `.opencode/...`
- `docs/...`

### 5. 参照解決ルール

Command 定義内の参照解決は以下とする。

- `.opencode/...` は repo root から解決する
- `docs/...` は repo root から解決する
- `templates/{command}/{variant}.md` は NG とする
- 期待値は `.opencode/commands/agentdev/templates/{command}/{variant}.md`

Skill 定義内の参照解決は以下とする。

- `.opencode/...` は repo root から解決する
- `docs/...` は repo root から解決する
- `references/...` は当該 skill directory から解決する
- `scripts/...` は当該 skill directory から解決する
- `templates/...` は当該 skill directory から解決する

### 6. 除外または特別扱い

以下は個別ファイル存在検査の対象外とする。

- glob: `*.md`, `**/*.md`
- placeholder: `{topic-slug}`, `{NNNN}`, `{command}`, `{variant}`
- runtime-generated artifact: `.agentdev/...`, `.sisyphus/...`
- GitHub Issue / PR 番号参照
- `/agentdev/...` コマンド名

ただし、glob / placeholder は親ディレクトリの存在確認対象とする。

## 対象外

- `.opencode/skills/agentdev-workflow-templates/templates/` 配下への完了報告テンプレート複製
- runtime-generated `.agentdev/` ファイルの個別存在保証
- `.sisyphus/` 配下の個別存在保証
- GitHub Issue / PR / branch など外部状態の存在保証
- Command / Skill の責務再設計
- template 本文の表現改善

## 受け入れ条件

- 全 command 定義の完了報告 template 参照が `.opencode/commands/agentdev/templates/{command}/{variant}.md` に統一されている
- `.opencode/skills/agentdev-workflow-templates/templates/` 配下に command 完了報告テンプレートを追加していない
- `integrity-check` が command-local template の存在を検査する
- `integrity-check` が command / skill 定義内の明示外部ファイル参照を検査する
- 存在しない参照は file / line / evidence / expected を含む NG として報告される
- `templates/{command}/{variant}.md` の相対表記は NG になる
- 既存の Issue / PR / コメントテンプレート参照は `.opencode/skills/agentdev-workflow-templates/templates/` として有効に扱われる
- glob / placeholder / runtime-generated artifact は過検出しない
- 追加した検査に対するテストが存在する
