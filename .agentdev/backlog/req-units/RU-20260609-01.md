# repo整合性問題の抜本是正

## 背景

docs、command、skill の整合性修正を繰り返した結果、現状には実不整合、docs-check / repo-agentdev-integrity の過検出、repo-agentdev-integrity 自体の検査漏れが混在している。

特に repo-agentdev-integrity が Windows junction 配下の `.opencode/skills/agentdev-*` を十分に検査できていない可能性があり、現在の検査結果をそのまま整合性の根拠として扱えない。

## 問題

- learning domain state の canonical path が REQ と command / skill の間で衝突している。
- learning promoted artifact の後処理が、backlog-review による削除と learning-pipeline 側の archive 移動で衝突している。
- retired ADR / ADR status の扱いに、現行根拠と履歴参照の境界揺れが残っている。
- active REQ 範囲、`.sisyphus/` 命名規則、配布対象 reference 内の固定 REQ/ADR ID に stale または配布境界違反がある。
- repo-agentdev-integrity が placeholder、番号帯、historical / supersedes、打ち消し線付き廃止履歴、禁止チェックリストを current-use finding として過検出している。
- repo-agentdev-integrity が Windows junction、repo-local 例外、reference 再帰スキャン、vocabulary registry 同期、report 再スキャン分離を十分に扱えていない。

## Source Summary

チャット内で、以下を合意済みの分析結果として整理した。

- 実不整合:
  - `REQ-0105` と `learning-promote` / `agentdev-learning-pipeline` の learning path が不一致。
  - learning promoted artifact の lifecycle が削除と archive 移動で不一致。

  - ADR status の旧形式案内が残る。

  - 配布対象 skill reference に self-hosting 固有の固定 REQ/ADR ID が残る。

  - retired ADR の historical / supersedes 文脈。

  - `req-backlog` / `Pattern A/B/C/D` の禁止チェックリスト例示。

  - Windows junction が `listDirs` で落ちる。

## 統合理由

これらは個別の文言修正だけでは再発を防げない。
docs の実不整合と、repo-agentdev-integrity の検査基盤不備が相互に影響しており、先に検査基盤を修正し、その後に docs / command / skill の意味整合を確認できる状態へ戻す必要がある。

## 要件化の方向

- repo-agentdev-integrity の junction 対応、repo-local 例外、reference 再帰スキャン、report 分離、historical 文脈判定、vocabulary registry 同期を強化する。
- learning domain state と promoted artifact lifecycle を command / skill 実体に合わせて正規化する。
- ADR status と retired ADR 参照の現行根拠 / 履歴参照境界を明確化する。
- active REQ 範囲、`.sisyphus/` 命名規則、配布対象 reference の固定 ID を更新または抽象化する。
- placeholder / 番号帯 / 禁止チェックリスト / 廃止履歴を current-use finding から除外する文脈判定を追加する。
