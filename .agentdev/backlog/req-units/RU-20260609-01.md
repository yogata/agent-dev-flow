# Command / Skill のトークン削減と分割基準の整理

## 背景

`src/opencode/commands/agentdev/` と `src/opencode/skills/` の command / skill について、サイズ・概算トークン量・既存目安基準の妥当性を評価した。

評価では、既存基準の方向性はおおよそ妥当と確認した。一方で、行数だけでは日本語 Markdown や長文行の重さを過小検出し、短い診断項目が多い command を過剰警告する可能性があることも確認した。

## 問題

一部の command / skill では、通常実行時に不要な詳細手順、例、エラー本文、conflict 対応、同型分岐の重複が本体に含まれている。

これにより、以下の問題がある。

- command / `SKILL.md` のホットパス読み込み量が大きい
- 本体と reference / skill 間で同じ invariant が繰り返される
- 分岐ごとの差分ではなく同型手順が重複記述されている
- `SKILL.md` が概要・判断基準・ナビゲーション以上の詳細を抱えている
- 分割しても毎回全文を読む場合、総トークン量削減にはならない

## Source Summary

この RU はチャット内で合意した以下の内容に基づく。

- 既存のサイズ目安基準は、おおよそ妥当である
- ただし行数基準だけでは不十分で、token / bytes の基準を併用する必要がある
- 分割だけでは総トークン量は減らない
- 分割に意味があるのは、詳細を条件付きで読むことで通常時のホットパス読み込み量を下げられる場合である
- セマンティクスを変えずに、重複表現・長い理由説明・同型分岐・例の過多を削ることで総量削減も可能である
- 高優先の削減対象は以下である
  - `agentdev-skill-authoring`
  - `agentdev-req-file-manager`
  - `agentdev-gh-cli`
  - `agentdev-learning-pipeline`
  - `case-open`
  - `req-save`
  - `learning-promote`
  - `agentdev-epic-tracker`

## 統合理由

本件は個別ファイルの単純な文面削減ではなく、command / skill 本体と `references/` / `templates/` の責務分離、ホットパス読み込み量削減、共通 invariant の集約をまとめて扱う必要がある。

そのため、個別の軽微修正ではなく、AgentDevFlow の command / skill authoring 方針に沿った一連の要件として統合する。

## 要件化の方向

以下の方向で要件化する。

- command 本体は、入力、出力、高レベル Steps、Guardrails、完了報告 variant に絞る
- `SKILL.md` は、概要、判断基準、読み先ナビゲーションを中心にする
- 詳細手順、長い判定表、Good/Bad例、エラー本文、conflict 対応、schema、archive rule は `references/` または `templates/` に分離する
- 分離した reference は、常時読み込みではなく条件付き読み込みにする
- 共通 invariant は重複記述せず、既存 skill / reference に集約する
- 同型分岐は個別 step の重複ではなく、共通手順 + 差分表で表現する
- セマンティクスを変えない範囲で、理由説明、例、SHALL/MUST の反復、長文表現を削減する
- token / line の before / after を計測し、`command-main` / `skill-main` / `skill-support` を分けて報告する

## 対象外

以下は対象外とする。

- command / skill の実行セマンティクス変更
- lifecycle state、command 名、skill 名の新設
- workflow の順序変更
- Issue 作成、PR 作成、既存 Issue / PR へのコメント
- 実装コードの変更
- `.opencode/` runtime projection の直接編集
- `reference/` 単数形ディレクトリの新設
- 単なる見た目の整形
- セマンティクス変更を伴う要件削除
- 全 reference を毎回読む前提の分割

## 受け入れ条件

- 高優先4 skill の `SKILL.md` が原則として 5,000 tokens 未満になる、または未達の場合は理由が明示されている
- `agentdev-skill-authoring/SKILL.md` が 500行未満になる
- `case-open.md` と `req-save.md` が 3,500 tokens 未満を目標に削減される、または未達の場合は理由が明示されている
- 分割された詳細は `references/` または `templates/` に配置される
- command / `SKILL.md` から、分割先を読む条件が明示されている
- `reference/` 単数形の新規参照が増えていない
- command 本文の runtime path は `.opencode/...` 形式を維持している
- `VERIFY`、`--body-file`、template必須セクション検証などの共通 invariant が不要に重複していない
- before / after の概算 token と行数が、`command-main` / `skill-main` / `skill-support` に分けて報告されている
- セマンティクス維持の確認メモが残されている
- 変更が `src/opencode/` の command / skill authoring 改善に直結する範囲に限定されている
