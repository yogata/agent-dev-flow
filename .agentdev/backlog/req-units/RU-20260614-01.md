# RU: agent-dev-flow 管理ドラフトを `.agentdev/drafts/` に統一し `.sisyphus/` 関与を排除する

## 背景

agent-dev-flow には、要件定義・レビュー・保存などのコマンド間で一時的または中間的なドラフトファイルを扱う場面がある。

これらのドラフトは agent-dev-flow の処理対象であり、agent-dev-flow の管理領域に配置される必要がある。

一方、`.sisyphus/` 階層は oh-my-openagent / Sisyphus / Prometheus などの harness 側が利用し得る領域であり、agent-dev-flow の仕様・運用・ライフサイクル管理対象ではない。

## 問題

現行 docs 内に、agent-dev-flow が扱うドラフトの保存先・参照先・コマンド間受け渡し先として `.sisyphus/` または `.sisyphus/drafts/` を想起させる記述が残ると、以下の問題が発生する。

- agent-dev-flow 管理領域と harness 管理領域の責務境界が曖昧になる
- req-define / req-save などの agent-dev-flow コマンド間ドラフトの配置先が不明確になる
- skill-review などのレビュー結果・是正ドラフトをどこに置くべきかが揺れる
- agent-dev-flow が `.sisyphus/` 配下のライフサイクルに関与するかのように読める

## Source Summary

本 RU は、チャット内で合意した以下の内容に基づく。

- agent-dev-flow が扱うドラフトファイルは `.agentdev/drafts/` に配置する
- req-define / req-save など、agent-dev-flow コマンド間で受け渡すドラフトは `.agentdev/drafts/` を使う
- skill-review のレビュー結果・是正ドラフトも、agent-dev-flow がファイルとして扱う場合は `.agentdev/drafts/` 配下を候補とする
- `.sisyphus/` は oh-my-openagent / Sisyphus / Prometheus など harness 側の管理領域であり、agent-dev-flow は生成・保存・参照・削除・保持期間・ライフサイクルに関与しない
- agent-dev-flow docs では `.sisyphus/` 階層そのものに原則言及しない
- `.sisyphus/` への言及を README 等で「存在するが対象外」と説明する案は提案レベルであり、本 RU では必須要件化しない

## 統合理由

本件は、単なるパス置換ではなく、agent-dev-flow と外部 harness の責務境界を明確にする文書体系上の修正である。

そのため、`.sisyphus/drafts/` を `.agentdev/drafts/` に一律置換するだけでは不十分である。agent-dev-flow が扱うドラフトについては `.agentdev/drafts/` に統一し、agent-dev-flow が関与しない `.sisyphus/` 階層の説明は docs から原則除去する必要がある。

## 要件化の方向

### 1. agent-dev-flow 管理ドラフトの配置先統一

agent-dev-flow が生成・保存・再読込・コマンド間受け渡しするドラフトファイルは `.agentdev/drafts/` 配下に配置する。

対象には少なくとも以下を含める。

- req-define から req-save へ渡す要件ドラフト
- req-save 前の保存対象ドラフト
- skill-review の結果をもとにした是正検討ドラフト
- agent-dev-flow コマンドが後続コマンドへ渡す中間ドラフト

### 2. `.sisyphus/` 階層への agent-dev-flow 関与排除

agent-dev-flow は `.sisyphus/` 配下を以下の対象として扱わない。

- 生成先
- 保存先
- 入力元
- コマンド間受け渡し先
- ライフサイクル管理対象
- 削除・保持・清掃対象
- agent-dev-flow 成果物の canonical または draft source

### 3. docs 内の `.sisyphus/` 言及整理

`docs/*` において、agent-dev-flow の仕様・運用・ドラフト・成果物管理として `.sisyphus/` または `.sisyphus/drafts/` に言及している箇所は削除または `.agentdev/drafts/` へ修正する。

修正は文脈別に行う。

- agent-dev-flow のドラフト保存先を示す記述は `.agentdev/drafts/` へ修正する
- agent-dev-flow のコマンド間受け渡し先を示す記述は `.agentdev/drafts/` へ修正する
- `.sisyphus/` の lifecycle、保持、削除、管理方針を agent-dev-flow docs が説明している記述は削除する
- oh-my-openagent / Sisyphus / Prometheus の内部仕様説明としての `.sisyphus/` 記述は、agent-dev-flow docs から原則削除する

### 4. `.sisyphus/` 対象外説明の扱い

`.sisyphus/` は agent-dev-flow の対象外である。

ただし、README 等で「存在するが agent-dev-flow の対象外」と説明することは、実装時の提案事項に留める。本 RU では、その説明を必須とはしない。

## 対象外

以下は本 RU の対象外とする。

- `.sisyphus/` 配下の作成、削除、保持期間、清掃、命名規則
- oh-my-openagent / Sisyphus / Prometheus 側の harness 実装
- `.sisyphus/drafts/` を harness が利用することの禁止
- agent-dev-flow 外部の runtime 作業領域設計
- `docs/*` 以外の実装修正
- Issue 作成、PR 作成、既存 Issue / PR へのコメント

- `docs/*` 内で、agent-dev-flow のドラフト保存先として `.sisyphus/` または `.sisyphus/drafts/` を指定している記述が残っていない

- `docs/*` 内で、agent-dev-flow が `.sisyphus/` 配下のライフサイクルに関与するように読める記述が残っていない

- `.sisyphus/` への対象外説明を残すかどうかは、本 RU の必須達成条件にしない
