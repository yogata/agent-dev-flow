# RU: REQ/ADR/SPEC 意味的不整合の横断是正

## 背景

本セッションでは、`docs/requirements/`、`docs/adr/`、`docs/specs/` を中心に、REQ / ADR / SPEC 間の意味的不整合を確認した。確認観点は、retired 文書の現行参照、active set の stale、ADR baseline の stale、旧 workflow 語彙、artifact owner の衝突、docs-check / integrity の read-only / write 境界、SPEC scope の衝突、guide / DOC-MAP の案内ずれである。

現行文書体系では、active REQ 一覧は `docs/requirements/README.md`、current ADR baseline は `docs/adr/README.md`、active SPEC 一覧は `docs/specs/README.md` を正本とする。retired REQ / ADR は履歴参照のみ許可し、現行根拠として扱わない。

## 問題

REQ / ADR / SPEC および関連案内文書に、現在の正本状態と矛盾する記述が残っている。主な問題は、retired 済み REQ / ADR の現行根拠化、active REQ set の古い固定範囲、ADR baseline の欠落、旧 workflow 語彙の残存、artifact / template / promoted artifact owner の衝突、docs-check / integrity 境界の衝突、SPEC scope の衝突、guide / DOC-MAP / glossary の案内ずれである。

## 統合理由

個別不整合は複数ファイルに分散しているが、根本原因は、active / retired の正本、ADR baseline、SPEC index、workflow 再基準化、artifact owner、docs-check / integrity 移管が参照側文書へ伝播していないことである。fixed list を複数文書に重複保持しているため、index と各文書が乖離している。したがって、個別ファイルごとの局所修正ではなく、REQ / ADR / SPEC の正本関係を基準に横断是正する Requirement Unit として統合する。

## 要件化の方向

1. active REQ set は `docs/requirements/README.md` を正本とし、固定範囲・固定件数・重複 active list を正本参照へ寄せる。
2. ADR baseline は `docs/adr/README.md` を正本とし、ADR-0112 accepted / ADR-0111 superseded を反映する。
3. active SPEC 一覧は `docs/specs/README.md` を正本とし、SPEC 内の固定 Living Specs 一覧を同期または参照化する。
4. retired REQ / ADR は履歴参照のみ許可し、active REQ / ADR / SPEC の現行根拠として参照しない。
5. Pattern A/B/C/D、maintenance-D、旧 Phase / Step、英字サブステップは現行仕様参照として残さない。
6. learning-promote の出力語彙は `promoted artifact` に統一する。
7. Command Template / workflow template / completion report template の owner を文書間で一致させる。
8. docs-check の read-only / write 境界を「検査対象 artifact 非変更」と「report / intake 生成の許可」に分けて定義する。
9. integrity severity taxonomy を一系統に統一する。
10. SPEC の scope は repo-internal 方針と衝突しないよう整理する。
11. guide / DOC-MAP / glossary は現行 REQ / command / namespace の案内に同期する。
12. 変更は意味的不整合の是正に限定し、無関係な表現改善・全面リライトは行わない。
