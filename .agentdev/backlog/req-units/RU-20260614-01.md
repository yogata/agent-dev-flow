# RU: Skill の See Also を補助導線に限定し、実行判断に必要な関連参照を本文へ移す

## 背景

各 skill の `SKILL.md` には、関連 skill や補助参照を示す `See Also` 相当の記述が存在する。

しかし、skill の関連性が実行判断、責務境界、禁止条件、委譲先、停止条件に関わる場合、末尾の関連一覧だけでは実行時の判断材料として不十分である。

`See Also` は読者またはエージェントに対する補助的な発見導線としては有効だが、実行時に参照すべき規則そのものを代替してはならない。

## 問題

現行 skill には、以下の問題が残る可能性がある。

- `See Also` が、単なる関連一覧を超えて、委譲先や責務境界を暗黙に表している
- `DO NOT USE FOR` 等に既に書かれている関連先が `See Also` に重複している
- 実行判断に必要な関連性が本文ではなく末尾導線に逃げている
- skill が全コマンド一覧など、別SSoTで管理すべき一覧を保持して stale 化しうる
- command / skill 間の関係が、内部 section 名、protocol 名、Step 名、Phase 名への依存に戻る余地がある

この状態では、エージェントが `See Also` を実行判断の根拠と誤認し、責務外の処理、過剰な参照、または参照漏れを起こす。

## Source Summary

本セッションでは、`See Also` は関連 skill の発見導線としては使えるが、実行判断・責務境界・委譲先・禁止条件を充足するものではないと合意した。

また、全 skill を対象に同様の観点で確認する必要があるとし、特に `agentdev-workflow-lifecycle/SKILL.md` では以下が修正候補として整理された。

- `DO NOT USE FOR` と `See Also` の重複
- lifecycle 判定 skill 内に全コマンド一覧を保持していることによる stale リスク
- ラベル体系や upstream handoff 判定など、本文側の判定要約が不足すると充足しない可能性
- `See Also` を委譲・判断根拠として扱う構造の排除

## 統合理由

本RUは、個別 skill の表現修正ではなく、skill 全体に共通する参照境界と責務記述の整理である。

対象は `See Also` という見出しそのものではなく、関連参照が実行判断を代替してしまう構造であるため、全 skill の `SKILL.md` を横断して扱う必要がある。

## 要件化の方向

Skill の関連参照は、以下の基準で整理する。

- `See Also` は補助的な発見導線に限定する
- 実行判断に必要な関係は、`USE FOR`、`DO NOT USE FOR`、責務境界、または該当ルール本文に記述する
- 委譲先は、単なる関連一覧ではなく、どの条件で委譲するかを本文に明記する
- 禁止条件は `See Also` に置かず、`DO NOT USE FOR` または同等の責務境界に置く
- skill 内部の section 名、protocol 名、Step 名、Phase 名を、他 command / skill が直接依存する参照として扱わない
- 全コマンド一覧など別SSoTで管理すべき一覧は、skill から除去するか、責務上必要な範囲に限定する

## 主対象REQまたは変更対象候補

主対象候補:

- `REQ-0103`
  - 参照境界ルールの拡張
  - 自然言語ラベル、skill 内部構造、`See Also` への実行判断委譲を禁止する方向

副対象候補:

- `REQ-0108`
  - integrity / docs-check 観点として、skill の `See Also` による充足不足、重複、責務外一覧保持を検出する方向

変更対象候補:

- `src/opencode/skills/**/SKILL.md`
- repo-local skill の `SKILL.md`
- skill authoring / integrity 関連の仕様またはガイド

## 対象外

以下は対象外とする。

- skill の実装ロジック変更
- command の処理順序変更
- Issue 作成
- PR 作成
- 未確認の個別 skill 内容を根拠にした断定的な削除
- `See Also` 見出しそのものの一律廃止
- 関連 skill への補助導線としての `See Also` の禁止
- 会話URLの記録

## 受け入れ条件

- 全 skill の `SKILL.md` が確認されている
- `See Also` が実行判断、禁止条件、委譲先、停止条件を代替していない
- 実行判断に必要な関連性が、該当本文、`USE FOR`、`DO NOT USE FOR`、責務境界のいずれかに記述されている
- `DO NOT USE FOR` 等と `See Also` の重複が、補助導線として必要なものを除いて整理されている
- 全コマンド一覧など、skill の責務外で stale 化しうる一覧が除去または責務上必要な範囲へ縮小されている
- 他 command / skill から、skill 内部の section 名、protocol 名、Step 名、Phase 名への直接依存が増えていない
- 修正後も skill 名による責務参照は維持されている
- `See Also` は補助的な発見導線としてのみ残っている
