# RU: active REQ の過渡期是正行を除去し現行状態要件へ再整理する

## 背景

active REQ には、過去の移行・是正・旧語彙除去・frontmatter 整理・参照整理など、過渡期の作業を直接表す要件行が残っている。

例として、`REQ-0108-095` は command frontmatter に `implementation_pattern` が存在することを禁止検査する要件である。この種の要件は、現行状態の恒久制約として残すべきものと、すでに完了した一時的な是正作業として active REQ から除去すべきものが混在している。

現行 repository 調査では、command frontmatter から `implementation_pattern` / `secondary_pattern` / `load_skills` は除去済みである一方、REQ / DOC-MAP / vocabulary registry / AGENTS.md 周辺には active REQ 範囲、旧名称、旧見出し、retire 未反映などの drift が残っていることが確認された。

## 問題

active REQ に一時是正作業が残ると、REQ が「現在満たすべき状態要件」ではなく「過去の作業履歴」を含む文書になる。

その結果、以下の問題が発生する。

- 完了済み作業が未完了要件のように見える
- 恒久検査と一時是正作業が重複する
- 旧語彙を検出するための記述と、旧語彙が現行説明として残っている記述を区別できない
- REQ 範囲や active/retired 状態に関する SSoT が drift する
- req-define / docs-check / backlog-review が不要な修正候補を生成し続ける

## 統合理由

対象となる修正は、個別の旧語置換や個別REQ削除ではなく、active REQ を「現在満たすべき状態要件」に戻すための同一目的の整理である。

そのため、以下を1つのRUに統合する。

- 完了済み一時是正行の除去
- 恒久制約への統合
- 重複要件行の集約
- 旧語彙・旧範囲・旧配置説明の現行化
- active / retired 状態の整合
- docs-check 等の再発防止検査が参照する基準の明確化

## 要件化の方向

active REQ は、現在満たすべき状態要件のみを保持する。

完了済みの過渡期是正作業、FPRトレース、Case/Issue/Wave履歴、具体ファイルの削除作業、具体件数の是正行は、active REQ の要件表から除去する。履歴として残す必要がある場合は、要件行ではなく Update Notes 等の履歴欄へ移す。

恒久的に維持すべき制約は、作業行ではなく状態要件として再記述する。

例:

- command frontmatter は `description` + `agent` のみを許可する
- command frontmatter に `implementation_pattern` / `secondary_pattern` / `load_skills` が存在しないことを検査する
- old vocabulary は、検出対象として扱う場合と現行説明として誤って残っている場合を区別する
- active REQ 範囲は repository 内の全入口文書で一致させる
- retired 扱いが合意された REQ は active path から除外する
