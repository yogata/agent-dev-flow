# RU: backlog-save 廃止と backlog-review への RU 作成統合

## 背景

現行の Backlog パイプラインでは、`promoted artifact` から RU を作成するまでの処理が `/agentdev/backlog-review` と `/agentdev/backlog-save` に分かれている。

現行仕様では、`backlog-review` は promoted artifact を分析・統合し、ユーザー承認済みの review draft を保存する。RU 生成は行わない。

`backlog-save` は、`backlog-review` が生成した承認済み review draft を読み込み、RU を生成し、RU 化成功分の promoted artifact を削除する。

## 問題

現在の責任範囲では、判断責務は `backlog-review` 側に集中している。

`backlog-review` は以下を担当している。

- promoted artifact の分析
- 統合・分割判定
- 矛盾検出
- ユーザー承認

一方、`backlog-save` は承認済み review draft を読み、追加のユーザー確認なしで RU を生成するだけである。

そのため、`backlog-save` は独立した判断責務を持たず、コマンドとして分離する必然性が弱い。

## Source Summary

この RU は、Backlog パイプラインにおける `backlog-review` / `backlog-save` の責務再編を扱う。

合意内容は、現行の2段階構成を見直し、`backlog-review` でユーザー承認を得た内容から、そのまま RU を作成する構成へ変更することである。

## 統合理由

`backlog-review` でユーザー承認が完了するなら、その承認は RU 作成承認を兼ねる。

`backlog-save` に残る処理は、承認済み内容の機械的な永続化であり、独立コマンドとして分離する理由がない。

したがって、`backlog-save` を廃止し、`backlog-review` に RU 作成まで統合する。

## 要件化の方向

`/agentdev/backlog-save` を廃止する。

`/agentdev/backlog-review` を、promoted artifact の分析・統合・ユーザー承認に加えて、承認済み内容から RU を生成するコマンドへ変更する。

変更後の `backlog-review` は以下を担当する。

- promoted artifact を読み込む
- 統合・分割を判定する
- 矛盾を検出する
- RU 化対象と RU 内容をユーザーに提示する
- ユーザー承認を得る
- 承認済み内容から `.agentdev/backlog/req-units/RU-*.md` を作成する
- RU 化成功分の promoted artifact を削除する
- `.agentdev/` 配下の変更を git commit / push する
- 次工程として `/agentdev/req-define` を案内する
