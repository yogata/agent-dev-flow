# RU: integrity-check 最新仕様準拠

## 背景

`integrity-check` は REQ-0108 に基づき、AgentDevFlow artifact の機械的整合性を検査する read-only diagnostic command である。

最新仕様では、command frontmatter は runtime 用の最小メタデータに限定される。許可フィールドは `description` と `agent` のみであり、`implementation_pattern`、`secondary_pattern`、`load_skills` は dev metadata として command frontmatter への混入を禁止する。

## 問題

現行の `agentdev-integrity` 実装には、旧仕様である `load_skills` 前提の検査が残っている。

確認済みの問題は以下。

- `checkCommandInventory` が `load_skills` を必須 frontmatter として扱っている
- `checkLoadSkillsExistence` が command frontmatter の `load_skills` を読み取って skill 存在確認を行っている
- `checkCompletionReportSkills` が `agentdev-workflow-reporting` の `load_skills` 参照を全 command に要求している
- `checkUnusedSkillsCategorized` が command frontmatter の `load_skills` から未使用 skill を算出している
- `checkCommandMapConsistency` が command-map と command frontmatter の `implementation_pattern` / `secondary_pattern` を照合している
- regression test の valid fixture に `implementation_pattern` / `load_skills` が残っている
- `agentdev-integrity/SKILL.md` に `Skill ↔ load_skills` など旧説明が残っている

このため、最新仕様では禁止すべき frontmatter を旧検査が要求・前提化する矛盾が発生している。

## Source Summary

- REQ-0108 は command frontmatter の `implementation_pattern` / `secondary_pattern` / `load_skills` を禁止し、許可フィールドを `description` と `agent` のみに定義している。
- `check_integrity.ts` には `load_skills` 必須確認、存在確認、完了報告 skill 参照必須確認、未使用 skill 算出、command-map と frontmatter pattern の照合が残っている。
- `check_integrity.test.ts` の valid fixture は旧 frontmatter 構造を正として扱っている。
- `agentdev-integrity/SKILL.md` の検査カテゴリ説明にも旧 `load_skills` 前提の表現が残っている。

## 統合理由

本件は個別の検査不具合ではなく、`integrity-check` が旧 command frontmatter 仕様と最新 command frontmatter 仕様を同時に前提としている整合性不良である。

`load_skills` 関連検査、frontmatter 許可フィールド検査、command-map 照合、regression fixture、SKILL.md 説明を一体で修正しないと、検査結果とテスト期待値が再び矛盾する。

## 要件化の方向

`integrity-check` を最新 REQ-0108 仕様に合わせ、command frontmatter の扱いを以下に統一する。

- 許可: `description`, `agent`
- 禁止: `implementation_pattern`, `secondary_pattern`, `load_skills`
- その他未知フィールド: NG
- `load_skills` を根拠にした存在確認・必須確認・過不足確認・完了報告 skill 参照確認は廃止
- command-map は pattern taxonomy の参照情報として扱い、command frontmatter との照合根拠にしない
