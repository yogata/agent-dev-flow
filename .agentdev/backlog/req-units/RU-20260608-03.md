# RU: 配布対象 command / skill から agent-dev-flow 固有ID参照を除去する

## 背景

agent-dev-flow の command / skill は配布先 consumer repository で実行される。

配布先では、agent-dev-flow リポジトリ内の具体的な ADR / REQ / SPEC 文書を参照できない。そのため、配布対象 command / skill に agent-dev-flow 固有の `ADR-ID` / `REQ-ID` / `SPEC-ID` が残っていると、利用者にとって追跡不能な参照になる。

## 問題

配布対象 command / skill 本文に、agent-dev-flow リポジトリ固有の具体ID参照が含まれている。

具体例:

- `REQ-0114-036`
- `REQ-0104-029`
- `REQ-0106-027`
- `ADR-0108`
- `ADR-0109`
- `SPEC-xxxx`

これらは配布先では参照不能であり、command / skill の実行時説明・判断基準・Guardrails としてノイズになる。

## Source Summary

チャット内で、以下を合意した。

- command / skill に agent-dev-flow リポジトリ固有の `ADR-ID` / `REQ-ID` / `SPEC-ID` の言及がないこと
- 配布先ではこれらの ADR / REQ / SPEC を参照できないこと
- 参照不能な具体IDはノイズであること
- 文書種別としての `REQ` / `ADR` / `SPEC` や、抽象的なファイル命名パターンは別扱いにすること

## 統合理由

この問題は、個別 command / skill の表現修正ではなく、配布対象成果物に共通する runtime / consumer 境界の問題である。

配布先で参照できない内部IDを削除し、必要な制約・判断基準を command / skill 本文内に自足させることで、consumer repository 上での実行可能性と説明の明瞭性を保つ。

## 要件化の方向

配布対象 command / skill 本文には、agent-dev-flow リポジトリ固有の具体 `REQ-ID` / `ADR-ID` / `SPEC-ID` を残さない。

具体ID参照に依存していた制約・判断基準は、ID参照ではなく本文内の自足した記述へ置換する。

例:

- 変更前: `REQ-0104-029〜033 で定義される case-open の Issue 構造ルールを使用する`
- 変更後: `case-open が出力した Issue 構造を再評価せず、そのまま使用する`

## 主対象REQまたは変更対象候補

主対象REQ候補:

- 配布対象 command / skill の runtime projection 境界を扱う既存REQ
- command / skill の品質・整合性検査を扱う既存REQ
- integrity / docs-check 系の検査条件を扱う既存REQ

変更対象候補:

- `src/opencode/commands/**/*.md`
- `src/opencode/skills/**/*.md`
- `.opencode/commands/**/*.md`
- `.opencode/skills/**/*.md`
- 配布対象生成・同期・検査に関わる integrity / docs-check 定義

## 対象外

以下は対象外とする。

- docs 配下の REQ / ADR / SPEC 本文における具体ID参照
- backlog / req-units / drafts / intake / learning など、agent-dev-flow リポジトリ内で処理される内部成果物
- `REQ` / `ADR` / `SPEC` という文書種別名そのもの
- `REQ-{NNNN}.md`、`ADR-{NNNN}.md`、`SPEC-{name}.md` のような抽象的な命名パターン
- consumer repository 内で作成される成果物としての `docs/requirements/REQ-{NNNN}.md` 等の抽象パス
- 具体IDが必要な内部ドキュメントの再設計
- 実装手順・コード差分案

## 受け入れ条件

- 配布対象 command / skill 本文に `REQ-[0-9]+` が残っていない。
- 配布対象 command / skill 本文に `ADR-[0-9]+` が残っていない。
- 配布対象 command / skill 本文に `SPEC-[0-9]+` が残っていない。
- 削除した具体ID参照が担っていた制約・判断基準は、本文内の自足した記述に置換されている。
- `REQ` / `ADR` / `SPEC` という文書種別名は削除対象として扱われない。
- `REQ-{NNNN}.md`、`ADR-{NNNN}.md` のような抽象命名パターンは削除対象として扱われない。
- integrity / docs-check 系で、配布対象 command / skill 内の具体 `REQ-ID` / `ADR-ID` / `SPEC-ID` 残存を検出できる。
- agent-dev-flow 内部文書である docs / backlog / drafts / intake / learning の具体ID参照は、本RUの受け入れ条件に含めない。
