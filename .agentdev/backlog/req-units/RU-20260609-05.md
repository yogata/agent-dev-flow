# RU: workflow 意味論と Epic status model の active source 競合を解消する

## 背景

workflow 系の current contract は、REQ-0104（workflow 分類）、REQ-0106（case-run / case-close / Epic）、REQ-0114（case-auto）と、それを説明する SPEC / skill / guide にまたがっている。

この層で active source 同士の意味矛盾が残ると、
command 実行基準と review 基準の両方がぶれる。

## 問題

`design-principles.md` は bugfix で docs 更新をスキップすると読める一方、`REQ-0106` / `system.md` / `case-run.md` / `req-case-flow.md` は、全 work_type 共通で矛盾する関連ドキュメント更新を要件達成対象としている。

また `system.md` は Epic status を 4 値としている一方、`agentdev-epic-tracker` と `glossary.md` は `⏭ スキップ` を正式状態として扱っている。

## Source Summary

- 関連ドキュメント更新は `REQ-0106-006`、`REQ-0106-010`、`case-run` Step 9、`case-close` Step 3 で全 work_type 共通の責務として読める。
- bugfix が REQ 作成不要であることと、関連 docs 更新不要であることは同義ではない。
- Epic status model は `system.md` と `agentdev-epic-tracker` で不一致である。
- active source 競合は、単純修文ではなく「どちらを current rule とするか」の決定を伴う。

## 統合理由

本件は workflow 意味論の正本を再確定し、REQ / SPEC / skill / guide / docs-check を揃える必要がある。

`REQ-0104` / `REQ-0106` / `REQ-0108` への APPEND / UPDATE として整理し、workflow semantics と Epic status model を一括で正規化するのが妥当である。

## 要件化の方向

以下を APPEND / UPDATE する。

- bugfix / maintenance / docs_chore は「新規 REQ 作成不要」であっても、変更後仕様と矛盾する関連 docs 更新は完了条件から外れないことを明記する。もし例外を認めるなら、その条件を REQ レベルで定義する。
- Epic status model について、`⏭ スキップ` を正式状態として認めるか、Orchestrator 内部概念に留めるかを決定し、REQ / SPEC / skill / guide を同期する。
- `docs-check` は active source 間で bugfix docs 更新ルールや Epic status 語彙が食い違う場
合に warning を出せるようにする。
