### 要件化候補

- implementation pattern は load_skills 判定の初期ヒントに留め、必須 skill の決定条件には command の実責務を含めること。
- `file-pipeline` であっても、git commit を行わない command に `agentdev-conventional-commits` を必須要求しないこと。
- REQ / ADR ファイルを作成・更新しない command に `agentdev-req-file-manager` / `agentdev-adr-file-manager` を必須要求しないこと。
- GitHub CLI を使わない command に `agentdev-gh-cli` を必須要求しないこと。
- read-only diagnostic command が意味的判定のために分析・ガイドライン系 skill を参照するケースを許容または別分類すること。
- `USE FOR` / `DO NOT USE FOR` との整合性検査は、単純な文字列一致ではなく、command 内の利用目的を根拠として分類すること。
