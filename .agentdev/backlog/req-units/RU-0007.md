# RU: req-backlog の backlog-* 分割

## 背景

現行 `/agentdev/req-backlog` は、intake / learning の promoted artifact を分析・統合し、RU（Requirement Unit）を `.agentdev/backlog/req-units/` に生成するコマンドである。

現行 REQ-0105 でも、`req-backlog` は promoted artifact を分析・統合し、RU を生成する責務として定義されている。

一方で、現行 `/agentdev/req-backlog` は単一コマンド内に以下のユーザー判断・確認を含む。

- Step 3: 統合・分割 grouping の妥当性確認
- Step 4: 矛盾検出時のユーザー判断
- Step 5: 各 RU 内容の再確認

このうち Step 3/4 は真の HITL boundary である。Step 5 の再確認は Step 3/4 の合意後に機械的に生成される内容の再承認であり、不要である。

また、`req-backlog-review` / `req-backlog-save` のように同一接頭辞を長く共有する命名は、コマンド補完時に使いづらい。

## 問題

現行 `/agentdev/req-backlog` には以下の問題がある。

1. HITL boundary がコマンド境界になっていない
2. Step 5 で不要な二重確認が発生している
3. 分割時に `req-backlog-*` 形式を採用すると補完性が悪い
4. 判断処理と保存・削除・git永続化処理が単一コマンドに混在している

## Source Summary

チャット内で以下を合意した。

- `backlog-*` コマンド群に寄せる
- 判断系コマンドは `/agentdev/backlog-review`
- 保存系コマンドは `/agentdev/backlog-save`
- 既存 `/agentdev/req-backlog` は即削除しない
- `/agentdev/req-backlog` は移行用 alias または非推奨コマンドとして扱う
- Step 5 のユーザー確認は削除する
- Step 3/4 の承認済み結果を `backlog-save` の入力とする
- `req-backlog-review` / `req-backlog-save` のような長い同一接頭辞コマンドは採用しない

## 統合理由

本件は、単なるコマンド名変更ではなく、以下を一体で扱う必要がある。

- RU 生成ワークフロー
- HITL boundary と command boundary の整合
- `req-backlog` の責務分割
- `backlog-*` 命名への移行
- Step 5 の不要確認削除
- 次コマンド提示・完了報告の更新

別RUに分けると、コマンド名・責務・RU lifecycle の整合が崩れるため、一体のRUとして扱う。

## 要件化の方向

REQ-0105 を UPDATE / APPEND する。

方向性は以下。

1. `req-backlog` 単独責務を廃止し、`backlog-review` / `backlog-save` に分割する
2. `backlog-review` は promoted artifact の分析、統合・分割判定、矛盾検出、承認待ち結果の提示までを担当する
3. `backlog-save` は承認済み review 結果を入力として RU 生成、成功 artifact 削除、git commit/push を担当する
4. `backlog-save` は RU 内容のユーザー再確認を行わない
5. `req-backlog` は移行用 alias または非推奨コマンドとして扱う
6. 補完性の観点から `req-backlog-review` / `req-backlog-save` 形式は採用しない

## 主対象REQまたは変更対象候補

### REQ-0105 UPDATE 候補

- REQ-0105-008
  - 現行: `req-backlog` が promoted artifact を分析・統合して RU を生成する
  - 修正後: `backlog-review` が promoted artifact を分析・統合・矛盾検出し、`backlog-save` が承認済み結果から RU を生成する

- REQ-0105-013
  - `req-backlog` 向け短期候補という表現を `backlog-review` / `backlog-save` ルート向けに更新する

- REQ-0105-028
  - 通常 intake 起点 item の後続ルートを `/agentdev/intake-promote` → `/agentdev/backlog-review` → `/agentdev/backlog-save` → `/agentdev/req-define` に更新する

### REQ-0105 APPEND 候補

- `backlog-review` は intake / learning の promoted artifact を読み込み、RU 生成前の統合・分割判定を提示すること（SHALL）
- `backlog-review` は artifact 間の矛盾を検出し、矛盾がある場合は RU 化判断をユーザーに委ねること（SHALL）
- `backlog-review` は RU ファイルを生成してはならない（MUST NOT）
- `backlog-save` はユーザー承認済みの `backlog-review` 結果を入力として RU を `.agentdev/backlog/req-units/` に生成すること（SHALL）
- `backlog-save` は RU 生成後に各 RU 内容の追加ユーザー確認を要求してはならない（MUST NOT）
- `backlog-save` は RU 生成に成功した promoted artifact のみを削除すること（SHALL）
- `backlog-save` は `.agentdev/` 配下の変更を git commit/push すること（SHALL）
- `req-backlog` は `backlog-review` / `backlog-save` への移行対象として扱うこと（SHALL）
- `req-backlog` を残す場合、非推奨または互換 alias として動作し、次に使うべき `backlog-*` コマンドを案内すること（SHOULD）
