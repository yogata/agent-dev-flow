# RU: コマンド・スキル間参照境界の正規化と曖昧参照防止

## 背景

`case-auto` 実行時、`case-open` 周辺の推論ログで以下の不存在ファイルを参照しようとしていた。

- `src/opencode/skills/agentdev-workflow-lifecycle/references/workflow-classification.md`
- `src/opencode/skills/agentdev-workflow-lifecycle/references/issue-generation-protocol.md`

原因は、コマンド・スキル内に自然言語ラベルだけの参照、旧称または曖昧な参照、実在ファイル名と一致しない参照、skill 内部構造への過剰依存が残っていることである。

`case-auto` は既存コマンド定義を authoritative source として読み、各コマンドの手順に従う設計であるため、参照元の曖昧さや責務境界の不明確さがそのまま実行時の誤読に直結する。

また、`case-open.md` では、`agentdev-workflow-lifecycle` skill の内部プロトコル名である「Issue 生成プロトコル」まで参照している記述が確認された。この記述は、skill を参照するだけでなく、skill 内部の見出し・protocol・Step 構造へ command が依存している状態である。

その後の確認で、同型の問題は `case-open.md` に限定されず、複数の command / skill に存在することが整理された。

## 問題

現行のコマンド・スキルには、次の問題がある。

1. `workflow classification` のような自然言語参照が、存在しない `workflow-classification.md` として推測されうる。
2. `Issue 生成プロトコル` のような自然言語参照が、存在しない `issue-generation-protocol.md` として推測されうる。
3. `agentdev-workflow-lifecycle の「Step X」` という参照があるが、現行の `agentdev-workflow-lifecycle/SKILL.md` は該当 Step 本文を持つ構成ではない。
4. 一部にインラインコードの閉じバッククォート欠落があり、参照範囲が壊れている。
5. command から skill の内部見出し、protocol 名、Step 番号、Section 名まで参照している。
6. command 固有の実行手順が skill 側へ漏れている。
7. skill 側が command 固有 Step 番号を一次情報として保持している。
8. `agentdev-workflow-lifecycle` が lifecycle 判定以外の汎用置き場になりうる。
9. 後続の command / skill 分割時に参照切れや責務重複が発生する。
10. 今後も同種の曖昧参照・内部構造参照が追加される可能性がある。

特に `agentdev-workflow-lifecycle` は、lifecycle / work_type / scale / upstream handoff などの判定系を扱うべきであり、Issue本文生成、Epic / child Issue生成、intake pipeline、docs-review診断、backlog-review統合手順などの command 固有処理を受け持つべきではない。

## Source Summary

チャット内で、以下が合意された。

- Phase 1: 不存在ファイルへ推測される参照の直接修正
- Phase 2: `agentdev-workflow-lifecycle の Step X` 参照の解消
- Phase 3: 今後同じ問題を防ぐ参照ルールの明文化

- `skill の <protocol>`、`skill の「Step N: ...」`、`skill の Section N` のような内部構造参照は修正対象とする
- `case-open.md` の `agentdev-workflow-lifecycle skill の Issue 生成プロトコル` 参照は過剰である
- 同様の問題を全 command / skill で確認し、修正候補を洗い出す

- 共有利用が確認できない段階で新 skill を作るのは避ける

また、Phase 4 として挙げた `/repo/docs-check` 導線の別件整合性は、今回の対象外と合意された。

## 統合理由

RU-20260613-02 は、不存在ファイルへ推測される参照、自然言語ラベル参照、壊れた Markdown 参照を直接修正する要件である。

RU-20260613-03 は、その原因を command / skill 間の参照境界問題として捉え、skill 内部構造への依存を禁止する要件である。

両者は別問題ではない。`workflow classification`、`Issue 生成プロトコル`、`agentdev-workflow-lifecycle の Step X`、壊れた `agentdev-workflow-routing` 参照は、いずれも「参照先をエージェントに推測させる」「skill 内部構造に command が依存する」「責務境界が曖昧になる」という同一原因から発生している。

したがって、個別ファイル修正ではなく、共通ルールを定義したうえで全 command / skill へ適用する単一の Requirement Unit として扱う。

## 要件化の方向

コマンド・スキル間参照について、以下の規則を導入する。

1. command から skill を参照する場合、原則として skill 名までにする。
2. command から skill 内部の protocol 名、Step 名、Section 名、見出し名を参照してはならない。
3. 例外は、実行時に明示的に読み込む必要がある template / reference / script の repo-root-relative path に限定する。
4. `references/*.md` を参照する場合は、実在する repo-root-relative path または skill-relative path を明記する。
5. skill 本体を参照する場合は、`agentdev-xxx/SKILL.md` または skill 名を明記する。
6. `workflow classification`、`Issue 生成プロトコル`、`Step X` のような自然言語ラベルだけの参照を禁止する。
7. 参照先ファイル名をエージェントに推測させる記述を禁止する。
8. command 固有の実行順序、Step 番号、Phase 名を skill 側の一次情報として保持してはならない。
9. skill の `USE FOR` / `DO NOT USE FOR` と参照元用途が矛盾する場合、参照元または参照先を修正する。
10. `agentdev-workflow-lifecycle` は lifecycle / work_type / scale / upstream handoff などの判定系に限定する。
11. Issue生成、intake処理、docs-review診断、backlog統合処理の詳細を `agentdev-workflow-lifecycle` に持たせない。
12. 複数 command で共有される詳細手順がある場合のみ、専用 skill または reference への分離を検討する。
13. Markdown のインラインコード範囲を壊さない。

## 要件

### 1. 不存在ファイルへ推測される参照の除去

`workflow classification`、`Issue 生成プロトコル` などの自然言語ラベルだけの参照を除去し、存在しない以下のファイル名を推測させない。

- `workflow-classification.md`
- `issue-generation-protocol.md`

修正後は、自然言語ラベルをそのまま参照名として残さず、以下のいずれかへ整理する。

- skill 名のみを参照する
- `SKILL.md` を参照する
- 実在する template / reference / script の repo-root-relative path を参照する
- command 本体へ手順を戻す

修正例:

````markdown id="e7jjz2"
修正前:
- `workflow classification` を参照する

修正後:

````

````markdown id="n3nhmr"
- lifecycle / work_type / scale / upstream handoff 判定のみ `agentdev-workflow-lifecycle` を参照する
````

### 2. skill 内部構造参照の禁止

command から skill 内部の protocol 名、Step 名、Section 名、見出し名を参照しない。

禁止する参照例:

````markdown id="cu8j1d"
- `agentdev-workflow-lifecycle` の Issue 生成プロトコル
- `agentdev-workflow-lifecycle` の「Step 4」
- `agentdev-workflow-lifecycle` の Section 3
- `agentdev-workflow-orchestration` の Self-Healing protocol
- `agentdev-workflow-lifecycle` の workflow classification
````

許容する参照例:

````markdown id="zpk88k"

- `src/opencode/skills/agentdev-workflow-lifecycle/references/<実在ファイル>.md`
- `src/opencode/commands/agentdev/templates/<command>/<variant>.md`
````

ただし、reference path を使う場合は、実在ファイルに限定する。存在しないファイル名を自然言語から推測させる記述は禁止する。

### 3. `agentdev-workflow-lifecycle` の責務境界明確化

`agentdev-workflow-lifecycle` は以下に限定する。

- lifecycle 判定
- workflow phase 判定
- work_type classification
- scale 判定
- upstream handoff 判定
- SSoT 遷移に関する共通判断

`agentdev-workflow-lifecycle` に以下を持たせない。

- Issue本文生成
- Issue作成
- Epic / child Issue 生成
- RU削除後処理
- intake pipeline
- docs-review 診断
- backlog-review 統合手順
- command 固有の Step 番号
- command 固有の実行順序
- command 固有の Phase 名

修正例:

````markdown id="t1jvfn"
修正前:
- `agentdev-workflow-lifecycle` の Issue 生成プロトコルに従い、Issue本文生成、Issue作成、Epic更新、RU削除を行う

修正後:
- Issue本文生成、Issue作成、Epic更新、RU削除は `case-open.md` の責務として実行する
- template 構造は `agentdev-workflow-templates` を参照する
- GitHub CLI 書込安全性は `agentdev-gh-cli` を参照する
- lifecycle / work_type / scale / upstream handoff 判定のみ `agentdev-workflow-lifecycle` を参照する
````

### 4. lifecycle Step 参照の解消

`agentdev-workflow-lifecycle の「Step X」` のように、実在しない Step 本文を参照している記述を解消する。

解消方法は、以下のいずれかとする。

- command 本体へ必要手順を戻す
- 実在する既存 reference へ接続する
- 複数 command で共有される場合のみ、専用 skill または reference への分離を検討する
- 単発用途の新規 skill は作成しない

修正例:

````markdown id="zr13nz"
- promoted artifact 読込、統合・分割、矛盾検出、RU生成は `backlog-review.md` 側の責務として実行する
````


### 8. authoring ルールの明文化

`agentdev-skill-authoring/SKILL.md` または同等の authoring ルールに、以下を追加する。

- `references/*.md` を参照する場合は、実在する相対パスまたは repo-root-relative path を明記する。
- skill 本体を参照する場合は、`agentdev-xxx/SKILL.md` または skill 名を明記する。
- 自然言語ラベルだけの「〜プロトコル」「〜分類」「Step X」「Section X」参照を禁止する。
- 参照先ファイル名をエージェントに推測させる記述を禁止する。
- command から skill 内部見出し、protocol 名、Step 名、Section 名へ依存する記述を禁止する。
- command 固有の Step 番号、Phase 名、実行順序を skill 側の一次情報として保持しない。

修正例:

````markdown id="fjgr1q"
## Reference writing rules

- When referencing a skill, use the skill name or `agentdev-xxx/SKILL.md`.
- When referencing a reference file, use an existing path under `references/*.md`.
- Do not refer to skill-internal protocol names, Step names, Section names, or headings from commands.
- Do not make agents infer filenames from natural language labels such as `workflow classification` or `Issue 生成プロトコル`.
- Do not store command-specific Step numbers or Phase names as primary information in skills.
````

### 9. integrity 観点の追加

`.opencode/skills/repo-agentdev-integrity/SKILL.md` に、`skill-internal-section-reference` 相当の検査観点を追加する。

検出対象例:

````text id="xg7ag2"
skill の .*protocol
skill の「Step
Section [0-9]
workflow classification
Issue 生成プロトコル
agentdev-workflow-lifecycle の Step
command 固有 Step 番号を skill 側に持つ記述
````

検査観点の目的は、以下の再発防止である。

- 存在しない `workflow-classification.md` を推測させる参照の再発防止
- 存在しない `issue-generation-protocol.md` を推測させる参照の再発防止
- command から skill 内部構造へ依存する記述の再発防止
- skill 側に command 固有 Step 番号を保持する記述の再発防止

## 対象外

以下は対象外とする。

- command / skill の実装ロジック変更
- GitHub Issue 作成
- PR 作成
- 実装計画の詳細化
- コード実装
- 未確認の既存REQ再編
- `agentdev-workflow-lifecycle` の全面再設計
- 単発用途の新規 skill 作成
- 文言改善のみを目的とした隣接記述の整形
- unrelated な表記ゆれ修正
- 参照不備と関係しない Markdown 整形
- 配布対象外ファイルの無関係な整理
- `case-auto` の自走制御や queue 制御の変更
- Issue 作成
- PR 作成
- 会話URLの記録
- `/repo/docs-check` 導線の別件整合性確認

ただし、repo-local の `/repo/docs-check` / `repo-agentdev-integrity` に同じ参照境界問題が存在する場合、参照境界の修正に直結する範囲のみ対象に含める。

## 受け入れ条件

1. `workflow-classification.md` という不存在ファイル名を推測させる参照記述が、対象コマンド・対象スキルから除去されている。
2. `issue-generation-protocol.md` という不存在ファイル名を推測させる参照記述が、対象コマンド・対象スキルから除去されている。
3. `src/opencode/commands/agentdev/**/*.md` に、skill 内部 protocol 名への参照が残っていない。
4. `src/opencode/commands/agentdev/**/*.md` に、`agentdev-workflow-lifecycle` の内部 Step 名参照が残っていない。
5. `src/opencode/commands/agentdev/**/*.md` に、`workflow classification` のような内部節名参照が残っていない。
6. command から skill を参照する場合、原則として skill 名までに正規化されている。
7. 実行時に明示的に読み込む template / reference / script は、実在する repo-root-relative path で参照されている。
8. 自然言語ラベルから reference ファイル名を推測させる記述が残っていない。
9. Issue本文生成、Issue作成、Epic / child Issue生成、RU削除後処理が `agentdev-workflow-lifecycle` の責務として扱われていない。
10. intake 固有処理が `agentdev-workflow-lifecycle` の責務として扱われていない。
11. docs-review 固有診断が `agentdev-workflow-lifecycle` の責務として扱われていない。
12. backlog-review 固有の promoted artifact 統合・RU生成処理が `agentdev-workflow-lifecycle` の責務として扱われていない。
13. skill 側に command 固有 Step 番号を一次情報として保持する記述が残っていない。
14. `agentdev-workflow-lifecycle` の `USE FOR` / `DO NOT USE FOR` と参照元用途が矛盾していない。
15. Markdown のインラインコード範囲が壊れていない。
16. `agentdev-skill-authoring/SKILL.md` または同等の authoring ルールに、実在パス明記、自然言語ラベル参照禁止、skill 内部構造参照禁止のルールが追加されている。
17. `.opencode/skills/repo-agentdev-integrity/SKILL.md` または同等の integrity 観点に、skill 内部 section / protocol / Step 参照を検出する観点が定義されている。
18. repo-local の `/repo/docs-check` / `repo-agentdev-integrity` も、参照境界に直結する範囲で同じ基準により確認されている。
19. 変更後、`case-auto` / `case-open` 実行時に、存在しない `workflow-classification.md` または `issue-generation-protocol.md` を読みに行く導線が残っていない。
20. 今回対象外とした `/repo/docs-check` 導線の別件整合性確認が、本RUの変更に混入していない。
21. unrelated な表記ゆれ修正、文体修正、整形変更が混入していない。
22. 変更は参照境界の修正に直結する箇所へ限定されている。
