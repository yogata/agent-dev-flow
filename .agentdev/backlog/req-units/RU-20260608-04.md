# RU: 配布用 command / skill における agent-dev-flow 固有パス・名称の混入防止

## 背景

agent-dev-flow は、consumer repository に配布される runtime artifact と、agent-dev-flow repository 自身でのみ使用する self-hosting artifact を分離している。

配布対象は `src/opencode/commands/agentdev/` および `src/opencode/skills/agentdev-*` であり、consumer 側では `.opencode/commands/agentdev/` と `.opencode/skills/agentdev-*` に投影される。

一方、`.opencode/commands/repo/` および `.opencode/skills/repo-*` は agent-dev-flow repository 固有の repo-local artifact であり、consumer へ配布されない。

## 問題

配布用 command / skill に、agent-dev-flow repository 固有のパス・名称・repo-local command が混入している。

確認済みの問題は以下。

1. `agentdev-workflow-lifecycle` に self-hosting 専用 command が対象コマンドとして含まれている
   - `integrity-check` または `docs-check`
   - `/repo/*` は repo-local command であり、consumer 配布対象ではない

2. `agentdev-workflow-orchestration` に source/authoring path として `src/opencode/...` が記載されている
   - runtime projection として `.opencode/...` を使うべき箇所に、開発リポジトリ内パスが併記されている
   - 実行時参照ではない注記であっても、配布用 skill 内に source path を残すと consumer 側で誤参照を誘発する

3. `case-auto.md` の `src/opencode/...` 記述は、runtime path として使用禁止を明記する guardrail であり、混入ではない

## Source Summary

チャット内で、配布用 command / skill に `src/opencode` や agent-dev-flow 固有の名称・パスが混入していないかを確認した。

確認結果として、配布仕様上許容されるものと、修正すべき混入を分離した。

許容されるもの:

- `/agentdev/*`
- `.agentdev/`
- `agentdev-*`
- `.opencode/commands/agentdev/`
- `.opencode/skills/agentdev-*`
- upstream handoff 判定における `agent-dev-flow`
- `src/opencode/...` を runtime path として使うことを禁止する guardrail

修正対象:

- 配布用 skill の対象コマンド一覧に repo-local command が含まれること
- 配布用 skill の See Also 等に authoring path `src/opencode/...` が現れること

## 統合理由

本 RU は、単なる個別文言修正ではなく、runtime 配布境界の再発防止に関わる。

`agentdev-workflow-lifecycle` と `agentdev-workflow-orchestration` はいずれも配布対象 skill であり、consumer repository 上で参照される。このため、self-hosting 専用 artifact や source repository 内パスを配布用 skill に残す問題は同一の責務境界違反として扱う。

## 要件化の方向

配布対象 command / skill に含めてよいパス・名称と、含めてはならない agent-dev-flow repository 固有のパス・名称を明文化する。

あわせて、既存の配布用 command / skill から確認済みの混入を除去する。

要件化では以下を区別する。

- runtime path として必要な `.opencode/...`
- consumer domain state として必要な `.agentdev/...`
- plugin identity として必要な `agentdev`
- upstream handoff の判定対象として必要な `agent-dev-flow`
- self-hosting repository 専用の `.opencode/commands/repo/`
- self-hosting repository 専用の `.opencode/skills/repo-*`
- source repository 内の authoring path `src/opencode/...`

主対象候補:

## 受け入れ条件

- 配布対象 `src/opencode/skills/agentdev-*` の本文に、self-hosting repo 専用 command を通常の対象コマンドとして列挙していないこと
- `agentdev-workflow-lifecycle` の対象コマンド一覧に `/repo/docs-check` または旧 `/repo/integrity-check` が含まれていないこと
- 配布対象 command / skill が runtime 参照先として `src/opencode/...` を案内していないこと
- `agentdev-workflow-orchestration` の See Also は consumer runtime で参照可能な `.opencode/...` を優先し、source/authoring path を配布用 skill の実行時導線として残していないこと
- `case-auto.md` の `src/opencode/...` 使用禁止 guardrail は維持されていること
- `.agentdev/`、`.opencode/commands/agentdev/`、`.opencode/skills/agentdev-*`、`/agentdev/*` は配布仕様上必要な語彙として残っていること

- repo-local artifact の説明が必要な場合は、repo-local 側または開発者向け docs に配置され、consumer runtime skill の通常実行導線に混入していないこと
