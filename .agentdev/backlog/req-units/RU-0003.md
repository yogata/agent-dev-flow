# RU-0003: RU削除責務のcase-open一本化

## Sources

### session:2026-05-31-case-open-owned-ru-deletion (intake)

現行定義では、RU削除処理が `req-save` と `case-open` の両方に存在している。

`req-save` 完了時点ではGitHub Issueがまだ作成されていない。通常フローでは実装に渡る単位はIssueであるため、RUの消費完了は `case-open` の Issue作成およびVERIFY成功時点とみなすほうが責務境界が明確である。

## Source Summary

RU削除責務を `case-open` に一本化する。

`req-save` はRUを削除せず、REQファイルに `Requirement Source` としてRUパスを保持する。`case-open` はREQファイルまたはセッション内要件docの `Requirement Source` をIssue本文へ転記し、Issue作成とVERIFY成功後に該当RUを削除する。
