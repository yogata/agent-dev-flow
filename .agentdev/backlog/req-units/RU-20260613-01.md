# RU: active REQ 全体における自明・同値記述の削減基準整備

## 背景

active REQ では、要件本文に補足説明、整合注記、対象外注記が追記されることがある。これらのうち一部は判定境界を追加する実質的な制約だが、一部は主語、適用範囲、既存 active REQ から自明に導ける再説明に留まっている。

本セッションでは、`REQ-0103-090` から `REQ-0103-092` に対して、`REQ-0103-092` の補足が `REQ-0103-090` の主語制約から自明であり、冗長ではないかという指摘があった。さらに、同種の自明・同値記述が active REQ 全体に存在するなら、意味を変えずに削減したいという要求が示された。

## 問題

active REQ に自明・同値の再説明が残ると、以下の問題が生じる。

- 判定に必要な本体制約と説明的補足の区別がつきにくくなる
- 同じ意味の制約が複数箇所に分散し、更新漏れや片更新の不整合リスクが上がる
- 新しい制約と既存制約の言い換えが混在し、要件境界が読み取りにくくなる
- `req-define` に渡す際、本質制約の抽出コストが上がる

## 統合理由

本RUでは、「active REQ 全体から自明・同値記述を削減したい」という要求を、個別文言修正ではなく、削除可否の基準整備として統合する。

理由は以下の通り。

- 個別 REQ ごとに場当たりで削ると、別の REQ では必要な例外条件まで削るおそれがある
- 削除対象を「主語、適用範囲、既存 active REQ から一意に導ける再説明」に限定すれば、意味を保ったまま文量だけを減らせる
- 一方で、対象外条件、例外条件、検査対象範囲、finding 条件、優先関係を変える文は残すべきであり、その線引きを先に要件化する必要がある
- 横断調査により、実際に削減候補と削減対象外の両方が active REQ 内に存在することを確認した

## 要件化の方向

- active REQ のリファクタリングでは、既存要件の主語、適用範囲、既存 active REQ から一意に導ける再説明を削除対象として扱う
- 既存要件の言い換えに留まり、新しい判定基準、例外、対象外、検査境界、優先関係を追加しない文は削除対象として扱う
- 既存制約との「整合する」「追従する」等の関係説明が、判定境界を増やさない場合は削除または要件本文外への整理対象として扱う
- 一方で、対象外条件、例外条件、検査対象範囲、finding 条件、出力粒度、優先関係を実際に変える文は削除対象に含めない
- `REQ-0103-090` から `REQ-0103-092` は本基準の代表事例として見直し対象とする。少なくとも `REQ-0103-092` の括弧内補足は削除候補とする
- `REQ-0113-009` の括弧内補足も、同基準に従う削除候補とする
- `REQ-0115-044` の括弧内補足も、同基準に従う削除候補とする
- 上記基準に基づき、active REQ 全体を横断して自明・同値記述の候補を抽出し、意味不変を前提に整理する

## 対象外

- retired REQ の整理
- 実装計画、編集手順、差分案の具体化
- docs-check ルール追加や自動検出ロジックの実装要件化
- 目的節や背景節の説明量最適化のみを目的とした編集
- 検査境界、対象外条件、例外条件、finding 条件を実際に変える文の削除
- 各 REQ の全面再構成や別テーマとの統合
