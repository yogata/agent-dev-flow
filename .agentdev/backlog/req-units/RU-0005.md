# RU-0005: integrity-check の intake item 作成時 git 永続化不備

## 背景

`/agentdev/integrity-check` 完了後、同一セッション内で抽出 Findings の対処を継続議論し、ユーザー判断により一部 Findings を intake item 化することがある。

`/agentdev/integrity-check` は検査対象 artifact を変更しない read-only diagnostic command だが、検出結果レポートと intake item の新規作成は許容されている。

## 問題

`/agentdev/integrity-check` が intake item を作成した場合でも、現行の完了報告 variant では `git 永続化: 該当なし` が固定されている。

その結果、`.agentdev/intake/inbox/` に作成された intake item が commit / push されず、セッション終了後に未永続化のまま残るリスクがある。

## Source Summary

- `integrity-check` は `.agentdev/intake/inbox/` への intake item 新規作成を許容している。
- finding の intake item 化はユーザー承認時のみ実行される。
- completion report variant は `git 永続化: 該当なし` を固定している。
- `intake-capture` には `.agentdev/intake/` 配下のみを commit / push する既存パターンがある。
- `.agentdev/integrity/reports/` はユーザー判断を含まない再生成可能な診断ログであるため git 永続化対象外とする。

## 統合理由

本件は、新規機能ではなく、既存 `/agentdev/integrity-check` の完了報告・git 永続化挙動の不整合修正である。

intake item はユーザー判断を経た後続 workflow artifact であり、git 永続化対象である。一方、integrity report は診断ログであり、同条件で再生成可能なため git 永続化対象外とする。

## 要件

### 1. integrity-check の git 永続化 Step を追加する

`/agentdev/integrity-check` は、intake item 作成後、完了報告前に以下を行う。

1. `git diff --name-only` で `.agentdev/intake/` 配下の変更を確認する
2. `.agentdev/intake/` 配下に変更がない場合は commit / push をスキップする
3. `.agentdev/intake/` 配下に変更がある場合は、その配下のみ `git add` する
4. commit message は `chore(agentdev): capture integrity intake items` とする
5. `git push` を実行する
6. push 失敗時は完了扱いにせず、構造化エラーを出す

### 2. integrity report を commit 対象に含めない

`git add` 対象は `.agentdev/intake/` 配下に限定する。

`.agentdev/integrity/reports/` は生成されていても commit 対象に含めない。

### 3. 完了報告の git 永続化固定文言を廃止する

現行の `git 永続化: 該当なし` 固定を廃止する。

出力は以下のいずれかに分岐する。

#### intake item 作成あり・push 成功

```markdown
git 永続化:
- commit: {commit_hash}
- push: 成功
- 対象: .agentdev/intake/inbox/{created_files}
- 除外: .agentdev/integrity/reports/（再生成可能な診断ログ）
```

#### intake item 作成なし

```markdown
git 永続化: 変更なし（commit/push スキップ）
```

#### push 失敗

```markdown
git 永続化:
- push: 失敗
- 対象: .agentdev/intake/inbox/{created_files}
- raw git output:
{git_error_output}
```

push 失敗時の検証結果は `⚠️ 注意` とする。

## 対象外

- `.agentdev/integrity/reports/` の git 永続化
- integrity report の archive / rotation 設計
- intake item の review / promote / req-define 化
- finding 分類ロジックの変更
- integrity-check の検査観点追加
- REQ 新設

## 受け入れ条件

- `/agentdev/integrity-check` で intake item が作成された場合、完了報告前に `.agentdev/intake/` 配下のみ commit / push される
- `.agentdev/integrity/reports/` が生成されても commit 対象に含まれない
- intake item 作成ありの場合、完了報告の `git 永続化` に commit hash と push 成否が表示される
- intake item 作成なしの場合、完了報告の `git 永続化` は `変更なし（commit/push スキップ）` となる
- push 失敗時は完了扱いにせず、構造化エラーまたは `⚠️ 注意` として報告される
- `git add` が `.agentdev/intake/` 配下に限定され、他ファイルを巻き込まない
- completion report variant から `git 永続化: 該当なし` 固定が除去される
