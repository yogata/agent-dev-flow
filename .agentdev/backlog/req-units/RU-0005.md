# RU: `/agentdev/case-auto` 最大自走モード追加

## 背景

AgentDevFlow の標準ワークフローは、機能追加では `/agentdev/req-define` → `/agentdev/req-save` → `/agentdev/case-open` → `/agentdev/case-run` → `/agentdev/case-close` である。

バグ修正・保守・ドキュメント作業では `/agentdev/req-save` をスキップし、`/agentdev/case-open` に進む。

この標準フローは維持する。

一方で、ユーザーが明示した場合に、`req-define` 完了後の後続工程を最大限自走させる入口が必要である。

## 問題

現状は、`req-save` 要否判定、`case-open`、`case-run`、`case-close` をユーザーが順に実行する必要がある。

`req-define` で要件が確定した後、repo に閉じる変更であれば、エージェントが自律的に実装・検証・PR作成・merge・close まで進められるべきである。

ただし、DBスキーマの実DB反映、deploy、cloud apply など、repo に閉じない変更は自走対象外とする必要がある。

## Source Summary

この RU は、チャット内で合意した以下の内容に基づく。

- 標準ワークフローは維持する
- 最大自走は明示指定時のみ使用する
- 実装対象は新規コマンド `/agentdev/case-auto` とする
- `req-save --auto` や `case-run --auto-close` は採用しない
- `req-define` と要件深掘りは自走対象外とする
- `req-define` 完了後の `req-save` 要否判定以降を自走対象とする
- repo に閉じる変更のみ自走対象とする
- DB migration 実行、deploy、cloud apply 等は自走対象外とする
- ドキュメント整備も実施対象とする

## 統合理由

`req-save` は REQ/ADR の保存コマンドであり、後続の `case-open` / `case-run` / `case-close` を制御する責務を持たせると責務が混在する。

`case-run` は Issue を入力として PR 作成までを担うため、`req-save` 要否判定や `case-open` 前段を扱えない。

そのため、既存コマンドの責務を変更せず、既存コマンド列を束ねる orchestration command として `/agentdev/case-auto` を追加する。

## 要件化の方向

新規コマンド `/agentdev/case-auto` を追加する。

`/agentdev/case-auto` は、`req-define` 完了後の要件docを入力として、既存の `req-save` / `case-open` / `case-run` / `case-close` 相当の工程を自律的に順次実行する orchestration command とする。

既存コマンドの責務は変更しない。

標準ワークフローも変更しない。

`/agentdev/case-auto` は、ユーザーが明示した場合のみ使用する追加入口とする。

## 自走対象

`/agentdev/case-auto` は以下を自走対象とする。

- `req-save` 要否判定
- `req-save` 実行またはスキップ
- `case-open` 相当処理
- `case-run` 相当処理
- repo に閉じる変更の self-healing
- PR 作成
- PR merge
- Issue comment 追加
- Issue close
- 作成した branch / worktree の削除
- `.agentdev` domain state の永続化
- docs / REQ / ADR / SPEC / command reference / guide の更新

## 分岐

### feature

- `req-save` 相当を実行する
- `case-open` 相当を実行する
- `case-run` 相当を実行する
- `case-close` 相当を実行する

### bugfix / maintenance / docs_chore

- `req-save` をスキップする
- `case-open` 相当を実行する
- `case-run` 相当を実行する
- `case-close` 相当を実行する

## 自走対象外

以下は `/agentdev/case-auto` の自走対象外とする。

- DBスキーマ変更の実DB反映
- DB migration の実行
- deploy / apply
- クラウドリソースの作成・変更・削除
- 外部SaaS設定変更
- 課金・権限・認証情報に関わる変更
- repo 外の実データ作成・更新・削除
- 通知送信など repo 外の状態を変更する操作

## 判定ルール

- repo にファイルとして残る変更は自走対象とする
- repo 外の実体を変更する操作は自走対象外とする
- migration ファイルの作成・修正は自走対象とする
- migration の実行は自走対象外とする
- IaC ファイルの作成・修正は自走対象とする
- IaC の apply は自走対象外とする
- GitHub Issue / PR / comment / merge / close は自走対象とする
- remote branch 削除は、当該 `case-auto` / `case-run` が作成した branch に限定する

## 停止条件

以下を検出した場合、`/agentdev/case-auto` は停止し、停止理由・現在地点・再開可能な次コマンドを報告する。

- `req-define` で合意した要件から外れる
- 要件未合意の scope 拡大が必要
- repo 外の実体変更が必要
- DB migration 実行が必要
- deploy / apply が必要
- 認証情報・秘密情報・権限変更を扱う必要がある
- CI / test / lint の失敗が self-healing で解決不能
- merge conflict または remote hash 不一致により安全に継続できない
- 作成元が不明な branch / user-owned branch / 他作業 branch を削除しようとしている
- 未コミット変更があり、当該 case の変更か判定できない
