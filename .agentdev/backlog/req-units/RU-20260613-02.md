# RU: ADR/REQ/SPEC から一時RU-ID参照を完全削除する

## 背景

RU ファイルは、一時的な要件整理入力であり、永続的な仕様・要件・設計根拠として参照される対象ではない。

ADR / REQ / SPEC は後続工程や保守時に参照される canonical な文書群であるため、本文や履歴欄に特定の RU-ID を残すと、一時入力への依存が残る。

## 問題

ADR / REQ / SPEC の一部に、`RU-0021`、`RU-20260613-02`、`RU-01` などの具体的な RU-ID 指定が残っている。

RU ファイルは一時的なものなので、ADR / REQ / SPEC から具体的な RU-ID を参照してはならない。

既に一部ファイルは削除対応済みだが、未処理の残分があるため、別エージェントで再チェックし、残存分を含めて完全削除する必要がある。

## 統合理由

対象はすべて「ADR / REQ / SPEC から一時RU-ID参照を削除する」という同一目的の文書整備である。

個別ファイルごとに要件を分ける必要はない。残存箇所の再検索、削除、検証を単一の操作単位として扱う。

## 要件化の方向

新しい恒久要件として定義しない。

この RU は、既存 ADR / REQ / SPEC から具体的 RU-ID 参照を完全削除するための cleanup input として扱う。

削除対象は、特定 RU ファイルまたは特定 RU-ID を指す記述に限定する。

以下のような一般概念としての `RU`、`Requirement Unit`、`req-units`、RU lifecycle の説明は削除対象に含めない。

## 主対象REQまたは変更対象候補

主対象は既存文書の直接修正。

変更対象候補:

- `docs/requirements/*.md`
- `docs/adr/*.md`
- `docs/specs/*.md`

既に削除済みのファイルも、再チェック対象に含める。

- `docs/requirements/REQ-0120.md`
- `docs/requirements/REQ-0122.md`
- `docs/requirements/REQ-0116.md`

## 対象外

以下は対象外。

- 新規 REQ / ADR / SPEC の作成
- RU-ID 参照禁止そのものの恒久要件化
- コマンド、スキル、スクリプトへの新規仕様追加
- 実装計画の策定
- Issue 作成
- PR 作成
- 一般概念としての `RU` / `Requirement Unit` / `req-units` の削除
- `.agentdev/backlog/req-units/` 配下の RU ファイル自体の削除
- 会話URLの記録

## 受け入れ条件

- ADR / REQ / SPEC 全文を対象に、具体的な RU-ID 参照を再検索している。
- `RU-[0-9]+`、`RU-YYYYMMDD-*`、`RU-XX` 形式の具体ID参照が ADR / REQ / SPEC に残っていない。
- 既に削除済みとされた `REQ-0120`、`REQ-0122`、`REQ-0116` も再チェック対象に含まれている。
- 残存候補として挙がった `REQ-0101`、`REQ-0102`、`REQ-0103`、`REQ-0104`、`REQ-0105`、`REQ-0106`、`REQ-0107`、`REQ-0108`、`REQ-0109`、`REQ-0112`、`REQ-0114`、`REQ-0121` の該当箇所が処理されている。
- 削除は該当する RU-ID 指定に限定され、隣接する無関係な本文、体裁、表現を変更していない。
- 一般概念としての `RU` / `Requirement Unit` / `req-units` は、具体的な一時RU-ID参照でない限り削除していない。
