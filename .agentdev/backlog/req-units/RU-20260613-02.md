# RU: コマンド・スキル間参照の実在パス化と曖昧参照防止

## 背景

`case-auto` 実行時、`case-open` 周辺の推論ログで以下の不存在ファイルを参照しようとしていた。

- `src/opencode/skills/agentdev-workflow-lifecycle/references/workflow-classification.md`
- `src/opencode/skills/agentdev-workflow-lifecycle/references/issue-generation-protocol.md`

原因は、コマンド・スキル内に自然言語ラベルだけの参照、旧称または曖昧な参照、実在ファイル名と一致しない参照が残っていることである。

`case-auto` は既存コマンド定義を authoritative source として読み、各コマンドの手順に従う設計であるため、参照元の曖昧さがそのまま実行時の誤読に直結する。

## 問題

現行のコマンド・スキルには、次の問題がある。

1. `workflow classification` のような自然言語参照が、存在しない `workflow-classification.md` として推測されうる。
2. `Issue 生成プロトコル` のような自然言語参照が、存在しない `issue-generation-protocol.md` として推測されうる。
3. `agentdev-workflow-lifecycle の「Step X」` という参照があるが、現行の `agentdev-workflow-lifecycle/SKILL.md` は該当 Step 本文を持つ構成ではない。
4. 一部にインラインコードの閉じバッククォート欠落があり、参照範囲が壊れている。
5. 今後も同種の曖昧参照が追加される可能性がある。

## Source Summary

チャット内で、以下の修正対象が合意された。

- Phase 1: 不存在ファイルへ推測される参照の直接修正
- Phase 2: `agentdev-workflow-lifecycle の Step X` 参照の解消
- Phase 3: 今後同じ問題を防ぐ参照ルールの明文化

また、Phase 4 として挙げた `/repo/docs-check` 導線の別件整合性は、今回の対象外と合意された。

## 統合理由

Phase 1〜3 はすべて、コマンド・スキル間参照を実在する参照先へ正規化し、実行時にエージェントが不存在ファイル名を推測しないようにする同一目的の変更である。

Phase 1 だけでは既存の直接不備を直すだけであり、Phase 2 の lifecycle Step 参照が残る。Phase 3 を含めなければ、同種の曖昧参照が再発する。

そのため、Phase 1〜3 を単一の Requirement Unit として扱う。

## 要件化の方向

コマンド・スキル内の参照記述を、以下の基準へ統一する。

1. `references/*.md` を参照する場合は、実在する相対パスを明記する。
2. skill 本体を参照する場合は、`agentdev-xxx/SKILL.md` を明記する。
3. `workflow classification`、`Issue 生成プロトコル`、`Step X` のような自然言語ラベルだけの参照を禁止する。
4. `agentdev-workflow-lifecycle` はハブとして扱い、実体を持たない詳細 Step 参照を置かない。
5. 詳細手順が必要な場合は、既存 reference へ明示的に接続するか、新規 reference を作成して接続する。
6. Markdown のインラインコード範囲を壊さない。

## 要件

### 1. workflow classification 参照の明示化

`workflow classification` または同等の自然言語参照を、実在する参照先へ明示化する。

修正後は、以下のような表現へ統一する。

- `agentdev-workflow-lifecycle/SKILL.md` の work_type classification を参照する

### 2. Issue 生成プロトコル参照の明示化

`Issue 生成プロトコル` または同等の自然言語参照を、実在する参照先へ置換する。

修正後は、以下の参照へ統一する。

- `agentdev-workflow-lifecycle/references/case-open-issue-creation.md`

### 3. 壊れたインラインコード参照の修正

閉じバッククォート欠落などにより参照範囲が壊れている記述を修正する。

特に、`agentdev-workflow-routing` への参照は、skill 名または実在 reference path として読める表記へ修正する。

### 4. lifecycle Step 参照の解消

`agentdev-workflow-lifecycle の「Step X」` のように、実在しない Step 本文を参照している記述を解消する。

解消方法は、以下のいずれかとする。

- command 本体へ必要手順を戻す
- 実在する既存 reference へ接続する
- 必要な場合のみ、新規 reference を作成して接続する

### 5. lifecycle skill のハブ責務の明確化

`agentdev-workflow-lifecycle/SKILL.md` は、workflow phase、work_type classification、SSoT遷移などのハブとして扱う。

詳細手順を外部 reference に分離する場合は、`SKILL.md` から参照可能にする。

### 6. 参照記述ルールの明文化

`agentdev-skill-authoring/SKILL.md` または同等の authoring ルールに、以下を追加する。

- `references/*.md` を参照する場合は、実在する相対パスを明記する
- skill 本体を参照する場合は、`agentdev-xxx/SKILL.md` を明記する
- 自然言語ラベルだけの「〜プロトコル」「〜分類」「Step X」参照を禁止する
- 参照先ファイル名をエージェントに推測させる記述を禁止する

## 対象外

以下は対象外とする。

- `/repo/docs-check` 導線の整合性確認
- Issue 作成
- PR 作成
- 実装計画の詳細化
- コード実装
- コマンド・スキルの責務再設計
- unrelated な表記ゆれ修正
- 参照不備と関係しない Markdown 整形
- 会話URLの記録

## 受け入れ条件

1. `workflow-classification.md` という不存在ファイル名を推測させる参照記述が、対象コマンド・対象スキルから除去されている。
2. `issue-generation-protocol.md` という不存在ファイル名を推測させる参照記述が、対象コマンド・対象スキルから除去されている。
3. `Issue 生成プロトコル` 参照は、実在する `agentdev-workflow-lifecycle/references/case-open-issue-creation.md` へ明示されている。
4. `workflow classification` 参照は、`agentdev-workflow-lifecycle/SKILL.md` の work_type classification へ明示されている。
5. `agentdev-workflow-lifecycle の「Step X」` のような、実在しない詳細 Step 参照が残っていない。
6. 必要な詳細手順は、command 本体または実在する reference path に置かれている。
7. `agentdev-workflow-lifecycle/SKILL.md` から、追加または整理した reference へ到達できる。
8. `agentdev-skill-authoring/SKILL.md` または同等の authoring ルールに、実在パス明記と自然言語ラベル参照禁止のルールが追加されている。
9. Markdown のインラインコード範囲が壊れていない。
10. 今回対象外とした `/repo/docs-check` 導線の整合性確認が、本RUの変更に混入していない。
11. unrelated な表記ゆれ修正、文体修正、整形変更が混入していない。
12. 変更後、`case-auto` / `case-open` 実行時に、存在しない `workflow-classification.md` または `issue-generation-protocol.md` を読みに行く導線が残っていない。
