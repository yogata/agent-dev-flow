# RU: ReferencePath検査の診断性改善と参照解決ルール明確化

## 背景

`/agentdev/integrity-check` の ReferencePath 検査で、`references/git-common-procedures.md`、`references/capture-boundaries.md`、`templates/doc_requirement.md` が存在しない参照として検出された。

当初、検出結果の確認コマンドが `x.filePath` を参照していたが、現行の `CheckResult` 型に存在する場所フィールドは `file` であり、`filePath` ではない。そのため、ReferencePath検査の生成ロジックが参照元ファイルと参照先文字列を混同しているとは断定できない。

## 問題

ReferencePath検査のNG結果について、参照元ファイル、行番号、抽出された参照文字列、実際に存在確認した解決後パスを区別して確認できていない。

この状態では、以下を切り分けられない。

- 参照記述そのものが誤っている
- パス解決ルールが誤っている
- ReferencePath検査のNG生成ロジックが誤っている
- 調査用コマンドが誤ったフィールドを見ている

また、別Skill参照と同一Skill内参照の記述ルールが曖昧な場合、`references/...` や `templates/...` の裸参照が誤検出または見逃しの原因となる。

## Source Summary

- `/agentdev/integrity-check` は `agentdev-integrity` の `scripts/check_integrity.ts` を実行する。
- `CheckResult` の場所フィールドは `file` / `line` であり、証拠値は `evidence`、期待値は `expected` として分離されている。
- Markdown出力も `file` を位置情報、`evidence` を証拠情報として扱う。
- 貼り付けログでは `x.filePath` を参照しており、現行スキーマと一致していない。
- ReferencePath NGとして `references/git-common-procedures.md`、`references/capture-boundaries.md`、`templates/doc_requirement.md` が検出されたが、参照元ファイルの確認が未確定である。

## 統合理由

本件は単なる参照パス修正ではない。

ReferencePath検査の診断出力が不十分なまま参照ファイルだけを修正すると、検査ロジックの欠陥、調査コマンドの誤り、参照記述ルールの曖昧さが残る。

そのため、以下を一体で要件化する。

1. ReferencePath NG結果の診断性改善
2. 調査時に参照すべきフィールドの明確化
3. 同一Skill内参照と別Skill参照のパス記述ルール明確化
4. ReferencePath検査のテスト追加

## 要件化の方向

ReferencePath検査は、NG結果において少なくとも以下を出力する。

- `file`: 参照を書いている元ファイル
- `line`: 行番号
- `evidence`: 抽出された参照文字列
- `expected`: 実際に期待または解決した参照先パス

調査・検証コマンドは `filePath` ではなく `file` を参照する。

パス解決ルールは以下とする。

- 同一Skill内参照では `references/foo.md` / `templates/foo.md` を許容する
- 別Skillのファイルを参照する場合は repo-root 相対パスで記述する
- 別Skill参照に裸の `references/foo.md` / `templates/foo.md` を使わない

## 受け入れ条件

- ReferencePath NGのJSON結果で `file`、`line`、`evidence`、`expected` が確認できる。
- 調査用コマンドまたはドキュメント例が `x.filePath` ではなく `x.file` を参照している。
- `file` には参照元ファイルが出力され、`evidence` には抽出された参照文字列が出力される。
- 同一Skill内の `references/foo.md` / `templates/foo.md` 参照は、該当Skill配下に実体があればOKとなる。
- 別Skillのファイルを裸の `references/foo.md` / `templates/foo.md` で参照した場合はNGとなる。
- 別Skillのファイルを repo-root 相対パスで参照した場合はOKとなる。
- ReferencePath検査について、少なくとも以下のテストが存在する。
  - NG時に `file` が参照元ファイルになる
  - NG時に `evidence` が抽出参照文字列になる
  - 同一Skill内参照がOKになる
  - 別Skillの裸参照がNGになる
  - 別Skillのrepo-root相対参照がOKになる
