# learning deferred pool の単一化

## 背景

`.agentdev/learning/` には、learning entry を一時保持し、`learning-promote` により評価、採用、保留、却下、重複判定へ進める運用がある。

現行整理では、learning の living pool を示す名称として `.agentdev/learning/deferred.md` と `.agentdev/learning/archive/active.md` が混在していた。さらに、`archive/active.md` は `archive/` ディレクトリ配下に置く意味が薄く、`archive` という語も終端保管の印象を持つため、再評価対象を保持する living pool の性質と合わない。

チャット上で、`.agentdev/learning/deferred.md` を唯一の living pool とする方針に合意した。

## 問題

learning の living pool を示すパスと語彙が揺れていると、以下の問題が発生する。

- `learning-promote` の入力、出力、prune 対象が読み取り手により異なる
- `archive/` が終端保管なのか再評価対象なのか不明瞭になる
- `archive/active.md` のような不要な階層が残り、実体のない運用単位を作る
- staged / rejected / duplicate の prune と deferred / 未処理 / 再評価対象の保持境界が曖昧になる

## Source Summary

チャット上で以下に合意した。

- `.agentdev/learning/archive/` は削除する
- `.agentdev/learning/archive/active.md` は廃止する
- `.agentdev/learning/archive.md` は採用しない
- `.agentdev/learning/deferred.md` を唯一の living pool とする
- `deferred.md` は deferred / 未処理 / 再評価対象の learning entry を保持する
- `staged / rejected / duplicate` は `learning-promote` の判定確定時に prune する
- `archive/` に関する記述は残さない
- 「archive/ を参照しない」「archive/ を前提にしない」のような否定形の要件にはしない

## 統合理由

本 RU は、learning の living pool 配置、名称、prune 境界を一体として扱う必要がある。

`archive/active.md` の廃止だけを単独で扱うと、`deferred.md` との関係が未整理のまま残る。`archive.md` へのリネーム案も検討されたが、最終的には `deferred.md` が living pool の意味に最も合うため、learning の保持先を `.agentdev/learning/deferred.md` に統一する。

## 要件化の方向

learning pipeline は、`.agentdev/learning/deferred.md` を唯一の living pool として扱う。

`deferred.md` は、deferred / 未処理 / 再評価対象の learning entry を保持する。`learning-promote` は判定確定時に、staged / rejected / duplicate を `deferred.md` から prune し、deferred / 未処理 / 再評価対象を保持する。

`.agentdev/learning/archive/`、`.agentdev/learning/archive/active.md`、`.agentdev/learning/archive.md` は learning の成果物配置として扱わない。learning 関連の文書、コマンド、スキル、テンプレート内の living pool 表現は `.agentdev/learning/deferred.md` に統一する。

## 主対象REQまたは変更対象候補

主対象REQまたは変更対象候補は不明。

変更対象候補は以下。

- `src/opencode/commands/agentdev/learning-promote.md`
- `src/opencode/skills/agentdev-learning-pipeline/SKILL.md`
- learning pipeline を参照する doc-input、template、関連文書
- `.agentdev/learning/archive/` 配下の実体が存在する場合、その削除
- `.agentdev/learning/archive.md` が存在する場合、その不採用整理
- `.agentdev/learning/deferred.md` の配置契約

## 対象外

- learning entry の評価ロジック変更
- 8軸評価の見直し
- promote / defer / reject / duplicate の判定基準変更
- backlog-review の RU 生成仕様変更
- intake / inspect の lifecycle 変更
- 新規 ledger、履歴ファイル、終端アーカイブの追加
- `archive/` を参照しないこと、または前提にしないことを独立した否定要件として定義すること

## 受け入れ条件

- learning の living pool が `.agentdev/learning/deferred.md` に統一されている
- `deferred.md` が deferred / 未処理 / 再評価対象を保持する living pool として説明されている
- `learning-promote` の判定確定時に staged / rejected / duplicate を prune する境界が説明されている
- deferred / 未処理 / 再評価対象は prune 対象外として説明されている
- learning 関連の記述から `.agentdev/learning/archive/` および `.agentdev/learning/archive/active.md` に関する記述が削除されている
- `.agentdev/learning/archive.md` は採用されていない
- `archive/` を参照しない、または前提にしないという否定形の要件が追加されていない
- 変更後の記述で、learning の living pool と終端アーカイブが混同されていない
