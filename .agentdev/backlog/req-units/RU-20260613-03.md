# RU-20260613-03 RFC2119ルールおよび記載の完全削除

## 背景
現行の AgentDevFlow では、RFC2119 規範語を「一部で使わない」方針に移している一方で、active REQ / SPEC / ADR、およびそれらに依存する integrity 定義・skill・command・test には、RFC2119 ルールや `SHALL` / `MUST` / `SHOULD` / `MAY` の記載が依然として残っている。

今回の合意は、「基本的に使わない」ではなく、RFC2119 ルールおよび記載を完全に削除する方向へ方針を切り替えるものである。

## 問題
現状は以下の不整合を含む。

- REQ / SPEC / ADR に RFC2119 の扱いに関するルール自体が残っている
- active 文書内に RFC2119 規範語の記載が残っている
- integrity 検査が RFC2119 marker や RFC2119 前提の判定基準を保持している
- runtime skill / command / test が RFC2119 前提の語彙や説明を保持している

この状態では、「RFC2119 を使わない」という運用方針と、実際の基準文書・検査・実行時定義が一致していない。

## Source Summary
チャット内で以下が合意された。

- RFC2119 ルール及び記載を全て完全に削除する
- 実施対象は REQ / SPEC / ADR にとどまらず、依存する integrity・skill・command・test まで含めて整合させる必要がある
- retired 文書や履歴アーティファクトまで一括修正する前提ではない
- 別エージェントに依頼するため、その作業を実行できる粒度の実行計画が必要である

## 統合理由
RFC2119 の「部分的削減」ではなく「完全削除」を行うには、要件文書だけを修正しても不十分である。基準文書、検査ルール、runtime 定義、テストまで同時に再基準化しないと、文書間の責務境界と integrity が再び崩れるため、単一の Requirement Unit として統合して扱う必要がある。

## 要件化の方向
以下の方向で要件化する。

- active REQ / SPEC / ADR から RFC2119 ルールおよび RFC2119 記載を完全に削除する
- RFC2119 前提の記述を、自然文ベースの必達要件・禁止事項・推奨事項・任意事項の表現へ再基準化する
- integrity 検査から RFC2119 marker 検査および RFC2119 前提の判定定義を除去または置換する
- `.opencode/` および `src/opencode/` の skill / command / reference / test に残る RFC2119 依存記述を追従修正する
- retired 文書や履歴アーティファクトは、現行基準として再利用しない前提で原則対象外とする
- 削除後は横断検索と integrity テストで残存ゼロを検証できる状態にする
