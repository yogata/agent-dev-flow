# RU: command / skill の存在しない参照パス修正と再発検査追加

## 背景

command / skill の薄型化・参照分離の過程で、実在しない script / template / completion report variant / 旧command への参照が残存している。

特に `/agentdev/integrity-check` は、過去には `agentdev-integrity` skill 配下の script を参照して実行できていたが、現行定義では root `scripts/check_integrity.ts` を参照しており、実行時に script を発見できない。

## 問題

以下の参照不整合により、該当 command 実行時に存在しないファイル・旧成果物を前提として失敗または誤誘導する。

1. `/agentdev/integrity-check` が `scripts/check_integrity.ts` を参照している
2. `/agentdev/req-define` が存在しない completion report variant を参照している
3. `/agentdev/case-update` が存在しない backlog 系 Issue template を参照している
4. `/agentdev/backlog-review` が削除済み `req-backlog` のエラー形式を参照している

## Source Summary

- `/agentdev/integrity-check` は root `scripts/check_integrity.ts` を参照しているが、過去に実装変更されていた実体は `.opencode/skills/agentdev-integrity/scripts/check_integrity.ts` だった。
- `/agentdev/req-define` は `bug-or-lightweight.md` / `maintenance.md` / `docs-or-chore.md` を参照しているが、現行 registry では `lightweight.md` に統合されている。
- `/agentdev/case-update` は `issue_desc_backlog_*.md` を参照しているが、現行 template 一覧には存在しない。
- `/agentdev/backlog-review` は削除済み `req-backlog` のエラー形式を参照している。

## 統合理由

いずれも command / skill の参照先が現行 artifact 実体と一致していない同一種別の不整合である。

個別修正だけでは再発するため、参照修正と integrity-check による再発検査を同一RUで扱う。

## 要件化の方向

既存 command / skill の参照先を現行実体に合わせて修正する。

あわせて、integrity-check に script / template / completion report variant / reference path の存在確認を追加し、同種の退行を検出できるようにする。

manual fallback は導入しない。参照先が存在しない場合は構造化 finding として検出する。
