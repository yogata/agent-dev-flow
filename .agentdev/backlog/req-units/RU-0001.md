# learning-promote の後続ルート不整合修正

## 背景

`/agentdev/learning-promote` の完了報告で、次のコマンドとして `/agentdev/req-define` が出力された。

しかし現行の workflow では、`.agentdev/learning/promoted/*.md` は `req-define` に直接渡さず、まず `/agentdev/req-backlog` で Requirement Unit（RU）へ変換する。

完了報告variant `completion-reports/learning-promote/standard.md` では、次のコマンドは `/agentdev/req-backlog` と定義済みである。

## 問題

`learning-promote.md` 本体に、`req-backlog` 導入前の古い後続ルート記述が残っている。

具体的には、以下の趣旨の記述が残存している。

- 生成した stub は `/agentdev/req-define` の明示入力ファイルとして扱う
- `/agentdev/req-define → /agentdev/req-save → /agentdev/case-open → /agentdev/case-run` のルートで実装に移行する
- `case-run` への直接受け渡しは禁止し、`req-define` を経由する

一方で、`req-define.md` では `.agentdev/intake/promoted/*.md` および `.agentdev/learning/promoted/*.md` の直接読み込みが禁止され、promoted artifact は `req-backlog` による RU 化を経由する必要がある。

そのため、`learning-promote.md` 本体・完了報告variant・`req-define.md` の間で後続ルートの定義が不整合になっている。

また、実際の完了報告では完了報告後に `# Todos` と補足文が出力されており、完了報告を最終出力とする reporting ルールにも違反している。

## Source Summary

本RUは、チャット内で確認した以下の事実に基づく。

- `completion-reports/learning-promote/standard.md` は、次のコマンドを `/agentdev/req-backlog` と定義している
- `learning-promote.md` 本体には、`/agentdev/req-define` 直行を前提とする古い記述が残っている
- `req-define.md` は promoted artifact の直接読み込みを禁止し、`req-backlog` 経由を求めている
- `agentdev-workflow-reporting` は、完了報告後の Todo 一覧・追加出力を違反条件として扱う

## 統合理由

本件は、単なる完了報告文言の誤りではなく、以下の複数箇所にまたがる同一原因の整合性不備である。

- learning-promote command 本体の後続ルート説明
- learning-promote 完了報告の実出力
- completion-report variant との一致
- req-define の promoted artifact 直読み禁止ルールとの整合

したがって、1つのRUとして統合し、`learning-promote` の後続ルート定義と完了報告適合性をまとめて修正対象とする。

## 要件化の方向

`learning-promote` の成果物は `.agentdev/learning/promoted/*.md` に生成される promoted artifact であり、次工程は `/agentdev/req-backlog` と定義する。

`req-backlog` により `.agentdev/backlog/req-units/RU-*.md` が生成された後、そのRUを `/agentdev/req-define` の入力として扱う。

修正後の標準ルートは以下とする。

```text
/agentdev/learning-promote
→ /agentdev/req-backlog
→ /agentdev/req-define
→ /agentdev/req-save
→ /agentdev/case-open
→ /agentdev/case-run
```
