# RU: integrity-check recommendation label 用語是正

## 背景

第2段階で `integrity-check` に command implementation pattern / load_skills / skill USE FOR 整合性の診断機能を追加した。

その際、診断結果の message に `[phase3: ...]` という分類ラベルを出力する実装が追加された。

しかし、Phase 1 / Phase 2 / Phase 3 は今回の command implementation pattern 見直しスレッド内の進行段階であり、AgentDevFlow 全体の恒久的な workflow / lifecycle 概念ではない。

一方で、`load_skills-remove-candidate`, `load_skills-add-candidate`, `skill-use-for-update-candidate`, `integrity-check-gap`, `no-action` などの診断分類そのものは、今後も `integrity-check` の汎用的な recommendation として有効である。

## 問題

`integrity-check` の恒久的な診断出力に `[phase3: ...]` が残ると、以下の誤解を招く。

- AgentDevFlow に普遍概念として Phase 1 / Phase 2 / Phase 3 が存在するように見える
- `integrity-check` に内部フェーズとして phase3 があるように見える
- 診断結果が特定セッションの第3段階実装にだけ紐づいているように見える

これは診断分類の問題ではなく、出力ラベルの用語問題である。
