## 要件化の方向

AgentDevFlow 配布 command と skill から、具体的な harness 名、エージェント名、`task()` の引数、実行 command、prompt、timeout、並列数、再試行、context 管理、ハーネス起動失敗の解析と救済、一時 harness 成果物の管理を除去する。

AgentDevFlow には、工程の進行条件と停止条件、`.agentdev/`、REQ、ADR、SPEC、Issue、PR の永続成果物、Quality Gate、実装実行の業務上の result 契約だけを残す。

実装実行の result 契約は、PR 作成済み、業務上の blocker、実行失敗、委譲起動不能を区別し、case-run と case-auto が次工程または停止を判断できる形式とする。

Harness の具体的な agent 選定と実行制御は AgentDevFlow 配布物の外部で定義する。

`.sisyphus/`、`.omo/`、ledger、実行計画、session state は harness の一時成果物として扱い、AgentDevFlow は作成、検査、cleanup を要求しない。

本 RU は AgentDevFlow 本体の責務境界変更を対象とするため、`agentdev_handoff: true` を付与する。
