# RU: integrity-check の load_skills 旧仕様残存修正

## 背景

`integrity-check` 実行結果で、全 command に対して `Missing frontmatter fields: load_skills` が NG として報告された。

しかし現行仕様では、command frontmatter の必須フィールドは `description` と `agent` のみであり、`load_skills` は dev メタデータとして runtime frontmatter から排除されるべき項目である。

## 問題

`integrity-check` の実装および `agentdev-integrity` の説明に、旧 `load_skills` 前提の検査が残っている。

具体的には以下が問題である。

- command frontmatter の必須項目として `load_skills` を要求している
- `load_skills` を根拠に skill 存在確認を実施している
- `load_skills` を根拠に未使用 skill 判定を実施している
- `agentdev-integrity/SKILL.md` に `Skill ↔ load_skills` 前提の説明が残っている

これにより、最新仕様に準拠した command 定義が誤って NG になる。

## Source Summary

- 添付された `2026-06-04-integrity-report.md` では、複数 command に対して `Missing frontmatter fields: load_skills` が NG として出力されている。
- 現行 `REQ-0108` では、command frontmatter の必須フィールドは `description` と `agent` のみと規定されている。
- 現行 `REQ-0108` では、`load_skills` を根拠とした存在確認・必須確認・過不足確認・完了報告 skill 参照必須確認を実行しないことが規定されている。
- 現行 `REQ-0108` では、未使用 skill の報告を残す場合でも command frontmatter の `load_skills` を根拠にしないことが規定されている。
- 現行 `check_integrity.ts` には、`load_skills` 抽出、`load_skills` 存在確認、`load_skills` 根拠の未使用 skill 判定、command 必須frontmatterとしての `load_skills` 要求が残っている。
- 現行 `agentdev-integrity/SKILL.md` には、`Skill ↔ load_skills` 前提の検査説明が残っている。

## 統合理由

本件は新規仕様追加ではなく、既存仕様 `REQ-0108-129`〜`REQ-0108-135` への未準拠を修正するものである。

したがって、個別findingの修正ではなく、`integrity-check` の実装・説明・回帰テストを最新仕様へ揃える1つの修正単位として扱う。

## 要件化の方向

既存 `REQ-0108` に基づき、`integrity-check` から旧 `load_skills` 前提の検査を除去する。

実装では以下を満たす。

1. command frontmatter の必須フィールドを `description` と `agent` のみにする。
2. `load_skills` 欠落を NG として報告しない。
3. `load_skills` を根拠とした skill 存在確認を実行しない。
4. `load_skills` を根拠とした未使用 skill 判定を実行しない。
5. `load_skills` が command frontmatter に存在する場合は、dev メタデータ混入として NG にする。
6. `agentdev-integrity/SKILL.md` から旧 `load_skills` 前提の説明を削除または現行仕様へ更新する。
7. regression test により、`description` と `agent` のみを持つ command frontmatter が valid になることを確認する。
8. regression test により、`load_skills` が存在する command frontmatter が NG になることを確認する。
