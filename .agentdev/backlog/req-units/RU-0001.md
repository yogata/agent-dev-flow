# RU-0001: repo-local integrity-check 分離

## 背景

現行の `/agentdev/integrity-check` と `agentdev-integrity` は、AgentDevFlow の配布対象である `src/opencode/commands/agentdev/` および `src/opencode/skills/agentdev-*` 配下に存在している。

しかし実際の検査内容は、`agent-dev-flow` リポジトリ固有の文書体系・運用状態を前提としている。

対象には以下が含まれる。

- REQ / ADR / SPEC / DOC-MAP
- retired REQ
- mapping-table
- source/projection
- vocabulary registry
- integrity rule catalog
- `.agentdev/integrity/` の reports / baseline
- self-hosting repo 固有の namespace / path / history

そのため、AgentDevFlow の consumer 向け配布物と、self-hosting repo 専用の自己監査 tooling が混在している。

## 問題

現状では以下の責務混線がある。

- consumer project に配布すべきでない repo 固有監査ロジックが `src/opencode/` 配下に存在している
- `/agentdev/integrity-check` が AgentDevFlow public workflow の一部に見える
- `agentdev-integrity` が consumer にも有効な汎用 skill のように見える
- `.agentdev/vocabulary-registry.md` が domain state 配下にあるが、実体は状態ではなく静的な検査レジストリである
- docs/ 配下で `integrity-check` の意味が「AgentDevFlow 配布機能」と「self-hosting repo 専用監査」の間で曖昧になっている

## Source Summary

この RU は、チャット内で合意した以下の内容を source とする。

- `integrity-check` の実装はこのリポジトリに特化している
- `src/opencode/` は AgentDevFlow consumer 向け配布 source として扱う
- repo 固有の integrity tooling は `src/opencode/` から分離する
- 分離後の command namespace は `/repo/*` とする
- 分離後の skill prefix は `repo-*` とする
- `vocabulary-registry.md` は `.agentdev/` ではなく repo-local integrity skill の `references/` に置く
- docs/ 配下は単なる名前変更ではなく、配布対象外化の意味に基づいて修正する

## 統合理由

本件は単なる rename ではない。

`/agentdev/integrity-check` から `/repo/integrity-check` への変更は、以下を同時に意味する。

- AgentDevFlow public workflow からの除外
- consumer 配布対象からの除外
- self-hosting repo 専用保守コマンドへの再分類
- `agentdev-integrity` から `repo-agentdev-integrity` への責務移管
- `vocabulary-registry.md` の domain state 外への移動
- docs/ 配下の意味修正

そのため、1つの RU として統合して扱う。

## 要件化の方向

`agent-dev-flow` リポジトリ固有の整合性検査を AgentDevFlow 配布対象から分離し、repo-local な自己監査 tooling として再定義する。

## 合意済み要件

### RU-0001-001: repo-local command namespace

`/repo/*` を `agent-dev-flow` self-hosting repo 専用 command namespace として定義すること。

### RU-0001-002: integrity-check の移管

現行の `/agentdev/integrity-check` を AgentDevFlow public workflow から除外し、repo-local command として `/repo/integrity-check` に移管すること。

```text

```

### RU-0001-003: repo-local skill prefix

`repo-*` を `agent-dev-flow` self-hosting repo 専用 skill prefix として定義すること。

### RU-0001-004: agentdev-integrity の移管

現行の `agentdev-integrity` を AgentDevFlow 配布対象 skill から除外し、repo-local skill として `repo-agentdev-integrity` に移管すること。

```text
  references/

```

### RU-0001-005: src/opencode からの分離

以下を `src/opencode/` から削除すること。


repo-local command / skill は `src/opencode/` に置かないこと。

### RU-0001-006: vocabulary-registry の移動

```text

```

```text

```

### RU-0001-007: .agentdev の責務整理

`.agentdev/` は domain state として扱い、静的 registry を置かないこと。

`.agentdev/` に置く対象は、reports / baseline / intake / learning / backlog などの実行結果・状態に限定すること。

### RU-0001-008: sync 対象外

`sync-opencode.ps1` は repo-local command / skill を junction 管理対象に含めないこと。

sync / junction 管理対象は以下に限定すること。

```text
.opencode/commands/agentdev/  → src/opencode/commands/agentdev/
.opencode/skills/agentdev-*/  → src/opencode/skills/agentdev-*/
```

以下は sync / junction 管理対象に含めないこと。

```text
.opencode/commands/repo/
.opencode/skills/repo-*/
```

### RU-0001-009: docs の意味修正

docs/ 配下では単なるパス置換ではなく、以下の意味に基づいて修正すること。

- `/agentdev/integrity-check` は AgentDevFlow public workflow ではない
- `/repo/integrity-check` は self-hosting repo 専用監査である
- `repo-agentdev-integrity` は consumer 配布対象ではない
- vocabulary registry は repo-local integrity skill の参照データである
- consumer 導入文書では repo-local integrity tooling を配布対象として案内しない

### RU-0001-010: docs 全体確認

docs/ 配下で以下を全文検索し、文脈に応じて修正すること。

```text

.opencode/skills/agentdev-integrity

.agentdev/vocabulary-registry.md
```

```text

.agentdev/vocabulary-registry.md
  → .opencode/skills/repo-agentdev-integrity/references/vocabulary-registry.md
```

consumer / AgentDevFlow 配布物としての記述は削除または「配布対象外」に変更すること。

履歴記述は残してよいが、historical / retired / old name と明記すること。
