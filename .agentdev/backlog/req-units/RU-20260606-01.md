# Runtime Template Resolution の修正

## 背景

case-auto の case-close 処理において、Issue コメントテンプレート `issue_comment_feature_implementation.md` を誤った場所から読み込もうとする事象が発生した。

発生した誤参照は以下。

```text
src\opencode\commands\agentdev\templates\issue_comment_feature_implementation.md
```

正しい参照先は以下。

```text
.opencode\skills\agentdev-workflow-templates\templates\issue_comment_feature_implementation.md
```

現行方針では、テンプレートは用途別に command-local と skill 側に分かれる。

- Issue本文 / Issueコメント / PR本文テンプレートは `agentdev-workflow-templates` skill 側
- コマンド完了報告テンプレートは `commands/agentdev/templates/{command}/` 側

また、runtime 実行時の参照先は `.opencode/...` であり、`src/opencode/...` は canonical source 配置または install/sync の入力元として扱う。

## 問題

case-auto / case-close 実行時に、以下の2種類の誤りが発生しうる。

1. Issueコメントテンプレートを command-local template 側に探索する
2. runtime 実行時に `.opencode/...` ではなく `src/opencode/...` を参照する

この結果、テンプレート探索で `File not found` が発生する。今回の実行では後続探索により正しいテンプレートへ到達したが、実行手順としては不正である。

## Source Summary

チャット内で以下を合意した。

- `commands/agentdev/templates/` はコマンド完了報告テンプレート用である
- `skills/agentdev-workflow-templates/templates/` は Issue本文、Issueコメント、PR本文テンプレート用である
- `issue_comment_feature_implementation.md` は case-close の Issue コメントテンプレートであり、workflow-templates skill 側を参照する
- runtime 実行時の commands / skills / templates 参照は `.opencode/...` を使う
- `src/opencode/...` は source 配置説明、install/sync 入力元、authoring context に限定する
- docs/* の `src/opencode` 表記を一括置換してはならない
- case-auto は既存 command 定義を authoritative source とし、工程固有手順を再実装しない

## 統合理由

テンプレート種別の誤探索と runtime path の誤参照は、どちらも case-auto / case-close 実行時の参照解決不備である。

個別のファイルパス修正だけでは、同種の誤探索が別テンプレートで再発する可能性がある。  
そのため、テンプレート種別ごとの参照先と runtime/source path の使い分けを同一の修正単位として扱う。

## 要件化の方向

以下を要件として明確化する。

- commands / skills の実行時 Read / Glob 対象は `.opencode/...` とする
- `src/opencode/...` は canonical source 配置、install/sync の入力元、authoring context に限定する
- Issue本文 / Issueコメント / PR本文テンプレートは workflow-templates skill 側から参照する
- command-local templates はコマンド完了報告テンプレートに限定する
- case-auto は既存 command 定義の runtime path を `src/opencode/...` に読み替えない
- case-close は Issueコメントテンプレートと完了報告テンプレートを用途別に正しく参照する

## 主対象REQまたは変更対象候補

主対象候補:

- case-auto の実行時参照解決に関する既存REQ
- case-close の後処理・コメント投稿に関する既存REQ
- template / artifact contract に関する既存SPECまたはREQ
- integrity / validation に関する既存REQ

変更対象候補:

- `src/opencode/commands/agentdev/case-auto.md`
- `src/opencode/commands/agentdev/case-close.md`
- `docs/specs/artifact-contracts.md`
- integrity / validation 関連の docs
- 必要に応じて integrity-check の検査条件

## 対象外

以下は対象外とする。

- テンプレート配置の統合
- `commands/agentdev/templates/` と `skills/agentdev-workflow-templates/templates/` の統合
- `docs/*` に存在する `src/opencode` 表記の一括置換
- source 配置説明として正しい `src/opencode/...` 記述の変更
- install / sync script における `src/opencode/...` 入力元参照の変更
- command / skill の大規模整理
- case-auto の工程再設計
- 既存 command の責務変更

## 受け入れ条件

- case-close 実行時に `issue_comment_feature_implementation.md` を command-local template 側へ探索しない
- Issueコメントテンプレートは `.opencode/skills/agentdev-workflow-templates/templates/issue_comment_*.md` から参照される
- コマンド完了報告テンプレートは `.opencode/commands/agentdev/templates/{command}/standard.md` から参照される
- commands / skills の実行時 Read / Glob 手順に `src/opencode/...` 固定参照が残らない
- `src/opencode/...` は source 配置説明、install/sync 入力元、authoring context に限定される
- docs/* では source path と runtime path の使い分けが明確である
- `docs/*` の正しい source 配置説明は不要に変更されない
- case-auto は既存 command 定義の runtime path を尊重する
- 同種の誤探索を検出できる検証条件が定義される
