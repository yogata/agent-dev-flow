# RU: learning-promote 後続ルートの整合性修正

## 背景

`learning-promote` 完了レポートで、生成された promoted artifact を `/agentdev/req-define` へ直接渡す後続ルートが提示された。

現行仕様では、`.agentdev/learning/promoted/` の artifact は `/agentdev/backlog-review` の入力であり、`/agentdev/backlog-save` による RU 化後に `/agentdev/req-define` へ合流する。

正規ルートは以下である。

````text
learning-promote
→ backlog-review
→ backlog-save
→ RU
→ req-define
→ req-save
→ case-open
→ case-run
````

## 問題

`learning-promote` 完了報告で、以下の誤った後続ルートが提示された。

````text
promoted/*.md → req-define → req-save → case-open → case-run
````

この出力は、promoted artifact を req-define が直接入力できるように誤認させる。

原因は、command 本体よりも周辺記述にある。

- `agentdev-learning-pipeline` の `promoted/` 説明が、`req-define` 直接入力と読める
- `staging stub` が `req-define` の汎用入力ファイルとして機能すると読める
- `req-define.md` の Input に `staging stub` と promoted 直読み禁止が併存している
- `learning-promote` 完了報告テンプレートが `backlog-review` 起点を示しているものの、`req-define` 直行禁止までは明示していない

## Source Summary

対象は、`learning-promote` 後の promoted artifact の取り扱いに関する command / skill / template の整合性修正である。

現行の責務分界は以下とする。

- `learning-promote`: learning item から promoted artifact / staging stub を生成する
- `backlog-review`: promoted artifact を分析・統合・分割判定し、review draft を作る
- `backlog-save`: reviewed draft から RU を生成する
- `req-define`: RU 化済みの Requirement Source を入力として要件定義する

## 統合理由

今回の不整合は、単一の完了報告ミスではなく、`learning-promote`、`agentdev-learning-pipeline`、`req-define`、完了報告テンプレートの間で後続ルートの表現が揺れていることに起因する。

個別に修正すると再発するため、REQ-0105 UPDATE として、Intake / Learning / Backlog pipeline の責務分界に合わせて一括で整合させる。

## 要件化の方向

REQ-0105 UPDATE として扱う。

promoted artifact の後続ルートを `backlog-review → backlog-save → RU → req-define` に固定し、`req-define` 直行と読める記述を排除する。
