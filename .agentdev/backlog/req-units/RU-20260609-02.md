# RU: 文書分類ポリシーの同期漏れを command/skill/docs-check まで解消する

## 背景

`REQ-0116` と `document-model.md` により、文書分類ポリシーが整理され、ADR の定義は「将来の設計・運用・文書システムを制約する決定」へ拡張された。

一方で、既存の REQ、skill、reference、checker の一部は旧定義・旧 status 形式・旧語彙のまま残っている。

## 問題

`REQ-0101-008` は ADR を「取り返しのつかない技術判断の記録」としており、`REQ-0116-010` と矛盾している。

また、`agentdev-skill-authoring`、`agentdev-adr-guidelines`、`agentdev-req-file-manager`、`vocabulary-registry` などに旧分類ルールが残っている。`docs-check` の分類ポリシー検査も `--classification` 指定時のみで、かつ深度が浅い。

## Source Summary

- 文書分類ポリシーは既に `document-model.md` に定義されている。
- active REQ 間で ADR 定義が競合している。
- `superseded_by` 正規形式、旧 REQ status 語彙、source/projection 用語の同期漏れが command / skill 側に残っている。
- `docs-check` は分類ポリシーを常時監視する水準まで達していない。

## 統合理由

本件は文書分類ポリシーの「文書本文への反映」だけでなく、「作成支援 skill」「レビュー command」「機械検査 docs-check」までを同期する作業である。

`REQ-0116` の Case 3 相当として、REQ-0101 / REQ-0108 / REQ-0116 をまたいで APPEND / UPDATE するのが妥当である。

## 要件化の方向

以下を APPEND / UPDATE する。

- `REQ-0101` の ADR 定義を `REQ-0116-010`
と整合する形に更新する。
- command / skill / reference から旧 ADR 定義、旧 `superseded-by:[ADR-XXXX]` 形式、旧 REQ status 語彙（`retained / partially superseded / superseded`）を除去・置換する。
- `docs-check` は分類ポリシー検査を常時実行対象に含め、少なくとも以下を検出する:
  - 旧 ADR 定義の残存
  - 旧 `superseded-by:[ADR-XXXX]` 形式
  - 旧 REQ status 語彙
  - source/projection 用語の逆転または誤用
- `docs-review` / skill-authoring は、新しい分類ポリシーに従った判断導線を前提とする。
