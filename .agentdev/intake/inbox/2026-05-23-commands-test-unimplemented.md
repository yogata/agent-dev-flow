# agentdev コマンド群のテスト（ユニットテスト・E2Eテスト）体系的未実装

## 観測
2026-05-16〜2026-05-23 にクローズされた多数の Issue/PR に、未チェックのテスト関連チェックボックスが残存している。ユニットテスト・E2Eテストの体系的なテストスイートが存在せず、各コマンドの動作検証が手動または未実施のままクローズされている。

## 今回扱わない理由
各 Issue/PR は機能実装・バグ修正自体をスコープとしており、テスト基盤の構築は別課題として扱われていた。テスト未実装は個別 Issue の完了をブロックしない慣例となっている。

## 影響
- コマンド・スキル・テンプレートの回帰テストが不可能
- リファクタリング時のデグレード検出が困難
- CI による自動品質ゲートが機能しない

## レビューで決めること
- テスト基盤（フレームワーク・ディレクトリ構造）の選定
- 優先的にテストを追加すべきコマンド/スキルの範囲
- テスト実装を既存 Issue に紐付けるか、独立した Epic として扱うか
