# REQ-0146-011 意味不一致（識別子中心評価との参照ズレ）

## 観測内容

PR #1559（Issue #1558 / AG-001..AG-007, feature）の case-close capture 回収（Step 10）で検出。capture 分類: intake（後続作業候補 = 別 Issue 検討推奨、user 確認候補）。

REQ-0146-015（Issue #1558 の req-save 工程、commit `c36d1c37` で追加）は、QG-4 観点9 の要件根拠として「識別子中心評価（REQ-0146-011）の運用実例集を同 reference に蓄積し」と明記する。しかし現 REQ-0146-011 の実際の内容は「case-open は子 Issue 本文に『前工程完了度』属性を埋め込む」であり、「識別子中心評価」とは意味的に不一致である。

PR #1559 の AG-004 実装では以下とした:

- REQ-0146-015 の後段（運用実例集の蓄積、主評価値と補助値の使い分け）に基づき qg-4 reference 観点9「識別子中心評価の運用実例集」を実装した。
- REQ-0146-011 への参照は qg-4 reference 観点9 に注意書きとして明記した（自律的な REQ 修正は行わない方針）。
- 実害は現状ない（qg-4 観点9 は REQ-0146-015 の後段に基づき機能する）が、REQ-0146-015 → REQ-0146-011 の参照が意味的に不正確なまま残存する。

## 影響

現状の機能的実害はない（qg-4 観点9 は REQ-0146-015 の後段に基づき機能する）。ただし REQ-0146-015 → REQ-0146-011 の参照が意味的に不正確なまま残留し、REQ 体系の意味整合性を損なう。REQ-0146-011 を別文脈で参照する将来の作業で誤解を生む可能性がある。

## 課題

REQ-0146-015 の REQ-0146-011 参照と、REQ-0146-011 の実内容との間の意味不一致を解消する。

## 既存要件・仕様との関連

- REQ-0146-011（現内容「case-open は子 Issue 本文に『前工程完了度』属性を埋め込む」）: 参照先。
- REQ-0146-015（REQ-0146-011 を「識別子中心評価の運用実例集」として根拠引用）: 参照元。
- 関係: 参照先 REQ-0146-011 の実内容と、参照元 REQ-0146-015 が期待する意味（識別子中心評価）が矛盾。
- qg-4-final-acceptance.md 観点9（L166）: 意味不一致の注記が残留。

## 対応方針の方向性

3案を列挙（item は (a) を推奨、user 確認が必要）。

- (b) REQ-0146-015 の参照先変更: REQ-0146-015 の参照を REQ-0146-011 から新規 REQ（例: REQ-0146-016「識別子中心評価の運用実例集」）に変更する。REQ-0146-011 は現状維持。
- (c) REQ-0146-011 の廃止と新規 REQ 移行: REQ-0146-011 を廃止し、新規 REQ（REQ-0146-016 等）へ要件を統合再編する。参照元（REQ-0146-015 等）は新 REQ へ参照更新する。
