# 文書品質 機械的置換ルールの skill references 明文化

## 観測内容

本成果物は単一 inbox 項目 `2026-06-23-docs-quality-mechanical-replacement-rules.md` を整理する。Epic #1075（REQ-0140-026 文書品質準拠の包括的修正）の Wave 2 で実施した 4 つの機械的テキスト置換 OU の各 PR Findings で、機械判定ルールの明文化が今後の類似作業の再現性を向上させると指摘された。各 OU と PR の対応は以下の通り。

| OU | Issue | PR | 対象規範 |
|----|-------|----|---------|
| OU-001 中黒 | #1078 | #1088 | 中黒許容範囲判定 |
| OU-002 em-dash | #1076 | #1089 | em-dash 置換判定 |
| OU-003 LLM 表現 | #1079 | #1090 | LLM 表現機械判定 |
| OU-004 一文一行 | #1080 | #1091 | 一文一行機械判定 |

各 PR の Findings から抽出した機械判定ルールは以下の通り。

1. **中黒許容範囲判定**（PR #1088）: 「SPEC 明示例 + 時系列対比ペア（X時・Y時）+ code block + YAML title」の組合せで機械判定可能。これらの文脈では中黒を許容し、それ以外の日本語並列では是正対象とする。
2. **em-dash 置換判定**（PR #1089）: ` — ` パターンの検出で大部分をカバーする。テーブルセルの N/A プレースホルダ（`| — |`）は `| - |` への置換が自然である。
3. **LLM 表現機械判定**（PR #1090）: 「において→で」「非常に→削除」等の表層形式の置換で大部分をカバーする。文脈判断が必要な「最も」「重要な」等は機械判定の対象外とし、サンプリング査読で対応する。
4. **一文一行機械判定**（PR #1091）: 複数の `。` を含む prose line の改行分割で大部分をカバーする。ただし括弧（`（）`）内の `。` は除外することで文境界を正確に判定できる。

## 影響

機械判定ルールが未明文化のままでは、次回以降の同種作業（inspect-docs での自動検出、CI での機械的検証、再発時の横断修正）で判定基準が実行者に依存し、再現性が揺らぐ。既存 SPEC `document-type-responsibilities.md` の用語政策節には人間向けの判定基準はあるが、機械適用向けのアルゴリズム的記述がない。そのため `inspect-docs` の自動検出や CI 検証を安定動作させるには、判定ルールを skill references に明文化する必要がある。

## 課題

中黒・em-dash・LLM 表現・一文一行の 4 規範について、機械的に適用可能な判定ルールを `agentdev-doc-writing` skill の references に明文化する。既存の `llm-expression-patterns.md` を拡張して 4 規範を集約するか、新規参照ファイルを分離するかは実装時に判断する。文脈判断が必要な部分（「最も」「重要な」等）は機械判定の対象外であることを明記し、サンプリング査読への委譲を明示する。

## 既存要件との関連

- REQ-0140-026: 文書品質準拠の包括的修正。本件は同要件が求める機械判定ルールの明文化にあたり、Epic #1075 の後続整備項目である。
- SPEC `document-type-responsibilities.md` 用語政策節: 人間向けの判定基準を保持するが、機械適用向けの記述は未整備。本件がその欠落を埋める。
- `agentdev-doc-writing` skill `references/llm-expression-patterns.md`: 既存の LLM 表現パターン参照。中黒・em-dash・一文一行を追加する拡張先の候補。
- 横断是正（`2026-06-24-cross-cutting-norm-patterns.md` 参照）: X-1〜X-4 の機械的修正を実行する際の判定基準として、本成果物の出力を利用する関係にある。
- 欠落: 中黒許容範囲・一文一行の括弧内例外等を含むアルゴリズム的判定基準の SPEC/Skill 化。

## 整形の方向性

backlog-review では以下を検討する。

1. **RU 構成**: 単一 RU として `agentdev-doc-writing` skill references の拡張・新規作成にまとめる。作業範囲が局所的（1 スキルディレクトリ内）であり、分割の利益が薄い。
2. **配置判断**: 既存 `llm-expression-patterns.md` への 4 規範集約と、規範別の参照ファイル分離（例: `mechanical-replacement-rules.md` 新規）を比較する。LLM 表現以外は判定アルゴリズムの性質が異なるため、新規ファイル `mechanical-replacement-rules.md` に集約し、`llm-expression-patterns.md` は現状維持とする構成が自然である。
3. **優先度**: 中。横断是正（X-1〜X-4）や inspect-docs 自動検出の前提となるため、これらより前に、または並行して着手する価値がある。REQ-0140-026 が後続作業の根拠となる。
4. **検証**: 明文化後に既存 OU（#1076/#1078/#1079/#1080）の修正サンプルを再適用し、ルールが再現性のある判定を出すことを確認する。
