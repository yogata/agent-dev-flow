# REQ-0114-004「draft が2件以上存在する場合」の閾値解釈

## 観測内容

PR #1299 は REQ-0140-029（件数排除）を適用し、並列上限「最大5件」の重複記載を REQ-0130-026 + SPEC 参照へ解消した。ただし REQ-0114-004「draft が2件以上存在する場合」の「2件」は、ドラフト処理の閾値（並列上限とは別）であり、REQ-0140-029 の対象解釈が分かれるため REQ に残した。

## 影響

- 実動作への影響なし
- 件数規定の分類基準が未明文化のため、今後の件数排除判定で解釈が揺れる可能性

## 課題

- REQ-0114-004「2件」が並列上限と独立した閾値として妥当か、SPEC への移送対象かの判定
- 件数規定の分類基準（並列上限 vs ドラフト処理閾値 vs 契約上の閾値）の明文化

## 既存要件との関連

- REQ-0114-004: 本件の主題。並列上限とは独立した閾値として残存
- REQ-0140-029: 件数排除を適用したが、本件の「2件」は対象外と判断
- REQ-0130-026: 並列上限の重複記載の解消先
