# REQ-0119 横断是正 残対応（6 command の重複定義解消未対応）

## 観測内容
- PR #1534 では「正規な定義元が既存であった明確な重複の是正」に限定し、9 箇所の同一契約再定義を解消
- 6 command には追加の Step X-Y 細分番号が含まれ、追加是正が可能なまま残っている

## 影響
- 6 command で同一契約再定義が残存しており、REQ-0119-033（正規定義）から逸脱
- 一部は command 固有の公開契約に密接し、移動により公開契約の意味が変わるリスクがある

## 課題
- 残り6 command の重複定義を解消する
- 各 command で REQ-0119-035（公開契約維持）を個別確認することが前提

## 既存要件との関連
- REQ-0119-033: 正規定義
- REQ-0119-034: 同一契約再定義抑止
- REQ-0119-035: 公開契約維持
- REQ-0119-036

## 対応方針の方向性
- 対象6 command をリストアップし、移動可能な Step X-Y 細分番号と公開契約に密接なものを分類
- 公開契約に密接なものは個別 case として扱い、移動可否を慎重判断
- PR #1534 のスコープ外として明示されたものであるため、段階的対応を前提とする
- 関連: artifact-responsibilities SPEC の重複許容基準（別成果物）と併せて判断する余地あり
