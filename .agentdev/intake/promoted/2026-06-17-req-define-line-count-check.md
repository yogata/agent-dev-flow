# 要件定義時に定量的前提データの実ファイル検証が欠如している

## 観測内容

REQ-0103-116〜119（skill SKILL.md 行数是正4件）が、前提としたファイル行数データの誤りにより無効化された。要件定義時に実ファイルの行数を検証するステップが存在しなかったため、実際には閾値（400行）を超えていないファイルの是正要件が立案された。

具体経緯:
- Issue #786 で skill-authoring（465行）、learning-pipeline（410行）、gh-cli（296行）、req-file-manager（377行）を行数超過として REQ-0103-116〜119 を立案
- PR #787 で実ファイル行数を再確認したところ、前提データが誤りだったことが判明し、4件とも無効化

## 課題

- 要件定義時（req-define）に定量的前提データ（行数、ファイル数等）を実ファイルで検証するステップが存在しない
- 行数前提の誤った要件が将来も同様に立案される可能性
- 要件定義→保存→実装の各工程で無効要件の確認コストが発生する

## 影響

- 低〜中程度: 無効要件の立案・確認コスト。今回の事例では REQ-0103-116〜119 の4件が無効化され、要件ファイル・Issue の無駄な編集が発生した。

## 既存要件との関連

- REQ-0102: REQ記述ルール（検証可能な必達要件に限定する方針）
- REQ-0103-035〜037: 行数ガバナンス（SKILL.md ≤400行）
- req-define command: 要件整理を行うコマンド。定量的前提データの検証ステップがない。

## 対応方向の候補

- req-define（要件整理）に、定量的前提データ（行数、ファイル数等）の実ファイル検証ステップを追加する
- req-save（要件保存）に、入力値の sanity check を追加する（行数閾値超過を主張する要件に対して実ファイル行数を確認）
- いずれもコマンド変更（maintenance work_type）となる
