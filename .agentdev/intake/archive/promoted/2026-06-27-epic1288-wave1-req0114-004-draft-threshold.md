# REQ-0114-004「draft が2件以上存在する場合」の閾値解釈

## 観察

PR #1299（Issue #1292「docs/requirements/配下REQ 既存規範横展開」）は REQ-0140-029（件数排除）を適用し、並列上限「最大5件」の重複記載を REQ-0130-026 + SPEC 参照へ解消した。ただし REQ-0114-004「draft が2件以上存在する場合」の「2件」は、ドラフト処理の閾値（並列上限とは別）であり、REQ-0140-029 の対象解釈が分かれる。本 Issue では並列上限（最大5件）の重複解消に限定し、「2件」は契約上の閾値として REQ に残した。

## 修正されなかった理由

「2件」は並列上限（最大5件）とは別の契約上の閾値（ドラフト処理の判定条件）として解釈され、REQ-0140-029（並列上限の件数排除）の対象外と判断。別途 inspect-docs で判定可能と明記。

## 課題

- REQ-0114-004「2件」が並列上限と独立した閾値として妥当か、SPEC への移送対象かの判定
- 件数規定の分類基準（並列上限 vs ドラフト処理閾値 vs 契約上の閾値）の明文化
