# integrity-rule-catalog.md 行964 の保護対象リストが陳腐化（RU-0011 で更新予定）

## 発生源

- Issue: #1109
- PR: #1119 (merged, squash 61c98d50)
- 発生日: 2026-06-24

## 観測

`docs/specs/integrity-rule-catalog.md` 行964 の IR-044 exemption 条件の回帰テスト設計において「保護対象の真陽性: REQ-0114-082、REQ-0144-008」と記載されている。本 Issue (#1109) の是正により両行から SPEC 詳細（Step 番号、件数、"fixture" キーワード）が除去され、両行は真陽性ではなくなった。よって行964 の記述は陳腐化している。

## 今回扱わない理由

PR #1119 では同箇所の更新を RU-0011（検出ロジック改良）のスコープと判断し未変更。RU-0011 が検出ロジック改良時に回帰テストを設計し直す際、保護対象リストから両行を除外する作業と一体化する方が自然であるため、個別修正を避けた。本 PR のスコープは REQ-0114-082 / REQ-0144-008 の是正と WARNING=0 確認であり、SPEC カタログのテスト設計記述の更新は含まない。

## 影響

現状の行964 は「保護対象の真陽性」として存在しない状態を指している。RU-0011 実施前に同箇所を根拠にテストを設計すると、既に是正済みの行を保護対象として扱い、テストの前提が崩れる可能性がある。

## レビューで決めること

- 行964 の更新を RU-0011（検出ロジック改良）のスコープに含めるか、本件から個別 Issue を起こして先行是正するか
- 保護対象リストから REQ-0114-082 / REQ-0144-008 を除去し、両行が「是正済み（非真陽性化）」となった経緯を併記するか

## 根拠

PR #1119 本文「SPEC確定候補」セクション。case-close Step 3-2 の SPEC 確定フローにて option (c) 見送り（RU-0011 スコープへ委譲）として記録。対象 SPEC は `docs/specs/integrity-rule-catalog.md`（draft 昇格対象ではなく、参照陳腐化の是正候補）。
