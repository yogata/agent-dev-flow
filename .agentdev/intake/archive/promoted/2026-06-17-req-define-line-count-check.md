# 要件定義時に実ファイル行数検証が欠如している

## 観測

REQ-0103-116〜119（skill SKILL.md 行数是正4件）が、前提としたファイル行数データの誤りにより無効化された。要件定義時に実ファイルの行数を検証するステップが存在しなかったため、実際には閾値（400行）を超えていないファイルの是正要件が立案された。

具体的経緯:
- Issue #786 で skill-authoring（465行）、learning-pipeline（410行）、gh-cli（296行）、req-file-manager（377行）を行数超過として REQ-0103-116〜119 を立案
- PR #787 で実ファイル行数を再確認したところ、前提データが誤りだったことが判明し、4件とも無効化

## 今回扱われなかった理由

Issue #786 のスコープ（既存非準拠インスタンスの是正）外のプロセス改善であり、個別作業として扱われた。

## 影響

- 行数前提の誤った要件が将来も同様に立案される可能性
- 要件定義→保存→実装の各工程で無効要件の確認コストが発生する

## レビューで決めること

- req-define（要件整理）に行数等の定量的前提データの実ファイル検証ステップを追加すべきか
- req-save（要件保存）に入力値の sanity check を追加すべきか
- 再発防止知見として learning-capture に記録すべきか、要件プロセスの変更候補として扱うべきか

## 根拠

- 観測元: Issue #786「Command/Skill 構造品質コンプライアンス是正（REQ-0103 APPEND: 112-125）」、PR #787
- 元テキスト（PR #787 品質メトリクス）:
  > REQ-0103-116〜119（行数是正4件）が前提データの誤りで無効化。要件定義時に実ファイル行数検証を含めるべき。req-save の1行ガードレールメッセージは構造化エラーではないため template 抽出対象外とした。
- 関連 REQ: REQ-0102（REQ記述ルール）、REQ-0103-035〜037（行数ガバナンス）
