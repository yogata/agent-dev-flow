# LLM 表現の検出→書き換え辞書

LLM が生成しがちな表現パターンの検出と書き換え辞書。`agentdev-doc-writing` スキルの査読観点「検出→書き換え」の具体例正誤表である。執筆規範の SSoT は `japanese-tech-writing` スキル（AGENTS.md 経由）。本ファイルは運用参照資料（REQ-0140-026「個別用語の正誤表は agentdev-doc-writing スキルの参照資料で管理」準拠）。

## 対象読者

docs/ 配下の REQ/ADR/SPEC/guides/DOC-MAP/README、および docs を生成・編集する command/skill の自然言語記述を査読する agentdev-doc-writing スキル利用者。

## パターン分類

### 1. 接続の型

文脈を円滑に繋ぐために LLM が過剰に使用する接続表現。直接の記述で置き換える。

| 検出パターン | 書き換え | 解説 |
|---|---|---|
| `〜で` | `〜で` | 「で」は文語調。散文では「で」で足りる |
| `〜で、` | `〜で、` | 同上 |
| `〜について` | `〜について` または直接記述 | 「関して」は迂遠 |
| `〜については、〜がある` | 直接内容を書く | 「には〜がある」は述語を先送りする型 |
| `〜には、〜がある` | `〜は〜である` または直接列挙 | 同上 |
| `〜を参照` の連打 | 関連リンク節にまとめる | 参照の連打は本文の流れを断つ |
| `〜の観点から` | 直接理由・判断を書く | 「観点から」は評価軸の表明であって結論ではない |
| `〜という観点で` | 直接理由を書く | 同上 |
| `〜に基づき` | `〜に従い` または直接記述 | 「基づき」は抽象的 |
| `〜に応じて` | 具体的条件を書く | 「応じて」は条件を明示しない |
| `〜で重要な` | `〜で重要な` | 「で」の濫用 |

### 2. 空虚な動詞

具体的な動作を示さず、文を完成させるためだけに使われる動詞。具体的動作へ置き換える。

| 検出パターン | 書き換え | 解説 |
|---|---|---|
| `〜を掘り下げる` | `〜を分析する`、`〜を検討する` | 「掘り下げる」は比喩で動作が不明 |
| `〜を深掘りする` | `〜を分析する`、`〜を検討する` | 同上 |
| `〜に触れる` | `〜を述べる`、`〜を説明する` | 「触れる」は軽く言及する意で深さを暗示するが曖昧 |
| `〜を決定する` | 具体的動作（`判定する`、`選択する`、`特定する`） | 「決定する」は対象により意味が変わる |
| `〜を解決する` | 具体的動作（`解釈する`、`対応する`、`修正する`） | 「解決する」は結果であって動作ではない |
| `〜を適用する` | 具体的動作（`従う`、`使用する`、`設定する`） | 「適用する」は対象により意味が変わる |
| `〜を行う` | 具体的動詞（`実行する`、`処理する`、`記述する`） | 「行う」は汎用すぎる |
| `〜を実施する` | 具体的動詞 | 同上 |
| `〜を確認する`（対象不明） | 具体的動詞（`検証する`、`評価する`、`読む`） | 「確認する」は動作の種類を明示しない |
| `〜をサポートする` | 具体的動作（`許可する`、`提供する`、`対応する`） | 「サポート」は外来語の万能語 |
| `〜を可能にする` | 具体的動作（`許可する`、`使用できる`） | 「可能にする」は原因と結果の結びつきが曖昧 |
| `〜を実現する` | 具体的動作 | 「実現する」は結果の表明で動作が不明 |

### 3. 空虚な形容・副詞

具体性を与えずに文を強調する修飾語。具体的内容で置き換えるか削除する。

| 検出パターン | 書き換え | 解説 |
|---|---|---|
| N/A | 具体的対象範囲を明示 | 対象表現の復元不可（解説引用語欠損、本来は範囲が不明となる表現） |
| N/A | 具体的方法を明示 | 同上 |
| `必要な〜` | 必要性を本文で説明（`〜が必要`、`〜が前提`） | 「必要」は主張であって根拠ではない |
| `主要な〜` | `主要な〜`、`中心となる〜` または必要性説明 | 「主要」は比喩 |
| `根本的に〜` | 具体的性質を明示 | 「根本的」は比喩 |
| N/A | 強調削除、重要性の根拠を本文で示す | 対象表現の復元不可（解説引用語欠損、本来は主観的強調の副詞） |
| N/A | 強調削除、理由を本文で示す | 同上 |
| `最も〜` | 範囲・比較対象を明示 | 「最も」は比較軸が不明な場合が多い |
| `より〜`（比較対象省略） | 比較対象を明示 | 「より」は単独では比較軸が不明 |
| `完全に〜` | 具体的状態を明示 | 「完全に」は基準が不明 |
| `幅広い〜` | 具体的範囲を明示 | 「幅広い」は範囲が不明 |
| `様々な〜` | 具体的種類を列挙 | 「様々な」は具体性を隠す |
| `重要な〜`（理由省略） | 重要性の理由を本文で示す | 「重要」は主張であって根拠ではない |
| `適切に〜` | 具体的方法を明示 | 「適切に」は基準が不明 |

### 4. ラベル前置き

本文前にラベル・見出しを置いて読者に予告する形式。見出し階層または本文化で置き換える。

| 検出パターン | 書き換え | 解説 |
|---|---|---|
| `> **Scope**: ...` | 本文化、見出し階層で表現 | ラベル前置きは読者の負荷を増やす |
| `> **注意**: ...` | 事実を述べる本文 | 同上 |
| `> **前提**: ...` | 本文で前提条件を記述 | 同上 |
| `> **対象**: ...` | 見出しまたは本文で対象明示 | 同上 |
| `> **要約**: ...` | 本文で要点を先に書く | 同上 |
| `> **ポイント**: ...` | 本文化 | 同上 |
| `**重要**: ...` | 本文で重要性の理由を書く | 同上 |
| `**注意**: ...` | 本文で制約・条件を書く | 同上 |
| `**参考**: ...` | 参照リンク節にまとめる | 同上 |
| `**ヒント**: ...` | 本文化または削除 | 同上 |

## 検出→書き換えの適用手順

1. パターン検出: 上記辞書の検出パターンで grep 検索または読み査読
2. 文脈確認: 検出箇所の前後関係を確認し、書き換えが適切か判断
3. 書き換え適用: 辞書の書き換え列を参考に具体的な表現へ置換
4. 周辺文調整: 書き換えによって文脈が不自然になった場合は周辺文も調整
5. 再査読: 書き換え後に別の LLM 表現パターンが混入していないか再確認

## 適用上の注意

- **機械的置換は避ける**: 全ての検出パターンが常に LLM 表現とは限らない。文脈で正当な使用（例: 「決定事項」は固定複合名詞）は保持する
- **書き換え候補は複数提示**: 辞書の書き換え列は例であり、文脈に応じて最も自然な表現を選ぶ
- **japanese-tech-writing SSoT**: 本辞書は具体例に特化した運用参照資料であり、規範の SSoT は `japanese-tech-writing` スキルである。判定の最終基準は SSoT に従う
