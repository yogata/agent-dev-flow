# 査読出力形式

> **正本**: [docs/specs/writing-style.md](../../../../../docs/specs/writing-style.md)「移送判断」。本ファイルは運用ビュー（査読結果の分類・出力形式）である。

## 4 分類

查読で問題箇所を検出した場合、以下のいずれかに分類する（REQ-0140-021）。

| 分類 | 意味 |
|---|---|
| 残す | 現在の文書種別・位置が適切。文章品質の修正のみで対応可能 |
| 分割 | 1行に複数関心が混在。関心ごとに別行へ分割 |
| 移送 | 別の文書種別（REQ↔SPEC↔ADR↔guide↔skill reference）への移動が必要 |
| 削除候補 | 作業記録・移行結果・現状構成の詳細説明・変更履歴等、要件として不要な内容 |

## 修正文案形式

各問題箇所について、以下の形式で提示する。**未合意事項を確定しない**（最終判断は親コマンド・ユーザーが行う: REQ-0140-021）。

```
### [分類] 問題の要約（1行）

- 対象: ファイルパス:行番号 または セクション名
- 問題: 検出した AI-slop 基準 / 品質違反の種別
- 根拠: なぜ問題か（該当箇所の引用）
- 修正文案: 書き換え後のテキスト（残す・分割の場合）
- 移送先候補: 移動先の文書種別・ファイル（移送の場合）
```

## 出力原則（5 順序）

查読結果は [ai-slop-detection.md](ai-slop-detection.md) の出力原則に従い、以下の順序でまとめる。

1. 結論（全体判定: 合格 / 修正必要 / 重大な問題あり）
2. 判断理由（検出した問題の概要と件数）
3. 根拠（各問題箇所の具体的な引用）
4. 具体的な対応（修正文案・移送先候補の一覧）
5. 不明点または残リスク（親コマンド・ユーザーの判断が必要な事項）

## read-only 系表現の分解出力

`read-only`・`advisor`・`architecture-affecting` 系表現を検出した場合、[rewrite-patterns.md](rewrite-patterns.md) の変換テーブル・6 確認に従い、以下を明記した修正文案を提示する（REQ-0140-008〜013）。

- 何を変更しないか
- 何を読み取ってよいか
- 何を生成してよいか（許可出力先）
- commit / push の可否と対象範囲
- Issue / PR 更新の可否
- 最終判断主体（親コマンドが保持するか）

## 制約事項

- 本スキルは查読提案を返すのみであり、ファイル保存・commit・push を行わない（REQ-0140-022）。
- 提示した修正文案・移送先は候補であり、採用の最終判断は親コマンド・ユーザーが行う（REQ-0140-021）。
