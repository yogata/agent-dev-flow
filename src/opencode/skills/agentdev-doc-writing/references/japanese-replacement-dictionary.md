# 日本語置換辞書（agentdev-doc-writing 運用資料）

`agentdev-doc-writing` スキルの查読観点（REQ-0140-033）と docs-check / inspect-docs の forbidden 語機械検出（IR-060）が参照する、不自然な日本語表現の置換辞書。
REQ-0140-026「個別用語の正誤表は `agentdev-doc-writing` スキルの参照資料で管理」に準拠する配置先。

## 運用区分

置換辞書は以下の4区分で運用する。

| 区分 | 機械検出 | 查読 | 取扱 |
|---|---|---|---|
| forbidden | 対象（IR-060） | 不要 | 横断置換で即時是正 |
| review | 対象外 | 対象（REQ-0140-033） | 文脈で推奨訳を選択 |
| allowed_identifier | 対象外 | 対象外 | 識別子・コード・YAML・パスでは許容 |
| reconstruct | 対象外 | 対象（REQ-0140-034 D） | 作成元 PR・commit・旧版から復元 |

## forbidden（禁止語、即時横断置換）

現行自然言語文書（Markdown 本文）での使用を禁止する語。IR-060 が機械検出し、推奨訳へ置換する。

### 中国語簡体字・中国語由来語句

| 禁止語 | 推奨訳 | 備考 |
|---|---|---|
| `而非` | 「〜ではなく」 | 中国語「而不是」の誤混入 |
| `统一` | 「統一」 | 中国語簡体字 |
| `陈述形式` | 「記述形式」「表明形式」 | 文脈で選択 |
| `定位` | 「位置づけ」「位置決め」 | 文脈で選択 |
| `候选` | 「候補」 | 中国語簡体字 |
| `路径` | 「パス」「経路」 | 文脈で選択 |
| `一致性` | 「一貫性」「整合性」 | 文脈で選択 |
| `来源` | 「出典」「由来」「発生元」 | 文脈で選択（review 区分へ移行候補） |

### 文字化け・誤字・変換失敗

| 禁止語 | 推奨訳 | 備考 |
|---|---|---|
| `破綾` | 「破綻」 | 文字化け（「綻」の誤変換） |
| `監査証跠` | 「監査証跡」 | 文字化け（「跡」の誤変換） |
| `成果成果物` | 「成果物」 | 重複表現 |
| `本来件` | 「本来」「件」 | 文脈で復元 |
| `測可能性` | 「テスト可能性」「測定可能性」 | 文脈で選択 |
| `進捰` | 「進捗」 | 文字化け（「捗」の誤変換） |

### 直訳独自語

| 禁止語 | 推奨訳 | 備考 |
|---|---|---|
| `単独根` | 「単一の連結成分」「要素数1の連結成分」 | graph connected component 文脈。ルートが1つではなく要素数1の連結成分を指す |
| `自己完束` | 「自己完結」 | 「完結」の誤記 |

### 英語混在（識別子以外）

| 禁止語 | 推奨訳 | 備考 |
|---|---|---|
| `source-of-trought` | 「唯一の情報源（SSoT）」 | 「source of truth」の誤記（thought → truth） |
| `要件doc` | 「要件文書」「REQ 文書」 | 日本語と英語の混在 |

## review（文脈確認、查読で推奨訳を選択）

文脈によって推奨訳が変わるため、機械検出対象外。`agentdev-doc-writing` 查読（REQ-0140-033）で人手確認する。

| 語 | 推奨訳（文脈別） | 備考 |
|---|---|---|
| `查読` | 「レビュー」「査読」 | 学術文脈では「査読」、開発文脈では「レビュー」 |
| `正規のXX` | 「正規のXX」「正統なXX」「正のXX」 | XX の内容で選択。「canonical」の訳揺れ |
| `局所物理分離` | 「ファイル分割」「モジュール分割」 | 「local physical separation」の直訳。分割の実態に合わせて選択 |
| `責務境界浄化` | 「責務境界の明確化」「責務境界の整理」 | 「浄化」は抽象的。具体的な操作へ |
| `純化` | 「整理」「精製」「絞り込み」 | 文脈で選択 |
| `浄化` | 「整理」「クリーニング」 | 文脈で選択 |
| `具象参照抽象化` | 「具体参照の一般化」「具体例の抽象化」 | 文脈で選択 |
| `repo-local` | 「リポジトリローカル」「リポジトリ固有」 | 文脈で選択 |

## allowed_identifier（識別子・コード・YAML・パスでは許容）

以下の文脈では英語のまま許容する。日本語本文中の普通名詞としての使用は forbidden または review 区分へ従う。

- 識別子: `REQ-NNNN`, `ADR-NNNN`, `IR-NNN`, `OU-NNN` 等
- enum 値・YAML キー・フロントマスターフィールド名
- コマンド名・スキル名（`agentdev-doc-writing`, `agentdev-req-analysis` 等）
- ファイルパス・ディレクトリパス
- backticks で囲まれたコード値（`` `source-of-trought` `` のように例示する場合）
- fenced code block 内の字句
- URL・固定リンク

## reconstruct（原文復元、作成元から復元）

機械的な置換が困難な語。作成元の PR・commit・旧版から原文を復元する（REQ-0140-034 D）。推測による置換は禁止する。

| 語 | 復元方法 | 備考 |
|---|---|---|
| `ターゲットング隠退化` | 作成元 PR・commit から原文を特定し復元 | 意味不明な圧縮表現。「ターゲティングの隠蔽」「対象範囲の隠蔽」等へ |
| 文字化け（Issue `#1167`, `#1169`, `#1369`, `#1373` 等） | Issue 作成元の commit・PR から原文を特定し復元 | 文字化けの推測置換は禁止 |
| `素流用` | 作成元から復元。「そのまま流用」「無改造で再利用」等へ | 助詞欠落の圧縮表現 |
| `未連リソース` | 作成元から復元。「未連携リソース」「未接続リソース」等へ | 助詞欠落の圧縮表現 |
| `平緒方針` | 作成元から復元。「基本方針」「持平の方針」等へ | 意味不明な表現 |

## 修正方式（A〜D）

検出された不自然表現の是正方式は以下の4種（REQ-0140-034）。

| 方式 | 内容 | 対象区分 |
|---|---|---|
| A | 安全な横断置換 | forbidden（推奨訳が一意のもの） |
| B | 文脈別置換 | forbidden（推奨訳が複数のもの）、review |
| C | 文全体の再記述 | P2（名詞の過剰連結）、review で文意が複雑なもの |
| D | 原文復元 | reconstruct（文字化け・意味不明語） |

## 検出分類（P0〜P4）

検出分類の原本は `document-type-responsibilities.md`「不自然表現検出分類（REQ-0140-033）」節を参照。本辞書の forbidden / review / reconstruct 区分が P0〜P4 の各分類に対応する。

## 更新運用

本辞書の更新は `agentdev-doc-writing` スキルの改訂として扱う。新たな forbidden 語、review 語の追加は以下の手順で行う。

1. 監査報告・学習事項から新たな不自然表現を抽出
2. 推奨訳、修正方式（A〜D）、運用区分（forbidden / review / reconstruct）を確定
3. 本辞書へ追加
4. forbidden 区分へ追加した場合は IR-060 の検出対象へ反映し、docs-check / inspect-docs で検出確認
