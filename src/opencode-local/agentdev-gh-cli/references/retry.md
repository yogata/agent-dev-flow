# リトライロジック（ローカル版）

VERIFY 失敗時の3段階リトライロジック（REQ-0150-001, ADR-0130 decision #4, #5）。
ローカル版は標準版（[../../skills/agentdev-gh-cli/references/retry.md](../../../skills/agentdev-gh-cli/references/retry.md)）と同一のリトライロジックを持つ。差分の種類に応じて対応を分類する。

ローカル版ではネットワーク揺れや gh CLI の一時不具合は発生しないが、Write tool の一時失敗、ファイルハンドル競合、BOM 付き書き出し事故等の同種の差分に対して本ロジックを適用する。

## 3段階分類

### (a) 同一内容リトライ

- **適用条件**: 書き込み tool の一時失敗、ファイルハンドル競合等、同一内容の再書き込みで解決する可能性がある差分
- **最大回数**: **2回**（初回 + 2回リトライ = 計3回）
- **手順**: 同一内容で書き込みを再実行し、再度検証する

### (b) 内容の再構築

- **適用条件**: 必須セクション欠落、Markdown 構造崩れ等、書き込み内容自体に問題がある差分。同一内容リトライ (a) で解決しなかった場合も含む
- **手順**: 書き込み元テキストを再構築して書き込み直し、再度検証する
- **再構築後は (a) に戻さない** — 再構築しても失敗する場合は (c) へ移行

### (c) 停止、ユーザー報告

- **適用条件**: (a) 同一内容リトライ上限到達、(b) 内容の再構築後も失敗、または差分が上記以外の原因
- **手順**: ユーザーに以下を報告して停止する:
  - 失敗した操作（例: Case ファイル本文更新 case-0042）
  - 検出された差分の内容
  - 実行した対応段階（(a)/(b)のどちらまで試行したか）
  - リトライ回数
  - ユーザーの指示を待つ（自動的な代替手段の実行は禁止）

## 遷移図

```
書き込み → VERIFY
              │
              ├─ PASS → 完了
              │
              └─ FAIL → (a) 同一内容リトライ (最大2回)
                          │
                          ├─ PASS → 完了
                          │
                          └─ FAIL (上限到達) → (b) 内容の再構築
                                                │
                                                ├─ PASS → 完了
                                                │
                                                └─ FAIL → (c) 停止、ユーザー報告
```
